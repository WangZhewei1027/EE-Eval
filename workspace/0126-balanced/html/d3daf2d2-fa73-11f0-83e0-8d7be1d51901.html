Log

This is a simulation of mark-and-sweep GC. Real JS engines use generational and incremental collectors and you can't reliably force GC from web pages. Use browser devtools to inspect memory.

How it works (brief)

- Roots (Global, DOM, Stack) are starting points the collector uses. Any object reachable from roots is "live".
- Mark phase: traverse references from roots and mark reachable objects.
- Sweep phase: unmarked objects are considered garbage and can be reclaimed.
- Common causes of leaks: closures that keep references, timers/event listeners not cleared, global references or DOM nodes still referenced in JS.

Interaction tips

- Allocate objects, create references between them, then click "Simulate GC" to mark reachable objects. Unmarked ones appear dimmed.
- Click "Sweep" to free garbage (removes them from the heap).
- Use examples to quickly create typical leak scenarios.
