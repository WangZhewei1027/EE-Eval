ACID Properties in Database Transactions

ACID (Atomicity, Consistency, Isolation, Durability) is a set of properties that guarantee database transactions are processed reliably.

Atomicity

Transactions are treated as a single "unit" which either completes fully or not at all.

Consistency

A transaction brings the database from one valid state to another, maintaining all defined rules.

Isolation

Concurrent transactions execute as if they were sequential, preventing interference.

Durability

Once a transaction is committed, it remains committed even in the event of a system failure.
