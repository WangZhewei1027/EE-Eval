Big-Omega (Ω) Notation — Interactive Demonstration

Definition (informal): We write f(n) ∈ Ω(g(n)) if there exist constants c > 0 and n0 such that for all n ≥ n0, f(n) ≥ c·g(n). Intuitively: g(n) is a lower bound for f(n) (up to constant factors) for large n.

Quick examples & formal proofs

Example 1: f(n)=3n+5 is Ω(n). Proof: choose c = 3 and n0 = 1. For n ≥ 1, 3n+5 ≥ 3n = c·n.

Example 2: f(n)=n^2+100n is Ω(n^2). Proof: for n ≥ 100, n^2+100n ≥ n^2 (so c=1,n0=100 works). Better: for n ≥ 1, n^2+100n ≥ n^2.

Counterexample: f(n)=n is NOT Ω(n^2) because n < c·n^2 for any fixed c>0 when n is large (divide both sides gives 1 < c·n, impossible for all large n unless c=0).

What the empirical test does

For each candidate n0 between start and max, the script computes the minimum value of f(n)/g(n) for all sampled n ≥ n0 (within the tested range). If that minimum is positive, we set c to that minimum and report that, on the tested range, f(n) ≥ c·g(n) for all sampled n ≥ n0. This is not a formal proof for all n→∞, but it often finds concrete c and n0 that serve as evidence or a counterexample.
