Integration Testing Demo

Concept Explanation

Integration Testing is a testing level where individual units or modules are combined and tested as a group. The purpose is to validate the interaction and data flow between modules to detect interface defects and ensure they work together correctly.

Unlike unit tests (testing components in isolation), integration tests focus on combined behavior.

Example Modules

We have three simple modules simulating user registration flow:

  1. validateUserInput – Validates user data (unit-level).
  2. saveUserToDB – Simulates saving to a database (mocked).
  3. sendWelcomeEmail – Simulates sending an email.

The integrated registerUser function calls all three to complete registration.
