P vs NP — Interactive Demonstration

This single-page interactive demo illustrates the difference between "P" (problems solvable in polynomial time) and "NP" (problems whose solutions can be verified in polynomial time). It does not resolve the open question "P vs NP" — that remains one of the major open problems in theoretical computer science. Use the controls below to experiment with small examples.

Quick definitions

  • Decision problem: answer is YES/NO (e.g., "Does this Boolean formula have a satisfying assignment?").
  • P: class of decision problems solvable by a deterministic algorithm in polynomial time (w.r.t. input size).
  • NP: class of decision problems where, if the answer is YES, there exists a short certificate (witness) that a polynomial-time algorithm can verify.
  • NP-complete: problems in NP that are at least as hard as every other problem in NP; if any NP-complete problem is in P, then P = NP.

SAT (Satisfiability) example — verify vs search

SAT is the canonical NP-complete problem: given a Boolean formula in CNF, does there exist an assignment of true/false to variables that makes the formula true? If someone hands you an assignment (a certificate), you can verify it quickly. Finding that assignment (without extra information) appears hard in general: known algorithms are exponential in the worst case.

Notes & tips

  • Verifier: given an assignment, checking every clause is linear in the formula size (fast).
  • Brute-force: tries all 2^n assignments. Time doubles when you add one variable — exponential growth.
  • Backtracking: assigns variables and prunes partial assignments when a clause becomes impossible; often much faster than naive brute force for structured formulas.

Explanation

Try increasing variables and see how brute-force time grows (estimation shown). The backtracking solver may quickly find a solution or prove unsatisfiable for many instances — but worst-case behavior still grows exponentially for general SAT.

Subset Sum (certificate vs search)

Given a list of integers and a target T, does a subset sum to exactly T? If someone gives you the subset (certificate), verifying is fast. Searching can be exponential (2^n). There's a dynamic programming algorithm that's pseudo-polynomial (O(n * T)).

Polynomial-time example: BFS shortest path (in unweighted graph)

Find shortest path in an unweighted graph using BFS — this runs in O(V + E) time (polynomial). Use this to contrast with exponential SAT/SubsetSum searches.

Key takeaway

P vs NP asks: is every problem whose solutions can be verified quickly (in polynomial time) also solvable quickly? Most researchers believe P != NP, but the question remains unproven. If someone produced a polynomial-time algorithm for any NP-complete problem, it would imply P = NP, with huge consequences — both positive (faster algorithms) and potentially negative (cryptography broken).
