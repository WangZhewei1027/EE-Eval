Big-O Notation — Visualizer & Interactive Demo

Big-O describes how an algorithm's runtime or memory grows as the input size (n) grows. Use the controls to explore common complexity classes, compare their growth, and run small timing experiments.

Micro-bench runs small operations to roughly illustrate runtime differences (not precise benchmarking).
When values grow very large (exponential/factorial), log scale reveals relative growth better.

Code examples

Short illustrations showing why the complexity is what it is.

O(1) — constant

// access an array element
const arr = [1,2,3,4];
return arr[0]; // always one step -> O(1)

O(n) — linear

// sum of array elements
let sum = 0;
for (let i = 0; i < n; i++) {
  sum += arr[i];
}
// ~n steps -> O(n)

O(n²) — quadratic

// naive pair checking
for (let i = 0; i < n; i++) {
  for (let j = 0; j < n; j++) {
    // constant work
  }
}
// ~n * n steps -> O(n²)

O(n log n) — divide/conquer sorts

// merge sort or quicksort average
// splits -> log n levels, work per level ~ n -> n log n

O(2ⁿ), O(n!) — exponential / factorial

// naive recursive subset enumeration (2ⁿ)
// permutations (n!) — grow extremely fast!
