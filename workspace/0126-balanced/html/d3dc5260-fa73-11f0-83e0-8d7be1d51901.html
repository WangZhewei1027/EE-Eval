Asymmetric Cryptography Demo

This page demonstrates public-key (asymmetric) cryptography using the Web Crypto API. You can generate key pairs for "Alice" (signing) and "Bob" (encryption), export/import public keys, encrypt a message with Bob's public key, and sign it with Alice's private key. Bob can then decrypt and verify the signature.

Note: For demo purposes keys are exportable. In a real application private keys should be non-exportable.
Quick notes: - RSA-OAEP is used for encryption/decryption; RSASSA-PKCS1-v1_5 is used for signing/verifying. - Public keys are exported as SPKI and encoded in Base64 to simulate exchanging keys. - This page uses extractable keys for demonstration; in production, keep private keys non-extractable and protected.
