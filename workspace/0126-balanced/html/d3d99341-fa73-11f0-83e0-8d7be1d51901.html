Indexing — interactive concepts

This page demonstrates common indexing concepts used in programming and search: array indexing (0 vs 1-based), using an index (map) to speed lookups vs linear scans, and building a simple inverted index for text search. Try the examples below.

Array Indexing (0-based vs 1-based)

Arrays store values at numeric positions called indices. Different systems sometimes use different base indexes (0 or 1).

Index (Map) vs Linear Search

We create many records and compare searching by scanning every item vs using an index (a Map) to jump directly to results.

Inverted Index (text search)

An inverted index maps terms (words) to the documents that contain them. This is the building block for fast full-text search.

Explanation & Quick Notes

  • Array indexing: Accessing arr[i] is O(1) because the location is computed from the base pointer + i × element-size.
  • Linear scan: To find a value without an index you may check every item → O(n).
  • Index (Map): A map from key → list of item locations lets you find matches in O(1) (average) for the lookup plus O(k) to return k matches.
  • Inverted index: Maps each term → set/list of document ids. For multi-term queries you typically intersect postings lists.
