ACID Properties — Interactive Demo

This small in-browser demo illustrates the four ACID properties (Atomicity, Consistency, Isolation, Durability) using simple account transfers and transactions.

Database (Committed State)

Committed values are persisted (durable). Transactions work on private snapshots/workspaces.
Serializable = optimistic concurrency check on commit (detects conflicts). No Isolation = commits overwrite without checks (shows anomalies).

Transactions

Open transactions and operations (atomic units)

Operations

Create transfer operations inside a selected transaction (atomic group).
Consistency rule: total sum must remain the same and balances cannot be negative. Commits that violate consistency will be rejected.

ACID Quick Guide

Atomicity: a transaction's operations either all happen (commit) or none (rollback).

Consistency: transactions must preserve database rules (here: total sum and non-negative balances).

Isolation: concurrent transactions do not interfere; serializable mode detects conflicts on commit. No isolation mode demonstrates anomalies.

Durability: once committed, changes survive crashes (persisted in localStorage).
Tip: Try creating two transactions that change the same account and test both isolation modes. Then simulate a crash after a commit vs after creating but not committing a transaction.
