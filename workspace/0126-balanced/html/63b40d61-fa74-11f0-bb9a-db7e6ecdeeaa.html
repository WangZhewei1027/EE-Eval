Static Typing Demonstration

What is Static Typing?

Static typing means that the type of a variable is known and checked at compile-time, before the program runs. Languages like TypeScript, C#, Java, and C++ use static typing. This allows for early detection of type-related errors.

JavaScript is a dynamically typed language, so types are checked at runtime and can change. To demonstrate static typing, we’ll use TypeScript (a superset of JavaScript with static typing).

Below is an example in TypeScript showing variable declarations with static types:

// TypeScript example
let name: string = "Alice";
let age: number = 30;
// name = 123;          // ❌ Compilation error: Type 'number' is not assignable to type 'string'
age = age + 1;         // ✅ OK

function greet(person: string): string {
  return "Hello, " + person;
}

console.log(greet(name));

Because static types are checked before runtime, the following faulty assignment would be caught by the TypeScript compiler:

name = 123; // Error: Type 'number' is not assignable to type 'string'
