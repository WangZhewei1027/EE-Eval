When does Greedy work? When can it fail?

- Greedy algorithms build a solution by repeatedly making a local optimum choice. When the local choice leads to a globally optimal solution, the algorithm is correct.
- They are usually fast (often O(n log n) due to sorting) and simple.
- They can fail if the problem doesn't have the "greedy-choice property" or "optimal substructure" aligned with the chosen rule — see the coin change counterexample in the demo (coins 1,3,4).
Complexities (typical):
  • Coin greedy: O(k + n) per run if coins sorted (k coins), optimal DP: O(amount * k).
  • Activity selection: O(n log n) dominated by sorting by finish times, greedy picks in O(n).
  • Fractional knapsack: O(n log n) for sorting by ratio, then O(n) to fill.
