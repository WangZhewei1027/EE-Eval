Quick Definitions

P: problems solvable by an algorithm in polynomial time (fast).

NP: decision problems where a candidate solution can be verified in polynomial time.

NP-Complete: problems in NP that are at least as hard as any other problem in NP — every NP problem can be reduced to an NP-Complete problem in polynomial time. If any NP-Complete problem is in P, then P = NP.

Why reductions?

Reductions translate instances of one problem to another so that a solution to the target gives a solution to the source. A single polynomial-time reduction from every problem in NP to one target problem makes that target NP-Complete.

Common NP-Complete problems

- SAT / 3-SAT (satisfiability of boolean formulas)
- CLIQUE, VERTEX COVER, INDEPENDENT SET (graph problems)
- SUBSET-SUM / KNAPSACK (numbers & targets)
- HAMILTONIAN CYCLE, TRAVELING SALESMAN (variants)

How the demo connects to NP-Completeness

3-SAT is a canonical NP-Complete problem (Cook–Levin theorem shows SAT is NP-complete; 3-SAT is a standard restriction still NP-Complete). The reduction to CLIQUE shown here is a textbook example: a polynomial-time transformation that preserves satisfiability. If you could solve CLIQUE quickly (in polynomial time), you could solve 3-SAT quickly by reducing and solving, implying P = NP.
