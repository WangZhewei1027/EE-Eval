Space Complexity Demonstration

Space complexity is the amount of memory an algorithm needs relative to the input size. Let's see this concept with two functions that calculate the sum of numbers from 1 to n:

Example 1: Iterative Approach (constant space)

function iterativeSum(n) {
  let sum = 0;          // uses a fixed amount of space
  for (let i = 1; i <= n; i++) {
    sum += i;
  }
  return sum;
}

Example 2: Create Array and Reduce (linear space)

function arraySum(n) {
  // creates an array of size n -> O(n) space
  let arr = Array.from({length: n}, (_, i) => i + 1);
  let sum = arr.reduce((acc, val) => acc + val, 0);
  return sum;
}

Explanation:

  • iterativeSum: Uses a constant amount of extra memory regardless of n. It only stores sum and i.
  • arraySum: Creates an array of n elements, which uses memory proportional to n (linear space complexity).

So even though both compute the same result, their space complexity differs:

  • iterativeSum: O(1) space
  • arraySum: O(n) space
