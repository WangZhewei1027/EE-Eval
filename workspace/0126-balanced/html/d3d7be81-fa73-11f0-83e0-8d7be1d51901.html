Recursion — Interactive Demonstrations

Recursion is when a function calls itself to solve a smaller piece of the same problem. Below are interactive examples: a factorial with an animated call stack, a visual recursion tree for Fibonacci, and a recursive drawing (Sierpinski triangle). Try small inputs first (recursion depth grows quickly).

Recursive implementation (conceptual): 
function factorial(n) {
  if (n <= 1) return 1;
  return n * factorial(n - 1);
}

Example 3 — Recursive drawing: Sierpinski triangle
A geometric fractal built by recursion: draw a triangle, then 3 smaller triangles inside, and repeat.
Try depths 0–6 for best performance.
Recursive idea (conceptual): 
function drawTriangle(a, b, c, depth) {
  if (depth === 0) fillTriangle(a, b, c);
  else {
    // compute midpoints and call drawTriangle on 3 corners
    drawTriangle(a, mid(a,b), mid(a,c), depth-1);
    drawTriangle(b, mid(b,a), mid(b,c), depth-1);
    drawTriangle(c, mid(c,a), mid(c,b), depth-1);
  }
}
Tip: Recursion is powerful for problems that can be broken into similar subproblems. Be careful with base cases and depth to avoid infinite recursion.
