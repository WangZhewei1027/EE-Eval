Dynamic Programming Demo: Fibonacci Numbers

Dynamic Programming (DP) is an optimization technique used to solve problems by breaking them down into simpler subproblems and storing the results to avoid redundant computations.

Here, we'll compute Fibonacci numbers using three different approaches to see how dynamic programming improves efficiency:

  • Naive recursion - exponential time (slow)
  • Memoization (Top-down DP) - cache results of recursion
  • Tabulation (Bottom-up DP) - iterative and fills a table
Result and performance will appear here.

Explanation

The Fibonacci sequence is defined as: 

Fib(0) = 0
Fib(1) = 1
Fib(n) = Fib(n-1) + Fib(n-2) for n ≥ 2

The naive recursive method computes many values repeatedly, causing exponential runtime. Memoization stores intermediate subproblem results to avoid recomputation, and tabulation builds the solution bottom-up iteratively.
