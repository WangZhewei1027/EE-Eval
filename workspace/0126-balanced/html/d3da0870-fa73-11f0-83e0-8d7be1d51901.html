HTTPS Concept — Interactive Simulation

This single-file demo simulates the main building blocks of HTTPS (certificates, trust, and an encrypted handshake) using the Web Crypto API. It is educational—no real TLS, but it demonstrates how certificates, signing, verification, and an encrypted session (ECDH + AES-GCM) work together.

High-level steps simulated:
  • Create a trusted Certificate Authority (CA).
  • Have the CA sign a server "certificate" that contains the server's public key.
  • Client verifies the certificate using the trusted CA's public key.
  • Client and server perform an ECDH key agreement to derive a symmetric AES-GCM session key.
  • Client sends an encrypted message; server decrypts it.
  • Try a tampered or attacker-signed certificate to see verification fail.
