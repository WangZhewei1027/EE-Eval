Transaction Demonstration

Interactive simulation of transactions (ACID): atomicity, consistency, isolation, durability. Try multiple transactions and see commits, rollbacks, locks and optimistic conflicts.

How it works:
  • Begin transactions on the right; add operations (deposit, withdraw, transfer).
  • In Pessimistic mode, write locks prevent concurrent writes.
  • In Optimistic mode, transactions check versions at commit and abort on conflict (you can retry).
  • Committed state is saved to localStorage with "Persist committed state". This demonstrates durability across reloads.
