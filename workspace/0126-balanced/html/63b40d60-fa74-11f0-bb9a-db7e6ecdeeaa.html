Understanding Type Systems in Programming

A type system is a set of rules that assigns a property called type to various constructs - such as variables, expressions, functions or modules - in a computer program. It helps detect errors, enforce constraints, and improve program reliability.

Key Concepts

  • Type Safety: Guarantees that operations only get applied to compatible data types.
  • Static Typing: Types are checked at compile time (e.g., Java, C#).
  • Dynamic Typing: Types are checked at runtime (e.g., JavaScript, Python).
  • Strong vs Weak Typing: Determines how strictly types are enforced or coerced.

Example: JavaScript's Dynamic & Weak Type System

JavaScript is dynamically typed and weakly typed, meaning variables can hold values of any type and implicit type coercion can occur.


// Variable can hold any type
let value = 42;       // number
value = "hello";      // now a string

// Implicit coercion in addition
console.log(2 + "3"); // "23" (number + string => string concatenation)

// Comparison quirks
console.log(0 == false);   // true (weak equality)
console.log(0 === false);  // false (strict equality)

Example: A Simple Static Type System Check

Let's simulate a very simple static type checker for a small expression language:


// Expressions can be numbers or booleans
// Addition only allowed with numbers

function typeCheck(expr) {
  if (typeof expr === "number") return "number";
  if (typeof expr === "boolean") return "boolean";
  if (expr.op === "+") {
    const leftType = typeCheck(expr.left);
    const rightType = typeCheck(expr.right);
    if (leftType === "number" && rightType === "number") return "number";
    throw new Error("TypeError: '+' requires number + number");
  }
  throw new Error("TypeError: Unknown expression");
}
