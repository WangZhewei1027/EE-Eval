Understanding Time Complexity

Time complexity measures how the runtime of an algorithm grows as the input size increases. Here are three common examples demonstrated in JavaScript:

1. Linear Time Complexity - O(n)

The function loops through an array once, so time increases linearly with input size. 
// Summing numbers in an array (O(n))
function sumArray(arr) {
  let sum = 0;
  for(let i = 0; i < arr.length; i++) {
    sum += arr[i];
  }
  return sum;
}

2. Quadratic Time Complexity - O(n²)

The function uses two nested loops which makes runtime grow quadratically. 
// Counting pairs with sum zero (O(n²))
function countZeroSumPairs(arr) {
  let count = 0;
  for(let i = 0; i < arr.length; i++) {
    for(let j = i + 1; j < arr.length; j++) {
      if(arr[i] + arr[j] === 0) count++;
    }
  }
  return count;
}

3. Logarithmic Time Complexity - O(log n)

Binary search halves the search space each step, making time grow logarithmically. 
// Binary search (O(log n))
function binarySearch(arr, target) {
  let left = 0, right = arr.length - 1;
  while(left <= right) {
    let mid = Math.floor((left + right) / 2);
    if(arr[mid] === target) return mid;
    else if(arr[mid] < target) left = mid + 1;
    else right = mid - 1;
  }
  return -1;
}
