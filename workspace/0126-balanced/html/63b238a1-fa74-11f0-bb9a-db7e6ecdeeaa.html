Understanding Big-O Notation

Big-O notation describes the upper bound of an algorithm's running time or space requirements in terms of the input size n. It helps compare how well algorithms perform as inputs become large.

Common Big-O Complexities

  • O(1) — Constant time: runtime does not depend on input size.
  • O(log n) — Logarithmic time: grows slowly as n increases.
  • O(n) — Linear time: runtime increases proportionally with n.
  • O(n log n) — Linearithmic time: typical of efficient sorting algorithms.
  • O(n²) — Quadratic time: runtime grows proportionally to square of n.
