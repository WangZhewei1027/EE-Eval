Key concepts demonstrated
  • Full table scan — examining every row costs O(N).
  • Indexes — lookup maps reduce cost to O(1) per probe (amortized), avoiding scanning the whole table.
  • Nested loop join — simple, but O(N*M) when naive.
  • Hash join — builds a hash table on the smaller input, yielding roughly O(N+M) work.
  • Predicate pushdown — filtering as early as possible reduces the amount of data operated on downstream.
This is a simulation to illustrate how planners choose algorithms and how choices affect cost and runtime.
