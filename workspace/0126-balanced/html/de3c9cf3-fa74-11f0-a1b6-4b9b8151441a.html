What is Amortized Analysis?

Amortized analysis is a method for analyzing a sequence of operations where, while some individual operations may be expensive, the average cost per operation is small when averaged over the entire sequence. It provides a worst-case guarantee on the average performance of each operation in the worst case.

This is different from average-case analysis because we're not making any probabilistic assumptions - the amortized analysis guarantees that any sequence of operations will have the stated average cost.

Dynamic Array Example

A common example is a dynamic array that grows when it becomes full. While some append operations may be expensive (when resizing is needed), the average cost per append operation is constant (O(1) amortized time).

Three Methods of Amortized Analysis

1. Aggregate Method

Determine total cost of n operations, then divide by n to get average cost per operation.

2. Accounting Method

Assign different charges to each operation ("amortized cost"). Some operations are overcharged to cover future expensive operations.

3. Potential Method

Define a "potential" function on the data structure's state. The amortized cost is the actual cost plus the change in potential.

Amortized Analysis of Dynamic Array

When the array is full, we:

  1. Allocate a new array of double the size (cost: n operations)
  2. Copy all elements to the new array (cost: n operations)
  3. Add the new element (cost: 1 operation)

Total cost for this operation: 2n + 1

But the next n-1 operations will only cost 1 each.

Total cost for n operations: (2n + 1) + (n - 1) = 3n

Amortized cost per operation: 3n / n = 3 (which is O(1))
