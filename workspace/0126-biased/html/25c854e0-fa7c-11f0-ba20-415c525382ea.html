Dynamic Arrays

An in-depth textual guide on the concept, theory, and practical algorithms behind Dynamic Arrays.

Introduction

A Dynamic Array is a fundamental data structure in computer science that provides the convenience of an array with the ability to grow or shrink dynamically at runtime. Unlike static arrays that have a fixed size defined upon creation, dynamic arrays adjust their capacity as elements are added or removed, offering a flexible and efficient way to store sequential elements.

Dynamic arrays underpin many higher-level structures and libraries, such as std::vector in C++, ArrayList in Java, and simply standard arrays in languages like Python or JavaScript with flexible resizing capabilities.

Why Dynamic Arrays?

Static arrays have an inherent limitation: their size must be determined in advance and cannot be modified during program execution. This limitation can be problematic:

  • Wasting memory if the array is allocated too large.
  • Running out of space if the array is too small.
  • Inability to store an unknown number of elements determined at runtime.

Dynamic arrays solve these issues by initially allocating some capacity and then resizing as needed, typically by allocating a larger block of memory and copying the existing data into it. This mechanism combines efficiency and flexibility.

Basic Concept

A dynamic array internally manages two main variables:

  • Size (n): The number of elements currently stored.
  • Capacity (c): The total number of elements the currently allocated memory can hold without resizing.

Invariant: 0 ≤ n ≤ c. When n == c, the array must be resized to accommodate additional elements.

Conceptually, you can think of a dynamic array as a wrapper around a static array where the wrapper handles resizing behind the scenes.

How Does Resizing Work?

The key to the dynamic array's efficiency is the resizing strategy. The typical approach is:

  1. Appending an element: When inserting a new element, if size < capacity, the element is placed in the next free slot and the size is incremented.
  2. Capacity full: If the array is full (size == capacity), typically:
    • Allocate a new array with a larger capacity, usually doubling the current capacity.
    • Copy all existing elements from the old array to the new array.
    • Release the old array's memory.
    • Place the new element in the new array and update size and capacity.

This doubling strategy amortizes the cost of resizing, keeping the average time complexity for insertion at O(1), called amortized constant time.

Algorithmic Details

Insertion (Append) Algorithm

function append(dynamicArray, element):
    if dynamicArray.size == dynamicArray.capacity:
        newCapacity = max(1, 2 * dynamicArray.capacity)
        newData = allocateArray(newCapacity)
        for i from 0 to dynamicArray.size - 1:
            newData[i] = dynamicArray.data[i]
        release(dynamicArray.data)
        dynamicArray.data = newData
        dynamicArray.capacity = newCapacity
    dynamicArray.data[dynamicArray.size] = element
    dynamicArray.size += 1

The above pseudocode:

  • Checks if a resize is necessary.
  • If yes, doubles the capacity, allocates new memory, copies elements, and frees the old memory.
  • Adds the new element at the next open index.

Access Algorithm

Accessing any element in a dynamic array is an O(1) operation since indices correspond directly to memory offsets: 

function get(dynamicArray, index):
    if index < 0 or index >= dynamicArray.size:
        raise IndexError("index out of bounds")
    return dynamicArray.data[index]

Deletion Algorithm (from the end)

Removing elements from the end is straightforward: 

function removeLast(dynamicArray):
    if dynamicArray.size == 0:
        raise Error("array is empty")
    dynamicArray.size -= 1
    // Optionally, shrink capacity here if needed, but shrinking is less common

Shrinking the capacity when many elements are removed is possible but often avoided or done conservatively to prevent frequent resizing on fluctuating array sizes.

Amortized Complexity Explanation

Although resizing an array is an O(n) operation due to copying elements, doubling the capacity amortizes this cost over multiple insertions.

Why? Consider inserting n elements starting from an empty dynamic array:

  • 1st insertion: No resize, cost O(1).
  • 2nd insertion: Resize from capacity 1 to 2; copying 1 element → cost ~2.
  • 3rd insertion: Resize from 2 to 4; copying 2 elements → cost ~3.
  • 5th insertion: Resize from 4 to 8; copying 4 elements → cost ~5.

Each resize doubles the capacity, and each element is copied at most once per resize step. This leads to a total copying cost proportional to 2n - 1, meaning the average (amortized) cost per insertion is O(1).

Additional Features and Considerations

1. Random Access

Dynamic arrays provide constant time random access, unlike linked lists which require traversal.

2. Memory Overhead

Since capacity is often larger than size, dynamic arrays may use more memory than strictly necessary. This trade-off is accepted for better performance on insertions.

3. Shrinking Arrays

Some implementations support shrinking the capacity when the size falls below certain thresholds (e.g., if size < capacity/4, reduce capacity to half). This avoids wasting too much memory but must be done cautiously to avoid frequent costly resizes.

4. Trade-offs versus Linked Lists

  • Dynamic Arrays: Fast random access, amortized constant time append, but expensive insertions/deletions in the middle.
  • Linked Lists: Constant time insertions/deletions anywhere (given references), but slow random access and higher memory per element.

Thus, dynamic arrays are preferable when element access speed and cache locality matter, and insertion/deletion mostly happens at the end.

Concrete Example

Let's consider a step-by-step example inserting integers from 1 to 10 into an initially empty dynamic array.

  1. Start: size = 0, capacity = 0, data = [].
  2. Insert 1: resize to capacity 1, data = [1], size = 1.
  3. Insert 2: array full, resize to capacity 2, copy existing 1 element, data = [1,2], size = 2.
  4. Insert 3: full again, resize to capacity 4, copy 2 elements, data = [1,2,3,_], size=3.
  5. Insert 4: no resize needed, data = [1,2,3,4], size=4.
  6. Insert 5: resize to capacity 8, copy 4 elements, data = [1,2,3,4,5,_,_,_], size=5.
  7. Continue inserting 6...10 without resize, because capacity is 8 > 9 (index 9 = 10th element), size increments each time.

At the end, size = 10 with capacity = 16 (resizing next time insert past 8).

Summary

Dynamic arrays provide an elegant solution to the problem of storing growing collections with a blend of performance and simplicity:

  • They maintain a size and a capacity, resizing by doubling capacity when needed.
  • Allow constant-time random access owing to contiguous memory layout.
  • Amortized constant time for append operations, balancing occasional costly resizes with many cheap insertions.
  • Commonly used as the basis for many language standard libraries' sequence containers.

Understanding dynamic arrays is foundational to appreciating memory management and algorithmic efficiency in higher-level data structures.
