1. Overview — What is backpropagation?

Backpropagation is the algorithm used to compute gradients of a scalar loss with respect to the parameters (weights and biases) of a feedforward neural network. It is the key computational tool that enables training neural networks by gradient-based optimization (for example, stochastic gradient descent).

In short: backpropagation applies the chain rule of calculus repeatedly to propagate derivative information from the loss backward through the network, layer by layer.

2. Intuition

Imagine a factory with successive stations (layers). At the end, a quality inspector provides a single error number (the loss). Backpropagation asks: how should each station's settings be changed to reduce that error? To answer this, it computes how sensitive the final error is to each setting. This sensitivity is the gradient.

Key points:

  • Each layer's contribution to the final loss depends on later layers. Backpropagation uses the chain rule to connect these dependencies.
  • The algorithm computes partial derivatives efficiently by reusing intermediate results: the local gradients and the error signals from the layer above.
  • Backpropagation itself is purely a gradient computation. The optimization step (e.g., applying gradients to update parameters) is separate.

3. Fundamental mathematics: chain rule and composition

Most neural networks are compositions of simpler functions. If y = f(g(x)), then the derivative dy/dx = (df/dg) * (dg/dx). This is the chain rule. For deeper compositions y = f_n(f_{n-1}(...f_1(x)...)), we repeatedly apply the chain rule.

Notation (informal):

  • Let L be the loss, a scalar.
  • Let a^l denote activations (outputs) at layer l, and z^l denote pre-activations (the linear transform before applying nonlinearity): z^l = W^l a^{l-1} + b^l
  • The activation function is a^l = phi^l(z^l).

We want gradients: dL/dW^l and dL/db^l for all layers l. Using chain rule, we propagate an error signal δ^l = dL/dz^l (partial derivative of loss w.r.t. pre-activation at layer l).

Core backpropagation identities (vector form):
  • δ^L = dL/da^L ⊙ phi'^L(z^L) — error at output layer (elementwise multiply)
  • dL/dW^l = δ^l (a^{l-1})^T
  • dL/db^l = δ^l (sum over examples)
  • δ^{l-1} = (W^l)^T δ^l ⊙ phi'^{l-1}(z^{l-1})

Notes: ⊙ denotes elementwise (Hadamard) product. When using mini-batches, average or sum gradients appropriately depending on chosen convention.

4. Single neuron worked-out derivative (very explicit)

Consider a single neuron with weights w (vector), bias b, input x (vector), activation phi, and scalar output y = phi(z) with z = w^T x + b. Suppose the loss is L(y). How do we get dL/dw and dL/db?

Apply chain rule step-by-step:

  1. Compute dL/dy (depends on loss). For example, for squared error L = 0.5*(y - t)^2, dL/dy = (y - t).
  2. dL/dz = dL/dy * dy/dz = dL/dy * phi'(z).
  3. dL/dw = (dL/dz) * dz/dw = (dL/dz) * x (vector scaled by scalar dL/dz).
  4. dL/db = dL/dz * 1 = dL/dz.

Resulting gradients are simple and reusable for many layers.

5. Multilayer network: step-by-step derivation (textual)

Consider a feedforward network with L layers. For each layer l (1 ≤ l ≤ L):

  • z^l = W^l a^{l-1} + b^l
  • a^l = phi^l(z^l)

We compute the forward pass first (store all z^l and a^l). Then compute gradients backward:

  1. Compute δ^L = dL/da^L ⊙ phi'^L(z^L). This uses the derivative of the loss with respect to the network's output.
  2. For l = L down to 1:
    • dL/dW^l = δ^l (a^{l-1})^T
    • dL/db^l = δ^l (summed/averaged over examples)
    • If l > 1: δ^{l-1} = (W^l)^T δ^l ⊙ phi'^{l-1}(z^{l-1})

Each δ^l is the sensitivity of the loss to the linear inputs z^l. Once δ^l is known, parameter gradients follow directly.

Important: Never compute gradients by recomputing everything from scratch for each parameter. Backpropagation reuses δ^l and local derivatives for efficiency.

6. Pseudocode — vanilla backpropagation (layerwise)

Given: input x, target t, network parameters {W^l, b^l}, activations phi^l and derivatives phi'^l.

Forward pass:
  a^0 = x
  for l = 1 to L:
    z^l = W^l a^{l-1} + b^l
    a^l = phi^l(z^l)
Compute loss L = Loss(a^L, t)

Backward pass:
  // Output layer error
  delta^L = dLoss/da^L ⊙ phi'^L(z^L)

  for l = L downto 1:
    dW^l = delta^l (a^{l-1})^T
    db^l = delta^l  // possibly summed over batch
    if l > 1:
      delta^{l-1} = (W^l)^T delta^l ⊙ phi'^{l-1}(z^{l-1})

Update parameters (SGD step example):
  for l = 1 to L:
    W^l = W^l - learning_rate * dW^l
    b^l = b^l - learning_rate * db^l

This is the central loop used in virtually all deep learning frameworks. They add batching, numerical stability, and optimized operations for speed.

7. Worked numeric example (small network) — step-by-step textual calculation

We demonstrate backpropagation numerically for a tiny network: 2 inputs → 2 hidden neurons (sigmoid) → 1 output (identity) with mean squared error loss. All numbers are chosen small to show arithmetic.

Network definition:
  • Input x = [x1, x2] = [0.5, 0.1]
  • Hidden layer weights W1 (2x2) and biases b1 (2): W1 = [[0.2, -0.3], [0.4, 0.1]] b1 = [0.0, 0.1]
  • Output layer weights W2 (1x2) and bias b2: W2 = [0.3, -0.2], b2 = 0.05
  • Hidden activation phi = sigmoid(s) = 1/(1+exp(-s)), phi'(s) = phi(s)*(1-phi(s))
  • Output is y = z_out (identity). Loss L = 0.5*(y - t)^2 with target t = 0.6

Step 1: Forward pass — compute z1, a1, z2, y

Compute hidden pre-activations z1 = W1 * x + b1:

z1_1 = 0.2*0.5 + (-0.3)*0.1 + 0.0 = 0.1 - 0.03 = 0.07

z1_2 = 0.4*0.5 + 0.1*0.1 + 0.1 = 0.2 + 0.01 + 0.1 = 0.31

Hidden activations a1 = sigmoid(z1):

a1_1 = sigmoid(0.07) ≈ 0.5175

a1_2 = sigmoid(0.31) ≈ 0.5779

Output pre-activation z2 = W2 ⋅ a1 + b2 = 0.3*0.5175 + (-0.2)*0.5779 + 0.05

z2 ≈ 0.15525 - 0.11558 + 0.05 = 0.08967

Output y = z2 ≈ 0.08967

Loss L = 0.5*(y - t)^2 = 0.5*(0.08967 - 0.6)^2 ≈ 0.5*( -0.51033 )^2 ≈ 0.1304

Step 2: Backward pass — compute gradients

dL/dy = y - t = 0.08967 - 0.6 = -0.51033

Output activation is identity, so delta2 = dL/dz2 = dL/dy = -0.51033

Gradients for W2: dL/dW2 = delta2 * a1

dL/dW2_1 = -0.51033 * 0.5175 ≈ -0.2640

dL/dW2_2 = -0.51033 * 0.5779 ≈ -0.2949

dL/db2 = delta2 ≈ -0.51033

Propagate error to hidden: delta1 = (W2^T * delta2) ⊙ phi'(z1)

W2^T * delta2 = delta2 * W2^T = -0.51033 * [0.3, -0.2] = [-0.1531, 0.1021]

phi'(z) for sigmoid: phi*(1-phi)

phi'(z1_1) ≈ 0.5175*(1-0.5175) = 0.2498

phi'(z1_2) ≈ 0.5779*(1-0.5779) = 0.2442

delta1_1 = -0.1531 * 0.2498 ≈ -0.0382

delta1_2 = 0.1021 * 0.2442 ≈ 0.0249

Gradients for W1: dL/dW1 = delta1 ⊗ x (outer product)

dL/dW1 row1 = delta1_1 * x = -0.0382 * [0.5, 0.1] = [-0.0191, -0.0038]

dL/dW1 row2 = delta1_2 * x = 0.0249 * [0.5, 0.1] = [0.01245, 0.00249]

dL/db1 = delta1 = [-0.0382, 0.0249]

Summary (approx):

dW2 ≈ [-0.2640, -0.2949], db2 ≈ -0.5103
dW1 ≈ [[-0.0191, -0.0038], [ 0.01245, 0.00249]], db1 ≈ [-0.0382, 0.0249]

These numeric gradients would be multiplied by -learning_rate and subtracted from the weights for an SGD update.

This example shows how forward computation produces activations and pre-activations that are then reused in backward computation to produce gradients efficiently.

8. Activation functions and their derivatives (textual summary)

Choice of activation affects the derivative used in backpropagation. Some common activations:

  • Sigmoid: phi(s) = 1/(1+e^{-s}); phi'(s) = phi(s)*(1-phi(s)). Risk: saturates for large |s| → very small gradients.
  • Tanh: phi(s) = tanh(s); phi'(s) = 1 - tanh^2(s). Centered at zero (often helpful).
  • ReLU: phi(s) = max(0, s); phi'(s) = 1 for s>0, else 0. Simpler, avoids saturation for positive s but dead neurons can occur.
  • Leaky ReLU: derivative is small nonzero for s<0 (helps avoid dead ReLU).
  • Softmax (used with cross-entropy): outputs probabilities across classes; backprop through softmax combined with cross-entropy yields a simple delta = (y_pred - y_true).

9. Loss functions and how they interact with backpropagation

Loss determines dL/da^L which kickstarts backprop. Common choices:

  • Mean squared error (regression): L = 0.5 * ||y - t||^2 ⇒ dL/dy = y - t
  • Cross-entropy with softmax (classification): when combined, derivative simplifies to y_pred - one_hot_target
  • Binary cross-entropy with sigmoid: derivative uses y_pred and target in a stable formula.

Important: Choosing the appropriate loss-activation pair often simplifies gradients and improves numerical stability.

10. Implementation notes & vectorization

In real implementations you will:

  • Compute forward pass for a batch of examples and store intermediate z^l and a^l matrices.
  • Use matrix operations to compute all gradients for a layer in one go: dW^l = (1/m) * delta^l * (a^{l-1})^T for m examples.
  • Avoid explicit loops over neurons when possible; modern frameworks use optimized BLAS libraries or autograd systems.

Batching detail: when computing db^l, summing delta^l over the batch dimension then dividing by batch size gives the mean gradient for the bias.

11. Common pitfalls and practical tips

  • Vanishing gradients: very small gradients in early layers (common with deep networks and saturating activations like sigmoid). Mitigations: ReLU variants, batch normalization, careful initialization, residual connections.
  • Exploding gradients: very large gradients that blow up updates. Mitigations: gradient clipping, proper initialization, normalization, or using stable architectures (e.g., LSTM for RNNs).
  • Numerical stability: use log-sum-exp and stable forms for softmax + cross-entropy.
  • Initialization: scale weights to keep signal variance stable (e.g., Xavier/Glorot, He initialization depending on activation).
  • Learning rate: a critical hyperparameter. Too large → divergence; too small → slow convergence.
  • Compute gradients once: frameworks compute gradients efficiently; avoid recomputing forward passes unnecessarily.

12. Variations and extensions

Backpropagation as described applies to feedforward networks. Variations/extensions include:

  • Backprop through time (BPTT): applies chain rule over time steps for recurrent networks.
  • Backprop through computational graphs: any differentiable program can be differentiated by applying the chain rule through its operations (this is the basis of autodiff).
  • Second-order methods: compute or approximate second derivatives (Hessian) for optimization (rare in large-scale deep learning).
  • Layer-wise alternatives: layerwise training, biologically motivated variants, or local learning rules (less common in practice).

14. Debugging gradient computations

Popular debugging techniques:

  • Gradient checking (finite differences): numerically approximate dL/dθ ≈ (L(θ + ε) - L(θ - ε)) / (2ε) for small ε. Compare with backprop results (use only for small models due to cost).
  • Check magnitudes: ensure gradients are not NaN or infinite. Print gradient norms during training.
  • Sanity check with tiny data: train on a tiny dataset (even 1 example) and verify loss decreases to near zero.
  • Inspect activations and pre-activations: ensure they are within reasonable ranges (not all zeros or saturating extremes).

15. Frequently asked conceptual questions

Q: Is backpropagation the same as training?

A: No. Backpropagation computes gradients. Training uses those gradients with an optimizer (SGD, Adam, etc.) to update parameters.

Q: Why is the chain rule enough?

A: Neural networks are compositions of differentiable functions. The chain rule provides an exact formula for derivatives of compositions, so applying it repeatedly yields correct gradients.

Q: Is backprop biologically plausible?

A: Standard backprop uses non-local information (weights of later layers, symmetric weight transport), which many argue is not biologically plausible. Research continues into more biologically plausible learning rules.

16. Short glossary

  • Activation (a): output of a neuron after nonlinearity.
  • Pre-activation (z): linear transform W a_prev + b before activation.
  • Delta (δ): error signal dL/dz at a layer.
  • Gradient: partial derivative of loss with respect to a parameter.
  • Learning rate: scalar controlling step size in parameter updates.

17. Final notes and best practices

  • Always store forward-pass intermediates you will need in backward pass (z^l, a^l).
  • Vectorize operations over batches for efficiency.
  • Use numerically stable formulas for combined operations (e.g., softmax + cross-entropy).
  • Use appropriate initialization and normalization to keep gradients healthy.
