Binary Search Tree (BST) - Comprehensive Guide

Introduction to Binary Search Trees

A Binary Search Tree (BST) is a fundamental data structure in computer science that provides efficient operations for storing, searching, and retrieving data. It combines the benefits of ordered arrays (fast searching) and linked lists (flexible insertion/deletion).

Formal Definition

A Binary Search Tree is a binary tree data structure where each node has at most two children (left and right) and satisfies the following properties:

  1. For any given node:
    • All values in the left subtree are less than the node's value
    • All values in the right subtree are greater than the node's value
  2. There are no duplicate nodes (though some implementations may allow duplicates in specific ways)

BSTs are particularly useful because they maintain their elements in sorted order, which enables efficient search operations that can be performed in O(log n) time in balanced trees.

BST Properties and Characteristics

Key Properties

  • Ordering Property: The left subtree of a node contains only nodes with values less than the node's value, and the right subtree contains only nodes with values greater than the node's value.
  • Unique Values: Typically, BSTs don't contain duplicate values (though some implementations may place duplicates in the right subtree).
  • Recursive Structure: Each subtree is itself a valid BST.

Terminology

  • Root: The topmost node in the tree
  • Leaf: A node with no children
  • Internal Node: A node with at least one child
  • Height: The number of edges on the longest path from the node to a leaf
  • Depth: The number of edges from the root to the node

Example BST

Consider the following BST with integer values:

8 / \ 3 10 / \ \ 1 6 14 / \ / 4 7 13

This tree satisfies all BST properties:

  • All left children are less than their parent (e.g., 3 < 8, 1 < 3, 6 > 3 but 4 < 6)
  • All right children are greater than their parent (e.g., 10 > 8, 14 > 10, 13 < 14)
  • Each subtree is itself a valid BST

BST Operations

Search Operation

Searching in a BST follows a simple recursive process:

  1. Start at the root node
  2. Compare the target value with the current node's value:
    • If equal, return the node (value found)
    • If less, search the left subtree
    • If greater, search the right subtree
  3. If you reach a null reference, the value is not in the tree

Search Example

Searching for value 6 in our example tree:

  1. Start at root (8) → 6 < 8 → go left to 3
  2. At node 3 → 6 > 3 → go right to 6
  3. At node 6 → found the value

This took 3 comparisons (O(h) time where h is tree height).

Insertion Operation

Insertion follows a similar process to search, but adds the new node where the search would terminate:

  1. Start at the root node
  2. Compare the new value with the current node's value:
    • If less, go to the left child
    • If greater, go to the right child
  3. When you reach a null child pointer, insert the new node there

Insertion Example

Inserting value 5 into our example tree:

  1. Start at root (8) → 5 < 8 → go left to 3
  2. At node 3 → 5 > 3 → go right to 6
  3. At node 6 → 5 < 6 → go left to 4
  4. At node 4 → 5 > 4 → go right (null) → insert 5 as right child of 4

The new tree would have 5 as the right child of 4.

Deletion Operation

Deletion is more complex, with three cases to consider:

  1. Node is a leaf: Simply remove it
  2. Node has one child: Replace the node with its child
  3. Node has two children:
    • Find the node's in-order successor (smallest in right subtree) or predecessor (largest in left subtree)
    • Copy the successor's value to the node
    • Recursively delete the successor

Deletion Examples

Case 1: Deleting 1 (a leaf) - simply remove it

Case 2: Deleting 14 (has one child) - replace with its child 13

Case 3: Deleting 3 (has two children):

  1. Find in-order successor (4) or predecessor (1)
  2. Let's choose successor 4
  3. Copy 4 to where 3 was
  4. Now delete the original 4 (which is now a Case 1 or 2 deletion)

BST Traversals

Tree traversal refers to visiting all nodes in the tree in a particular order. BSTs support several standard traversal methods:

In-order Traversal

Visits nodes in ascending order (left-root-right):

  1. Traverse the left subtree
  2. Visit the root
  3. Traverse the right subtree

For our example tree: 1, 3, 4, 6, 7, 8, 10, 13, 14

Pre-order Traversal

Visits root before children (root-left-right):

  1. Visit the root
  2. Traverse the left subtree
  3. Traverse the right subtree

For our example tree: 8, 3, 1, 6, 4, 7, 10, 14, 13

Post-order Traversal

Visits root after children (left-right-root):

  1. Traverse the left subtree
  2. Traverse the right subtree
  3. Visit the root

For our example tree: 1, 4, 7, 6, 3, 13, 14, 10, 8

Level-order Traversal (Breadth-first)

Visits nodes level by level from top to bottom:

  1. Visit the root
  2. Visit all nodes at depth 1 (left to right)
  3. Visit all nodes at depth 2 (left to right)
  4. Continue until all depths are visited

For our example tree: 8, 3, 10, 1, 6, 14, 4, 7, 13

Time Complexity Analysis

The efficiency of BST operations depends on the tree's height:

Operation Average Case (Balanced BST) Worst Case (Skewed BST)
Search O(log n) O(n)
Insertion O(log n) O(n)
Deletion O(log n) O(n)
Traversal O(n) (must visit all nodes)

The worst case occurs when the tree becomes a linked list (completely unbalanced). This can happen if elements are inserted in sorted order. To maintain O(log n) performance, self-balancing BST variants (like AVL trees, Red-Black trees) are used.

BST Implementation (Pseudocode)

Node Structure

class Node {
    value
    leftChild
    rightChild
    
    constructor(value) {
        this.value = value
        this.leftChild = null
        this.rightChild = null
    }
}

Search Operation

function search(root, target) {
    if root is null or root.value == target
        return root
    
    if target < root.value
        return search(root.leftChild, target)
    else
        return search(root.rightChild, target)
}

Insertion Operation

function insert(root, value) {
    if root is null
        return new Node(value)
    
    if value < root.value
        root.leftChild = insert(root.leftChild, value)
    else if value > root.value
        root.rightChild = insert(root.rightChild, value)
    
    return root
}

Deletion Operation

function deleteNode(root, value) {
    if root is null
        return root
    
    if value < root.value
        root.leftChild = deleteNode(root.leftChild, value)
    else if value > root.value
        root.rightChild = deleteNode(root.rightChild, value)
    else
        // Node with only one child or no child
        if root.leftChild is null
            return root.rightChild
        else if root.rightChild is null
            return root.leftChild
        
        // Node with two children
        root.value = minValue(root.rightChild)
        root.rightChild = deleteNode(root.rightChild, root.value)
    
    return root
}

function minValue(node) {
    let current = node
    while current.leftChild is not null
        current = current.leftChild
    return current.value
}

BST Variations and Advanced Concepts

Self-Balancing BSTs

To maintain O(log n) performance, self-balancing BSTs automatically adjust their structure:

  • AVL Trees: Maintain balance factor (height difference between subtrees) of at most 1
  • Red-Black Trees: Use color coding and rotation rules to maintain balance
  • Splay Trees: Recently accessed elements move to the root

Threaded BSTs

Store in-order predecessor/successor information in null pointers to allow efficient in-order traversal without recursion or stack.

Applications of BSTs

  • Database indexing
  • Filesystem directories
  • Routing tables in networking
  • Implementation of ordered maps and sets
  • Auto-completion and suggestion systems
