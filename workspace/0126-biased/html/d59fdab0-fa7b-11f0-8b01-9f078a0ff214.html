Dynamic Arrays

A dynamic array is an array that can change in size during runtime. Unlike static arrays, whose size must be defined at compile time, dynamic arrays can grow and shrink as needed.

Overview

In programming, data structures like arrays are vital tools for storing collections of items. A static array has a fixed size defined when it is created. However, dynamic arrays allocate memory during execution, making them much more flexible and useful for scenarios where the quantity of data isn't known upfront.

Characteristics of Dynamic Arrays

  • Resizing: Dynamic arrays can resize as elements are added or removed.
  • Random Access: Similar to static arrays, dynamic arrays allow random access, meaning any element can be accessed in constant time, O(1).
  • Amortized Cost: While resizing costs O(n) (where n is the number of elements), the average cost of appending an element is O(1) due to amortization.
  • Storage: They require additional memory for storing metadata about the array size and capacity.

How Dynamic Arrays Work

The implementation of a dynamic array typically involves two key components: the array itself, and a management system for its size and capacity.

Basic Operations

1. Initialization

Initially, a dynamic array is created with a starting capacity, which can be a small fixed number (e.g., 4). This defines the size of the underlying storage array.

2. Insertion

When an element is added, the algorithm checks if there is enough capacity.

  • If there is enough capacity, the element is simply inserted.
  • If not, a new larger array (usually double the size of the current array) is created, and the existing elements are copied over. The new element is then added.

3. Deletion

When an element is removed, the remaining elements are shifted (if necessary), and if the array becomes less than a certain threshold of its capacity (e.g., 25% used), it may also shrink the size of the array.

Example

        // Pseudocode for adding an element to a dynamic array
        function addElement(dynamicArray, element):
            if dynamicArray.size == dynamicArray.capacity:
                // double the capacity
                newCapacity = dynamicArray.capacity * 2
                newArray = new Array(newCapacity)
                
                // copy old elements to new array
                for i from 0 to dynamicArray.size - 1:
                    newArray[i] = dynamicArray.array[i]
                
                dynamicArray.array = newArray
                dynamicArray.capacity = newCapacity
            
            // add new element
            dynamicArray.array[dynamicArray.size] = element
            dynamicArray.size += 1

Advantages and Disadvantages

Advantages

  • Flexibility in size management reduces the need for knowing the maximum size in advance.
  • Efficient memory usage as it can grow and shrink.

Disadvantages

  • Overhead of memory allocation and copying when resizing.
  • Potential performance hit during resizing operations.

Conclusion

Dynamic arrays represent a powerful data structure that combines the benefits of arrays with the flexibility of dynamic memory allocation. They are widely used in programming languages and libraries, providing support for array-like data management while allowing for scalability.
