Concepts, theory, algorithms, step-by-step examples, pseudocode, and practical advice for classification and regression trees.

Overview: What is a Decision Tree?

A decision tree is a predictive model that maps observations about an item (represented by features) to conclusions about the item's target value (class label or numeric value). It is a tree-structured model where:

  • Internal nodes represent tests on features (e.g., "Humidity > 75%?" or "Outlook = Sunny?").
  • Branches represent outcomes of tests.
  • Leaves (terminal nodes) represent predicted values (class labels for classification, numeric values for regression) or distributions over labels.

Decision trees are intuitive, interpretable, and form the basis for many ensemble methods (random forests, gradient boosting).

Core Concepts and Terminology

  • Root node: the top-most node that contains the entire dataset.
  • Internal node: tests on a feature that partition data.
  • Leaf node: provides a prediction. For classification it may give a class or probability distribution; for regression it gives a numeric prediction (mean or median of target values in that leaf).
  • Depth: number of edges on the longest path from root to a leaf; also called tree height.
  • Splitting criterion: measure used to decide the best feature/split at each node (e.g., information gain, Gini impurity, variance reduction).
  • Pruning: reducing the size of a fully grown tree to avoid overfitting.

Why trees?

  • Interpretable: rules correspond to human-readable decisions.
  • No need to scale features for many implementations.
  • Can handle heterogeneous features (categorical and continuous).
  • Nonparametric: no assumption of linearity or fixed functional form.

Limitations

  • Prone to overfitting if grown deep without constraints.
  • Small changes in data can produce different trees (high variance).
  • Greedy top-down splits may not find global optimum tree.

Classification uses entropy or Gini to select splits.

Impurity and Split Criteria (Mathematical Definitions)

The goal at each node is to select a split that makes child nodes "purer" than the parent. Common measures:

Entropy (information gain)

For a node with classes c in C and probabilities p(c), entropy is:

Entropy = - Σ_c p(c) * log2(p(c))

Information gain for choosing a split is:

Gain = Entropy(parent) - Σ_k (|D_k| / |D_parent|) * Entropy(child_k)

Gini impurity

Gini = 1 - Σ_c p(c)^2

Gini represents the probability of incorrectly labeling a randomly chosen element if it was randomly labeled according to class distribution in the node. Used by CART for classification.

Variance reduction (regression)

Variance = (1 / |D|) * Σ_i (y_i - mean(y))^2

Reduction in variance (or decrease in mean squared error) is used as splitting criterion for regression trees.

Gain ratio (C4.5)

Information gain is biased toward attributes with many values. Gain ratio compensates by dividing gain by split information:

GainRatio = Gain / SplitInfo
SplitInfo = - Σ_k (|D_k| / |D_parent|) * log2(|D_k| / |D_parent|)

Which metric to use? For classification, Gini and entropy typically yield similar trees. CART (Gini) produces binary splits; ID3/C4.5 may produce multi-way splits on categorical features.

Classic Example: "Play Tennis" (ID3)

We use a frequently cited small example that helps explain information gain calculations. Target: Play Tennis (Yes/No). The standard dataset has 14 examples with outcomes: Yes = 9, No = 5.

Entropy of entire dataset:

Entropy(parent) = - (9/14) * log2(9/14) - (5/14) * log2(5/14) ≈ 0.940

Consider attribute "Outlook" with three values: Sunny (5 examples: 2 Yes, 3 No), Overcast (4 examples: 4 Yes, 0 No), Rain (5 examples: 3 Yes, 2 No).

Child entropies: 

Entropy(Sunny) = - (2/5) log2(2/5) - (3/5) log2(3/5) ≈ 0.971
Entropy(Overcast) = - (4/4) log2(4/4) = 0
Entropy(Rain) = - (3/5) log2(3/5) - (2/5) log2(2/5) ≈ 0.971
Weighted entropy after splitting on Outlook: 
Weighted = (5/14)*0.971 + (4/14)*0 + (5/14)*0.971 = (10/14)*0.971 ≈ 0.694
Information gain: 
Gain(Outlook) = 0.940 - 0.694 = 0.246

Similar calculations are performed for other attributes; the attribute with largest information gain is selected for the root split.

Step-by-Step Building Process (Greedy Top-Down)

  1. Start with all training data at the root.
  2. For each candidate feature (and for continuous features, candidate thresholds), compute the impurity measure (entropy, Gini, variance) for the resulting partitions and compute impurity decrease (gain).
  3. Select the feature/split that maximizes impurity decrease (or gain ratio in C4.5).
  4. Partition the data into child nodes and recurse: repeat the process independently for each child node.
  5. Stopping criteria may include: all samples in node have same label, maximum depth reached, minimum number of samples in a node, or impurity decrease below a threshold.
  6. If needed, prune the fully grown tree to reduce overfitting (see pruning section).

Choosing thresholds for continuous features

Common approach:

  • Sort unique values of the continuous feature.
  • Consider candidate thresholds as midpoints between consecutive unique values where the class label changes.
  • Evaluate impurity decrease for each candidate threshold and choose best.

Handling categorical features

For a categorical attribute with k values you may:

  • Create a multi-way split that creates k child nodes (ID3/C4.5 support this).
  • For binary-split algorithms (CART), evaluate possible groupings of categorical values into two sets. For many categories this is combinatorial; heuristics or ordering by target statistics are used.

Algorithms and Pseudocode

ID3 (basic)

function ID3(dataset, features):
    if all examples have same label:
        return leaf with that label
    if features is empty:
        return leaf with majority label
    best = feature that maximizes InformationGain(dataset, feature)
    tree = new node labeled best
    for each value v of best:
        subset = examples where best == v
        if subset is empty:
            attach leaf with majority label of dataset
        else:
            attach subtree = ID3(subset, features - {best})
    return tree

C4.5 (improvements over ID3)

  • Uses GainRatio instead of plain information gain (to reduce bias toward many-valued attributes).
  • Can handle continuous attributes by selecting thresholds.
  • Handles missing values by using fractional counts or distributing instances probabilistically.
  • Prunes the tree (post-pruning) using estimated error rates.

CART (Classification And Regression Trees)

  • Produces binary trees (splits into two children) for both classification and regression.
  • Classification: uses Gini impurity as split criterion (commonly).
  • Regression: uses reduction in mean squared error (variance reduction).
  • Often uses cost-complexity (α) pruning to select simplified subtree.

CART Pseudocode (recursive)

function CART(dataset, features):
    if stopping_condition(dataset):
        return leaf with prediction (majority class or mean)
    for each feature f in features:
        for each candidate binary split s on f:
            compute impurity decrease ΔI(s)
    choose split s* with maximum ΔI
    left, right = partition(dataset, s*)
    node.left = CART(left, features)
    node.right = CART(right, features)
    return node

Overfitting, Pruning, and Generalization

Trees grown to pure leaves (zero training error) often overfit the training data. Pruning reduces complexity to improve generalization.

Pre-pruning (early stopping)

  • Stop splitting when the node has fewer than min_samples_split examples.
  • Limit tree depth (max_depth).
  • Require minimum impurity decrease to split.

Post-pruning (prune after full growth)

  • Reduced error pruning: replace a subtree with a leaf if performance on validation set does not degrade.
  • Cost-complexity pruning (CART): define cost(T) = R(T) + α * |T| where R(T) is error on training set (or estimated error), |T| is number of leaves. Choose α via cross-validation; prune subtrees that reduce cost.

Bias-Variance tradeoff

Shallow trees have high bias (underfit) and low variance. Deep trees have low bias (fit training data well) but high variance (sensitive to noise). Pruning aims to balance bias and variance.

Handling Practical Issues

Missing values

  • Imputation prior to training (mean/mode, or model-based imputation).
  • Assign instances with missing attribute to child nodes using fractional instance counts (C4.5 approach).
  • Use surrogate splits: find alternative features that split data similarly and use them when the primary split feature is missing.

Categorical features with many levels

  • High-cardinality categories can bias information gain. Consider grouping infrequent categories into "other", target encoding, or using gain ratio.

Imbalanced classes

  • Use class weights to penalize misclassification of minority class more heavily.
  • Use sampling strategies: oversample minority class or undersample majority class.

Feature preprocessing

Decision trees do not require feature scaling. Encoding categorical variables (one-hot, ordinal, target encoding) should be chosen carefully depending on algorithm specifics.

Complexity, Scalability, and Implementation Notes

Typical time complexity for greedy tree building depends on implementation details and whether continuous features are present.

  • Naive approach: for each node, for each feature, evaluate all possible splits. If N is number of samples and M features, and we examine F candidate splits per feature, complexity per node is O(M * F). Building tree on all nodes leads to roughly O(M * N * log N) when sorting continuous features once per node is used (practical heuristics reduce repeated sorting).
  • Space complexity: O(N) to store data; additional memory for recursion and node structures.
  • For large datasets: use approximate splitting (histograms), sampling, or distributed implementations (e.g., in Spark) to scale.

Modern libraries (scikit-learn, XGBoost, LightGBM) include optimized implementations with performance improvements: vectorized operations, efficient sorting, histogram-based splitting, and parallelization.

Interpreting Trees and Feature Importance

Interpretation is a major strength of decision trees. A path from root to leaf forms a rule (e.g., "If Outlook = Sunny and Humidity > 75% then Play = No").

Feature importance

One common measure: total impurity decrease contributed by all splits on the feature, averaged (or normalized). This gives a global importance score but may be biased in presence of correlated features.

For local explanation of a prediction, read the path the instance follows from root to leaf: which tests were decisive, which feature thresholds were crossed.

Related Methods: Ensembles

Decision trees are often used as base learners in ensemble algorithms that reduce variance or bias:

  • Random Forests: build many decision trees on bootstrap samples and random subsets of features, then average (regression) or majority vote (classification). This reduces variance dramatically.
  • Gradient Boosting Machines (GBM): sequentially add trees that correct errors of previous ensemble (e.g., XGBoost, LightGBM). Trees are usually shallow (weak learners) and combined additively.
  • Bagging: bootstrap aggregating of trees to improve stability and accuracy.

When to Use Decision Trees

  • When interpretability is important (business rules, regulatory scenarios).
  • When you have mixed feature types and interactions that are nonlinear.
  • As a baseline model: quick to train and easy to explain.
  • Avoid relying on a single deep tree for production-critical predictive accuracy; prefer ensembles when accuracy matters more than interpretability.

Practical Hyperparameters (Common in Libraries)

  • criterion: "gini" or "entropy" (classification) / "mse" or "mae" (regression).
  • max_depth: maximum depth of the tree.
  • min_samples_split: minimum samples required to split an internal node.
  • min_samples_leaf: minimum samples required to be at a leaf.
  • max_features: number of features to consider when looking for best split.
  • class_weight: balanced weighting to cope with class imbalance.
  • ccp_alpha (cost complexity pruning alpha): for pruning in CART implementations.

A Worked Classification Example (Small, Step-by-step)

Consider a tiny synthetic dataset for a binary classification problem:

Dataset (6 examples):
ID | Temperature (°C) | Windy | Play?
---+------------------+-------+------
1  | 30               | False | No
2  | 25               | False | Yes
3  | 28               | True  | No
4  | 22               | False | Yes
5  | 18               | True  | Yes
6  | 16               | True  | No

We will show conceptually how one might build a tree with a single continuous feature "Temperature" and boolean "Windy".

  1. Root entropy: compute class proportions: Yes = 3, No = 3 → entropy = 1.0 (maximal uncertainty).
  2. Evaluate splitting on "Windy" (binary):
    • Windy = False: examples {1(No),2(Yes),4(Yes)} → Yes=2,No=1 → entropy ≈ 0.918
    • Windy = True: examples {3(No),5(Yes),6(No)} → Yes=1,No=2 → entropy ≈ 0.918
    • Weighted entropy = 0.918 → gain = 1.0 - 0.918 = 0.082
  3. Evaluate splits for "Temperature": sort values and consider thresholds between sorted unique temps: (16,18,22,25,28,30) → candidate thresholds (17,20,23.5,26.5,29).
    • Try threshold 23.5: Temperature ≤ 23.5 → examples {4(Yes),5(Yes),6(No)} → Yes=2,No=1 → entropy=0.918
    • Temperature > 23.5 → {1(No),2(Yes),3(No)} → Yes=1,No=2 → entropy=0.918
    • Weighted = 0.918, gain = 0.082 ≈ same as Windy for this split.
  4. We pick one split (tie-breaking by convention). Suppose we split on Windy. Both children have mixed labels, so we may continue splitting until stopping criteria satisfied (e.g., min_samples_leaf = 1 or max_depth). Further splits can use Temperature thresholds within each child.

This small example illustrates how candidate splits are evaluated and chosen by impurity decrease. In real datasets, calculations are similar but on larger scales.

Checklist: Building and Using Decision Trees

  • Understand your features: continuous vs categorical; handle encoding appropriately.
  • Choose a split criterion appropriate to problem (entropy/Gini for classification, variance for regression).
  • Decide pruning or early stopping strategy to control overfitting.
  • Use cross-validation to tune hyperparameters (max_depth, min_samples_leaf, etc.).
  • Interpret tree paths for domain insights; use feature importance to summarize predictors.
  • When in doubt about stability or accuracy, consider ensembles (random forests, boosting).
