Overview & Definition

A linked list is a linear data structure in which elements are stored in nodes, and each node contains a reference (often called a pointer) to the next node in the sequence. Unlike arrays, linked lists do not require contiguous memory and support efficient insertions and deletions at known node positions.

Key ideas (high level)

  • Each element is wrapped in a node that stores the element's value and a pointer/reference to another node.
  • The list is accessed via a reference to the first node, commonly called head. An empty list has head = null (or equivalent).
  • Traversal requires following pointers from one node to the next until you reach null (end) or when the list loops (in circular lists).

Linked lists are fundamental building blocks for many higher-level data structures (stacks, queues, adjacency lists for graphs) and are used when dynamic memory allocation and fast insert/remove operations are required.

Node Structure & Memory

Conceptually, a node holds two pieces of information:

  1. Data (payload): The value or object you want to store (e.g., integer, string, struct).
  2. Pointer/Reference: A reference to another node (often called next). In doubly linked lists, a previous pointer also exists.

Memory layout explanation

Imagine the heap as a collection of individually allocated blocks. When you create a node, you allocate one block and store the value and pointer inside that block. The pointer contains the memory address (or runtime reference) of another node, not the contents of that other node.

Example (singly linked list nodes): 
[ Node A ] -> [ Node B ] -> [ Node C ] -> null
  value       value       value
  next ------> next ------> next

Because nodes can be placed anywhere in memory, linked lists avoid the cost of shifting elements on insertion/deletion, at the cost of extra memory (pointers) and less cache-friendly access patterns.

Types of Linked Lists

  • Singly linked list: Each node has data and a single next pointer. Traversal is forward only.
  • Doubly linked list: Nodes have next and prev pointers allowing traversal in both directions. Slightly more memory, easier deletions when you have the node reference.
  • Circular linked list: The last node's next pointer points back to the head (or some node), forming a loop. Can be singly or doubly circular.
  • Sentinel (dummy) node: A special node used as head or tail to simplify boundary case handling.
  • Skip list: A probabilistic multilayered linked structure built on top of linked lists to support faster searches (logarithmic average).

Basic Operations — Step-by-step Descriptions

Traversal (visit every node)

Goal: Visit each node once and possibly inspect or process its value.

Step-by-step:

  1. Set current = head.
  2. While current != null:
    1. Process current.value (print, compute, etc.).
    2. Set current = current.next.
  3. When current becomes null, traversal is complete.

If the list is circular, ensure you stop after a full cycle (for example, stop when you've returned to the starting node).

Insert at head (prepend)

Goal: Add a new value at the beginning of the list.

Step-by-step:

  1. Create newNode with the data value.
  2. Set newNode.next = head.
  3. Set head = newNode.

Edge cases: if the list was empty (head was null), newNode.next remains null. For circular lists, set next appropriately.

Insert at tail (append)

Goal: Add a new value at the end of the list.

Step-by-step (no tail pointer):

  1. Create newNode with the data value and newNode.next = null.
  2. If head == null, set head = newNode (empty list).
  3. Set current.next = newNode.

If you maintain a tail pointer, you can do append in O(1) by setting tail.next = newNode and updating tail.

Insert at position (index)

Goal: Insert at a specified position/index (0-based).

Step-by-step:

  1. If index == 0, perform insert-at-head.
  2. Else, set current = head and move current forward index-1 times (or until null). Keep track of position count.
  3. If current is null before reaching index-1, the index is out of bounds (handle error or append at end depending on policy).
  4. Create newNode, set newNode.next = current.next, then current.next = newNode.

Delete by value

Goal: Remove the first node that matches a target value.

Step-by-step:

  1. If head == null, nothing to delete.
  2. If head.value == target, set head = head.next (node is removed by losing reference to it).
  3. Else, use previous and current pointers: prev = head; current = head.next; while current != null and current.value != target: prev = current; current = current.next.
  4. If current != null (found), set prev.next = current.next to unlink current.

For doubly linked lists, you can update prev pointers too; deletion when you have direct node reference is O(1) by adjusting neighbors' links.

Search (find element)

Goal: Determine whether a value exists and possibly return its node or index.

Step-by-step:

  1. Set current = head, index = 0.
  2. While current != null:
    1. If current.value == target, return (current, index).
    2. Else current = current.next; index++.
  3. If loop finishes without finding target, return not found.
Note: Most singly linked list operations that require locating a node by index or value are O(n) because they involve traversal. Operations that operate at the head (insert/remove) can be O(1).

Pseudocode examples

FUNCTION traverse(head):
    current = head
    WHILE current != null:
        visit(current.value)
        current = current.next

FUNCTION insertHead(head, value):
    newNode = Node(value)
    newNode.next = head
    head = newNode
    RETURN head

FUNCTION deleteValue(head, target):
    IF head == null: RETURN head
    IF head.value == target:
        temp = head
        head = head.next
        free(temp)   # conceptual
        RETURN head
    prev = head
    current = head.next
    WHILE current != null and current.value != target:
        prev = current
        current = current.next
    IF current != null:
        prev.next = current.next
        free(current)
    RETURN head

Code Samples

C (singly linked list basic operations)

#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int value;
    struct Node *next;
} Node;

Node* insert_head(Node *head, int val) {
    Node *n = malloc(sizeof(Node));
    n->value = val;
    n->next = head;
    return n; // new head
}

Node* delete_value(Node *head, int target) {
    if (!head) return NULL;
    if (head->value == target) {
        Node *next = head->next;
        free(head);
        return next;
    }
    Node *prev = head;
    Node *cur = head->next;
    while (cur && cur->value != target) {
        prev = cur; cur = cur->next;
    }
    if (cur) {
        prev->next = cur->next;
        free(cur);
    }
    return head;
}

Java (singly linked list node and traversal)

class Node {
    int value;
    Node next;
    Node(int v) { value = v; next = null; }
}

void traverse(Node head) {
    Node cur = head;
    while (cur != null) {
        System.out.println(cur.value);
        cur = cur.next;
    }
}

Python (simple, idiomatic singly linked list)

class Node:
    def __init__(self, value, next=None):
        self.value = value
        self.next = next

def insert_head(head, value):
    return Node(value, next=head)

def traverse(head):
    cur = head
    while cur:
        print(cur.value)
        cur = cur.next

Variants & Advanced Topics

Doubly linked lists

Nodes contain next and prev pointers. Deleting a node when you have its direct reference takes constant time because you can update its neighbors directly.

XOR linked list

An XOR linked list stores the XOR of next and previous pointers in one field to save space. It is more memory efficient for pointer storage but difficult to implement in safe, managed languages and complicates debugging and serialization.

Skip lists

Skip lists are layered linked lists with probabilistic "express lanes" that let you skip ahead, offering O(log n) expected search, insertion, and deletion.

Persistent (immutable) linked lists

In functional programming, lists are often immutable. Prepending is O(1) because a new node can share the tail with the old list. This avoids mutation and enables safe sharing of structure across versions.

Common Pitfalls & Practical Tips

  • Null pointer errors: Always handle empty list (head is null) and single-element list as special cases or use sentinel nodes to simplify code.
  • Memory leaks: In manual memory languages (C/C++), ensure you free removed nodes and avoid double frees.
  • Lost references: When deleting, update references carefully. For example, not keeping a handle on next before freeing the current node loses the rest of the list.
  • Use tail pointer when necessary: If you need frequent appends, maintain a tail pointer to achieve O(1) appends.
  • Avoid unnecessary traversals: Combine operations when possible rather than traversing multiple times.
  • Be mindful of cache misses: Linked lists are pointer-heavy and less cache-friendly than arrays; for tight loops and performance-critical code, arrays or contiguous vectors may be faster.

Exercises & Learning Tasks

  1. Implement a singly linked list in your preferred language with insert, delete, search, and traverse operations.
  2. Modify your implementation to add a tail pointer for O(1) appends; measure or reason about differences.
  3. Implement reverse(list) that reverses the list in-place. Provide a step-by-step explanation of pointer changes.
  4. Implement a function to detect a cycle in a linked list (Floyd's Tortoise and Hare). Explain why the algorithm works and its complexity.
  5. Implement merge of two sorted linked lists into a single sorted linked list without allocating extra nodes.

Hints (brief)

  • Reverse in-place: iterate with three pointers: prev, current, next. At each step, set current.next = prev, then advance all pointers.
  • Cycle detection: use two pointers moving at different speeds. If they meet, a cycle exists.
