Heap (Min) - Comprehensive Guide

Introduction to Min-Heaps

A min-heap is a specialized tree-based data structure that satisfies the heap property: in a min-heap, for any given node C, if P is a parent node of C, then the key (the value) of P is less than or equal to the key of C. This means the smallest element is always at the root of the heap.

Key Property: In a min-heap, the parent node is always less than or equal to its child nodes.

Min-heaps are a type of binary heap, which is a complete binary tree (all levels are completely filled except possibly the last level, which is filled from left to right).

Heap Representation

Heaps are typically represented as arrays. For a node at index i:

  • Parent index: floor((i-1)/2)
  • Left child index: 2i + 1
  • Right child index: 2i + 2

Example: Min-Heap Representation

Consider the following min-heap:

                      1
                    /   \
                   3     6
                  / \   /
                 5   9 8

Array representation: [1, 3, 6, 5, 9, 8]

Index 0 1 2 3 4 5
Value 1 3 6 5 9 8

Basic Operations

1. Insertion (Heapify Up)

To insert a new element into the heap:

  1. Add the new element to the end of the array (bottom-right of the tree)
  2. Compare the added element with its parent
  3. If the parent is larger, swap them
  4. Continue this process until the heap property is satisfied
function insert(heap, value):
    heap.append(value)          // Add to end
    i = len(heap) - 1           // Current index
    while i > 0:
        parent = floor((i-1)/2) // Parent index
        if heap[parent] <= heap[i]:
            break               // Heap property satisfied
        swap(heap[parent], heap[i])
        i = parent

2. Extract Minimum (Heapify Down)

To remove and return the minimum element (root):

  1. Remove the root element (minimum value)
  2. Move the last element to the root position
  3. Compare the new root with its children
  4. If it's larger than either child, swap with the smaller child
  5. Continue this process until the heap property is satisfied
function extractMin(heap):
    if heap is empty:
        return null
    min = heap[0]               // Store minimum
    heap[0] = heap.pop()        // Move last to root
    i = 0                       // Current index
    while true:
        left = 2*i + 1          // Left child
        right = 2*i + 2         // Right child
        smallest = i
        if left < len(heap) and heap[left] < heap[smallest]:
            smallest = left
        if right < len(heap) and heap[right] < heap[smallest]:
            smallest = right
        if smallest == i:
            break               // Heap property satisfied
        swap(heap[i], heap[smallest])
        i = smallest
    return min

Time Complexity Analysis

Operation Time Complexity Description
Insertion O(log n) May need to bubble up the new element to maintain heap property
Extract Min O(log n) May need to bubble down the new root to maintain heap property
Find Min O(1) Minimum is always at the root
Build Heap O(n) Can be done by heapifying all non-leaf nodes from bottom up

Applications of Min-Heaps

  • Priority Queues: Min-heaps efficiently support operations needed for priority queues
  • Heap Sort: Can be used to sort elements in O(n log n) time
  • Graph Algorithms: Used in Dijkstra's algorithm and Prim's algorithm
  • Kth Smallest Element: Can find the kth smallest element efficiently
  • Merge Sorted Lists: Efficiently merge multiple sorted lists

Min-Heap vs Max-Heap

Feature Min-Heap Max-Heap
Root Element Minimum value Maximum value
Heap Property Parent ≤ Children Parent ≥ Children
Common Use Priority queues where lower values have higher priority Priority queues where higher values have higher priority

Common Variations

Binary Heap

The standard heap implementation with each node having at most two children.

Binomial Heap

A collection of binomial trees that satisfies the heap property, allowing efficient merge operations.

Fibonacci Heap

A more advanced heap with better amortized time complexity for some operations, though with higher constant factors.

Implementation Considerations

  • Array vs Tree: While conceptually a tree, arrays are more efficient for storage and access
  • Dynamic Resizing: Many implementations automatically resize the underlying array when needed
  • Generic Types: Can be implemented to work with any comparable data type
  • Custom Comparators: Some implementations allow custom comparison functions for more complex ordering
