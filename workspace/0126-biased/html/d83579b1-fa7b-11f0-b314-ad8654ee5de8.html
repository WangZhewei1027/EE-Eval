Ternary Search — A Comprehensive Guide

Definitions, algorithms, step-by-step explanations, worked examples, pseudocode, complexity, pitfalls and best practices. A primarily textual and pedagogical resource; a very small interactive demonstration is provided at the end for illustration.

What is Ternary Search?

"Ternary search" refers to techniques that split a search domain into three parts instead of two. There are two related ideas commonly called ternary search:

  • Ternary search on unimodal functions (continuous or discrete optimization): used to find the maximum or minimum of a unimodal function by repeatedly evaluating two interior points and discarding one third of the interval each iteration. This is the most common algorithmic use in competitive programming and numerical optimization for unimodal functions.
  • Ternary division for searching sorted arrays: analogous to binary search, but divides the array into three parts using two pivot indices and reduces the search space to one of three parts. This is less common in practice and often has worse comparison counts than binary search.

In this guide we focus on the unimodal-function version (the classical "ternary search") and also explain the alternate "three-way divide" interpretation because learners sometimes confuse them.

Key idea and definitions

Unimodal function: a function f defined on an interval [l, r] is unimodal if it increases (monotonically non-decreasing) up to some point (the maximum), and then decreases (monotonically non-increasing) afterwards. For minimization, flip the direction or negate the function. The function has a single local (and global) extremum in the interval.

Ternary search exploits unimodality by sampling two interior points m1 and m2 (typically dividing the interval into three equal parts), evaluating f(m1) and f(m2), and using the comparison to discard one third of the interval where the extremum cannot lie.

Important: Ternary search is only correct when the function is unimodal on the considered interval. If that assumption fails, results can be incorrect.

Continuous unimodal ternary search — step-by-step

Suppose we want the maximum of a unimodal function f on [l, r]. Each iteration:

  1. Compute two points: 
    m1 = l + (r - l) / 3
m2 = r - (r - l) / 3
  2. Evaluate f(m1) and f(m2).
  3. If f(m1) < f(m2), the maximum lies in (m1, r], so set l = m1 (discard left third).
  4. Otherwise (f(m1) >= f(m2)), the maximum lies in [l, m2), so set r = m2 (discard right third).
  5. Repeat until the interval width r - l becomes smaller than a predefined tolerance ε. The approximate maximum is at (l + r) / 2 or the better of f(m1), f(m2), etc.

For minimization, reverse the inequality: if f(m1) > f(m2) then set l = m1; else set r = m2.

Note: When f(m1) == f(m2) you may shrink to [m1, m2] (both points remain candidates), but in practice using either branch yields correct convergence for strict unimodality.

Pseudocode (continuous unimodal, maximization)

function ternary_search_max(f, l, r, eps):
    while (r - l) > eps:
        m1 = l + (r - l) / 3
        m2 = r - (r - l) / 3
        if f(m1) < f(m2):
            l = m1
        else:
            r = m2
    return (l + r) / 2  // approximate argmax

Complexity: each iteration reduces interval length to 2/3 of previous, so number of iterations to reach width ≤ ε is O(log((r-l)/ε)).

Discrete/uniform integer domain variant

When the domain is discrete (integers) and the function is unimodal over integers, use a similar approach but be careful about rounding and termination:

  • Compute m1 = l + (r - l) / 3, m2 = r - (r - l) / 3 using integer division.
  • When r - l is small (for example ≤ 5 or 10), stop iterating and inspect all remaining integer points directly to find the maximum.
  • Reason: integer rounding and equal-value plateaus can cause off-by-one errors if you continue shrinking too far without brute force verification.

Pseudocode is similar, but after termination search the remaining integer points explicitly.

Ternary search vs Binary search (comparison summary)

Binary search divides the search interval into two parts and is the standard choice for locating elements in a sorted structure or for root-finding and monotone predicates. Ternary search divides into three parts and is tailored to unimodal optimization.

Key differences:

  • Use case: Binary search — monotone predicate, finding boundaries, searching in sorted arrays. Ternary (unimodal) — finding maximum/minimum of a unimodal function.
  • Comparisons per iteration: Binary uses about 1 comparison to decide half (sometimes 2 when comparing to a target). Ternary uses 2 evaluations of f per iteration (often more expensive if f is heavy to compute).
  • Asymptotic iterations: Binary reduces length by 1/2 each iteration; ternary reduces length by 2/3 each iteration. For equal per-iteration cost, binary often converges faster. For function optimization where only function values matter and the function is unimodal, ternary is appropriate.

Conclusion: Use the method that matches your problem. For optimization of unimodal functions, ternary search is simple and effective. For searching in sorted discrete data, binary search is almost always preferable.

Worked numerical example (continuous)

Consider f(x) = - (x - 2)^2 + 3, on interval [-2, 6]. This is a concave parabola with maximum at x = 2 and f(2) = 3. We'll run a few ternary steps conceptually (maximization).

  1. Initial: l = -2, r = 6. m1 = -2 + 8/3 ≈ 0.6667, m2 = 6 - 8/3 ≈ 3.3333.
  2. Compute f(m1) = - (0.6667 - 2)^2 + 3 ≈ -1.7778 + 3 = 1.2222. f(m2) = - (3.3333 - 2)^2 + 3 ≈ -1.7778 + 3 = 1.2222. They are equal (symmetry), so we can set r = m2 and now interval becomes [-2, 3.3333].
  3. Repeat: new m1 ≈ -2 + 5.3333/3 ≈ -0.2222, m2 ≈ 1.4444. Evaluate: f(-0.2222) ≈ -4.938 + 3 = -1.938 (lower), f(1.4444) ≈ -0.3086 + 3 = 2.6914. Since f(m1) < f(m2), set l = m1 -> interval becomes [-0.2222, 3.3333].
  4. Continuing this process rapidly narrows toward x = 2. After several iterations the interval is small and the center approximates the maximum.

This example demonstrates the symmetric shrinkage when the parabola is symmetric about the maximum.

Algorithmic complexity and precision

Each ternary iteration reduces the interval length by a constant factor 2/3. If the initial interval length is L0 and we want final length ≤ ε, the number of iterations k satisfies: 

 (2/3)^k * L0 ≤ ε  =>  k ≥ log(ε / L0) / log(2/3)  = O(log(L0/ε))

So convergence is logarithmic in the desired precision. For integer domains, complexity measured in evaluations is O(log n) with base (3/2) factor per step, but practical costs depend on how expensive each function evaluation is.

Pitfalls and practical considerations

  • Do not apply ternary search if the function is not unimodal on the interval — the result may be wrong.
  • For discrete integer domains, use cautious termination and brute-force check when interval is small.
  • Be careful with floating point comparisons and precision; choose ε appropriately relative to function scale.
  • When function evaluations are expensive, minimize the number of evaluations — caching can help (m1 and m2 from previous iteration can be reused in some implementations to avoid duplicate evaluations).
  • Ternary search on a sorted array (three-way divide) rarely beats binary search in practice because it requires two key comparisons per iteration while binary uses one; binary search minimizes comparisons in the decision tree model.

Code examples (for reference)

Python (continuous unimodal maximization):

def ternary_search_max(f, left, right, eps=1e-7):
    while right - left > eps:
        m1 = left + (right - left) / 3
        m2 = right - (right - left) / 3
        if f(m1) < f(m2):
            left = m1
        else:
            right = m2
    return (left + right) / 2

C++ (discrete integer unimodal):

long long ternary_search_int(function f, long long l, long long r){
    while (r - l > 3){                 // when small, brute force
        long long m1 = l + (r - l) / 3;
        long long m2 = r - (r - l) / 3;
        if (f(m1) < f(m2)) l = m1;
        else r = m2;
    }
    long long best = l;
    for (long long x = l; x <= r; ++x)
        if (f(x) > f(best)) best = x;
    return best;
}

When to prefer ternary search

Use ternary search when your objective satisfies unimodality and you need a simple, robust method to find an extremum (max or min) with only function evaluations available and no derivative information. Examples:

  • Finding the maximum of a smooth concave function on a closed interval.
  • Optimizing a unimodal cost function that is expensive to evaluate but where derivative information is unavailable.
  • Competitive programming problems that explicitly state or guarantee unimodality.
