Big-O Notation

Visualizing algorithmic time complexities — how runtime grows relative to input size n. Observe the elegant curves depicting the performance of common complexity classes.

Big-O Complexity Classes

O(1) - Constant Time
Execution time remains constant regardless of input size. Example: Accessing an element in an array.

O(log n) - Logarithmic Time
Time grows logarithmically with input size. Example: Binary search in a sorted array.

O(n) - Linear Time
Time grows linearly with input size. Example: Simple loops through data.

O(n log n) - Linearithmic Time
Typical in efficient sorting algorithms like mergesort and heapsort.

O(n²) - Quadratic Time
Time grows with the square of input size. Examples: Simple nested loops.

O(2ⁿ) - Exponential Time
Time doubles with each element added. Examples: Recursive calculation of Fibonacci numbers naïvely.
