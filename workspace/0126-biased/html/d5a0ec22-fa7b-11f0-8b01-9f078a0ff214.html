Understanding Counting Sort

What is Counting Sort?

Counting Sort is a non-comparison based sorting algorithm that sorts elements by counting the occurrences of each unique value in the input. It is particularly effective for sorting integers or objects that can be expressed as integers. The key principle of Counting Sort is to determine the position of each value in the sorted output using counts of the occurrences of each value.

Theoretical Foundations

Counting Sort is efficient when the range of potential values (k) is not significantly larger than the number of elements (n) to be sorted. The overall time complexity of Counting Sort is O(n + k), where:

  • n is the number of elements in the input array.
  • k is the range of the input values (the difference between the maximum and minimum values plus one).

Algorithm Steps

Counting Sort can be broken down into the following steps:

  1. Find the Range: Determine the maximum value in the input array to establish the range of values.
  2. Create Count Array: Create an array (the count array) of size k + 1 initialized to zeros to hold the count of each unique value in the input.
  3. Count Occurrences: Iterate through the input array and for each value, increment its corresponding index in the count array.
  4. Cumulative Count: Modify the count array such that each element at each index stores the sum of previous counts. This helps determine the position of each element in the output array.
  5. Build Output Array: Create an output array. Iterate through the original array from last to first and place each element at its correct position in the output array based on the counts. This ensures the stability of the sort.
  6. Copy Back: Finally, copy the output array back to the input array to reflect the sorted order.

Example of Counting Sort

Let’s consider an array of integers: [4, 2, 2, 8, 3, 3, 1]. Here’s how Counting Sort would process this array:

  1. The maximum value is 8.
  2. Create a count array of size 9 (from 0 to 8): [0, 0, 0, 0, 0, 0, 0, 0, 0].
  3. Count the occurrences: 
                    [0, 1, 2, 2, 1, 0, 0, 0, 1]
  4. Update the count array to become cumulative: 
                    [0, 1, 3, 5, 6, 6, 6, 6, 7]
  5. Build the output array from the original array: 
                    [1, 2, 2, 3, 3, 4, 8]
  6. Copy the output back to the original array, which is now sorted.

When to Use Counting Sort

Counting Sort is particularly useful when:

  • The maximum number in the input is not significantly larger than the number of elements.
  • Data contains integers within a specific known range.
  • Performance is critical, and you need a linear time sorting algorithm.

Limitations of Counting Sort

There are some limitations to consider:

  • Not suitable for sorting large ranges of integer values, as memory usage can be significant.
  • Cannot handle non-integer data types directly.

Conclusion

Counting Sort is a straightforward yet powerful algorithm for sorting integers or categorical data efficiently. Understanding its operation involves grasping fundamental array manipulation concepts and can form the basis of more complex algorithms and data structures.
