Directed Graphs: Theory and Applications

Introduction to Directed Graphs

A directed graph (or digraph) is a fundamental data structure in computer science and mathematics that represents relationships between objects where the connections have a specific direction.

Formal Definition

A directed graph G is an ordered pair G = (V, E) where:

  • V is a set of vertices (also called nodes)
  • E is a set of ordered pairs of vertices called directed edges (or arcs)

For an edge (u, v), we say that the edge is directed from u to v, where u is called the tail and v is called the head of the edge.

Example

Consider a social network where "follow" relationships are directional. If Alice follows Bob, but Bob doesn't necessarily follow Alice back, this can be represented as a directed edge from Alice to Bob.

Terminology and Properties

Basic Terminology

  • Out-degree: Number of edges leaving a vertex
  • In-degree: Number of edges entering a vertex
  • Source: Vertex with in-degree zero
  • Sink: Vertex with out-degree zero
  • Path: Sequence of vertices where each adjacent pair is connected by an edge in the correct direction
  • Cycle: Path that starts and ends at the same vertex

Strongly Connected Components

A directed graph is strongly connected if there is a path from every vertex to every other vertex. A strongly connected component (SCC) is a maximal strongly connected subgraph.

Graph Representation Methods

Method Description Space Complexity
Adjacency Matrix A V×V matrix where A[i][j] = 1 if there's an edge from i to j O(V²)
Adjacency List An array of lists where each list stores the neighbors of a vertex O(V + E)
Edge List A list of all edges as ordered pairs O(E)

Algorithms on Directed Graphs

Depth-First Search (DFS)

DFS explores as far as possible along each branch before backtracking. For directed graphs, it's particularly useful for:

  • Topological sorting
  • Finding strongly connected components
  • Cycle detection
function DFS(G, v):
    mark v as visited
    for each neighbor u of v in G:
        if u is not visited:
            DFS(G, u)

Topological Sorting

A topological sort of a directed acyclic graph (DAG) is a linear ordering of vertices such that for every directed edge (u, v), u comes before v in the ordering.

Algorithm steps:

  1. Perform DFS on the graph
  2. As each vertex is finished (all neighbors processed), add it to the front of a list
  3. The final list is the topological order

Time complexity: O(V + E)

Kosaraju's Algorithm for Strongly Connected Components

This algorithm finds all strongly connected components in a directed graph in linear time.

Steps:

  1. Perform DFS on the original graph, pushing vertices to a stack in order of completion
  2. Compute the transpose graph (reverse all edges)
  3. Perform DFS on the transpose graph in the order defined by the stack
  4. Each DFS tree in the second pass is an SCC

Time complexity: O(V + E)

Applications of Directed Graphs

Computer Science Applications

  • Dependency resolution: Package managers use DAGs to resolve dependencies
  • Task scheduling: Representing precedence constraints between tasks
  • Data flow analysis: Modeling information flow in compilers
  • Web crawling: Representing links between web pages

Other Applications

  • Transportation networks: One-way streets in road networks
  • Social networks: Following relationships on platforms like Twitter
  • Epidemiology: Modeling disease spread through contact networks
  • Game theory: Representing possible moves in games

Advanced Topics

Transitive Closure

The transitive closure of a directed graph is a new graph with an edge from u to v if there is a path from u to v in the original graph.

Flow Networks

Directed graphs where edges have capacities and vertices have flow conservation constraints, used in maximum flow problems.

Tournaments

A complete oriented graph where for every pair of distinct vertices, there is exactly one directed edge between them.
