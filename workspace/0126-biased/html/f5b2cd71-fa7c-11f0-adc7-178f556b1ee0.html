Understanding the OSI Model

The OSI model is a conceptual framework for understanding how data communications are made. It consists of 7 layers, each with its own set of functions and protocols.

Layer 1: Physical Layer

The Physical Layer (Layer 1) defines the physical means of transmitting data over a network. It includes the transmission media, network topology, and protocols for error detection and correction.

Example: Ethernet, Wi-Fi

Layer 2: Data Link Layer

The Data Link Layer (Layer 2) provides error-free transfer of data frames between two devices on the same network. It includes protocols such as Ethernet and PPP.

Example: Ethernet, PPP

Layer 3: Network Layer

The Network Layer (Layer 3) routes data between different networks. It includes protocols such as IP and ICMP.

Example: IP, ICMP

Layer 4: Transport Layer

The Transport Layer (Layer 4) provides reliable data transfer between devices. It includes protocols such as TCP and UDP.

Example: TCP, UDP

Layer 5: Session Layer

The Session Layer (Layer 5) establishes and manages connections between devices. It includes protocols such as SSH and Telnet.

Example: SSH, Telnet

Layer 6: Presentation Layer

The Presentation Layer (Layer 6) converts data into a format that can be understood by the receiving device. It includes protocols such as SSL and TLS.

Example: SSL, TLS

Layer 7: Application Layer

The Application Layer (Layer 7) provides services to end-user applications. It includes protocols such as HTTP and FTP.

Example: HTTP, FTP
