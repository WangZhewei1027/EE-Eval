Bucket Sort Algorithm

Introduction to Bucket Sort

Bucket sort is a distribution sort algorithm that works by distributing elements of an array into a number of buckets. Each bucket is then sorted individually, either using a different sorting algorithm or by recursively applying the bucket sort algorithm.

Bucket sort is mainly useful when the input is uniformly distributed over a range. It's not a comparison-based sort, which gives it an advantage in certain scenarios where comparison-based sorts would be less efficient.

Key Characteristics:

  • Non-comparison based sorting algorithm
  • Works well when input is uniformly distributed
  • Can achieve linear time complexity O(n) in best cases
  • Uses buckets to distribute elements

How Bucket Sort Works

The bucket sort algorithm can be broken down into several distinct steps:

1. Bucket Creation

Create an array of empty buckets. The number of buckets is typically determined based on the range of input values and the desired distribution.

2. Scatter Phase

Go through the input array and place each element into its corresponding bucket. The bucket for each element is determined by a bucket index calculation, often using a formula like:

bucketIndex = (value - minValue) * (numberOfBuckets - 1) / (maxValue - minValue)

3. Sort Individual Buckets

Sort each non-empty bucket. This can be done using any sorting algorithm, but insertion sort is commonly used for small bucket sizes.

4. Gather Phase

Visit the buckets in order and put all elements back into the original array.

Example Walkthrough

Let's sort the array [0.42, 0.32, 0.75, 0.12, 0.98, 0.63] using bucket sort with 4 buckets:

  1. Create 4 empty buckets (ranges: [0-0.25), [0.25-0.5), [0.5-0.75), [0.75-1.0))
  2. Scatter elements:
    • 0.42 → Bucket 1 (0.25-0.5)
    • 0.32 → Bucket 1 (0.25-0.5)
    • 0.75 → Bucket 3 (0.75-1.0)
    • 0.12 → Bucket 0 (0-0.25)
    • 0.98 → Bucket 3 (0.75-1.0)
    • 0.63 → Bucket 2 (0.5-0.75)
  3. Sort each bucket:
    • Bucket 0: [0.12] (already sorted)
    • Bucket 1: [0.32, 0.42]
    • Bucket 2: [0.63] (already sorted)
    • Bucket 3: [0.75, 0.98]
  4. Gather: [0.12, 0.32, 0.42, 0.63, 0.75, 0.98]

Bucket Sort Algorithm Pseudocode

function bucketSort(array, numberOfBuckets): // Find minimum and maximum values minValue = min(array) maxValue = max(array) // Initialize buckets buckets = array of empty lists for i from 0 to numberOfBuckets-1: buckets[i] = new List() // Distribute elements into buckets for each element in array: bucketIndex = (element - minValue) * (numberOfBuckets - 1) / (maxValue - minValue) buckets[bucketIndex].append(element) // Sort each bucket for each bucket in buckets: sort(bucket) // typically using insertion sort // Concatenate all buckets sortedArray = new List() for each bucket in buckets: sortedArray.appendAll(bucket) return sortedArray

Time and Space Complexity

Scenario Time Complexity Space Complexity
Best Case (uniform distribution) O(n) O(n + k)
Average Case O(n + n²/k + k) O(n + k)
Worst Case (all elements in one bucket) O(n²) O(n + k)

Where:

  • n = number of elements in the input array
  • k = number of buckets

Important Notes on Complexity:

  • The best case occurs when elements are uniformly distributed across buckets
  • The worst case occurs when all elements are placed in a single bucket
  • Space complexity includes the storage for the buckets and the output array
  • Bucket sort is efficient when k ≈ n, making the average case O(n)

Implementation Considerations

Choosing the Number of Buckets

The number of buckets is a crucial parameter that affects the performance of bucket sort:

  • Too few buckets may result in many elements per bucket, making the sorting of individual buckets expensive
  • Too many buckets may result in many empty buckets, wasting memory
  • A common choice is to make the number of buckets equal to the number of elements (k = n)

Sorting Within Buckets

The choice of sorting algorithm for the individual buckets affects performance:

  • Insertion sort is commonly used because it's efficient for small lists
  • For larger buckets, more efficient algorithms like merge sort or quicksort might be better
  • Recursively applying bucket sort is also an option

Data Distribution

Bucket sort works best when the input data is uniformly distributed:

  • Non-uniform distributions can lead to some buckets having many more elements than others
  • For non-uniform data, adaptive bucket sizing might be necessary

Applications of Bucket Sort

Bucket sort is particularly useful in the following scenarios:

  • Sorting floating-point numbers in a known range (e.g., [0.0, 1.0))
  • External sorting when data doesn't fit in memory (buckets can represent files)
  • Histogram generation where you need to count elements in ranges
  • As a subroutine in more complex algorithms like radix sort
  • When the input is uniformly distributed across a known range

Real-world Example: Grading System

Imagine sorting student grades (0-100) into letter grade buckets:

  • A: 90-100
  • B: 80-89
  • C: 70-79
  • D: 60-69
  • F: 0-59

Bucket sort naturally fits this problem by placing each grade in the appropriate bucket, then sorting within each grade range if needed.

Advantages and Disadvantages

Advantages

  • Can achieve linear time complexity O(n) in best cases
  • Efficient for uniformly distributed data
  • Stable if the sorting algorithm used for buckets is stable
  • Works well for floating-point numbers
  • Can be parallelized (different buckets can be processed in parallel)

Disadvantages

  • Performance degrades with non-uniform distributions
  • Requires additional memory for buckets (not in-place)
  • Not efficient for small datasets due to overhead
  • Requires knowledge of the input range
  • Worst-case performance is O(n²) when all elements are in one bucket

Conclusion

Bucket sort is a powerful non-comparison based sorting algorithm that excels when the input data is uniformly distributed across a known range. While it has some limitations, particularly with non-uniform data distributions, its ability to achieve linear time complexity in ideal cases makes it an important algorithm in certain scenarios.

Understanding bucket sort provides valuable insight into distribution-based sorting approaches and serves as a foundation for more complex algorithms like radix sort. When the problem characteristics match its strengths (uniform distribution, known range), bucket sort can outperform traditional comparison-based sorting algorithms.
