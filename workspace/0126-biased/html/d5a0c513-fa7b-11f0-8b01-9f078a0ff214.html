Insertion Sort: A Comprehensive Guide

What is Insertion Sort?

Insertion Sort is a simple and efficient sorting algorithm that builds a sorted array (or list) one element at a time. It is much less efficient on large lists than more advanced algorithms such as quicksort, heapsort, or merge sort. However, it has its advantages: it is easy to implement and works well for small datasets or nearly sorted datasets.

How Does Insertion Sort Work?

The algorithm divides the input list into two parts: a sorted part and an unsorted part. Initially, the sorted part consists of the first element, and the unsorted part consists of the rest of the elements. The algorithm proceeds by taking an element from the unsorted part and inserting it into the correct position in the sorted part. This process is repeated until no elements remain in the unsorted part.

Step-by-Step Process

To further understand how Insertion Sort operates, let’s follow the following step-by-step description:

  1. Start with the second element (the first element is trivially sorted). Compare it to the first and place it in the correct position.
  2. Take the next element from the unsorted part and compare it with the elements in the sorted part, moving larger elements one position to the right until the correct position is found.
  3. Insert the current element at its proper position.
  4. Repeat this process until all elements in the list have been sorted.

Algorithm Pseudocode

The Insertion Sort algorithm can be represented in pseudocode as follows:

        procedure insertionSort(A : array of items)
            for i = 1 to length(A) - 1 do
                key = A[i]
                j = i - 1
                while j >= 0 and A[j] > key do
                    A[j + 1] = A[j]
                    j = j - 1
                A[j + 1] = key

Time Complexity

The time complexity of Insertion Sort is:

  • Best Case: O(n) - when the array is already sorted.
  • Average Case: O(n²) - when the elements are in random order.
  • Worst Case: O(n²) - when the array is sorted in reverse order.

Why Use Insertion Sort?

Despite its O(n²) worst-case time complexity, Insertion Sort has several key advantages:

  • It is simple and easy to implement.
  • It is efficient for small data sets or partially sorted data.
  • It is a stable sort: it maintains the relative order of equal elements.
  • It is an in-place sort: it requires a constant amount of additional space.

Example

Let's consider an array of numbers: [5, 2, 4, 6, 1, 3].

  1. Start with 5 (already sorted).
  2. Insert 2: [2, 5, 4, 6, 1, 3].
  3. Insert 4: [2, 4, 5, 6, 1, 3].
  4. Insert 6: [2, 4, 5, 6, 1, 3].
  5. Insert 1: [1, 2, 4, 5, 6, 3].
  6. Insert 3: [1, 2, 3, 4, 5, 6].

After completing the process, we see that the array is now sorted!

Conclusion

Insertion Sort is a fundamental algorithm that provides an excellent example of a simple sorting method. Understanding this algorithm lays the groundwork for learning more complex sorting techniques and algorithms. With its practical applications in certain scenarios, Insertion Sort remains a noteworthy topic in computer science.
