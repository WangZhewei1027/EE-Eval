Paging in Operating Systems

Definition

Paging is a memory management scheme that eliminates the need for contiguous allocation of physical memory. This scheme permits the physical address space of a process to be non-contiguous. Paging avoids external fragmentation and the need for compaction.

Fundamental Concepts

Basic Terminology

  • Page: A fixed-size block of data in virtual memory.
  • Frame: A fixed-size block of data in physical memory (same size as a page).
  • Page Table: A data structure used by the virtual memory system to store the mapping between virtual addresses and physical addresses.
  • Page Fault: An interrupt that occurs when a program accesses a page that is mapped in virtual address space but not loaded in physical memory.

How Paging Works

Paging divides both the virtual address space (what the program sees) and physical memory into fixed-size blocks called pages and frames respectively. The operating system maintains a page table for each process that maps the program's pages to physical frames.

Address Translation

The key mechanism in paging is the translation of virtual addresses (generated by the CPU) to physical addresses (used by the memory hardware). This translation is performed by the Memory Management Unit (MMU).

Address Translation Process

1. CPU generates a virtual address

2. MMU splits the address into:

- Page number (p)

- Page offset (d)

3. Page number is used as an index into the page table

4. Page table contains the base address of the frame in physical memory

5. Physical address is formed by combining frame base address with page offset

Example: Address Translation

Assume:

  • Page size = 4KB (2^12 bytes)
  • Virtual address = 0x3A7F

Calculation:

1. Convert to binary: 0011 1010 0111 1111

2. Split into page number (upper bits) and offset (lower 12 bits)

3. Page number = 0011 (page 3)

4. Offset = 1010 0111 1111 (0xA7F)

5. If page table shows page 3 maps to frame 7:

6. Physical address = frame 7 + offset 0xA7F

7. Final physical address = 0x7A7F

Page Table Structures

Different operating systems use various page table structures to manage the virtual-to-physical mappings:

Hierarchical Page Tables

A multi-level page table breaks the virtual address into multiple parts, each indexing a different level of the page table hierarchy. This approach saves space by not requiring entries for unused address ranges.

Hashed Page Tables

Common in address spaces larger than 32 bits, this approach uses a hash function to map virtual page numbers to a hash table containing chains of elements.

Inverted Page Tables

Instead of mapping virtual pages to physical frames, inverted page tables map physical frames back to virtual pages, significantly reducing the space needed for page tables.

Page Replacement Algorithms

When a page fault occurs and all frames are in use, the operating system must select a page to replace. Several algorithms exist for this purpose:

Algorithm Description Advantages Disadvantages
FIFO Replaces the oldest page in memory Simple to implement Poor performance, suffers from Belady's anomaly
LRU Replaces least recently used page Good performance in most cases Difficult to implement efficiently
Clock Approximation of LRU using a circular list and reference bit Good balance of performance and implementation complexity Not as precise as true LRU
Optimal Replaces page that won't be used for longest time (theoretical) Provides best possible performance Not implementable in practice (requires future knowledge)

Performance Considerations

Translation Lookaside Buffer (TLB)

The TLB is a special cache that stores recent address translations to avoid having to access the page table for every memory reference. It's a critical component for paging performance.

Page Size Trade-offs

Choosing an appropriate page size involves balancing several factors:

  • Larger pages: Reduce page table size, increase TLB coverage, but may lead to internal fragmentation
  • Smaller pages: Reduce internal fragmentation, allow better memory utilization, but increase page table size

Advantages of Paging

  • Eliminates external fragmentation
  • Allows efficient use of memory through demand paging
  • Simplifies memory allocation (any free frame can be used)
  • Enables memory protection through page-level permissions
  • Supports virtual memory and swapping

Disadvantages of Paging

  • Internal fragmentation can occur
  • Memory reference overhead due to address translation
  • Page tables consume memory resources
  • Complex page replacement algorithms may be needed

Conclusion

Paging is a fundamental memory management technique that provides numerous benefits for modern operating systems. By dividing memory into fixed-size pages and frames, it solves the problem of external fragmentation while enabling efficient memory allocation and virtual memory capabilities. Understanding paging is essential for grasping how operating systems manage memory in today's computing environments.
