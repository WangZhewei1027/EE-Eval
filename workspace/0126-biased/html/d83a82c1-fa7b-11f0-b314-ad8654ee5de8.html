1. What is linear regression?

Linear regression is a statistical method for modeling the relationship between one dependent variable (response, target) and one or more independent variables (features, predictors) by fitting a linear equation to observed data. The goal is to predict the target from the features and to understand how changes in features are associated with changes in the target.

Core idea

Represent the expected value of the response as a linear function of the inputs. For example, in simple (single-variable) linear regression:

y ≈ β0 + β1 x

Here β0 is the intercept and β1 is the slope (coefficient). The "≈" indicates that the true observations differ from this ideal line because of noise or unmodeled effects.

When to use

  • When you believe the relationship between variables is well approximated by a linear function (or a linear function of transformed features).
  • When you want interpretable coefficients and a simple, fast model.
  • As a baseline model or part of larger pipelines (feature engineering, residual modeling).

2. Simple linear regression — intuition and equations

Consider paired observations (x1, y1), (x2, y2), ..., (xn, yn). We model

yi = β0 + β1 xi + εi

where εi is an error term (noise). The parameters β0 and β1 are chosen to make the line "fit" the observed points.

Objective: Ordinary Least Squares (OLS)

OLS chooses parameters to minimize the sum of squared residuals (differences between observed and predicted y):

minimize over β0,β1: S(β0,β1) = Σ (yi − (β0 + β1 xi))^2

Closed-form solution (one predictor)

Let x̄ and ȳ be sample means. Then the OLS estimates are:

β1_hat = Σ (xi − x̄)(yi − ȳ) / Σ (xi − x̄)^2

β0_hat = ȳ − β1_hat x̄

Intuition: β1_hat is the covariance divided by the variance of x. The result is the slope that minimizes squared errors.

3. Multiple linear regression and matrix form

When there are multiple features x1, x2, ..., xp, we write the model in vector/matrix form:

y = X β + ε

where:

  • y is an n×1 vector of responses
  • X is an n×(p+1) design matrix (often including a column of ones for the intercept)
  • β is a (p+1)×1 vector of parameters
  • ε is an n×1 vector of errors

Explicitly, each row of X is [1, xi1, xi2, ..., xip] if including intercept.

Normal equation (closed-form OLS)

β_hat = (Xᵀ X)^{-1} Xᵀ y

This gives the least-squares solution provided Xᵀ X is invertible (no exact multicollinearity).

4. OLS derivation — step-by-step

We minimize the objective function S(β) = (y − Xβ)ᵀ(y − Xβ) (sum of squared residuals).

  1. Write S(β) = yᵀy − 2βᵀ Xᵀ y + βᵀ Xᵀ X β.
  2. Differentiate with respect to β (matrix calculus): ∇β S = −2 Xᵀ y + 2 Xᵀ X β.
  3. Set gradient to zero: −2 Xᵀ y + 2 Xᵀ X β = 0 ⇒ Xᵀ X β = Xᵀ y.
  4. If Xᵀ X is invertible, solve: β = (Xᵀ X)^{-1} Xᵀ y.

Remarks

  • If Xᵀ X is singular (non-invertible), regularization (Ridge) or variable selection is required.
  • Numerical stability: use QR decomposition or SVD rather than forming (Xᵀ X) directly for stable computations.

5. Alternative algorithms and regularization

Gradient descent (GD)

Instead of closed-form, one can minimize the cost using iterative gradient-based methods. For the mean squared error (MSE) objective:

J(β) = (1/2n) Σ (yi − x_iᵀ β)^2

Gradient: ∇J = −(1/n) Xᵀ (y − Xβ)

Update rule (batch gradient descent):

// pseudocode
initialize β = 0
for iter = 1..T:
  gradient = -(1/n) * Xᵀ * (y - X * β)
  β = β - α * gradient
end

Choose learning rate α small enough for convergence. Variants include stochastic GD (SGD), mini-batch GD, and momentum-based optimizers.

Regularized linear models

Regularization penalizes large coefficients to control variance and improve conditioning.

  • Ridge (L2): minimize Σ (yi − x_iᵀ β)^2 + λ Σ βj^2. Closed form: β = (Xᵀ X + λ I)^{-1} Xᵀ y.
  • Lasso (L1): minimize Σ (yi − x_iᵀ β)^2 + λ Σ |βj|. No closed-form; use coordinate descent or convex solvers. Encourages sparse coefficients.
  • Elastic Net: combination of L1 and L2 penalties.

6. Assumptions and diagnostics

For OLS inference (standard errors, hypothesis tests) and optimality, several assumptions are commonly cited — the Gauss–Markov conditions:

  • Linearity: The conditional expectation of y given X is linear in parameters: E[y|X] = Xβ.
  • Full rank: X has full column rank (no perfect multicollinearity).
  • Exogeneity: Errors have zero mean and are uncorrelated with predictors: E[ε|X]=0.
  • Homoscedasticity: Var(εi | X) = σ^2 (constant variance).
  • No autocorrelation: errors are uncorrelated across observations (important for time-series).

If these hold, OLS is the Best Linear Unbiased Estimator (BLUE). Violations change properties and require different methods (robust standard errors, GLS, transformations).

Common diagnostics

  • Residual plots: plot residuals vs fitted values to see patterns (nonlinearity, heteroscedasticity).
  • Q-Q plot of residuals: check normality assumption for small-sample inference.
  • Variance inflation factor (VIF): detect multicollinearity.
  • Cook's distance / leverage: identify influential points.

7. Interpretation of coefficients

Each coefficient βj represents the expected change in y for a one-unit increase in xj, holding other predictors constant (ceteris paribus). Important nuances:

  • Units matter: scale of x influences coefficient magnitude.
  • Centering variables (subtracting mean) can make intercept interpretable as predicted y at average predictors.
  • Dummy variables: coefficients represent differences relative to a reference category.
  • Interactions: include product terms to model changing effects, e.g., β3 x1*x2.

8. Measures of fit and inference

R-squared

R^2 = 1 − (SSR / SST) = 1 − Σ (yi − ŷi)^2 / Σ (yi − ȳ)^2

R^2 measures the proportion of variance explained by the model. Adjusted R^2 penalizes adding predictors:

R^2_adj = 1 − (1 − R^2) * (n−1)/(n−p−1)

Standard errors and t-tests

Under standard assumptions, variance of β_hat is σ^2 (Xᵀ X)^{-1}. Estimate σ^2 by residual mean square:

σ^2_hat = SSR / (n − p − 1), where SSR = Σ (yi − ŷi)^2

Then standard error SE(βj_hat) = sqrt(σ^2_hat * (jth diagonal element of (Xᵀ X)^{-1})). The t-statistic for testing H0: βj = 0 is

t = βj_hat / SE(βj_hat)

Confidence intervals

e.g., a 95% CI: βj_hat ± t_{α/2, n−p−1} * SE(βj_hat).

9. Practical considerations and tips

  • Feature scaling: not required for OLS, but helps optimization when using gradient-based methods.
  • Categorical variables: encode with dummy/one-hot encoding; avoid the dummy variable trap (drop one category).
  • Outliers: can strongly influence OLS. Consider robust regression or transform variables.
  • Collinearity: high correlation among predictors inflates SEs. Use VIF to detect, or apply PCA/regularization.
  • Model selection: use cross-validation, AIC/BIC, or regularization paths to choose variables.
  • Nonlinearity: include polynomial or basis expansions, or transform variables (log, sqrt) to capture nonlinear patterns while keeping linear-in-parameters.
  • Use QR or SVD for numerical stability when solving normal equations.

Computational complexity

Direct closed-form solution involves computing Xᵀ X and its inverse — cost roughly O(p^3 + np^2). For very large p, iterative methods (gradient-based) or incremental solvers are preferred.

Multiple OLS (matrix)

Checklist before trusting OLS

  • Plot data and residuals
  • Check multicollinearity (VIF)
  • Check for heteroscedasticity (Breusch–Pagan)
  • Consider transformations for skewed variables
  • Split data or cross-validate

Common pitfalls

  • Interpreting correlation as causation
  • Overfitting by adding many predictors without validation
  • Ignoring measurement error in predictors (attenuation bias)
  • Extrapolating far outside the observed range
