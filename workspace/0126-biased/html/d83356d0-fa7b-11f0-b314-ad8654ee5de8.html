Set — Thorough Explanation, Theory, and Algorithms

This document is a comprehensive, textual guide to the mathematical concept of a set, its notation, properties, key theorems, and practical algorithms for computing with finite sets. It emphasizes clear step-by-step explanations and algorithmic descriptions. Interactivity is intentionally minimal — a single demonstration button at the end illustrates a few basic operations.

1. What is a set?

A set is one of the most fundamental concepts in mathematics: it is an unordered collection of distinct objects, called elements or members. The defining features are:

  • Unordered: the order of elements does not matter. {1,2,3} is the same as {3,2,1}.
  • Distinct: an element can appear at most once. {1,1,2} is the same set as {1,2}.
  • Membership: for any object x, either x ∈ S (x is in S) or x ∉ S (not in S).

Examples

Common finite sets: {1, 2, 3}, {a, e, i, o, u}, {red, green, blue}.

Infinite sets: the set of all natural numbers N = {0, 1, 2, 3, ...}; the set of real numbers R (uncountably infinite).

2. Notation and basic terms

  • Membership: x ∈ A means x is an element of A; x ∉ A means x is not in A.
  • Subset: A ⊆ B means every element of A is also in B. Proper subset A ⊂ B means A ⊆ B and A ≠ B.
  • Equality: A = B if each is a subset of the other (same elements).
  • Empty set: ∅ is the set that contains no elements.
  • Cardinality: |A| denotes the number of elements when finite.
  • Set-builder notation: {x | P(x)} denotes the set of all x satisfying predicate P.

3. Common operations on sets

Given sets A and B (in some universal context U), common operations include:

Union

A ∪ B = { x | x ∈ A or x ∈ B } — all elements that are in either set.

Intersection

A ∩ B = { x | x ∈ A and x ∈ B } — elements common to both sets.

Difference (Relative complement)

A \ B = { x | x ∈ A and x ∉ B } — elements in A that are not in B.

Complement

When a universal set U is fixed, the complement of A is U \ A: elements not in A.

Symmetric difference

A Δ B = (A \ B) ∪ (B \ A) — elements in exactly one of A or B.

Cartesian product

A × B = { (a,b) | a ∈ A, b ∈ B } — the set of ordered pairs formed from A and B.

Power set

P(A) is the set of all subsets of A. If |A| = n then |P(A)| = 2^n for finite A.

4. Laws and identities (important)

Commutative: A ∪ B = B ∪ A; A ∩ B = B ∩ A.
Associative: (A ∪ B) ∪ C = A ∪ (B ∪ C); (A ∩ B) ∩ C = A ∩ (B ∩ C).
Distributive: A ∩ (B ∪ C) = (A ∩ B) ∪ (A ∩ C) and A ∪ (B ∩ C) = (A ∪ B) ∩ (A ∪ C).
De Morgan's laws: (A ∪ B)' = A' ∩ B'; (A ∩ B)' = A' ∪ B' (where ' denotes complement).
Identity and null: A ∪ ∅ = A; A ∩ ∅ = ∅; A ∪ U = U; A ∩ U = A.

5. Types of sets and related concepts

  • Finite vs Infinite sets.
  • Countable vs Uncountable: N is countable; R is uncountable (Cantor's diagonal argument explains why).
  • Multiset: elements may repeat; differs from set in that multiplicity matters.
  • Ordered tuple vs set: (a,b) differs from {a,b} because order and repetition are meaningful in tuples.
  • Fuzzy sets: membership is a degree between 0 and 1 rather than boolean.

6. A brief theoretical note — Cantor and cardinality

Cantor's theorem: for any set A, |P(A)| > |A| (the power set of A has strictly larger cardinality than A). The reals can be shown to have the same cardinality as the power set of the naturals; therefore the real numbers are uncountable.

7. Practical representation of sets (computing)

When implementing sets in code or algorithms, the intended operations, size, and element type influence representation choices. Common structures:

  • Hash-based sets (hash table): O(1) average membership, insertion, deletion. Example: Java HashSet, Python set.
  • Balanced tree sets (BSTs): O(log n) operations, ordered iteration. Example: C++ std::set (Red-Black tree).
  • Bitset / bitmap: compact, very fast bitwise operations for small integer universes (union as bitwise OR).
  • Sorted arrays / lists: good for memory and for fast set operations via two-pointer algorithms (union/intersection in O(n+m)).

8. Common algorithms and step-by-step descriptions

8.1 Union with two sorted lists (two-pointer algorithm)

Preconditions: A and B are sorted arrays with distinct elements each.

Step-by-step description:

  1. Initialize i = 0, j = 0, result = empty list.
  2. While i < len(A) and j < len(B):
    1. If A[i] < B[j]: append A[i] to result, i++.
    2. Else if A[i] > B[j]: append B[j], j++.
    3. Else (equal): append A[i] (or B[j]), i++, j++ (skip duplicates).
  3. After loop, append any remaining elements from A or B.
  4. Return result as the union (sorted, distinct).

Time complexity: O(n + m) where n = |A|, m = |B|.

8.2 Intersection with hash table (one unsorted and one used as hash)

Preconditions: sets A and B are represented as arrays or lists.

Step-by-step description:

  1. If |A| > |B| swap them (we build a hash set on the smaller one to reduce memory/time).
  2. Build a hash table H containing all elements of A (O(|A|) expected time).
  3. Initialize result = empty list.
  4. For each element x in B: if x is in H and not yet added to result, add x.
  5. Return result.

Time complexity: expected O(|A| + |B|); worst-case depends on hash behavior but typically O(n+m).

8.3 Set difference (A \\ B) using hash table

Step-by-step:

  1. Build hash table H for B.
  2. For each element x in A: if x ∉ H, include x in the result.
  3. Return result.

8.4 Generating the power set (all subsets) — iterative bitmask method

Preconditions: finite set A with n elements labelled a0...a{n-1}.

Step-by-step:

  1. There are 2^n subsets. For each integer mask from 0 to 2^n - 1:
    1. Initialize an empty subset S.
    2. For k from 0 to n-1: if the k-th bit of mask is 1, include ak in S.
    3. Output S.

Time complexity: O(n * 2^n) overall (each subset requires up to n checks). Memory: can be streamed or stored.

8.5 Recursive generation of subsets (backtracking)

Step-by-step:

  1. Define a recursive function generate(i, current): i is index; current is current subset.
  2. If i == n: output current; return.
  3. Exclude element i: call generate(i+1, current).
  4. Include element i: add ai to current; call generate(i+1, current); remove ai from current (backtrack).

Equivalent complexity to bitmask but often more flexible for constraints like early pruning.

9. Correctness and proofs (short)

Proof sketch for correctness of the two-pointer union algorithm: at each step, the smallest unprocessed element from A ∪ B must be either A[i] or B[j]. Comparing them and appending the smaller (or equal) ensures elements are added in increasing order and no element is missed. Because arrays contain distinct elements and we skip duplicates when equal, the result is distinct.

10. Worked example — manual walkthrough

Let A = {1, 3, 4, 6}, B = {2, 3, 5, 6, 8}. Both sorted and distinct.

Union by two-pointer:

  1. i=0 (A[i]=1), j=0 (B[j]=2): 1 < 2 -> append 1, i=1.
  2. A[i]=3, B[j]=2: 3 > 2 -> append 2, j=1.
  3. A[i]=3, B[j]=3: equal -> append 3, i=2, j=2.
  4. A[i]=4, B[j]=5: 4 < 5 -> append 4, i=3.
  5. A[i]=6, B[j]=5: 6 > 5 -> append 5, j=3.
  6. A[i]=6, B[j]=6: equal -> append 6, advance both.
  7. B has remaining element 8 -> append 8.

Result: {1,2,3,4,5,6,8}.

11. Advanced topics (brief)

  • Relations and functions are built from sets: a relation is a subset of A × B. A function is a relation with special properties (each input maps to exactly one output).
  • Algebra of sets leads to Boolean algebra. Many logical identities correspond to set identities via characteristic functions.
  • Set theory foundations (ZF, ZFC) provide axioms for rigorous treatment of sets, including axioms such as Extensionality, Foundation, Replacement, etc.
  • Cardinality theory leads into ordinals, transfinite induction, and continuum hypothesis (advanced, beyond this text's scope).

12. Step-by-step algorithmic checklist when implementing set operations

  • Decide representation: hash vs sorted vs bitset vs tree.
  • Choose algorithm to match representation: two-pointer for sorted arrays; hash-set scanning for unsorted collections; bitwise ops for bitsets.
  • Consider memory vs time trade-offs: bitsets are compact for small universes; hash tables have overhead but fast average times.
  • Be explicit about equality and hashing for complex elements (objects, tuples).
  • Handle duplicates in input collections by normalizing (e.g., convert lists to sets) if required by the semantics.
