Mutex Simulator

Overview
This simulator models mutexes and threads. You can create threads and locks, write small thread scripts, pick scheduling policies, and explore behaviors: contention, starvation, priority inversion, reentrancy, try-locks, timed locks, nested locks, deadlocks, and more. Use Start/Step/Run to control execution. Logs and metrics update live.

Thread Script Language
Simple commands (one per line). Parameters separated by spaces. Commands: acquire LOCKNAME - blocks until lock obtained (or respects try/timeout if specified) tryacq LOCKNAME - try-lock, does not block timeacq LOCKNAME ms - try to acquire with timeout (ms) release LOCKNAME hold ms - simulate working inside critical section or outside sleep ms - non-CPU wait (yields) yield - voluntarily yield scheduler nested - placeholder for nested acquisitions in script setprio N - change this thread's priority end - thread terminates Comments start with #
