Overview

A Random Forest is an ensemble machine learning method primarily used for classification and regression. It builds many decision trees at training time and combines their outputs to produce a single, more robust prediction. The central idea is to reduce variance (overfitting) of single deep trees by averaging many trees, each built from a randomized subset of the data and features.

At a conceptual level:

  • Each tree is a "weak" or "noisy" learner—a decision tree trained on a bootstrap sample.
  • Trees are de-correlated by selecting a random subset of features (variables) at each split.
  • Predictions are aggregated: majority vote for classification; average for regression.

When to use Random Forests

  • Tabular data with mixed feature types (numerical, categorical).
  • Problems where interpretability by single features is helpful but a small loss of transparency is acceptable.
  • When you want strong baseline performance with relatively little hyperparameter tuning.

How it works — Step-by-step narrative

1. Build many trees (training)

  1. Repeat T times (T is number of trees):
    1. Draw a bootstrap sample from the training set (sample N items with replacement). Some original examples will be missing (out-of-bag).
    2. Grow a decision tree on this sample:
      • At each internal node, instead of considering all features for the best split, randomly select mtry features (a small subset).
      • Choose the best split among those mtry features according to a split criterion (Gini impurity, entropy for classification; mean squared error for regression).
      • Continue splitting until a stopping rule: maximum depth, minimum samples per leaf, or nodes are pure.

2. Make predictions (inference)

  1. For a new example, send it down every tree to reach a leaf node.
  2. Collect each tree’s prediction:
    • Classification: each tree votes for a class; the forest prediction is the class with most votes. You can also compute class probabilities as fraction of trees voting for each class.
    • Regression: average the numeric outputs from all trees.

Why bootstrapping and random features?

Decision trees are high-variance models: small changes in training data can yield different trees. Bagging (bootstrap aggregating) averages out this variance by training different trees on different bootstrap samples. However, when strong predictors dominate, trees can become correlated: if one strong predictor is always chosen for top splits, different trees become similar. Randomly sampling features (mtry) at each split reduces correlation between trees, improving the ensemble's variance reduction.

Key hyperparameters (informal)

  • Number of trees (n_estimators): more trees -> lower variance, up to computing limits.
  • mtry (max features): how many features to consider at each split. Typical defaults: sqrt(p) for classification, p/3 for regression.
  • max_depth / min_samples_leaf: control tree size to balance bias and variance.
  • bootstrap: whether to use bootstrap sampling (typical true) or use the full sample for each tree (sometimes used in extra-trees).

Algorithm and Pseudocode

High-level pseudocode (training)

Inputs:
  - Training data D = {(x_i, y_i)} for i = 1..N
  - Number of trees T
  - Number of features considered per split mtry
  - Stopping criteria (max depth, min samples per leaf, etc.)

Procedure:
  forest = []
  for t in 1..T:
    D_t = bootstrap_sample(D)   # sample N examples with replacement
    tree = grow_tree(D_t)
    forest.append(tree)

Function grow_tree(data):
  if stopping_condition(data):
    return leaf_node(predict(data))
  else:
    features = random_subset(all_features, mtry)
    best_split = find_best_split(data, features)
    left, right = split(data, best_split)
    return node(best_split, grow_tree(left), grow_tree(right))

Pseudocode (prediction)

Function predict_forest(forest, x):
  predictions = []
  for tree in forest:
    predictions.append(predict_tree(tree, x))
  if classification:
    return majority_vote(predictions)
  else if regression:
    return mean(predictions)

Split criteria (brief)

  • Gini impurity (classification): G = 1 - sum_k p_k^2 (lower is better)
  • Entropy (classification): H = -sum_k p_k log p_k
  • Regression (MSE): choose split minimizing sum of squared residuals in children

Out-of-Bag (OOB) estimation

For any tree t, roughly 36% of the original samples are not included in its bootstrap sample (they are "out-of-bag"). We can use those OOB samples to estimate prediction error without a separate validation set:

  1. For each training example i, collect predictions from trees for which i was OOB.
  2. Aggregate those predictions to produce an OOB prediction for i.
  3. Compute error across training set using those OOB predictions. This yields an unbiased estimate of test error in many cases.

Theory & Properties

Bias–Variance perspective

Random forests primarily reduce variance through averaging. A single deep tree often has low bias (can fit complex functions) but high variance. Ensemble averaging reduces variance. Random feature selection reduces correlation between trees, making averaging more effective. The bias may increase slightly compared to an optimally tuned single tree, but total error often decreases because variance drops considerably.

Consistency and asymptotics

Under certain conditions (tree depth, splitting rules, sample sizes), random forests can be shown to be consistent estimators: their error goes to zero as the number of samples and trees increase. Formal proofs require technical assumptions; in practice, strong empirical performance is the main reason for their popularity.

Complexity

  • Training complexity roughly: O(T * N * log N * mtry) (very rough; depends on implementation and number of splits).
  • Prediction complexity roughly: O(T * depth) per sample.
  • Parallelism: Trees are independent; training/prediction can be parallelized across trees.

Statistical properties

  • Robust to irrelevant features because randomization limits the influence of any single feature across the forest.
  • Can handle unscaled features (tree splits are scale-invariant).
  • Not prone to strong model misspecification; trees capture interactions automatically.

Practical guidance and tips

Hyperparameter tuning

  • n_estimators: set large enough until OOB error stabilizes; often 100–1000.
  • mtry: default often good; try tuning with cross-validation if performance is sensitive.
  • min_samples_leaf / max_depth: limit tree size if overfitting or for speed/memory.
  • max_features: consider sqrt(p) for classification, p/3 for regression, or tune via CV.

Feature engineering

  • One-hot encoding categorical variables for some implementations; other libraries accept categorical splits directly.
  • Missing values: some implementations handle them natively; otherwise impute beforehand.

Limitations & pitfalls

  • Less interpretable than single decision trees, though feature importance helps.
  • Cannot extrapolate beyond the range seen in training (regression trees predict averages).
  • Large forests can be memory- and time-intensive.
  • Biased variable importance measures can favor variables with many categories or high cardinality.

When not to use

  • When model interpretability at the decision-path level is critical and must be simple to explain.
  • When data are sequential/time-series where temporal structure must be modeled explicitly (unless features encode time).

Explanation of the demo

Each stump is a very weak learner: it uses a single feature and a single threshold to partition the space. Bootstrapping causes different stumps to see different subsets of training examples. Randomly picking the feature for each stump simulates the mtry step used in Random Forests. Averaging (majority voting) across stumps often improves robustness compared to any single stump.

Interpretability & Variable Importance

1. Feature importance measures

  • Gini importance (mean decrease in impurity): sum of impurity decreases from splits using the feature, averaged over trees. Fast but biased toward variables with more categories or continuous variables with many cut points.
  • Permutation importance (mean decrease in accuracy): measure how model error increases when the values of a feature are randomly permuted. Less biased, more computationally expensive.

2. Partial dependence and ICE plots

Partial dependence plots show the marginal effect of a feature on predicted outcome, averaging over other features. Individual Conditional Expectation (ICE) plots show effects per instance. These help interpret feature influence in non-linear models.

3. Local explanations

Methods like SHAP or LIME can produce local explanations for random forest predictions by approximating the model around a single prediction.

Variants and Extensions

  • Extra-Trees (Extremely Randomized Trees): randomize cut points as well as feature choice; often increases randomness and reduces variance faster at cost of slightly higher bias.
  • Gradient Boosted Trees: sequentially build trees to correct previous errors (different family, often better accuracy but more tuning).
  • Quantile Regression Forests: estimate conditional quantiles, not just mean prediction.
  • Unsupervised Random Forests: use split score based on similarity to create proximity matrices for clustering.

FAQ, Glossary & Quick Reference

Q: How many trees should I use?
A: Use as many as necessary until performance stabilizes and computing resources allow. Many applications use 100–1000 trees. Monitor OOB error or validation error.
Q: Does random forest overfit?
A: Random forests are resistant to overfitting as the number of trees increases—the generalization error tends to stabilize. However, overfitting can occur with poorly chosen parameters or when irrelevant features overwhelm useful signals in extreme cases.
Q: Can I get class probabilities?
A: Yes: proportion of trees predicting a class gives an empirical class probability. Calibration may still be needed.

Glossary

Bootstrap sample
Sample of the same size as original dataset, drawn with replacement.
Bagging
Bootstrap aggregating: training multiple models on bootstrap samples and averaging their predictions.
mtry / max_features
Number of features randomly sampled at each split.
Out-of-bag (OOB)
Training examples not included in a tree's bootstrap sample; used for validation of that tree.
Gini impurity
A measure of node impurity for classification based on class probabilities.
