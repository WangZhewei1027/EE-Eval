Optimizing a JOIN-heavy query

Observe how the optimizer rewrites the plan, reorders joins, and uses indexes to reduce cost.

-- Original SQL
SELECT o.order_id, c.customer_name, SUM(i.amount) total
FROM orders o
JOIN customers c ON o.customer_id = c.id
JOIN items i ON i.order_id = o.order_id
WHERE o.created_at > '2025-01-01'
GROUP BY o.order_id, c.customer_name
