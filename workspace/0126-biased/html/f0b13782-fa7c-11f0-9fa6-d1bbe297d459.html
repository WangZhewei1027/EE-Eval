Comprehensive Guide to Hash Tables

Introduction to Hash Tables

A hash table (also called a hash map) is a fundamental data structure in computer science that implements an associative array abstract data type, a structure that can map keys to values. It uses a hash function to compute an index into an array of buckets or slots, from which the desired value can be found.

Hash tables offer average-case constant time complexity (O(1)) for search, insert, and delete operations, making them one of the most efficient data structures for these operations.

Key Characteristics:
  • Stores key-value pairs
  • Uses a hash function to determine storage location
  • Provides fast lookup, insertion, and deletion
  • Handles collisions (when two keys hash to the same index)

Components of a Hash Table

1. Hash Function

A hash function takes a key as input and returns an integer called the hash code. The hash code is used to determine the index where the value will be stored in the underlying array.

Properties of a good hash function:

  • Deterministic: Same input always produces the same output
  • Uniform distribution: Distributes keys evenly across the array
  • Efficient to compute: Should be fast to calculate

Example Hash Function (Simple Modulo):

hash(key) = key % array_size

For a key 42 and array size 10: 42 % 10 = 2 (stored at index 2)

2. Array (Buckets)

The underlying data structure where values are stored. The size of the array is typically a prime number to help with uniform distribution.

3. Collision Resolution Mechanism

When two different keys produce the same hash (called a collision), the hash table needs a way to handle this situation. Common methods include:

  • Separate chaining (using linked lists at each index)
  • Open addressing (probing to find next available slot)

Hash Table Operations

1. Insertion

To insert a key-value pair:

  1. Compute the hash of the key to get the index
  2. If the slot is empty, store the value there
  3. If the slot is occupied (collision), use the collision resolution method

Insertion Example:

Insert ("apple", 25) into a hash table with size 5:

  1. Compute hash("apple") = 1 (for this example)
  2. Store value 25 at index 1

2. Lookup

To find a value by its key:

  1. Compute the hash of the key to get the index
  2. Check if the key at that index matches
  3. If not (due to collision), follow the collision resolution method

3. Deletion

To remove a key-value pair:

  1. Find the item using the lookup process
  2. Remove the item while maintaining the integrity of the collision resolution method

Collision Resolution Methods

1. Separate Chaining

Each bucket contains a linked list of entries. When collisions occur, items are added to the list at that bucket.

Separate Chaining Example:

Index Values (Linked List)
0 → "pear" (3) → "kiwi" (8)
1 → "apple" (25)
2 → "banana" (7)
3 → "orange" (12)
4 (empty)

2. Open Addressing

All entries are stored in the bucket array itself. When a collision occurs, the algorithm searches for the next available slot.

Common probing methods:

  • Linear probing: Check next slots sequentially
  • Quadratic probing: Check slots at increasing quadratic intervals
  • Double hashing: Use a second hash function to determine the step size

Linear Probing Example:

Inserting keys 10, 20, 30 into a table of size 5:

  1. 10 % 5 = 0 → store at 0
  2. 20 % 5 = 0 → collision, store at next available (1)
  3. 30 % 5 = 0 → collision, next available is 2

Resulting table: [10, 20, 30, _, _]

Time Complexity Analysis

Operation Average Case Worst Case
Insertion O(1) O(n)
Lookup O(1) O(n)
Deletion O(1) O(n)

The worst case occurs when all keys hash to the same bucket, essentially turning the hash table into a linear list. This is why a good hash function and proper sizing are crucial.

Load Factor: The ratio of the number of elements to the number of buckets (n/k). When the load factor grows beyond a certain threshold (typically 0.7), the table is resized (rehashed) to maintain efficiency.

Applications of Hash Tables

  • Database indexing: Quickly locate data without searching every row
  • Caching: Memoization, browser caching, DNS caching
  • Sets: Checking for membership (is this item in the set?)
  • Object representation: Many programming languages use hash tables to implement objects/dictionaries
  • Cryptography: Password verification, digital signatures
  • Compilers: Symbol tables for variables and functions

Sample Implementation (Pseudocode)

Hash Table Class Structure

class HashTable:
    def __init__(self, size):
        self.size = size
        self.table = [[] for _ in range(size)]  # Using separate chaining
        
    def hash_function(self, key):
        return hash(key) % self.size  # Using Python's built-in hash()
        
    def insert(self, key, value):
        hash_key = self.hash_function(key)
        bucket = self.table[hash_key]
        
        # Check if key already exists
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)  # Update existing
                return
        bucket.append((key, value))  # Add new
        
    def get(self, key):
        hash_key = self.hash_function(key)
        bucket = self.table[hash_key]
        
        for k, v in bucket:
            if k == key:
                return v
        return None  # Not found
        
    def delete(self, key):
        hash_key = self.hash_function(key)
        bucket = self.table[hash_key]
        
        for i, (k, v) in enumerate(bucket):
            if k == key:
                del bucket[i]
                return

Conclusion

Hash tables are one of the most versatile and efficient data structures in computer science, offering average constant-time operations for insertion, deletion, and lookup. Their performance depends heavily on:

  • The quality of the hash function (uniform distribution)
  • The size of the underlying array
  • The collision resolution method chosen

Understanding hash tables is fundamental to computer science and software development, as they form the basis for many higher-level constructs in programming languages and are widely applied in system design and algorithms.
