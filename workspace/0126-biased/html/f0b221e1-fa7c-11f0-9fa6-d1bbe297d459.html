Radix Sort: A Comprehensive Guide

Introduction to Radix Sort

Radix sort is a non-comparative integer sorting algorithm that sorts data with integer keys by grouping keys by the individual digits which share the same significant position and value. Unlike comparison-based sorting algorithms (like quicksort or mergesort), radix sort distributes elements into buckets according to their radix (base).

Key Characteristics:
  • Non-comparative sorting algorithm
  • Works on integer keys (or strings/other data that can be represented as integers)
  • Uses digit-by-digit sorting starting from least significant digit (LSD) or most significant digit (MSD)
  • Time complexity: O(nk) where n is number of elements and k is number of digits
  • Stable sorting algorithm (maintains relative order of equal elements)

How Radix Sort Works

Radix sort processes the digits of numbers one by one from the least significant digit (LSD) to the most significant digit (MSD) or vice versa. The algorithm works as follows:

  1. Find the maximum number: Determine the number of digits in the largest number (let's call this k).
  2. Initialize buckets: Create 10 buckets (for base 10 numbers) labeled 0 through 9.
  3. Digit-wise sorting: For each digit position (from LSD to MSD):
    • Distribute numbers into buckets based on the current digit
    • Collect numbers from buckets in order (0 to 9)
  4. Final sorted array: After processing all digits, the array is sorted.

Radix Sort Algorithm Pseudocode

// Function to get the maximum number in an array function getMax(arr): max = arr[0] for i from 1 to n-1: if arr[i] > max: max = arr[i] return max // Function to perform counting sort based on digit at exp function countingSort(arr, exp): n = length(arr) output = array of size n count = array of size 10 initialized with 0 // Store count of occurrences in count[] for i from 0 to n-1: digit = (arr[i] / exp) % 10 count[digit]++ // Change count[i] to contain actual position in output for i from 1 to 9: count[i] += count[i - 1] // Build the output array for i from n-1 down to 0: digit = (arr[i] / exp) % 10 output[count[digit] - 1] = arr[i] count[digit]-- // Copy output to arr[] for i from 0 to n-1: arr[i] = output[i] // Main radix sort function function radixSort(arr): max = getMax(arr) // Do counting sort for every digit exp = 1 while max / exp > 0: countingSort(arr, exp) exp *= 10

Time and Space Complexity

The time complexity of radix sort depends on several factors:

Case Time Complexity Explanation
Best Case O(nk) When all numbers have the same number of digits (k)
Average Case O(nk) Typical case with varying digit lengths
Worst Case O(nk) When numbers have widely varying digit lengths

Space Complexity: O(n + k) where k is the range of input (10 for base 10)

Comparison with Other Sorts: Radix sort can be faster than O(n log n) comparison-based sorts when k is less than log n. For very large numbers where k approaches n, it becomes less efficient.

Variations of Radix Sort

1. Least Significant Digit (LSD) Radix Sort

This is the most common implementation, processing digits from right to left. It's stable and works well for fixed-length keys.

2. Most Significant Digit (MSD) Radix Sort

Processes digits from left to right. It can be implemented recursively and is more efficient for variable-length keys. However, it's more complex to implement and may not be stable.

3. In-Place Radix Sort

A variation that uses O(1) extra space by performing swaps within the array, though it's more complex to implement.

When to Use Radix Sort

Radix sort is particularly effective in these scenarios:

  • Sorting integers or fixed-length data
  • When the range of input data is not significantly greater than the number of elements
  • When you need a stable sort
  • In applications like suffix array construction, IP routing tables, or other specialized domains

Limitations: Radix sort is not suitable for sorting floating-point numbers (without modification) or data with variable-length representations where the length varies significantly.

Implementation in JavaScript

// Function to get the digit at a specific place function getDigit(num, place) { return Math.floor(Math.abs(num) / Math.pow(10, place)) % 10; } // Function to count the number of digits in a number function digitCount(num) { if (num === 0) return 1; return Math.floor(Math.log10(Math.abs(num))) + 1; } // Function to get the maximum digit count in an array function mostDigits(nums) { let maxDigits = 0; for (let num of nums) { maxDigits = Math.max(maxDigits, digitCount(num)); } return maxDigits; } // Radix sort implementation function radixSort(nums) { const maxDigitCount = mostDigits(nums); for (let k = 0; k < maxDigitCount; k++) { // Create 10 buckets (0-9) const digitBuckets = Array.from({length: 10}, () => []); // Distribute numbers into buckets based on current digit for (let num of nums) { const digit = getDigit(num, k); digitBuckets[digit].push(num); } // Rebuild the array by concatenating all buckets nums = [].concat(...digitBuckets); } return nums; }

Conclusion

Radix sort is a powerful non-comparative sorting algorithm that can outperform traditional comparison-based sorts in specific scenarios. Its O(nk) time complexity makes it particularly effective when sorting integers with a limited number of digits. While it has some constraints (primarily working with integer data), its stability and predictable performance make it valuable in many applications.

Understanding radix sort provides insight into alternative approaches to sorting beyond the comparison-based paradigm, expanding your algorithmic toolkit for solving problems efficiently.
