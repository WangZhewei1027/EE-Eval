Understanding Big-Theta Notation

Big-Theta (Θ) provides a tight bound on the growth rate of an algorithm's running time, describing both the upper and lower bounds.

Θ

What is Big-Theta Notation?

Big-Theta notation is used in computer science to describe the asymptotic behavior of functions. It provides a tight bound on the growth rate of an algorithm's running time, meaning it describes both the upper and lower bounds.

A function f(n) is Θ(g(n)) if there exist positive constants c₁, c₂, and n₀ such that for all n ≥ n₀, c₁·g(n) ≤ f(n) ≤ c₂·g(n).

In simpler terms, Big-Theta means the function grows exactly at the same rate as g(n), within constant factors. It's the most precise asymptotic notation when you can prove both Big-O and Big-Ω bounds.

Common Big-Theta Examples

Here are some common algorithm complexities expressed in Big-Theta notation:

Constant Time
Runtime doesn't depend on input size
Θ(1) - Array access, arithmetic operations
Linear Time
Runtime grows linearly with input size
Θ(n) - Simple search, iterating through array
Logarithmic Time
Runtime grows logarithmically with input size
Θ(log n) - Binary search, balanced BST operations
Quadratic Time
Runtime grows with the square of input size
Θ(n²) - Bubble sort, insertion sort (worst case)
Linearithmic Time
Between linear and quadratic
Θ(n log n) - Merge sort, heap sort, quick sort (average)
Exponential Time
Runtime grows exponentially with input size
Θ(2ⁿ) - Recursive Fibonacci, brute-force algorithms
