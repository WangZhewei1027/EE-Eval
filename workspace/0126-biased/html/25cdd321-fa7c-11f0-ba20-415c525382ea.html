Logistic Regression: A Comprehensive Guide

Introduction

Logistic Regression is a fundamental machine learning technique used primarily for binary classification problems, where the goal is to classify inputs into one of two possible categories. Although it has the word "regression" in its name, logistic regression is fundamentally a classification algorithm — it predicts the probability that a given instance belongs to a particular class.

This guide aims to thoroughly explain the theory behind Logistic Regression, detail its mathematical formulation, and describe the algorithmic aspects, step-by-step, to build a solid conceptual understanding.

What is Logistic Regression?

Logistic Regression models the probability that an input vector x belongs to a particular class (y = 1) as a function of x. Typically, in the binary classification setting, y is either 0 or 1.

Unlike linear regression which outputs any real number, logistic regression outputs values strictly between 0 and 1 representing probabilities. It uses the logistic function (also known as the sigmoid function) to "squash" the linear combination of inputs into this probability range.

Formally:

  • Given data points (x^{(i)}, y^{(i)}), where i = 1, 2, ..., m
  • x^{(i)} ∈ ℝⁿ is the feature vector
  • y^{(i)} ∈ {0, 1} is the class label

Logistic regression assumes the following model for the probability that y=1 given x:

P(y=1 | x) = h_θ(x) = σ(θᵀx) = 1 / (1 + e^{−θᵀx})

Here:

  • θ ∈ ℝⁿ are the model parameters (weights)
  • θᵀx = θ₀ + θ₁x₁ + ... + θₙxₙ is the linear combination of features (with θ₀ often treated as bias/intercept term)
  • σ(z) = 1 / (1 + e^{−z}) is the sigmoid function

The Sigmoid (Logistic) Function

The sigmoid function is central to logistic regression. It takes any real-valued number and "squashes" it into the range (0, 1), making it suitable for modeling probabilities.

Mathematical definition:

σ(z) = 1 / (1 + e−z)

Properties:

  • Range: (0,1), never exactly 0 or 1
  • S-shaped ("sigmoid") curve
  • Derivative expressed in terms of itself: σ'(z) = σ(z)(1 - σ(z))

Because the sigmoid function outputs a probability, logistic regression makes a prediction by thresholding this probability at 0.5 (or other customizable thresholds):

  • If h_θ(x) ≥ 0.5, predict class 1
  • If h_θ(x) < 0.5, predict class 0

Why Not Use Linear Regression for Classification?

At first glance, one might think of applying linear regression to classification by simply coding classes as 0 or 1. However, linear regression has several drawbacks for classification:

  • Its output can be any real number, not limited between 0 and 1, so it cannot represent valid probabilities directly.
  • It is sensitive to outliers and may produce nonsensical predictions beyond the expected class ranges.
  • Linear regression assumes constant variance and normally distributed residuals, assumptions violated in classification tasks.

Logistic regression avoids these issues by modeling the log-odds of the positive class as a linear function, and then mapping these odds back to a probability via the sigmoid function.

Link to Odds and Log-Odds (Logit)

Logistic regression models the log-odds (also called the logit) of the probability of class 1 as a linear function of the features:

log( P(y=1|x) / P(y=0|x) ) = θᵀx

The odds represent the ratio:

Odds = P(y=1|x) / P(y=0|x)

Applying the logistic function to the linear model gives a probability between 0 and 1. This connection to odds ratios and the natural logarithm of odds provides interpretability: each coefficient θ_j corresponds to the change in log-odds per unit increase in feature x_j.

Training Logistic Regression: The Loss Function

To learn the parameters θ, we use a dataset of labeled samples and try to fit the model so that it best predicts the probabilities of the observed labels.

Likelihood and Log-likelihood:

Each labeled sample contributes a probability to the likelihood of the parameters:

L(θ) = ∏i=1m P(y^{(i)} | x^{(i)}; θ) = ∏i=1m h_θ(x^{(i)})^{y^{(i)}} (1 - h_θ(x^{(i)}))^{1 - y^{(i)}}

Because product of probabilities can be very small and difficult to optimize directly, it is more convenient to work with the log-likelihood:

ℓ(θ) = log(L(θ)) = ∑i=1m [ y^{(i)} log(h_θ(x^{(i)})) + (1-y^{(i)}) log(1 - h_θ(x^{(i)})) ]

The goal of training is to maximize this log-likelihood (equivalent to maximizing the probability of observing the given labels).

Loss function: In practice, algorithms minimize the negative log-likelihood, often called the logistic loss or cross-entropy loss:

J(θ) = - (1/m) ∑i=1m [ y^{(i)} log(h_θ(x^{(i)})) + (1-y^{(i)}) log(1 - h_θ(x^{(i)})) ]

Minimizing J(θ) finds parameters θ that model the data well.

Optimization Algorithms

Since the cost function J(θ) is convex in terms of θ, optimization methods like Gradient Descent can reliably find a global minimum.

Gradient Descent

The gradient of the cost function w.r.t parameters is:

∇J(θ) = (1/m) ∑i=1m ( h_θ(x^{(i)}) - y^{(i)} ) x^{(i)}

The update rule for gradient descent is:

θ := θ - α ∇J(θ)

  • α is the learning rate (step size)

We iteratively adjust θ until convergence.

Other Optimization Methods

  • Newton's Method / Iteratively Reweighted Least Squares (IRLS): Uses second derivatives (Hessian) to achieve faster convergence than gradient descent. Popular for smaller datasets.
  • Stochastic Gradient Descent (SGD): Updates parameters based on a single or mini-batch of samples. Useful for very large datasets.

Step-by-Step Description of Training Logistic Regression

  1. Initialize parameters: Set initial values for θ, often starting at zero or small random numbers.
  2. Calculate the hypothesis: For each training example, compute h_θ(x^{(i)}) = σ(θᵀ x^{(i)}).
  3. Compute the loss: Evaluate the logistic loss function J(θ) over all samples.
  4. Compute the gradient of loss: Calculate the partial derivatives of J(θ) w.r.t each θ_j.
  5. Update parameters: Adjust θ by moving in the direction opposite to the gradient (gradient descent).
  6. Repeat: Iterate steps 2–5 until parameters converge or a stopping criterion is met.

Example: Simplified Conceptual Illustration

Suppose we have a single feature x and a dataset consisting of points labeled 0 or 1."

We want to find parameters θ₀ and θ₁ so that the hypothesis

h_θ(x) = σ(θ₀ + θ₁ x)

outputs probability estimates near 1 for positive class points and near 0 for negative class points.

As training proceeds, the logistic cost function is minimized by adjusting θ coefficients, shaping the sigmoid curve to best separate classes by probability.

Once trained, for any new input x, we compute h_θ(x) and classify by thresholding at 0.5.

Interpretation of Coefficients

Each coefficient θ_j can be interpreted as how the log-odds of the outcome change with a one-unit increase in the feature x_j, holding all other variables constant.

In other words:

e^{θ_j} = multiplicative change in odds for a 1-unit increase in x_j.

For example, if θ_j = 0.5, then the odds are multiplied by e^{0.5} ≈ 1.65 when x_j increases by 1.

Multiclass Logistic Regression (Softmax Regression)

While the classical logistic regression applies to binary classification, extensions allow handling more than two classes.

Softmax regression, also called multinomial logistic regression, generalizes logistic regression to multiple classes by modeling the probability of each class as:

P(y = k | x) = e^{θ_kᵀ x} / ∑_{j} e^{θ_jᵀ x}

This model uses the softmax function to produce a valid probability distribution over classes.

Limitations of Logistic Regression

  • Linearity in the Parameters: Logistic regression assumes a linear relationship between the input features and the log-odds of the outcome. It cannot naturally model complex nonlinear relationships without feature engineering or basis expansions.
  • Sensitivity to Outliers: Extreme feature values can disproportionately influence parameter estimates.
  • Multicollinearity: Highly correlated input features can cause instability in the coefficient estimates.

Despite these limitations, logistic regression remains widely used due to its simplicity, interpretability, and effectiveness in many practical problems.
