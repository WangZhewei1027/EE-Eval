Understanding Linear Search

What is Linear Search?

Linear search, also known as sequential search, is a simple algorithm used for finding a specific value within a list or array. It checks each element one by one until the desired element is found or the end of the list is reached. This method is straightforward and easy to implement, making it suitable for small datasets.

How Does Linear Search Work?

The linear search algorithm follows these steps:

  • Start from the first element of the list.
  • Compare the current element with the target value.
  • If they match, return the index of the element.
  • If not, move to the next element in the list.
  • Repeat the process until the target value is found or the end of the list is reached.

Algorithm Example

Consider the following array: [4, 2, 7, 1, 3]. If we want to find the number 1, the steps would be as follows:

1. Start at index 0: Check 4 (not a match).

2. Move to index 1: Check 2 (not a match).

3. Move to index 2: Check 7 (not a match).

4. Move to index 3: Check 1 (match found!).

The target value 1 is found at index 3.

Time Complexity

The time complexity of a linear search is O(n), where n is the number of elements in the array. In the worst-case scenario, the algorithm needs to check every element in the list, which makes it less efficient for large datasets compared to other search algorithms like binary search.

When to Use Linear Search?

Linear search is often used in the following situations:

  • When the list or array is small.
  • When the list is unsorted, as linear search does not require the data to be sorted.
  • As a simple introductory algorithm for educational purposes to teach search techniques.
