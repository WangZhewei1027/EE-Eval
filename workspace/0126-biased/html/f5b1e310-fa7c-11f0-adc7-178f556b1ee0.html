Big-O Notation

Big-O notation is a way to measure the amount of time or space an algorithm takes as the input size increases.

Big-O notation is used to describe the time or space complexity of an algorithm. It helps us understand how an algorithm scales as the input size increases.

It's a measure of the worst-case scenario, which is the scenario that occurs most often.

Examples of Big-O notation include:

  • O(1)
  • O(log n)
  • O(n)
  • O(n log n)
  • O(n^2)
  • O(2^n)

Time Complexity

Time complexity is the amount of time an algorithm takes as the input size increases.

O(1) means the algorithm takes constant time, regardless of the input size.

O(log n) means the algorithm takes logarithmic time, as the input size increases by a factor of n.

O(n) means the algorithm takes linear time, as the input size increases by a factor of n.

O(n log n) means the algorithm takes time proportional to the product of the input size and its logarithm.

O(n^2) means the algorithm takes time proportional to the square of the input size.

O(2^n) means the algorithm takes time proportional to 2 raised to the power of the input size.

Space Complexity

Space complexity is the amount of space an algorithm uses as the input size increases.

O(1) means the algorithm uses constant space, regardless of the input size.

Example

Example: Bubble Sort

Bubble Sort has a time complexity of O(n^2) and a space complexity of O(1). This means that as the input size increases, Bubble Sort takes linear time and constant space.
