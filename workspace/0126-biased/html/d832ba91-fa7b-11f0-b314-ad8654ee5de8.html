Stack — Complete Educational Guide

A stack is one of the simplest and most powerful abstract data types. This page thoroughly explains what a stack is, how it works, common implementations, key algorithms that use stacks, step-by-step examples, pseudocode, complexities, and practical use-cases.

1. Definition and Core Concept

A stack is an abstract data type that represents a collection of elements with two principal operations:

  • Push(x): Insert element x onto the top of the stack.
  • Pop(): Remove and return the element at the top of the stack.

The stack follows the Last-In, First-Out (LIFO) discipline. The last element pushed onto the stack is the first one removed. Imagine a stack of plates: you add (push) plates to the top and remove (pop) from the top.

Other common operations

  • Peek / Top: Return (but do not remove) the top element.
  • isEmpty: Check whether the stack contains no elements.
  • size: Return the number of elements in the stack.
  • clear: Remove all elements.

2. Abstract Properties and Guarantees

A stack provides:

  • Deterministic LIFO access—only the top element is accessible in O(1) time.
  • Constant-time basic operations (in typical implementations): push, pop, peek often run in O(1).
  • Logical ordering used primarily to reverse sequences or remember intermediate states.

3. Implementations

Two common practical representations of a stack:

3.1 Array-based stack

Store elements in a contiguous array. Maintain an integer index (top) pointing to the next free slot or the current top element. 

// Pseudocode: array-based stack (0-based)
initialize array A of capacity CAP
top = 0  // number of elements, also index of next free slot

function push(x):
    if top == CAP:
        error "stack overflow"  // or resize array
    A[top] = x
    top = top + 1

function pop():
    if top == 0:
        error "stack underflow"
    top = top - 1
    return A[top]

function peek():
    if top == 0:
        error "empty"
    return A[top - 1]

Array stacks are simple and fast. If you allow automatic resizing (e.g., doubling capacity), amortized push time remains O(1).

3.2 Linked-list stack

Use a singly linked list where new nodes are inserted at the head. Each push creates a new node and points it to the previous head. 

// Pseudocode: linked-list based stack
top = null

function push(x):
    node = new Node(x)
    node.next = top
    top = node

function pop():
    if top == null: error "empty"
    value = top.value
    top = top.next
    return value

function peek():
    if top == null: error "empty"
    return top.value

Linked stacks avoid resizing but use extra memory per element for pointers.

4. Time and Space Complexity

  • push: O(1) time (amortized O(1) for dynamic array)
  • pop: O(1)
  • peek: O(1)
  • size/isEmpty: O(1)
  • Space: O(n) where n is number of elements (plus pointer overhead for linked implementation)

5. Common Use Cases and Examples

Stacks are ubiquitous. Several canonical applications:

5.1 Call stack (function invocation)

Programming languages use a call stack to manage active function calls. When a function is called, the runtime pushes a stack frame (arguments, local variables, return address). When the function returns, its frame is popped. This is why recursion depth is bounded by stack space.

5.2 Expression evaluation and parsing

Stacks are fundamental for converting infix expressions (like "3 + 4 * 2 / (1 - 5)") to postfix (Reverse Polish Notation) and then evaluating them. The Shunting Yard algorithm (by Dijkstra) uses an operator stack to manage operator precedence and parentheses.

5.3 Parentheses matching and syntax checking

To check balanced parentheses, push opening tokens and pop when you see a matching closing token. If anything mismatches, the expression is invalid.

5.4 Depth-first search (DFS) — iterative

DFS can be implemented using an explicit stack rather than recursion. This is particularly useful when recursion depth is limited or when you want explicit control over backtracking.

5.5 Undo/Redo

Many editors keep an "undo" stack of changes. For redo functionality, maintain two stacks: apply changes push to undo; when undoing, pop from undo and push to redo.

6. Key Algorithms Demonstrated (textual step-by-step)

6.1 Parentheses balancing (step-by-step)

Algorithm idea: iterate characters; push opening brackets; when a closing bracket appears, pop and check types match.

// Example string: "{ [ ( ) ] }"
stack = empty
for each char c in string:
  if c is opening bracket: push(c)
  else if c is closing bracket:
    if stack empty: invalid
    top = pop()
    if top and c not matching pair: invalid
after loop:
  if stack not empty: invalid else valid

Walkthrough for "{ [ ( ) ] }":

  1. Read '{' → push '{' → stack ['{']
  2. Read ' ' ignore whitespace
  3. Read '[' → push '[' → stack ['{','[']
  4. Read '(' → push '(' → stack ['{','[','(']
  5. Read ')' → pop '(' → matches → stack ['{','[']
  6. Read ']' → pop '[' → matches → stack ['{']
  7. Read '}' → pop '{' → matches → stack [] → balanced

6.2 Evaluate postfix (Reverse Polish) expressions

Postfix evaluation uses a value stack. When encountering a number, push it. On an operator, pop operands, apply operator, push result.

// Evaluate "3 4 + 2 * 7 /"
stack = []
read 3 => push 3         stack [3]
read 4 => push 4         stack [3,4]
read + => pop 4, pop 3   compute 3+4=7, push 7   stack [7]
read 2 => push 2                    stack [7,2]
read * => pop 2, pop 7   compute 7*2=14, push 14  stack [14]
read 7 => push 7                    stack [14,7]
read / => pop 7, pop 14  compute 14/7=2, push 2   stack [2]
end => result is 2

6.3 Shunting Yard (infix → postfix) — conceptual steps

High-level idea: use an operator stack; output queue collects numbers and final postfix tokens.

  1. Read tokens left to right.
  2. If token is a number/operand: append to output.
  3. If token is an operator (o1):
    • While top of operator stack is operator o2 with greater precedence, or equal precedence and o1 is left-associative, pop o2 to output.
    • Then push o1 to operator stack.
  4. If token is '(': push it.
  5. If token is ')': pop operators to output until '(' is popped.
  6. At end: pop remaining operators to output.

Walkthrough example: "3 + 4 * 2 / ( 1 - 5 ) ^ 2 ^ 3"

Detailed token-level demonstration is available in many resources; the important point is that the operator stack controls precedence and associativity to produce correct postfix order.

6.4 Iterative DFS on a graph (textual)

// Pseudocode using adjacency lists
function iterativeDFS(start):
  stack = empty
  push(start)
  visited = set()
  while stack not empty:
    node = pop()
    if node in visited: continue
    visit(node)
    add node to visited
    for each neighbor in adjacency[node]:
      if neighbor not in visited:
        push(neighbor)

Note: the neighbor iteration order affects traversal order; pushing neighbors in the reverse desired order yields the same order as recursive DFS.

7. Variants and Practical Considerations

7.1 Bounded vs Unbounded

An array-based stack may be bounded (fixed capacity). If you need growth, use dynamic resizing (double capacity when full). A linked-list stack is effectively unbounded until memory runs out.

7.2 Thread-safety and concurrency

In multithreaded programs, stacks used by many threads must be synchronized. Techniques:

  • Coarse-grained locks (mutex around push/pop)
  • Lock-free algorithms (compare-and-swap operations) for high-performance concurrent stacks

7.3 Memory and safety

When using stacks for recursion, be mindful of stack overflow (limited call-stack memory). For general stacks in programs, monitor memory usage when stacking large objects.

8. When Not to Use a Stack

Use a stack when LIFO semantics are appropriate. If you need FIFO behavior (first-in, first-out), use a queue. For random access, an array or list is better. For prioritized removal, use a priority queue.

9. Sample Code Snippets

These are minimal examples showing basic stack behavior in common languages.

Java (array-based, simplified)

class Stack {
  private int[] a;
  private int top = 0;
  public Stack(int cap) { a = new int[cap]; }
  public void push(int x) { if(top==a.length) throw new RuntimeException("overflow"); a[top++]=x; }
  public int pop() { if(top==0) throw new RuntimeException("empty"); return a[--top]; }
  public boolean isEmpty(){return top==0;}
}

Python (using list)

# Python list already supports stack operations:
stack = []
stack.append(1)   # push
x = stack.pop()   # pop
top = stack[-1]   # peek (if not empty)

JavaScript (using array)

const stack = [];
stack.push(10);     // push
const v = stack.pop(); // pop
const top = stack[stack.length - 1]; // peek

11. Practical Example: Undo / Redo with Two Stacks (conceptual)

Maintain two stacks: UndoStack and RedoStack.

  1. On user action A: push A onto UndoStack; clear RedoStack.
  2. On Undo: pop action from UndoStack, apply inverse action, push that inverse onto RedoStack.
  3. On Redo: pop from RedoStack, apply it, push corresponding inverse back to UndoStack.

This LIFO behavior ensures the most recent change is undone first.
