Overview & Intuition

K-Nearest Neighbors (KNN) is one of the simplest machine learning algorithms. It is instance-based and non-parametric: rather than learning a global model mapping features to targets, KNN keeps the training instances and makes predictions based on the labels or values of nearby training examples in feature space.

Key intuitive points:

  • If similar inputs have similar outputs, the label/value of nearby training points provides a good guess for a new input.
  • "Similarity" is quantified by a distance (or dissimilarity) metric in the feature space.
  • It is lazy learning: no explicit training phase (other than storing the data); prediction cost occurs at query time.

Formal definition

Given a training set of n labeled examples {(x_i, y_i)} for i = 1..n, where x_i is a vector of features and y_i is a label (discrete for classification or continuous for regression), and given a query point x_q, the KNN predictor works as follows:

  1. Compute distances d(x_q, x_i) between x_q and all training points.
  2. Select the K training points with the smallest distances (the K nearest neighbors).
  3. Aggregate their labels to make a prediction: majority vote for classification, average (or weighted average) for regression.

Notation:

Distance between two points: d(x, z).

Algorithm — Step-by-step textual description

At prediction time (single query x_q):

  1. Ensure the query point x_q is represented in the same feature space and scaled/encoded consistently with training data.
  2. Compute a distance (or similarity) between x_q and every training example x_i. Choose a metric appropriate to the data (Euclidean for continuous, Hamming for binary categorical, etc.).
  3. Sort the training examples by increasing distance to x_q, or use a selection algorithm to find the K smallest distances without full sorting.
  4. Select the top K neighbors. If ties occur at the boundary, decide a tie-breaking policy (e.g., smaller distances, random, or increase K).
  5. Aggregate neighbors' outputs:
    • Classification: majority vote among neighbors' labels. If using weights, weight each neighbor's vote by a function of distance.
    • Regression: compute the simple mean of neighbors' target values or a weighted mean.
  6. Return the aggregated prediction (and optionally confidence estimates like fraction of votes for the chosen class).

Notes on ties and deterministic behavior

To keep behavior deterministic, specify tie-breaking rules (e.g., use nearest average distance, fixed ordering), or use odd values of K for binary classification to reduce ties.

Distance metrics — which to use and why

The distance metric is crucial because it encodes the notion of "near". Common choices:

  • Euclidean distance (L2): d(x, z) = sqrt(sum_j (x_j - z_j)^2). Best for continuous, isotropic features when squared differences are meaningful.
  • Manhattan distance (L1): d(x, z) = sum_j |x_j - z_j|. Less sensitive to outliers; can be better if features are additive.
  • Minkowski distance: generalization: (sum_j |x_j - z_j|^p)^(1/p). p=2 -> Euclidean, p=1 -> Manhattan.
  • Cosine distance: 1 - (x·z) / (||x|| ||z||). Measures angle rather than magnitude; useful for text embeddings or high-dimensional sparse vectors.
  • Hamming distance: proportion or count of differing categorical positions. Used for binary/categorical data.
  • Gower distance: a mixed-type distance that can handle numeric, categorical, ordinal features.

Important: scale-sensitive metrics like Euclidean require features to be on comparable scales.

Choosing K and validation

The hyperparameter K controls smoothness:

  • Small K (e.g., 1) leads to low bias, high variance — the decision function can be noisy and sensitive to outliers.
  • Large K increases bias (over-smoothing) but reduces variance — predictions reflect broader neighborhood trends.

Typical strategies to choose K:

  • Cross-validation: test several K values (e.g., odd numbers up to sqrt(n)) and pick the one that optimizes validation performance.
  • Use domain knowledge: if the data is expected to have local structure, choose smaller K; for noisy labels, increase K.
  • Rule-of-thumb: K ≈ sqrt(n) is sometimes used as a starting point but is not a replacement for validation.

Weighted KNN

Instead of giving each neighbor equal weight, weight neighbors by their distance so closer neighbors influence the prediction more. Two common weighting functions:

  • Inverse distance: weight_i = 1 / (d(x_q,x_i) + epsilon). Epsilon prevents division by zero.
  • Gaussian (or other kernels): weight_i = exp(-d(x_q,x_i)^2 / (2*sigma^2)).

Weighted classification uses weighted votes; weighted regression uses weighted averages. Weighting often improves robustness in regions with varying density.

Classification vs Regression

Classification: labels are discrete classes. Predict the class with the highest (weighted) vote among neighbors. Optionally return class probabilities as normalized weighted votes.

Regression: labels are continuous. Predict the (weighted) mean of neighbors' target values. Use median if robust estimation desired.

Multi-class behavior: KNN naturally extends to multi-class by voting among neighbors. For imbalanced classes, weighting or different decision thresholds may be necessary.

Preprocessing & Practical considerations

Feature scaling

Scale numeric features (standardization to zero mean & unit variance, or min-max scaling) when using distance metrics sensitive to scale (Euclidean, Manhattan). Without scaling, features with larger ranges dominate distances.

Encoding categorical variables

Binary/one-hot encoding for nominal categories works with Euclidean or Hamming distance. For high-cardinality categories, careful encoding or specialized distances (Gower) may be preferable.

Missing values

Impute missing values (mean/mode, KNN imputation itself, or model-based imputation) before applying KNN. Alternatively, use distances that ignore missing components and normalize by the number of available dimensions.

Dimensionality

KNN suffers in high dimensions (curse of dimensionality). Distances become less informative as dimensionality grows. Dimensionality reduction (PCA, feature selection) can help.

Feature selection

Remove irrelevant or redundant features. KNN treats all features equally unless you introduce feature weighting or use learned metric learning.

Imbalanced classes

In classification with imbalance, the majority class may dominate local neighborhoods. Use weighted votes, stratified sampling, or adjust K per class or use different distance metrics.

Computational complexity & optimizations

Brute-force KNN requires computing distances to all n training examples for each query and sorting/selecting K. Complexity per query:

  • Distance computation: O(n * d) where d is dimensionality.
  • Selecting K smallest: O(n log n) if sorting, or O(n) expected using selection algorithms or heaps (maintain size-K max-heap).

Thus naive query cost is O(n d). Space complexity is O(n d) to store training data.

Optimizations

  • KD-Trees / Ball-Trees: partition space to reduce candidate points. Effective in low to moderate dimensions (~d < 20).
  • Approximate Nearest Neighbors (ANN): algorithms like locality-sensitive hashing (LSH), Annoy, FAISS trade accuracy for speed in high dimensions.
  • Precompute and cache distances for repeated queries if appropriate.
  • Use optimized BLAS libraries and vectorized operations for distance computations.

Pseudocode & Implementation notes

Simple pseudocode for classification KNN:

function KNN_Predict(x_q, TrainingSet, K, distance):
    distances = []
    for each (x_i, y_i) in TrainingSet:
        d = distance(x_q, x_i)
        append (d, y_i) to distances
    sort distances by d ascending
    neighbors = first K entries of distances
    // majority vote (optionally weighted)
    votes = {}
    for each (d, y) in neighbors:
        weight = 1 / (d + epsilon)   // or weight = 1 for unweighted
        votes[y] = votes.get(y, 0) + weight
    return argmax_y votes[y]

Implementation tips:

  • Vectorize distance computations when possible (matrix operations) for speed.
  • Handle zero distances (exact matches) with care — you might immediately return the matched label or use epsilon in weights.
  • When memory is limited, consider streaming or disk-backed structures for extremely large datasets.

Short Python example (illustrative)

# Very small illustrative KNN
def euclidean(a,b):
    return sum((ai-bi)**2 for ai,bi in zip(a,b))**0.5

def knn_predict(x_q, X_train, Y_train, K=3):
    dist_label = [(euclidean(x_q, x), y) for x,y in zip(X_train, Y_train)]
    dist_label.sort(key=lambda t: t[0])
    neighbors = dist_label[:K]
    # unweighted majority vote
    votes = {}
    for d,y in neighbors:
        votes[y] = votes.get(y,0)+1
    return max(votes.items(), key=lambda kv:kv[1])[0]

Worked example — manual calculation (classification)

We walk through a small 2D example to make the mechanics clear.

Training points (x, y), label:

PointCoordinates (x1, x2)Label
A(1.0, 2.0)Red
B(2.0, 1.0)Red
C(3.0, 4.0)Blue
D(6.0, 5.0)Blue
E(7.0, 3.0)Blue

Query point Q = (3.5, 3.0). We'll use Euclidean distance and K = 3.

Step-by-step distances:

  1. d(Q, A) = sqrt((3.5-1.0)^2 + (3.0-2.0)^2) = sqrt(2.5^2 + 1^2) = sqrt(6.25 + 1) = sqrt(7.25) ≈ 2.6926
  2. d(Q, B) = sqrt((3.5-2)^2 + (3-1)^2) = sqrt(1.5^2 + 2^2) = sqrt(2.25 + 4) = sqrt(6.25) = 2.5
  3. d(Q, C) = sqrt((3.5-3)^2 + (3-4)^2) = sqrt(0.5^2 + (-1)^2) = sqrt(0.25 + 1) = sqrt(1.25) ≈ 1.1180
  4. d(Q, D) = sqrt((3.5-6)^2 + (3-5)^2) = sqrt((-2.5)^2 + (-2)^2) = sqrt(6.25 + 4) = sqrt(10.25) ≈ 3.2016
  5. d(Q, E) = sqrt((3.5-7)^2 + (3-3)^2) = sqrt((-3.5)^2 + 0) = 3.5

Sorted by distance: C (Blue, 1.118), B (Red, 2.5), A (Red, 2.693), D (Blue, 3.202), E (Blue, 3.5).

K=3 nearest neighbors: C (Blue), B (Red), A (Red). Majority vote: Red (2 votes) vs Blue (1 vote) ⇒ Predict "Red".

If we used weighted voting with inverse distance weights: weight(C)=1/1.118≈0.895, weight(B)=1/2.5=0.4, weight(A)=1/2.693≈0.371. Total Red weight ≈ 0.4 + 0.371 = 0.771; Blue weight ≈ 0.895 ⇒ Predict "Blue" under inverse-distance weighting because the closest Blue point dominates.

Practical tips, trade-offs and pitfalls

  • Scale your features — without scaling, certain features may dominate the distance computation and ruin performance.
  • Beware of high dimensionality — distances become less informative in high dimensions; reduce dimensionality when needed.
  • Use appropriate distance for data type — mixing categorical and continuous features requires care or specialized distances.
  • Consider approximate methods for large datasets: exact KNN can be too slow at query time.
  • Model interpretability: KNN is easy to explain: predictions are based on actual nearby training examples — you can inspect them directly.
  • Storage & memory: KNN stores all training data, so memory can be large for big datasets.
