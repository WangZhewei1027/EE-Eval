Virtual Memory

Virtual memory is a key component of computer architecture that allows a computer to use both physical RAM and hard disk storage simultaneously.

It is a memory management system that manages the allocation and deallocation of memory on the computer's physical RAM, as well as the storage of data on the hard disk.

How it Works

When a program is running, it allocates memory on the physical RAM. However, as the program uses more and more memory, the physical RAM becomes full. To address this, the operating system checks if there is enough free memory on the physical RAM and if there is enough free space on the hard disk.

If there is enough free memory, the operating system can continue to use the program as usual. However, if there is not enough free memory, the operating system will request more physical RAM from the system's hard disk.

Algorithms

There are several algorithms used by the operating system to manage virtual memory, including:

  • Moving
  • Paging
  • Swapping

Moving is an algorithm that involves moving a program from the physical RAM to the hard disk when the physical RAM becomes full. This is done by mapping the program to a contiguous block of free physical RAM.

Paging is an algorithm that involves dividing the physical RAM into smaller blocks and mapping each program to a different block. This is done to prevent a single program from using up too much memory at once.

Swapping is an algorithm that involves moving a program from the physical RAM to the hard disk when the physical RAM becomes full. This is done by mapping the program to a block of free physical RAM on the hard disk.

Example

Let's say we have a program that uses 1000 bytes of physical RAM. If we run the program for 10 minutes, it will use up the entire block of physical RAM. To address this, the operating system will request 1000 more bytes of physical RAM from the system's hard disk.
