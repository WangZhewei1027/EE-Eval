Prim's Algorithm — Minimum Spanning Tree (MST)

A complete, textual and pedagogical guide explaining the idea, correctness, algorithms, examples, pseudocode, complexity, and practical notes about Prim's algorithm. The page is primarily static text; a single button reveals a short worked demonstration for a small fixed graph.

What is the problem?

Given an undirected, connected graph G = (V, E) with a weight function w: E → ℝ (weights are real numbers, typically non-negative), a spanning tree is a subset of edges T ⊆ E that connects all vertices (it is acyclic and has exactly |V|−1 edges). A minimum spanning tree (MST) is a spanning tree whose total weight (sum of the weights of edges in the tree) is as small as possible.

High-level idea of Prim's algorithm

Prim's algorithm builds an MST by growing a single tree one edge at a time. Start from an arbitrary vertex. At each step, among all edges that have exactly one endpoint in the tree (the "cut" edges or the current frontier), pick the edge of minimum weight and add it to the tree, bringing a new vertex into the tree. Continue until all vertices are included.

Intuition: imagine expanding a region that initially contains one vertex. At every step, extend the region using the cheapest edge that reaches a vertex not yet in the region. Because the algorithm only adds edges connecting the region to outside vertices and always picks the cheapest such edge, the resulting edge set will form an MST (proof sketch below).

Key definitions and terminology

  • Frontier / cut edges: edges with one endpoint inside the currently built tree and the other endpoint outside.
  • Greedy choice: choose the minimum-weight edge crossing the cut.
  • Priority queue: a data structure commonly used to implement Prim's algorithm efficiently; it keeps candidate edges or candidate vertex keys ordered by smallest weight.
  • Key of a vertex: the smallest edge weight that connects that vertex to the tree so far. The algorithm maintains these keys and updates them as the tree grows.

Step-by-step algorithmic description (textual)

Below is a detailed stepwise narrative of what happens during one run of Prim's algorithm.

  1. Pick any start vertex r. Mark it as part of the growing tree T. Initialize a set S = {r}.
  2. For every vertex v not in S, maintain a value key[v] = +∞ and a parent pointer parent[v] = null.
  3. For every neighbor u of r, set key[u] = weight(r,u) and parent[u] = r. (u becomes a candidate to add.)
  4. Repeat until S contains all vertices:
    1. Select a vertex v not in S with the minimum key[v]. This is the cheapest way to connect v to S using a single edge.
    2. Add v to S and add edge (parent[v], v) to the MST.
    3. For every neighbor x of v that is not in S: if weight(v,x) < key[x], update key[x] = weight(v,x) and parent[x] = v. This updates the frontier / candidate edges.

At termination, the set of chosen edges (parent pointers for all vertices except the root) forms an MST for the connected component containing the start vertex. If the graph is disconnected, running Prim starting from one vertex produces a minimum spanning forest for that component; repeat for other components to obtain a full MST forest.

Pseudocode (priority-queue based)

Prim(G = (V,E), w):
  choose arbitrary r in V
  for each v in V:
    key[v] = +∞
    parent[v] = null
    inTree[v] = false
  key[r] = 0

  // Priority queue keyed by key[v]; supports decrease-key
  Q = all vertices V in priority queue keyed by key[v]

  while Q not empty:
    u = extract-min(Q)          // vertex with smallest key
    inTree[u] = true
    if parent[u] != null:
      add edge (parent[u], u) to MST
    for each neighbor v of u:
      if not inTree[v] and w(u,v) < key[v]:
        key[v] = w(u,v)
        parent[v] = u
        decrease-key(Q, v, key[v])

  return the set of edges formed by parent pointers (excluding root)

Remarks about the implementation

  • Priority queue implementation matters for performance. Common choices:
    • Binary heap or pairing heap with adjacency list: O(E log V) time.
    • Fibonacci heap: O(E + V log V) time (theoretical improvement, less used in practice).
    • Dense graphs (e.g., adjacency matrix): simple array-based selection of min key gives O(V^2) time and is often simpler and faster for dense graphs.
  • Edge weights may be equal; Prim's algorithm still produces a minimum spanning tree, though there can be multiple MSTs.
  • Prim's algorithm works with negative weights as well (no problem), provided the graph is undirected and connected. Negative edge weights don't break the greedy reasoning.
  • For disconnected graphs, the algorithm builds a minimum spanning forest. To produce a full forest, run Prim from any unvisited vertex until all vertices are included.

Correctness sketch (why Prim's greedy choice is safe)

We provide an informal proof using the cut property and an exchange argument.

Cut property: For any partition of vertices into two sets (S, V\S), the minimum-weight edge crossing the cut (an edge with one endpoint in S and one in V\S) belongs to some MST. Prim's algorithm always picks such an edge where S is the current tree. Therefore, each choice can be extended to an MST.

Exchange argument: Suppose we have an MST T* and Prim's algorithm picks edge e across the cut. If e is already in T* we are fine. If e is not, adding e to T* creates a cycle; that cycle must contain at least one other edge f that also crosses the cut (because e crosses the cut but all other edges in T* connect vertices within S or within V\S except f). Because e is the minimum-weight edge across the cut, w(e) ≤ w(f). If w(e) < w(f) we can replace f with e to get another spanning tree with no larger total weight. Thus, we can transform T* step by step to include Prim's choices without increasing total weight, proving Prim's choices never prevent building an MST.

Complexity summary

  • Using binary heap or similar with adjacency lists: O(E log V) time, O(V + E) space.
  • Using Fibonacci heap: O(E + V log V) time (theoretical).
  • Using a simple array (no heap) is O(V^2) time and O(V + E) space — often simpler and efficient for dense graphs where E ≈ V^2.

Comparison with Kruskal's algorithm

Both Prim's and Kruskal's produce Minimum Spanning Trees using greedy choices but proceed differently:

  • Prim's: Grows a single tree by choosing the cheapest edge crossing from the tree to outside vertices (like a region expansion). Uses a priority queue of candidate vertices/edges keyed by cheapest incident weight.
  • Kruskal's: Considers edges globally in non-decreasing weight order and adds an edge if it doesn't form a cycle (using union-find / disjoint set union). It can produce a forest that later becomes a spanning tree.

Which to use?

  • Prim's is often preferable for dense graphs and when you have a good adjacency structure and want a tree rooted at a particular vertex.
  • Kruskal's is often simpler to implement with sorted edges and is attractive if edges are already sorted or when using union-find is straightforward.

Sample implementations and notes

    
Here is concise sample code in Python (adjacency list + heapq). This is meant to illustrate the structure; production code often uses decrease-key via additional bookkeeping or specialized heaps.

    
# Python (heap-based, lazy updates)
import heapq

def prim_mst(adj, start=0):
    # adj: adjacency list as dict or list of lists: adj[u] = [(v,w), ...]
    n = len(adj)
    visited = [False]*n
    parent = [None]*n
    key = [float('inf')]*n
    key[start] = 0
    heap = [(0, start)]  # (key, vertex)
    mst_edges = []

    while heap:
        k,u = heapq.heappop(heap)
        if visited[u]: continue
        visited[u] = True
        if parent[u] is not None:
            mst_edges.append((parent[u], u, k))
        for v,w in adj[u]:
            if not visited[v] and w < key[v]:
                key[v] = w
                parent[v] = u
                heapq.heappush(heap, (w, v))   # lazy; may push duplicates

    return mst_edges
