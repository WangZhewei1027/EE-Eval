PAGING

The Concept of Paging

Paging is a memory management scheme that eliminates the need for contiguous allocation of physical memory. This allows physical address space of a process to be noncontiguous, solving the problems of fitting varying sized chunks of data into the memory efficiently.
PAGE 1 / 5

Logical & Physical Pages

The process is divided into fixed-size pages, and the physical memory is divided into fixed-size frames. Pages are mapped onto frames dynamically, allowing better memory utilization and avoiding fragmentation.
PAGE 2 / 5

Page Table

Each process has a page table which keeps track of the mapping between the page number and the frame number where it resides. It acts as a directory, facilitating efficient address translation.
PAGE 3 / 5

Page Faults

When a process accesses a page not currently in a frame, a page fault occurs. The operating system then loads the required page into memory, possibly replacing another page using an algorithm like LRU (Least Recently Used).
PAGE 4 / 5

Benefits of Paging

Paging minimizes external fragmentation, supports the efficient and safe sharing of memory, and simplifies the loading of processes by allowing noncontiguous memory allocation.
