Depth-First Search (DFS)

A comprehensive, text-driven explanation of Depth-First Search: theory, algorithmic descriptions, step-by-step walkthroughs, edge classifications, complexity, applications, and implementation notes.

What is Depth-First Search?

Depth-First Search (DFS) is a fundamental graph traversal algorithm used to explore nodes and edges of a graph systematically. Starting from a source vertex, DFS explores as far as possible along each branch before backtracking. It can be implemented recursively (natural with the call stack) or iteratively using an explicit stack. DFS applies to both directed and undirected graphs.

Key idea: go deep first, backtrack when you cannot continue, and repeat until all reachable vertices are visited.

Where DFS is used

  • Discovering connected components (undirected graphs).
  • Topological sorting of DAGs (via DFS postorder).
  • Cycle detection (directed and undirected graphs).
  • Finding strongly connected components (Kosaraju's algorithm uses DFS twice).
  • Finding articulation points and bridges (Tarjan-style DFS).
  • Maze solving and backtracking search where exploring a branch completely before others is desired.

Formal description and terminology

Consider a graph G = (V, E) where V is a set of vertices and E is a set of edges (directed or undirected). DFS maintains a set of visited vertices and explores edges leaving each vertex to find new vertices.

Important concepts

  • Visited / discovery: When DFS first reaches a vertex, it is marked discovered (visited).
  • Finish: When DFS has explored all edges leaving a vertex and backtracks, that vertex is finished.
  • Discovery/finish times: We often record timestamps (integers) when a vertex is discovered and finished.
  • DFS tree: The parent pointers established during DFS form a forest (collection of trees) that spans the reachable vertices.
  • Edge classification: During DFS we can classify edges as tree, back, forward, or cross edges (useful in directed graphs).

Discovery and finish times

If we increment a global time counter on discovery and finish, each vertex v receives a pair (d[v], f[v]) where d is discovery time and f is finish time. For any vertex v, d[v] < f[v]. For any two vertices u and v, their intervals [d[u], f[u]] either are disjoint or one is nested inside the other, which corresponds to ancestor/descendant relations in the DFS tree.

Algorithms and pseudocode

Recursive DFS (standard)

Recursive DFS is the simplest to express. It uses the program call stack to remember the path taken.


// Pseudocode: recursive depth-first search
procedure DFS(G):
    for each vertex u in G.V:
        color[u] ← WHITE        // WHITE = unvisited
        parent[u] ← NIL
    time ← 0
    for each vertex u in G.V:
        if color[u] = WHITE:
            DFS-Visit(u)

procedure DFS-Visit(u):
    color[u] ← GRAY           // GRAY = discovered (in recursion stack)
    time ← time + 1
    d[u] ← time               // discovery time
    for each v in Adj[u]:
        if color[v] = WHITE:
            parent[v] ← u
            DFS-Visit(v)
    color[u] ← BLACK          // BLACK = finished
    time ← time + 1
    f[u] ← time               // finish time

Colors: WHITE = not visited, GRAY = discovered but not finished, BLACK = finished. This color scheme helps edge classification.

Iterative DFS using a stack

The iterative version simulates the recursion with an explicit stack. There are multiple valid ways to write it; one common pattern pushes the (node, iterator) pair to preserve which neighbors were already explored.


// Pseudocode: iterative DFS (stack with iterator state)
procedure DFS-Iterative(G):
    for each u in G.V:
        color[u] ← WHITE
        parent[u] ← NIL
    time ← 0
    for each u in G.V:
        if color[u] = WHITE:
            push stack (u, iterator over Adj[u])
            color[u] ← GRAY
            time ← time + 1
            d[u] ← time
            while stack is not empty:
                (x, it) ← top of stack
                if it has next neighbor v:
                    if color[v] = WHITE:
                        parent[v] ← x
                        color[v] ← GRAY
                        time ← time + 1
                        d[v] ← time
                        push (v, iterator over Adj[v])
                    else:
                        // optionally classify edge (x,v) using colors/times
                        // do nothing if already discovered/finished
                        continue
                else:
                    // all neighbors processed for x
                    pop stack
                    color[x] ← BLACK
                    time ← time + 1
                    f[x] ← time

Simple Python-like implementations

# Recursive (Python-like)
def dfs(graph):
    color = {u: 'white' for u in graph}
    parent = {u: None for u in graph}
    d, f = {}, {}
    time = 0

    def visit(u):
        nonlocal time
        color[u] = 'gray'
        time += 1
        d[u] = time
        for v in graph[u]:
            if color[v] == 'white':
                parent[v] = u
                visit(v)
        color[u] = 'black'
        time += 1
        f[u] = time

    for u in graph:
        if color[u] == 'white':
            visit(u)
    return parent, d, f

Many real implementations omit explicit color arrays and use boolean visited sets, but colors provide more power for edge classification.

Edge classification

While performing DFS on directed graphs, edges can be classified with respect to the DFS tree using discovery/finish times or colors:

  • Tree edge: an edge that is used by DFS to discover a new vertex (u → v when v is WHITE).
  • Back edge: an edge from a vertex to an ancestor in the DFS tree (u → v where v is GRAY). Back edges indicate cycles in directed graphs.
  • Forward edge: a non-tree edge from a vertex to a descendant in the DFS tree (u → v where v is BLACK and d[u] < d[v]).
  • Cross edge: any other edge between nodes in different branches (can appear in directed graphs).

In undirected graphs, every non-tree edge is effectively a back edge (because the undirected edge connects to an ancestor). Careful handling is needed to avoid classifying the parent edge as a back edge.

Complexity and resource usage

Time complexity depends on how the graph is represented:

  • Adjacency list: O(|V| + |E|) time — every vertex is discovered once and every edge is explored once (directed) or twice (undirected, counted appropriately).
  • Adjacency matrix: O(|V|^2) time — scanning all possible neighbors per vertex.

Space complexity:

  • Auxiliary memory: O(|V|) for visited array/color, parent pointers, and discovery/finish arrays.
  • Stack (recursion or explicit): O(|V|) in the worst case (path graph) for deepest recursion depth.

When DFS is expensive

If the graph is extremely deep, recursion can overflow the call stack; prefer an explicit stack or tail recursion elimination if available. If many vertices have large adjacency lists but few reachable nodes from a source, you may avoid scanning unreachable parts by starting from specific sources.

Proof sketch of correctness

DFS explores reachable vertices from a starting vertex by always following unexplored edges until no more unvisited vertices are found on that path, then backtracking. This process ensures that every vertex in the same connected component (undirected) or reachable via directed paths (directed) will be discovered.

In more formal terms: DFS only marks a vertex visited when it is first reached by a path of tree edges from the root. For any vertex v reachable from u, there exists a path u → ... → v; DFS follows that path (since each step explores new, unvisited neighbors) until v is discovered.

Applications and variations

Topological sort

In a directed acyclic graph (DAG), performing DFS and recording vertices in order of decreasing finish times yields a valid topological ordering. Implementation: push each vertex onto a stack when it finishes (f[v] assigned). After DFS completes, popping the stack gives a topological sort.

Cycle detection

For directed graphs, encountering a back edge (edge to a GRAY vertex) implies a directed cycle. For undirected graphs, encountering an edge to a visited vertex that is not the parent implies a cycle.

Strongly connected components (SCC)

Kosaraju's algorithm: run DFS on G, compute finish times, reverse graph G^T, then process vertices in order of decreasing finish times and run DFS again to collect SCCs.

Articulation points and bridges

Tarjan-style DFS computes low values (the earliest discovery reachable from a subtree via a back edge). Using discovery times and low values we can detect articulation points (cut vertices) and bridges (cut edges) in O(V+E) time.

Backtracking and puzzle solving

DFS is the natural choice when a solution requires exploring an entire branch before trying the next one (e.g., depth-first backtracking for constraint satisfaction problems).

Implementation notes and practical tips

  • Prefer adjacency lists for sparse graphs (space O(V+E) and time O(V+E)).
  • Be careful with recursion depth: use iterative approach for very deep graphs or increase recursion limit where safe.
  • When performing edge classification, maintain either color states or discovery/finish times for robust checks.
  • For undirected graphs, when seeing a neighbor v of u that is visited, ignore the case when v is the parent of u — this is the same undirected edge in reverse.
  • If ordering matters (e.g., lexicographically smallest topological order), sort adjacency lists before running DFS.
  • When multiple starting points are needed, run DFS from each unvisited vertex in the outer loop to get a full forest covering all components.

Common pitfalls

  • Forgetting to mark nodes as visited immediately on stack push (iterative) or on entry (recursive). This can cause an infinite loop.
  • Mixing up discovery and finish times when classifying edges — use the correct inequalities.
  • Using DFS where breadth-first properties are needed (e.g., shortest path in unweighted graph) — BFS is the correct tool for shortest unweighted paths.
