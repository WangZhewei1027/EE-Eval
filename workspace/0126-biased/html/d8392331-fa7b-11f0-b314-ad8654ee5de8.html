Routing — A Comprehensive Educational Guide

Definitions, theory, algorithms, protocol overviews, step-by-step descriptive examples, and a focused demonstration of a classical shortest-path algorithm.

What is Routing? — A Precise Definition

Routing is the process of selecting paths in a network along which to send data from a source to a destination. At its core, routing decides "which next hop" a packet or datagram should take so that it ultimately reaches its intended endpoint.

Routing appears in multiple contexts:

  • Packet routing in computer networks (IP routing).
  • Routing in distributed systems and overlay networks.
  • Pathfinding on graphs (transportation, robotics).
  • Application-level routing (web frameworks, client-side routers).

This guide focuses primarily on routing in packet-switched networks and related routing algorithms for path selection, but many principles apply across contexts.

Core Concepts and Vocabulary

Node (router, switch)
Device that forwards data. Maintains a routing table and performs forwarding based on destination addresses.
Link (edge)
Physical or logical connection between nodes with attributes (bandwidth, delay, cost).
Routing table
Local data structure that maps destination prefixes to next-hops and interface choices.
Forwarding
The act of sending a packet out a specific interface according to the routing table.
Control plane vs. data plane
The control plane discovers and maintains routes; the data plane forwards packets using installed rules.
Metric
A value representing path quality (cost, delay, hop count, bandwidth) used to choose among routes.

Routing Goals and Tradeoffs

Key objectives when designing or choosing routing strategies:

  • Correctness: packets reach the intended destination.
  • Optimality: choose a "best" path according to a metric.
  • Scalability: handle growing network size without excessive overhead.
  • Convergence: adapt to changes (failures) quickly and reliably.
  • Stability and policy: routes reflect administrative policies and avoid oscillation.

Tradeoffs occur: faster convergence might require more bandwidth for protocol messages; shortest-path optimality might conflict with administrative policies; global optimality may be expensive to compute in large networks.

Control Plane vs Data Plane — A Clarifying Note

Modern architectures separate concerns:

  • Control plane runs routing algorithms, exchanges routing information, and populates routing tables (e.g., OSPF, BGP).
  • Data plane forwards packets at wire speed using entries provisioned by the control plane.

Note: Software-Defined Networking (SDN) centralizes part of the control plane (a controller) to compute forwarding state for many devices; the data plane remains on network devices.

Types of Routing Algorithms (Broad Categories)

Routing algorithms vary in how they learn and compute routes. Principal categories:

  • Shortest-path algorithms compute minimal-cost paths on a graph (Dijkstra, Bellman-Ford, Floyd-Warshall).
  • Distance-vector (distributed): nodes exchange distance vectors containing distances to destinations; each node updates via Bellman-Ford style relaxation (example: RIP).
  • Link-state (distributed): nodes flood link-state information and compute shortest paths locally using Dijkstra (example: OSPF, IS-IS).
  • Path-vector: similar to distance-vector but carries path information to avoid loops and express policy (example: BGP).
  • Hierarchical routing: aggregate and summarize to scale (Internet uses hierarchical addressing and AS-level routing).

Shortest-Path Algorithms — Theory & Algorithms

Many routing decisions reduce to solving a shortest-path problem on a weighted graph G = (V, E) where edge weights model cost.

Dijkstra's Algorithm (single-source, non-negative weights)

Dijkstra finds the shortest path from a source node s to all other nodes when all edge weights are non-negative.

High-level steps (descriptive):

  1. Initialize distances: dist[s] = 0, dist[v] = ∞ for v ≠ s. Mark all nodes as unvisited. Set predecessor array to reconstruct paths.
  2. Repeat: select the unvisited node u with the smallest dist[u] and mark it visited (this node's shortest distance is finalized).
  3. For each neighbor v of u, consider the path through u: newDist = dist[u] + weight(u,v). If newDist < dist[v], update dist[v] and set predecessor[v] = u.
  4. Stop when all nodes are visited or the smallest unvisited dist is ∞ (unreachable nodes remain ∞).

Complexity:

  • With a binary heap: O((|V| + |E|) log |V|).
  • With a Fibonacci heap: O(|E| + |V| log |V|).

Bellman–Ford Algorithm (single-source, allows negative weights)

Bellman–Ford handles graphs with negative edge weights (but no negative cycles reachable from the source). It relaxes every edge |V|-1 times.

Descriptive step-by-step:

  1. Initialize dist[s] = 0, others ∞.
  2. Repeat |V|-1 times: for each edge (u,v) with weight w, if dist[u] + w < dist[v], set dist[v] = dist[u] + w and pred[v] = u.
  3. Optionally, run one more iteration: if any distance improves, a negative cycle exists.

Complexity: O(|V|·|E|). Used in distributed distance-vector scenarios conceptually (Bellman-Ford forms the basis of the distance-vector update rule).

Floyd–Warshall (all-pairs shortest paths)

Floyd–Warshall computes shortest paths between all pairs of nodes via dynamic programming. It is simple but cubic time: O(|V|^3). Useful for dense graphs or theoretical analysis. 

Simple pseudocode — Dijkstra (descriptive)
Given: Graph G(V,E) with non-negative weights w(u,v). Source s.
Initialize:
  for v in V:
    dist[v] = +∞
    prev[v] = undefined
  dist[s] = 0
  Q = set of all nodes (or priority queue keyed by dist)

While Q not empty:
  u = node in Q with minimum dist[u]
  remove u from Q
  for each neighbor v of u:
    alt = dist[u] + w(u,v)
    if alt < dist[v]:
      dist[v] = alt
      prev[v] = u
End

Distance-Vector vs Link-State — Conceptual Comparison

Two broad distributed approaches to dynamic routing:

Distance-Vector

  • Each router periodically sends its distance vector (best-known distances) to neighbors.
  • Routers update their own vector by considering neighbors' reports (Bellman-Ford relaxation).
  • Advantages: simple, low computational requirements.
  • Disadvantages: slow convergence, susceptible to routing loops and count-to-infinity problems; requires loop-avoidance tricks (split horizon, route poisoning).

Link-State

  • Each router floods the network with link-state advertisements describing its neighbors and link costs.
  • Every router builds a complete topology map and runs Dijkstra locally to compute shortest paths.
  • Advantages: faster convergence, global view, easier to reason about optimality.
  • Disadvantages: more memory and CPU, control-plane flooding overhead (scales with topology size).

Routing Protocols — Practical Overviews

RIP (Routing Information Protocol)

An early distance-vector protocol. Uses hop count as metric (max 15). Simple but limited scale. Implements periodic updates and split horizon/poison reverse to mitigate loops.

OSPF (Open Shortest Path First)

A widely used intra-domain link-state protocol. Routers flood link-state advertisements (LSAs). Supports areas to scale, route types (intra-area, inter-area), and multiple metrics. Uses Dijkstra for shortest-path trees.

BGP (Border Gateway Protocol)

The Internet's inter-domain path-vector protocol. BGP exchanges reachability (prefixes) along with path attributes (AS path, MED, local-pref) and the full AS path — enabling policy-based route selection rather than pure metric optimization.

IS-IS

Another link-state protocol similar to OSPF, originally designed for ISO networks but widely used in large IP networks. Also supports hierarchical areas and flexible metric settings.

Routing Table Structure and Lookup

In IP routing, a routing table contains entries mapping destination prefixes (e.g., 10.1.0.0/16) to next-hops and outgoing interfaces. Lookup uses longest-prefix match: the most specific route that matches the packet's destination is selected.

Key fields in a routing entry:

  • Destination prefix and mask
  • Next hop (IP or interface)
  • Metric and administrative distance
  • Route origin (protocol that installed it)
  • Timestamp, lifetime, and policy tags

Loop Avoidance and Convergence Techniques

Common strategies:

  • Split horizon and poison reverse (distance-vector): avoid sending an advertisement back on the interface from which it was learned.
  • Hold-down timers: delay acceptance of potentially unstable routes.
  • Sequence numbers and versioning (link-state): ensure freshness of flooded updates.
  • Path information (BGP): use AS path to detect and prevent loops.

Convergence speed depends on protocol design; link-state typically converges faster at the cost of higher flooding overhead.

Hierarchical Routing and Aggregation

To scale, networks use hierarchy: aggregate prefixes and split the network into areas or autonomous systems. Aggregation reduces the size of routing tables and control-plane traffic but may hide detail needed for finer-grained path selection.

Other Routing Types

Multicast

Routing mechanisms to deliver a single source stream to multiple receivers (e.g., PIM). Multicast routing maintains distribution trees rather than per-destination shortest paths.

Anycast

Same address announced from multiple locations; routing directs the packet to the "closest" instance based on routing metrics, used for redundancy and load distribution.

Ad-hoc and Mobile Routing

Special routing algorithms for networks with dynamic topology (e.g., MANETs): AODV, DSR, OLSR. These emphasize quick route discovery and reactiveness to topology changes.

Metrics and Policy

Metrics quantify "cost" — hop count, delay, jitter, loss, monetary cost, or composite values. Administrative policy, however, often overrides pure metric-based decisions: operators prefer specific links for commercial, security, or regulatory reasons.

Advanced Topics — MPLS and SDN

Multiprotocol Label Switching (MPLS) forwards packets based on short fixed-length labels rather than destination addresses. Labels are used to implement traffic engineering and fast reroute.

SDN centralizes route computation: a controller computes paths according to global network objectives and programs forwarding devices via a southbound API (e.g., OpenFlow). This separation enables flexible policies and optimization at the cost of new failure modes and controller scalability considerations.

Sample Pseudocode — Bellman-Ford (Descriptive)

Bellman-Ford (single-source)
Given: source s, edges E
Initialize:
  for each vertex v:
    dist[v] = +∞
    prev[v] = undefined
  dist[s] = 0

for i from 1 to |V|-1:
  for each edge (u, v, w) in E:
    if dist[u] + w < dist[v]:
      dist[v] = dist[u] + w
      prev[v] = u

Troubleshooting & Practical Considerations

  • When routes do not converge, inspect timers, update intervals, and check for loops or persistent oscillation due to conflicting policies.
  • High CPU on routers can result from too-frequent flooding or large topologies — use areas or aggregation and tune timers.
  • In BGP, incorrect prefix announcements can cause widespread reachability issues — validate with RPKI and filters.
  • Use multiple metrics and link monitoring to avoid using a path that is "short" in hops but suffers from high packet loss.

Guided Exercises for the Reader

  1. Manually run Dijkstra on a seven-node graph with varying weights. Track distance and predecessor arrays on paper.
  2. Implement a simple distance-vector simulator: nodes exchange distance vectors; observe count-to-infinity on a topology with a failed link, then implement split-horizon and poison reverse to see the effect.
  3. Study BGP path selection rules: given multiple available paths with different AS paths, local preference, and MED, determine the selected route.
  4. Explore tradeoffs: design a metric that favors low-latency paths but penalizes links that are heavily utilized; discuss consequences on load balancing and stability.
