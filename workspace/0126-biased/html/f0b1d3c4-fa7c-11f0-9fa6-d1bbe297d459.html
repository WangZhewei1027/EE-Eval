Bubble Sort Algorithm

Introduction to Bubble Sort

Bubble Sort is one of the simplest sorting algorithms that works by repeatedly swapping adjacent elements if they are in the wrong order. This algorithm gets its name because smaller elements "bubble" to the top of the list (beginning of the array) while larger elements sink to the bottom (end of the array) with each iteration.

While Bubble Sort is easy to understand and implement, it is not efficient for large datasets. Its average and worst-case time complexity is O(n²), making it impractical for sorting large numbers of elements. However, it serves as an educational tool to introduce sorting algorithms and their basic concepts.

How Bubble Sort Works

Bubble Sort Algorithm Steps

  1. Start with an unsorted array of elements.
  2. Compare the first element with the second element.
  3. If the first element is greater than the second element, swap them.
  4. Move to the next pair of elements (second and third) and repeat the comparison and potential swap.
  5. Continue this process until the end of the array is reached. This completes one pass.
  6. After each pass, the largest unsorted element will have "bubbled up" to its correct position at the end of the array.
  7. Repeat the process for the remaining unsorted portion of the array (excluding the last element which is already sorted).
  8. Continue until no more swaps are needed in a complete pass, indicating the array is sorted.

Optimization: Early Termination

A common optimization for Bubble Sort is to terminate the algorithm early if no swaps are made during a pass. This means the array is already sorted, and no further passes are needed. This optimization can significantly improve performance for nearly-sorted arrays.

Pseudocode

procedure bubbleSort(A : list of sortable items)
    n = length(A)
    for i from 0 to n-1
        swapped = false
        for j from 0 to n-i-1
            if A[j] > A[j+1] then
                swap(A[j], A[j+1])
                swapped = true
            end if
        end for
        if not swapped then
            break
        end if
    end for
end procedure

Example Walkthrough

Let's sort the following numbers using Bubble Sort: [5, 1, 4, 2, 8]

First Pass:

(5 1 4 2 8) → (1 5 4 2 8) Swap since 5 > 1

(1 5 4 2 8) → (1 4 5 2 8) Swap since 5 > 4

(1 4 5 2 8) → (1 4 2 5 8) Swap since 5 > 2

(1 4 2 5 8) → (1 4 2 5 8) No swap since 5 ≤ 8

Second Pass:

(1 4 2 5 8) → (1 4 2 5 8) No swap since 1 ≤ 4

(1 4 2 5 8) → (1 2 4 5 8) Swap since 4 > 2

(1 2 4 5 8) → (1 2 4 5 8) No swap since 4 ≤ 5

8 is already in its correct position from first pass

Third Pass:

(1 2 4 5 8) → (1 2 4 5 8) No swap since 1 ≤ 2

(1 2 4 5 8) → (1 2 4 5 8) No swap since 2 ≤ 4

5 and 8 are already in correct positions

No swaps were made in the third pass, so the array is now sorted.

Final sorted array: [1, 2, 4, 5, 8]

Time Complexity Analysis

Worst-case and Average-case Time Complexity: O(n²)

In the worst case (when the array is sorted in reverse order), Bubble Sort requires n-1 passes, with each pass making n-i comparisons (where i is the pass number). The total number of comparisons is:

(n-1) + (n-2) + ... + 1 = n(n-1)/2 = O(n²)

Best-case Time Complexity: O(n)

When the array is already sorted, the optimized version of Bubble Sort (with early termination) will make just one pass through the array with n-1 comparisons and no swaps, then terminate.

Space Complexity: O(1)

Bubble Sort is an in-place sorting algorithm, meaning it only requires a constant amount of additional space for temporary variables used in swapping.

Characteristics of Bubble Sort

  • Stable: Bubble Sort is a stable sorting algorithm as it maintains the relative order of equal elements.
  • In-place: It doesn't require additional memory space beyond what's needed for the original array.
  • Adaptive: With the early termination optimization, it can be efficient for nearly sorted data.
  • Simple to implement: The algorithm is straightforward to understand and code.

When to Use Bubble Sort

While Bubble Sort is rarely used in practice for large datasets due to its O(n²) time complexity, it might be appropriate in certain situations:

  • When the dataset is small (few elements to sort)
  • When the data is nearly sorted (with early termination optimization)
  • When simplicity of implementation is more important than efficiency
  • For educational purposes to teach sorting algorithm concepts

Implementation in JavaScript

function bubbleSort(arr) {
    let n = arr.length;
    let swapped;
    
    for (let i = 0; i < n - 1; i++) {
        swapped = false;
        
        for (let j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                // Swap elements
                [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
                swapped = true;
            }
        }
        
        // If no two elements were swapped, array is sorted
        if (!swapped) break;
    }
    
    return arr;
}

// Example usage:
const unsortedArray = [64, 34, 25, 12, 22, 11, 90];
const sortedArray = bubbleSort(unsortedArray);
console.log(sortedArray); // [11, 12, 22, 25, 34, 64, 90]

Comparison with Other Sorting Algorithms

Bubble Sort is often compared with other simple sorting algorithms:

  • Selection Sort: While both have O(n²) time complexity, Selection Sort typically performs better as it makes fewer swaps (O(n) swaps vs O(n²) for Bubble Sort).
  • Insertion Sort: Insertion Sort is generally more efficient than Bubble Sort, especially for small or nearly sorted datasets, though both have the same worst-case time complexity.
  • Quick Sort and Merge Sort: These more advanced algorithms (O(n log n) time complexity) are significantly more efficient for large datasets.

Common Variations of Bubble Sort

Cocktail Shaker Sort (Bidirectional Bubble Sort)

This variation sorts in both directions alternately, which can be more efficient for certain data distributions.

Odd-Even Sort

A parallel version of Bubble Sort that compares all odd/even indexed pairs of elements in the array.

Comb Sort

An improvement over Bubble Sort that compares elements separated by a certain gap, which decreases over time.
