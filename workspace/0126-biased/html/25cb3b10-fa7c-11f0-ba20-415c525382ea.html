Understanding the Sliding Window Technique

The Sliding Window is a fundamental algorithmic technique commonly employed to solve a range of problems involving sequences such as arrays or strings. Its main utility lies in efficiently finding subproblems that require inspecting contiguous sequences (windows) without redundant recomputation.

1. What is the Sliding Window Technique?

At its core, the Sliding Window technique involves maintaining a window (a contiguous subset) over data and moving this window through the entire structure to compute answers. Rather than recalculating everything for each new position of the window, the technique typically updates the previous results incrementally, leading to improved time complexities, often from O(n²) naïve approaches down to O(n) linear approaches.

“Sliding Window allows us to look at a subset of data, efficiently shift our view as we progress, and perform computations without starting over each time.”

Typical use case:

  • Find the maximum/minimum/sum/average of all contiguous subarrays of a fixed length.
  • Find substrings with certain properties (e.g., containing unique characters, at most K distinct characters).
  • Detect patterns and compute frequencies in streaming or sequential data.

2. How the Sliding Window Works Conceptually

Imagine you have a window of fixed size k starting at the beginning of your array or string. Initially, you compute the desired value (sum, max, etc.) within this window. You then move the window forward by one position at a time until you reach the end of the sequence. For each movement:

  1. You remove the effect or contribution of the element going out of the window (the leftmost element).
  2. You add the effect/contribution of the new element entering the window (immediately to the right of the window).
  3. You update your answer or result accordingly.

This process avoids recalculating from scratch for every window.

For example, for sum of subarrays of length k:
sum_new = sum_old - arr[i - 1] + arr[i + k - 1]

where i is the starting index of the new window.

3. Types of Sliding Windows

3.1 Fixed-size Sliding Window

The window size k stays constant throughout the traversal. This is the simpler form of the technique.

Example problem: Find the maximum sum of any subarray of size k.

3.2 Variable-size or Dynamic Sliding Window

Here, the window size can expand or shrink based on a particular condition, such as maintaining certain constraints inside the window (e.g., at most K distinct characters).

Example problem: Find the length of the longest substring without repeating characters.

4. Step-by-step Description of a Fixed-size Sliding Window Example

Problem: Given an integer array arr and a number k, find the maximum sum of any contiguous subarray of size k.

Step 1: Compute the sum of the first k elements because this is the first window.

sum = arr[0] + arr[1] + ... + arr[k-1]

Step 2: Initialize a variable maxSum with the value of this initial sum.

Step 3: Start sliding the window by moving one position at a time. For each new window:

  • Subtract the element leaving the window (left-most element).
  • Add the element entering the window (the next element in the array).
  • Update maxSum if the new sum is greater.

Step 4: Continue this until the window reaches the end of the array.

Step 5: maxSum is the final answer.

5. Variable-size Sliding Window: Concept and Steps

When constraints depend on the contents of the window, dynamic adjustment of the window is necessary:

  1. Initialize two pointers, often called start and end, at the beginning of the sequence.
  2. Move end forward, expanding the window, while maintaining necessary information to check constraints.
  3. When constraints are violated, move start forward to shrink the window until constraints are satisfied again.
  4. Track or update your result based on the window meeting the criteria.

6. Common Algorithms and Data Structures Used

  • Queues or Deques: For efficiently tracking max/min elements in windows (e.g., sliding window maximum).
  • Hash Maps/Sets: To maintain counts of elements or characters within the window.
  • Prefix sums: Sometimes combined with sliding windows for sum/range queries.

7. Illustrative Example: Longest Substring Without Repeating Characters

Problem: Find the length of the longest substring without repeating characters in a string s.

Approach (Variable-size Sliding Window):

  1. Use two pointers start and end to define the current substring window.
  2. Use a hash map (or dictionary) to keep track of characters and their latest indices.
  3. Incrementally move end forward, adding characters:
    • If a repeating character is found within the current window, move start right after the previous occurrence to maintain uniqueness.
  4. Keep track of the maximum length of substring encountered that contains unique characters.

Step-by-step textual explanation:

Suppose the string is "abcabcbb":

  • Start with start = 0, end = 0, substring = "a".
  • Move end to 1, substring = "ab" (unique chars).
  • Move end to 2, substring = "abc" (unique chars).
  • Move end to 3, char = "a" repeats.
  • Since "a" was at index 0, set start to 1 to exclude previous "a". Substring now "bca".
  • Continue, updating max length as substrings grow or adjusting start on repeats.

8. Simple Code Illustration (JavaScript)

This snippet finds the maximum sum of contiguous subarrays of size k:

function maxSumSubarray(arr, k) {
  let windowSum = 0;
  // Sum initial window
  for (let i = 0; i < k; i++) {
    windowSum += arr[i];
  }
  let maxSum = windowSum;

  for (let end = k; end < arr.length; end++) {
    windowSum += arr[end] - arr[end - k]; // Slide the window
    if (windowSum > maxSum) {
      maxSum = windowSum;
    }
  }
  return maxSum;
}

10. Key Takeaways

  • Sliding Window optimizes problems involving contiguous subproblems, reducing redundant calculations.
  • Fixed-size windows move systematically with a constant size, while variable-size windows adapt dynamically to constraints.
  • Data structures like hash maps, sets, queues, and deques often support sliding window implementations efficiently.
  • Applications span across arrays, strings, streaming data, and many classic algorithmic challenges.
