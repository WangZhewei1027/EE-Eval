Understanding Binary Search Trees (BST)

A Binary Search Tree (BST) is a fundamental data structure in computer science used for efficiently storing and retrieving data that can be ordered. It combines the properties of a binary tree and a searchable structure, enabling operations like search, insertion, and deletion to be performed quickly on average.

1. What is a Binary Search Tree?

A BST is a specialized form of a binary tree with the following key property:

For every node in the tree, all values in the left subtree are less than the node’s value, and all values in the right subtree are greater than the node’s value.

This property ensures that the tree is sorted, allowing for efficient searching analogous to binary search on sorted arrays.

Important: BSTs typically do not contain duplicate values. If duplicates are allowed, a convention must be chosen (e.g., duplicates go to the right subtree).

Binary Tree vs. Binary Search Tree

  • A binary tree is any tree where each node has at most two children (left and right), with no ordering requirements.
  • A binary search tree adds ordering constraints on the values in the children nodes relative to each node.

2. Structure and Terminology

  • Node: Each element in the tree containing a value, along with pointers/references to its left and right children.
  • Root: The topmost node of the tree.
  • Left subtree: The subtree consisting of all nodes reachable from the left child of a node.
  • Right subtree: The subtree consisting of all nodes reachable from the right child of a node.
  • Leaf node: A node with no children.
  • Height of the tree: The length of the longest path from the root to a leaf.

3. How Does a BST Maintain Order?

At each node N, the tree enforces:

  • Every element in the left subtree of N is strictly less than N.value.
  • Every element in the right subtree of N is strictly greater than N.value.

This condition is verified recursively at all nodes — this ensures the entire tree is sorted with respect to in-order traversal.

4. Core Operations on a BST

BSTs support several important operations efficiently (in average time complexity O(log n) if the tree is balanced):

4.1 Searching for a Value

To find a value X in a BST:

  1. Start at the root.
  2. If the current node’s value equals X, the search is successful.
  3. If X is less than the node’s value, search recursively in the left subtree.
  4. If X is greater than the node’s value, search recursively in the right subtree.
  5. If a leaf is reached without finding X, it means X is not in the tree.

Example: Searching for value 7 in the BST:

      10
     /  \
    5    15
   / \     \
  3   7     20

Start at 10 → 7 < 10 → go left to 5 → 7 > 5 → go right to 7 → value found.

4.2 Insertion of a Value

To insert a value X into the BST:

  1. Start at the root.
  2. Compare X to the node’s value.
  3. If X is less, proceed to the left child; if the left child is null, insert here.
  4. If X is greater, proceed to the right child; if the right child is null, insert here.
  5. Repeat recursively until the correct spot is found.

Inserting maintains the BST property since the value is placed so ordering holds at every node.

4.3 Deletion of a Value

Deleting a node from a BST is more complex due to maintaining the BST property post-removal. Three cases exist when deleting a node with value X:

  • Case 1: Node is a leaf (no children)
    Simply remove this node from its parent.
  • Case 2: Node has one child
    Remove the node and replace it by its child (left or right subtree).
  • Case 3: Node has two children
    Find the in-order successor (smallest value in the right subtree) or in-order predecessor (largest in left subtree), replace the node’s value with it, and then delete that successor/predecessor node (which falls under case 1 or 2).

This process ensures the BST ordering remains intact after the deletion.

5. Traversing a BST

Traversal means visiting all nodes in some systematic order. The three main traversals are:

  • In-order traversal: left subtree → node → right subtree.
    This will visit the nodes' values in sorted ascending order due to BST property.
  • Pre-order traversal: node → left subtree → right subtree.
    Often used for copying or expressing the tree structure.
  • Post-order traversal: left subtree → right subtree → node.
    Used when deleting or freeing nodes.

6. Time Complexity Summary

Note: The worst case occurs when the tree degenerates into a linked list, e.g., inserting already sorted data without balancing.

8. BST Algorithm Pseudocode

8.1 Search Algorithm Pseudocode

function BST_Search(node, key):
    if node is null:
        return false  // key not found
    if key == node.value:
        return true   // key found
    else if key < node.value:
        return BST_Search(node.left, key)
    else:
        return BST_Search(node.right, key)

8.2 Insertion Algorithm Pseudocode

function BST_Insert(node, key):
    if node is null:
        node = new Node(key)
        return node
    if key < node.value:
        node.left = BST_Insert(node.left, key)
    else if key > node.value:
        node.right = BST_Insert(node.right, key)
    // if key == node.value, duplicates typically not inserted
    return node

8.3 Deletion Algorithm Pseudocode

function BST_Delete(node, key):
    if node is null:
        return null
    if key < node.value:
        node.left = BST_Delete(node.left, key)
    else if key > node.value:
        node.right = BST_Delete(node.right, key)
    else:
        // node with value to delete found
        if node.left is null:
            return node.right
        else if node.right is null:
            return node.left
        else:
            // node with two children:
            successor = findMin(node.right)
            node.value = successor.value
            node.right = BST_Delete(node.right, successor.value)
    return node

function findMin(node):
    while node.left is not null:
        node = node.left
    return node

9. Summary and Conclusion

Binary Search Trees offer a simple yet powerful structure to maintain dynamically changing sorted data. Their key ordering property allows search, insertion, and deletion to be much faster than in unordered data structures, especially when the tree is balanced. Understanding BSTs builds a foundation for more advanced tree structures such as AVL trees, Red-Black trees, and B-Trees, which ensure balanced heights and consistent logarithmic performance even in the worst case.

Mastering BSTs is invaluable for computer scientists and programmers working with sorted data, databases, and searching algorithms.
