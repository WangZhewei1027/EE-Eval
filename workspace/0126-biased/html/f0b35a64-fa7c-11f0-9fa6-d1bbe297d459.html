CPU Scheduling: A Comprehensive Guide

Introduction to CPU Scheduling

CPU scheduling is a fundamental concept in operating systems that determines how the CPU is allocated to processes. The primary goal of CPU scheduling is to maximize CPU utilization by ensuring the CPU is always executing a process.

Definition: CPU Scheduling is the process by which the operating system selects which process in the ready queue is to be allocated the CPU for execution. The selection is made by the CPU scheduler.

When multiple processes are in the ready state, the operating system must decide which process to execute next. This decision-making process is called CPU scheduling.

Key Concepts in CPU Scheduling

  • CPU-I/O Burst Cycle: Process execution consists of cycles of CPU execution and I/O wait.
  • Preemptive Scheduling: The CPU can be taken away from a process before it completes.
  • Non-preemptive Scheduling: Once the CPU is allocated to a process, it keeps it until termination or I/O request.
  • Dispatcher: The module that gives control of the CPU to the process selected by the scheduler.

Scheduling Objectives

CPU scheduling algorithms are designed with several objectives in mind:

Objective Description
CPU Utilization Keep the CPU as busy as possible (ideally 100%)
Throughput Number of processes completed per time unit
Turnaround Time Time from submission to completion of a process
Waiting Time Time a process spends waiting in the ready queue
Response Time Time from submission until first response (for interactive systems)

Scheduling Algorithms

Various algorithms have been developed to achieve the scheduling objectives. Each has its own advantages and disadvantages.

First-Come, First-Served (FCFS)

Description: Processes are executed in the order they arrive in the ready queue (FIFO).

Characteristics: Non-preemptive, simple to implement, can lead to convoy effect.

Example:

Processes arrive in order P1 (burst time 24), P2 (burst time 3), P3 (burst time 3).

P1 executes for 24 units, then P2 for 3, then P3 for 3.

Average waiting time = (0 + 24 + 27)/3 = 17 units.

Shortest-Job-First (SJF)

Description: The process with the smallest next CPU burst is selected first.

Characteristics: Can be preemptive or non-preemptive, optimal for minimizing waiting time.

Example (Non-preemptive):

Processes: P1 (6), P2 (8), P3 (7), P4 (3) arrive at time 0.

Execution order: P4 (3), P1 (6), P3 (7), P2 (8).

Average waiting time = (3 + 0 + 6 + 9)/4 = 4.5 units.

Priority Scheduling

Description: Each process is assigned a priority, and the highest priority process is selected.

Characteristics: Can be preemptive or non-preemptive, may lead to starvation.

Round Robin (RR)

Description: Each process gets a small unit of CPU time (time quantum), then is preempted and added to the end of the ready queue.

Characteristics: Preemptive, fair allocation, performance depends on time quantum size.

Example: Time quantum = 4, Processes: P1 (24), P2 (3), P3 (3).

Execution order: P1 (4), P2 (3), P3 (3), P1 (4), P1 (4), etc.

Multilevel Queue Scheduling

Description: Ready queue is partitioned into separate queues (e.g., foreground/interactive, background/batch). Each queue can have its own scheduling algorithm.

Multilevel Feedback Queue

Description: Similar to multilevel queue but allows processes to move between queues based on their behavior (CPU-bound vs I/O-bound).

Algorithm Comparison

Algorithm Advantages Disadvantages
FCFS Simple, fair Poor performance with varying burst times, convoy effect
SJF Optimal (minimal average waiting time) Difficult to predict burst times, may starve long processes
Priority Flexible, can prioritize important processes Starvation, priority inversion problem
Round Robin Fair, good response time Performance depends on time quantum size
Multilevel Queue Can optimize for different process types Complex to implement

Real-World Considerations

Context Switching

When the CPU switches from one process to another, the system must save the state of the old process and load the saved state for the new process. This is called a context switch.

Dispatcher

The dispatcher is the module that gives control of the CPU to the process selected by the short-term scheduler. Its functions include:

  • Switching context
  • Switching to user mode
  • Jumping to the proper location in the user program to restart it

Thread Scheduling

In systems that support threads, the operating system schedules threads rather than processes. Thread scheduling can occur at two levels:

  • Process Contention Scope (PCS): Scheduling done locally within the process
  • System Contention Scope (SCS): Scheduling done system-wide among all threads

Conclusion

CPU scheduling is a critical component of modern operating systems that directly impacts system performance and user experience. The choice of scheduling algorithm depends on the specific requirements of the system, such as whether it needs to optimize for throughput, response time, or fairness.

While simple algorithms like FCFS are easy to implement, more complex algorithms like multilevel feedback queues can provide balanced performance for a variety of process types. Understanding these scheduling concepts is fundamental to operating system design and performance optimization.
