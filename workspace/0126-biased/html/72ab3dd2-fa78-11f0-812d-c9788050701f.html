Time Complexity Visualized

Explore the elegance of algorithmic efficiency through these visual representations of common time complexities in computer science.

O(1)

Constant Time

The algorithm takes the same amount of time regardless of input size. Ideal for operations that access a single element.

O(log n)

Logarithmic Time

The time grows logarithmically with input size. Common in divide-and-conquer algorithms like binary search.

O(n)

Linear Time

Time increases linearly with input size. Found in simple iterations through data structures.

O(n²)

Quadratic Time

Time grows quadratically with input size. Often seen in nested loops over the same dataset.
