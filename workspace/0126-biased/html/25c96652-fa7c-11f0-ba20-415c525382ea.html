The Max Heap Data Structure: A Comprehensive Explanation

The concept of a Heap (Max) is fundamental in computer science, especially when it comes to efficient priority queue implementations, sorting algorithms like Heapsort, as well as graph algorithms such as Dijkstra’s and Prim’s. This explanation provides a thorough overview of the Max Heap data structure, how it is defined, its core properties, common operations, and algorithms.

What is a Heap?

In computer science, a heap is a special tree-based data structure that satisfies the heap property. A heap can be classified into two types depending on the heap property it satisfies:

  • Max Heap: In a max heap, for every node i, the value of i is greater than or equal to the values of its children. Thus, the maximum element is always at the root (top) of the tree.
  • Min Heap: In a min heap, for every node i, the value of i is less than or equal to the values of its children, meaning the minimum element is at the root.

Here we focus exclusively on the Max Heap variant.

Basic Properties of a Max Heap

  • Shape Property: A max heap must be a complete binary tree. That means it is completely filled on all levels except possibly the lowest, which is filled from the left up to a point.
  • Heap Property: The key or value of each node is greater than or equal to those of its children.
  • Root Node: Always contains the maximum element of the entire heap.

Representation of Max Heap

Although conceptually a max heap is a binary tree, it is often implemented using a simple array or list due to its shape property (completeness). This allows efficient indexing and reduces overhead.

Nodes are stored level-by-level from left to right starting at index 0 (sometimes 1, based on implementation).

For any node at index i in the array:

  • The left child of node i is at index 2i + 1
  • The right child is at index 2i + 2
  • The parent node is at index ⌊(i - 1) / 2⌋

Example Array Representation

Consider the heap: [90, 85, 70, 75, 60, 65, 50]
This corresponds to the following tree: 
          90
         /  \
       85    70
      /  \  /  \
    75  60 65  50

Core Operations on Max Heap

The two fundamental operations that maintain and utilize max heaps are Insertion and Extraction of the maximum. Additionally, we often need to build a max heap from arbitrary data and maintain the heap property using a procedure called heapify.

1. Max Heapify (Restore Heap Property)

The max-heapify operation is used to maintain the max-heap property. If the value of a node violates the heap property (i.e., is smaller than at least one of its children), max-heapify corrects it:

  1. Compare the node with its left and right children.
  2. If the node is smaller than one or both children, swap the node with the largest child.
  3. Recursively call max-heapify on the subtree affected by the swap to ensure heap property is restored.

This operation takes O(log n) time in the worst case, where n is the number of nodes, as it traverses from the node down to a leaf.

The Max-Heapify Algorithm (Pseudocode)

MAX-HEAPIFY(A, i):
  left = 2*i + 1
  right = 2*i + 2
  largest = i

  if left < length(A) and A[left] > A[largest]:
    largest = left
  if right < length(A) and A[right] > A[largest]:
    largest = right
  if largest != i:
    swap A[i] with A[largest]
    MAX-HEAPIFY(A, largest)

2. Building a Max Heap

Given an unordered array, it can be transformed into a valid max heap by calling max-heapify starting from the lowest non-leaf nodes up to the root:

  • Non-leaf nodes have indices from ⌊n/2⌋ - 1 down to 0.
  • For each index in this range, run max-heapify.

This process builds a max heap in O(n) time, a surprising fact since repeatedly calling max-heapify takes O(log n) time but it runs fewer times on upper levels where the tree is shallow.

3. Inserting an Element

When inserting a new element into a max heap:

  1. Add the element at the end of the array (maintaining shape property).
  2. Bubble Up / Heap Increase Key: Compare the new element to its parent; if the new node is larger, swap it with its parent.
  3. Repeat step 2 until the heap property is restored or the root is reached.

This operation works in O(log n) time.

4. Extracting the Maximum Element

Removing the maximum element (the root) involves:

  1. Replace the root element with the last element in the heap.
  2. Remove the last element (which was moved to root).
  3. Call max-heapify on the root to restore the heap property.

This also works in O(log n) time.

Practical Example Walkthrough

Let's understand the Max Heapify process step by step with an example array:

Initial Array (index-based): [27, 17, 33, 14, 7, 10, 12]
Corresponding Tree: 
           27
          /  \
        17    33
       /  \  /  \
     14   7 10  12

Notice at the root the value is 27, but the right child is 33, which is greater. The heap property is violated at the root.

Applying max-heapify at index 0:

  1. Compare 27 with left child 17 and right child 33.
  2. 33 is the largest, so swap 27 with 33.
  3. Recurse at index 2 where 27 now resides.
  4. At index 2, children are 10 (left) and 12 (right). 27 > 12, so no swaps needed, heap property restored.

The final max heap array becomes: [33, 17, 27, 14, 7, 10, 12].

Use Cases of Max Heap

  • Priority Queues: Max heaps efficiently extract the highest priority element.
  • Heapsort Algorithm: Uses repeated extraction from a max heap to perform sorting in O(n log n) time.
  • Graph Algorithms: Algorithms like Dijkstra’s shortest path or Prim’s MST rely on priority queues implemented with heaps.
  • Median Finding: Median can be found using two heaps - a max heap (for the low half) and min heap (for the high half).

Summary of Key Points

  • A Max Heap is a complete binary tree where every node’s value ≥ the values of its children.
  • Stored efficiently as an array, with parent and child positions calculated by simple arithmetic.
  • Max-Heapify is the key procedure for maintaining heap property, recursively fixing violations.
  • Insertion and extraction operate in logarithmic time because of the tree height.
  • Building a max heap from an unordered array is linear time.
