Ternary Search

Understanding Ternary Search: Theory, Algorithms, and Applications

1. Introduction and Definition

Ternary Search is a divide-and-conquer algorithm used to find the maximum (or minimum) of a unimodal function or to search an element in a sorted array in a less commonly used way than binary search. The key concept behind the algorithm is that it divides the search space into three parts (instead of two as in binary search) and eliminates one third of the search interval in each step.

While binary search splits the search interval in half, ternary search splits it in thirds. It is particularly efficient in the following two scenarios:

  • Finding the extremum (maximum or minimum) in a unimodal function over a defined interval, when the function is known to first increase and then decrease (or vice versa).
  • Searching a value in a sorted sequence, but this use is less common compared to binary search because binary search tends to be simpler and often faster in practice.
Important: The function or array property in focus highly influences the efficiency and applicability of ternary search.

2. Theoretical Background and Unimodal Functions

Before delving into the algorithm, it is crucial to understand the type of problems ternary search solves most naturally: unimodal function optimization.

What is a Unimodal Function?

A function f(x) defined on an interval [a, b] is called unimodal if there exists a point m ∈ [a,b] such that:

  • f(x) is strictly increasing for all x < m, and
  • f(x) is strictly decreasing for all x > m.

More informally, unimodal means the function has exactly one "peak" (maximum) or one "valley" (minimum) within the interval.

If the function has multiple peaks or valleys, ternary search will not guarantee the correct extremum.

Why Use Ternary Search for Unimodal Functions?

Ternary Search strategically narrows down the interval by dividing it into three parts and comparing function values at two interior points to identify which segment contains the extremum. By eliminating one-third of the interval each iteration, ternary search efficiently zooms in on the maximum or minimum point.

3. Ternary Search Algorithm — Step-by-Step Explanation

The main idea is to repeatedly reduce the search interval [L, R] by evaluating the function at two interior points and comparing their values.

Algorithm for Finding Maximum of a Unimodal Function on Continuous Domain:

Suppose we want to find the maximum value of a unimodal function f(x) on the interval [L, R]. The ternary search proceeds as follows:

  1. Calculate two points dividing the interval into three equal parts: 
    m1 = L + (R - L) / 3
m2 = R - (R - L) / 3
  2. Evaluate f(m1) and f(m2).
  3. Compare:
    • If f(m1) < f(m2), the maximum lies in [m1, R]. Update L = m1.
    • Otherwise (if f(m1) ≥ f(m2)), the maximum lies in [L, m2]. Update R = m2.
  4. Repeat the above steps until the interval is sufficiently small (e.g., the length R - L is less than a small epsilon).
  5. The approximate maximum is at any point within the final interval [L, R].

Note: To find the minimum of a unimodal function, invert the comparison:

  • If f(m1) > f(m2), update L = m1.
  • Otherwise, update R = m2.

Pseudocode for Maximum Search

function ternarySearch(L, R, f, epsilon):
    while R - L > epsilon:
        m1 = L + (R - L) / 3
        m2 = R - (R - L) / 3

        if f(m1) < f(m2):
            L = m1
        else:
            R = m2

    return (L + R) / 2  # Approximate location of maximum

4. Ternary Search on Discrete Sorted Arrays

Ternary search can also be applied to discrete problems such as searching an element in a sorted array. However, this use-case is mostly theoretical or for didactic interest because binary search is simpler and usually faster on average and in practice.

How Does It Work on a Sorted Array?

Given a sorted array A of size n and a value key, ternary search divides the search interval into three parts to locate key.

  1. Let the current interval be [L, R].
  2. Calculate two mid-points: 
    mid1 = L + (R - L) / 3
mid2 = R - (R - L) / 3
  3. Compare key to A[mid1] and A[mid2]:
    • If key == A[mid1], return mid1.
    • If key == A[mid2], return mid2.
    • If key < A[mid1], update R = mid1 - 1.
    • If key > A[mid2], update L = mid2 + 1.
    • Otherwise, the key lies in (mid1, mid2), so update L = mid1 + 1 and R = mid2 - 1.
  4. Repeat until L > R or the element is found.

Pseudocode for Discrete Search

function ternarySearchArray(A, L, R, key):
    while L <= R:
        mid1 = L + (R - L) // 3
        mid2 = R - (R - L) // 3

        if A[mid1] == key:
            return mid1
        if A[mid2] == key:
            return mid2

        if key < A[mid1]:
            R = mid1 - 1
        else if key > A[mid2]:
            L = mid2 + 1
        else:
            L = mid1 + 1
            R = mid2 - 1

    return -1  # key not found
Note: Although possible, this search is usually slower and less practical than binary search.

5. Comparison of Ternary Search and Binary Search

Aspect Binary Search Ternary Search
Search Domain Sorted discrete arrays / monotonic functions Unimodal continuous or discrete functions; sorted arrays (less common)
Division of Interval Split into 2 parts Split into 3 parts
Time Complexity O(log₂ n) O(log₃ n)
Use Case Search for values (key existence/position) Find max/min of unimodal functions
Implementation Complexity Simple and widely used More complex, niche applications
Practical Performance Generally faster and more popular Used mainly in optimization problems

6. Practical Considerations & Mathematical Insights

Continuous Domain Version

In many problems, especially optimization problems without closed-form derivatives, ternary search is practical for approximating extrema of unimodal functions when:

  • You know the function is unimodal on the interval [L, R].
  • You want a result within a precision ε.
  • The function evaluations f(x) are expensive or no analytic derivative is available.

How precise is the output? Each iteration reduces the search space by a factor of ⅓, so after k iterations, the interval size is roughly (R - L) * (1/3)^k. To achieve precision ε, at most:

k ≈ log₃((R - L) / ε)

iterations are needed.

Discrete Domain Version

Note that if the domain is discrete (like array indices), care must be taken with integer division and updating interval bounds to ensure no infinite loops or missed elements.

Warning: Ternary search requires strict unimodality for maximum/minimum search; if the function has multiple local maxima/minima, it may find a local extremum instead of the global one.
