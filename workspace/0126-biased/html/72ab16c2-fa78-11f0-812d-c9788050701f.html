Common Time Complexities

O(1)

Constant time - operations execute in the same time regardless of input size

O(log n)

Logarithmic time - operations halve the problem space with each step

O(n)

Linear time - operations grow linearly with input size

O(n²)

Quadratic time - operations grow with the square of input size

O(2ⁿ)

Exponential time - operations double with each addition to input

What is Big-O?

Big-O notation describes the upper bound of an algorithm's time or space complexity in relation to its input size. It provides a theoretical measure of an algorithm's efficiency, helping developers choose the most optimal solution for their problems.

Why it Matters

Understanding Big-O helps predict how an algorithm will scale. An O(n²) algorithm might work fine with 10 items but become unusable with 10,000. Choosing the right complexity class can mean the difference between an instant response and a frozen application.

Visual Interpretation

The height of each bar represents how the time complexity grows relative to input size. Notice how O(1) remains constant while O(2ⁿ) grows exponentially - quickly becoming impractical for large inputs.
