Circular Linked List

In computer science, a circular linked list is a data structure consisting of a sequence of nodes in which each node points to the next node, and the last node points back to the first node, forming a circle. This makes the list cyclical and allows traversals that loop infinitely unless explicitly stopped.

What is a Circular Linked List?

A linked list is a linear data structure made up of nodes where each node contains two parts:

  • Data: the actual value or information stored.
  • Pointer (or Reference): a link to the next node in the sequence.

Unlike a standard singly linked list where the last node points to null (or None), a circular linked list has its last node's pointer redirect to the first node, thus creating a continuous loop.

Types of Circular Linked Lists

  • Single Circular Linked List: Each node has a single pointer to the next node, and the last node points back to the first node.
  • Doubly Circular Linked List: Each node has two pointers: one to the next node and one to the previous node. The first node’s previous pointer points to the last node, and vice versa, forming a bidirectional circle.

Why Use Circular Linked Lists?

Circular linked lists offer several advantages over linear linked lists:

  • Efficient traversal: You can traverse from any node and eventually loop through all nodes without worrying about reaching a null reference.
  • Continuous cycling: Ideal for applications requiring repetitive iteration over data, such as in round-robin scheduling, buffering data, or multiplayer games.
  • Easy insertion and deletion: Inserting or deleting nodes in a circular linked list especially around the start or end is straightforward without needing separate head or tail conditions.

Structure and Components

Each node of a circular linked list typically looks like:

Node {
  data;    // the actual value
  next;    // pointer/reference to next node (or head if last node)
}

To maintain the circular nature, we usually keep a reference to a node in the list — commonly the head node (first inserted node) or sometimes a tail node (last node, to simplify some insertions).

For example, if tail points to the last node, then tail.next points to the head, making the loop: 

tail -> firstNode -> ... -> lastNode
   |_______________________________|
             (tail.next points back to firstNode)

How Does Traversal Work?

Traversing a circular linked list involves starting from a node (commonly the head) and following the next pointers. Since there is no null or None to mark the end, we must check if we have returned to the starting node to avoid infinite looping. 

// Example (in pseudocode) for traversing a single circular linked list:

start_node = head
current_node = start_node

do {
    process(current_node.data)
    current_node = current_node.next
} while (current_node != start_node)

This ensures each node is visited exactly once.

Common Operations and Algorithms

1. Insertion

There are different places where insertion can be done:

  • Insert at beginning: Insert before head and update tail's next.
  • Insert at end: Insert after tail and update tail pointer.
  • Insert after a given node: Insert after a certain node elsewhere in the list.

Here’s a step-by-step outline to insert a node at the end of a single circular linked list when using a tail pointer:

  1. Create the new node (with given data).
  2. Set the new node’s next pointer to point to the tail->next (which is head).
  3. Change the current tail’s next pointer to the new node.
  4. Update the tail pointer to the new node.
// Pseudocode for insertion at end
new_node = create_node(data)
if tail is null
{
    tail = new_node
    tail.next = tail   // points to self, single element circular list
}
else
{
    new_node.next = tail.next
    tail.next = new_node
    tail = new_node
}

2. Deletion

Deleting a node requires:

  1. Finding the node before the one to delete.
  2. Change its next pointer to skip over the node being deleted.
  3. Special care if deleting the only node, the head node, or the tail node.

Example: deleting a node with a given value from a circular singly linked list with a tail pointer:

  1. If list is empty (tail is null), no action.
  2. Traverse the list to find the node to delete and its previous node.
  3. If node found:
    • If only one node (tail.next == tail), set tail to null.
    • If deleting tail, update tail to previous node.
    • Change previous node’s next pointer to skip the deleted node.

Illustration with an Example

Suppose we have a circular linked list of integers: 10 → 20 → 30 → 10 (head)

  • tail points to node with data 30.
  • tail.next points to node with data 10 (head).

If we insert 40 at the end:

  1. Create node 40 pointing to 10.
  2. 30’s next pointer updated to 40.
  3. Update tail to 40.

Resulting list: 10 → 20 → 30 → 40 → 10...

Comparison Table

Aspect Singly Linked List Circular Linked List
Last node pointer Points to null Points to head (first node)
Traversal ends when Next pointer is null Current node loops back to start node
Use cases General linear data Round-robin scheduling, buffers, cyclic iteration
Insertion at end Need to traverse entire list if no tail pointer Tail pointer helps insert at the end in O(1)
Deletion complexity Linear time to find node Same, but careful with circularity

Summary

A circular linked list is a variant of the linked list with the primary characteristic that its last node points back to the first node, creating a circular chain of nodes. This property enables easy continuous traversal and is useful in specific scenarios like scheduling or buffering.

Understanding its structure, traversal method, and operations such as insertion and deletion, is fundamental for programmers working with dynamic data and algorithms requiring cyclic ordering or iteration.
