Support Vector Machine (SVM)

A thorough, text-focused explanation of the concept, mathematics, algorithms, and practical guidance for Support Vector Machines.

Overview — What is an SVM?

A Support Vector Machine (SVM) is a supervised learning model used primarily for classification and regression tasks. The fundamental idea is geometric: SVM finds a decision boundary (a hyperplane) that best separates classes by maximizing the margin — the distance between the boundary and the nearest labelled training points. Those nearest points are the "support vectors".

SVMs are powerful because they combine geometric intuition (maximum margin) with convex optimization, and can be extended to non-linear decision boundaries through the kernel trick.

Key Concepts and Definitions

  • Hyperplane: In d-dimensional space, a hyperplane is {x | w · x + b = 0} where w is the normal vector and b is the bias.
  • Margin: The distance between the hyperplane and the closest data points from each class. SVM aims to maximize this margin.
  • Support vectors: Training points that lie on (or violate) the margin boundaries; they determine the position of the optimal hyperplane.
  • Hard-margin SVM: Assumes data is linearly separable. We seek the hyperplane with maximum margin without misclassification.
  • Soft-margin SVM: Allows some misclassification or margin violations, controlled by a regularization parameter C.
  • Kernel: A function K(x, x') that computes an inner product in some (possibly high-dimensional) feature space without explicit mapping — enabling non-linear separation.

Mathematical Formulation — Primal Problem (Hard Margin)

Consider binary classification labels y_i ∈ {+1, −1} and feature vectors x_i ∈ R^d.

Hard-margin SVM (linearly separable): 

Minimize over (w, b):
    1/2 ||w||^2
Subject to:
    y_i (w · x_i + b) ≥ 1   for all i

Geometric interpretation: minimizing 1/2 ||w||^2 maximizes the margin 1/||w|| subject to correct classification constraints.

Soft-Margin SVM (Practical)

For non-separable data, introduce slack variables ξ_i ≥ 0 to allow violations, and penalty parameter C > 0: 

Minimize over (w, b, ξ):
    1/2 ||w||^2 + C Σ ξ_i
Subject to:
    y_i (w · x_i + b) ≥ 1 - ξ_i
    ξ_i ≥ 0

C controls the trade-off: large C → penalize errors strongly (lower bias, potentially higher variance); small C → allow more violations (higher bias, lower variance).

Dual Formulation

Writing the Lagrangian and deriving the dual yields: 

Maximize over α_i:
    W(α) = Σ α_i - 1/2 ΣΣ α_i α_j y_i y_j (x_i · x_j)
Subject to:
    0 ≤ α_i ≤ C
    Σ α_i y_i = 0

The solution uses α_i (Lagrange multipliers). The classifier becomes: 

f(x) = Σ α_i y_i (x_i · x) + b

Only points with α_i > 0 contribute — these are the support vectors.

The Kernel Trick — Non-linear Boundaries

Replace the inner product (x_i · x_j) with a kernel function K(x_i, x_j) that implicitly maps inputs into a higher-dimensional feature space: 

Dual with kernel:
    W(α) = Σ α_i - 1/2 ΣΣ α_i α_j y_i y_j K(x_i, x_j)
Decision function:
    f(x) = Σ α_i y_i K(x_i, x) + b

Common kernels:

  • Linear: K(x, x') = x·x' (no mapping)
  • Polynomial: K(x, x') = (γ x·x' + r)^d
  • RBF / Gaussian: K(x, x') = exp(-γ ||x - x'||^2)
  • Sigmoid: K(x, x') = tanh(γ x·x' + r)

Mercer's theorem provides conditions under which a function is a valid kernel (i.e., corresponds to an inner product in some Hilbert space).

Optimization and Algorithms

The SVM optimization is a convex quadratic program (QP). Common solvers and algorithms:

  • SMO (Sequential Minimal Optimization): Decomposes the QP into smallest possible QP subproblems (two variables at a time) and solves analytically — widely used (e.g., LIBSVM).
  • Quadratic programming solvers: Generic QP solvers can be used but may not scale well to large datasets.
  • Primal methods: Use gradient-based solvers or coordinate descent for linear SVMs (e.g., LIBLINEAR).
  • SGD / Pegasos: Stochastic methods that approximate the solution by minimizing hinge loss with L2 regularization. Useful for very large data.

Complexity: classic SVM with kernel scales between O(n^2) and O(n^3) in training for n training points (depending on implementation), though many solvers have shortcuts and approximate methods.

Hinge Loss Perspective

SVM can be seen as minimizing empirical hinge loss plus regularization: 

Minimize over (w, b):
    1/2 ||w||^2 + C Σ max(0, 1 - y_i (w · x_i + b))

This perspective is helpful for stochastic optimization: hinge loss is convex but not differentiable at the hinge; subgradient methods apply.

Step-by-step Training Narrative (Soft-margin)

  1. Prepare data: features X, labels y in {+1, −1}. Standardize/scale features since SVM is sensitive to feature scales.
  2. Choose kernel: start with linear for high-dimensional sparse data (e.g., text). Use RBF or polynomial for complex non-linear patterns.
  3. Select hyperparameters: C (regularization), and kernel params (e.g., γ for RBF, degree for polynomial).
  4. Choose solver: for moderate n and kernel, use SMO-based solver; for very large n and linear case, use primal solvers or SGD.
  5. Train: solve the convex optimization problem to obtain α (dual) or w (primal) and b.
  6. Identify support vectors: data points with α_i > 0 (or nearly > 0 in practice).
  7. Predict: compute s = Σ α_i y_i K(x_i, x) + b (or w·x + b for linear); assign label sign(s).
  8. Evaluate and tune: use cross-validation to choose C and kernel parameters. Monitor overfitting/underfitting.

Practical Tips and Common Pitfalls

  • Feature scaling matters: Standardize features (zero mean, unit variance) or scale to [0,1]. Kernels like RBF are sensitive to distance scales.
  • Use linear SVM for high-dimensional sparse data: Text classification often benefits from linear SVMs (fast and effective).
  • RBF as a default kernel: If unsure and features are reasonably scaled, RBF is a common starting point. Beware of tuning γ and C together.
  • Class imbalance: SVMs optimize margin; with imbalanced data, adjust class weights or use different C per class.
  • Interpretability: Linear SVM weights are interpretable (feature importance); kernel SVMs are less interpretable because support vectors live in feature-transformed space.
  • Sensitivity to outliers: Hard-margin SVM will fail with even one outlier — use soft-margin and tune C.

Multiclass Strategies

SVM is intrinsically a binary classifier. Multiclass extensions include:

  • One-vs-Rest (OvR): Train K binary classifiers for K classes; for prediction, choose class with highest decision function.
  • One-vs-One (OvO): Train K(K-1)/2 classifiers between each pair of classes; predict by voting or pairwise aggregation.
  • Direct multiclass formulations: There are formulations that optimize a multiclass objective directly, but many implementations use OvR or OvO due to simplicity.

Interpretation and Support Vectors

Support vectors are the training examples that "support" or define the margin. If you remove non-support vectors, the solution (hyperplane) does not change. Inspecting support vectors can help understand which examples are critical or ambiguous.

Advantages and Limitations

Advantages

  • Works well in high-dimensional spaces.
  • Effective when number of dimensions > number of samples.
  • Memory efficient: uses subset of training points (support vectors).
  • Flexible with kernel functions for non-linear decision boundaries.

Limitations

  • Can be slow on large datasets (training complexity).
  • Choice and tuning of kernel and hyperparameters can be non-trivial.
  • Outputs are not probabilistic by default (but can be calibrated using methods like Platt scaling).
  • Less straightforward to scale to millions of examples without approximations.

Implementation Notes

Popular libraries and tools:

  • LIBSVM — widely used kernel SVM implementation with SMO.
  • LIBLINEAR — efficient linear SVM and logistic regression for large datasets.
  • scikit-learn — Python library exposing SVMs (wrappers to LIBSVM/LIBLINEAR), convenient for prototyping.

When implementing SVM from scratch, consider using the dual form when kernels are needed; use primal solvers or SGD for linear SVM when scaling matters.

Worked Example (Conceptual)

Suppose we have 2D points from two classes that are roughly linearly separable: 

Class +1: (2,3), (3,4), (4,3)
Class -1: (1,1), (2,0), (0,2)

Intuitively, an SVM will place a line between clusters that maximizes the distance to the closest points. The support vectors will be the closest points on either side — e.g., (2,3) and (1,1) — and they determine the line.

In practice you'd scale features, pick C (if you want to allow misclassification), and solve an optimization to compute weights w and bias b.

Frequently Asked Questions

Q: Should I always use SVM?

A: SVMs are a strong choice when you need a robust classifier with good generalization and when data is not extremely large. For very large datasets or for end-to-end deep learning settings, other methods may be preferred.

Q: How to choose between linear and kernel SVM?

A: If data is high-dimensional and sparse (text), try linear SVM first. If features are low-dimensional and you suspect non-linear separation, try RBF or polynomial kernels and validate performance.

Q: What is gamma in RBF?

A: Gamma controls the "reach" of a single training example. Low gamma = large radius (smoother decision boundary). High gamma = tight radius (complex boundary that may overfit).

Concise Summary of Formulas

Hard-margin primal:
  min 1/2 ||w||^2
  s.t. y_i (w·x_i + b) ≥ 1

Soft-margin primal:
  min 1/2 ||w||^2 + C Σ ξ_i
  s.t. y_i (w·x_i + b) ≥ 1 - ξ_i, ξ_i ≥ 0

Dual:
  max Σ α_i - 1/2 ΣΣ α_i α_j y_i y_j K(x_i,x_j)
  s.t. 0 ≤ α_i ≤ C, Σ α_i y_i = 0

Decision:
  f(x) = Σ α_i y_i K(x_i, x) + b   (or w·x + b)
          sign(f(x)) gives the predicted class
          margin width = 2/||w||
          support vectors: α_i > 0

When to use which solver

  • SMO / kernel QP: moderate data (n up to a few tens of thousands), powerful kernels.
  • LIBLINEAR / primal solvers: high-dimensional linear problems (text, sparse features).
  • SGD / Pegasos: extremely large-scale approximate solutions.

Quick Checklist Before Training

  1. Inspect and clean data (outliers, missing values).
  2. Scale features (standardize or min-max).
  3. Choose kernel and initial hyperparameters.
  4. Use cross-validation to tune C and kernel params.
  5. Inspect support vectors and performance metrics.
Tip: For interpretability, prefer linear SVM — coefficients w indicate feature importance (larger magnitude → more influence).
