Threads in Computer Science

Definition: A thread is the smallest sequence of programmed instructions that can be managed independently by a scheduler, which is typically a part of the operating system. Threads exist within a process and share the process's resources, including memory and file handles.

Fundamental Concepts

Process vs. Thread

To understand threads, we must first distinguish them from processes:

Characteristic Process Thread
Definition An instance of a program in execution A lightweight sub-process within a process
Memory Has its own address space Shares memory with other threads in the same process
Creation Heavyweight operation (resource intensive) Lightweight operation (uses existing process resources)
Communication Inter-process communication (IPC) required Direct communication through shared memory
Context switching More expensive (requires saving entire process state) Less expensive (only thread-specific registers need saving)

Thread Components

Each thread maintains its own:

  • Thread ID: Unique identifier for the thread
  • Program Counter: Tracks which instruction to execute next
  • Register Set: Stores current working variables
  • Stack: Contains execution history and local variables

Thread States and Lifecycle

A thread transitions through several states during its lifetime:

Thread State Diagram:

New → Runnable → Running → Blocked/Waiting → Terminated

Detailed State Descriptions

  1. New: The thread has been created but not yet started.
  2. Runnable: The thread is ready to run and waiting for CPU time.
  3. Running: The thread is currently executing on the CPU.
  4. Blocked/Waiting: The thread is not running because:
    • It's waiting for I/O operations to complete
    • It's waiting for a lock to be released
    • It's explicitly put to sleep
  5. Terminated: The thread has completed execution or was terminated.

Thread Scheduling

Thread scheduling determines which thread runs when. Common approaches include:

Preemptive Scheduling

The operating system can interrupt a running thread to give CPU time to another thread. This prevents any single thread from monopolizing the CPU.

Time Slicing

Each thread gets a small time quantum to execute before the scheduler switches to another thread. This creates the illusion of parallel execution.

Priority Scheduling

Threads are assigned priorities, and higher priority threads get preference in scheduling. However, this can lead to starvation of lower priority threads.

Thread Types

User-Level Threads

Managed entirely by user-space libraries without kernel support. Advantages include:

  • Fast creation and context switching
  • Customizable scheduling policies

Disadvantages:

  • If one thread blocks, all threads in the process block
  • Cannot take advantage of multiple CPUs

Kernel-Level Threads

Managed directly by the operating system kernel. Advantages include:

  • Can run on multiple CPUs simultaneously
  • If one thread blocks, others can continue

Disadvantages:

  • Slower creation and context switching
  • More resource intensive

Hybrid Approaches

Many systems use a combination, where user-level threads are mapped to kernel threads, providing a balance between flexibility and performance.

Thread Synchronization

When multiple threads access shared resources, synchronization mechanisms are needed to prevent race conditions and ensure data consistency.

Critical Sections

Portions of code that access shared resources and must be executed atomically.

Common Synchronization Primitives

  • Mutexes (Mutual Exclusion): Allows only one thread to access a resource at a time
  • Semaphores: Generalization of mutexes that allow a specified number of threads to access a resource
  • Condition Variables: Allow threads to wait for certain conditions to become true
  • Monitors: Higher-level synchronization construct that combines mutual exclusion with condition variables
Example of a Race Condition:

Consider two threads incrementing a shared counter:

Thread 1: Reads counter (value = 5)
Thread 2: Reads counter (value = 5)
Thread 1: Increments to 6 and writes back
Thread 2: Increments to 6 and writes back

Despite two increment operations, the counter only increases by 1. This is why synchronization is crucial.

Thread Safety

A piece of code is thread-safe if it functions correctly during simultaneous execution by multiple threads. Common approaches to achieve thread safety include:

  • Using synchronization primitives
  • Making data immutable
  • Using thread-local storage
  • Employing atomic operations

Thread Pools

A thread pool is a collection of pre-initialized threads that stand ready to execute tasks. Benefits include:

  • Eliminates thread creation overhead for each task
  • Controls the number to prevent resource exhaustion
  • Provides task queueing and scheduling

Multithreading Models

Many-to-One Model

Many user-level threads mapped to a single kernel thread. Simple but limited by the inability to run threads in parallel.

One-to-One Model

Each user-level thread maps to a kernel thread. Provides better concurrency but creates more overhead.

Many-to-Many Model

Multiplexes many user-level threads to a smaller or equal number of kernel threads. Offers the best balance between flexibility and efficiency.

Conclusion

Threads are a fundamental concept in modern computing that enable efficient utilization of system resources and responsive applications. Understanding thread concepts, synchronization mechanisms, and proper thread management is crucial for developing robust concurrent applications.

While threads offer significant performance benefits, they also introduce complexity in the form of race conditions, deadlocks, and other synchronization challenges. Modern programming languages and frameworks provide various abstractions to make working with threads safer and more manageable.
