Breadth-First Search (BFS)

Breadth-First Search (BFS) is a fundamental algorithm in computer science used to traverse or search through a graph or tree data structure. BFS explores vertices or nodes of a graph level by level, starting from a given source node, visiting all of its neighbors before moving on to nodes at the next level of depth.

What is BFS?

BFS is a graph traversal method that systematically explores all the neighbor nodes at the present depth prior to moving on to nodes at the next depth level.

"BFS visits nodes in layers moving outward from the source vertex, ensuring the shortest path in an unweighted graph."

This characteristic makes BFS particularly useful in:

  • Finding the shortest path (minimum number of edges) between two nodes in an unweighted graph.
  • Checking connectivity and discovering components.
  • Level-order traversal in trees.
  • Solving puzzles and games represented as graphs (e.g., shortest path in mazes).

Basic Concepts and Terminology

  • Graph: A set of vertices (nodes) connected by edges.
  • Vertices (Nodes): The individual elements or points in the graph.
  • Edges: Connections between nodes, can be directed or undirected.
  • Adjacency: Two nodes are adjacent if there is an edge between them.
  • Queue: A First-In-First-Out (FIFO) data structure used internally by BFS to keep track of nodes to visit.
  • Visited Set: Keeps track of nodes already explored to avoid processing them multiple times.

How BFS Works: Conceptual Overview

BFS starts from a source node and explores all neighbors of that node. Once the neighbors are processed, it moves on to explore the neighbors' neighbors, and continues in this way level by level.

Steps summary:

  1. Start by enqueueing the source node and mark it as visited.
  2. While the queue is not empty:
    • Dequeue the front node u.
    • For each unvisited neighbor v of u:
      • Mark v as visited.
      • Enqueue v into the queue.
  3. Repeat until all reachable nodes are visited and the queue is empty.

Because BFS explores equally outward from the source node, it finds the shortest path to each node it visits, assuming all edges have the same cost.

BFS Algorithm in Pseudocode


BFS(Graph, start_vertex):
    let Q be a queue
    create a set visited
    enqueue start_vertex onto Q
    add start_vertex to visited

    while Q is not empty:
        current = dequeue from Q
        process(current)  // e.g., print or store

        for each neighbor in Graph.neighbors(current):
            if neighbor not in visited:
                add neighbor to visited
                enqueue neighbor onto Q

Notes:

  • Q: holds vertices waiting to be explored.
  • visited: ensures each node is processed once.
  • process(current): can be customized to define what happens when a node is visited.

Example Explanation

Consider the following undirected graph represented as adjacency lists:


A: B, C
B: A, D, E
C: A, F
D: B
E: B, F
F: C, E

If we run BFS starting from node A, the visit order will be:

  1. Start at A. Enqueue A. Visited = {A}
  2. Dequeue A. Visit neighbors: B and C. Enqueue both and mark visited.
  3. Dequeue B. Visit neighbors: D, E (skip A as already visited). Enqueue D, E, mark visited.
  4. Dequeue C. Visit neighbors: F (skip A). Enqueue F, mark visited.
  5. Dequeue D. Neighbors: B (visited), no new nodes.
  6. Dequeue E. Neighbors: B (visited), F (visited), no new nodes.
  7. Dequeue F. Neighbors: C (visited), E (visited), no new nodes.

Final Traversal Order: A → B → C → D → E → F

Applications of BFS

BFS is a versatile algorithm with many uses, including:

  • Shortest Path in Unweighted Graphs: BFS guarantees the shortest distance measured by number of edges.
  • Finding Connected Components: Identifying all nodes reachable from a source.
  • Level Order Traversal of Trees: Visiting nodes by level or depth.
  • Cycle Detection in Undirected Graphs: Using BFS to detect back edges.
  • Web Crawlers: Systematic exploration of webpages starting from a source URL.
  • Social Networking: Finding people within certain degrees of separation.

Complexity Analysis

Given V vertices and E edges, the time and space complexity of BFS are:

  • Time Complexity: O(V + E)
  • Space Complexity: O(V) due to the queue and visited set.

This efficiency makes BFS practical on large sparse graphs.

Summary

Breadth-First Search (BFS) is a cornerstone algorithm in computer science used for graph traversal. By exploring neighbors level-by-level using a queue and a visited set, BFS efficiently visits all nodes reachable from a starting point and finds shortest paths in unweighted graphs.

Understanding BFS deeply enables one to apply it in various domains such as networking, search problems, artificial intelligence, and more.
