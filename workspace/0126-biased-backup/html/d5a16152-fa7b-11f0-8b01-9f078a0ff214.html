The Bellman-Ford Algorithm

1. Introduction

The Bellman-Ford algorithm is a dynamic programming algorithm used to find the shortest path from a single source vertex to all other vertices in a weighted graph. It can handle graphs with negative weight edges, making it more versatile than some other shortest-path algorithms, such as Dijkstra's algorithm, which cannot process negative weights.

2. Theory of Bellman-Ford Algorithm

The main idea behind the Bellman-Ford algorithm is to relax the edges of the graph multiple times. "Relaxing" an edge means updating the shortest known distance to a vertex that can be reached through that edge only if the distance is less than the previously known distance.

The algorithm proceeds in the following steps:

  1. Initialize the distance to the source vertex to 0 and all other vertices to infinity (∞).
  2. Iterate through all edges up to V-1 times (where V is the number of vertices). For each edge, update the distance to the target vertex if the current distance can be improved through the source vertex.
  3. Perform an additional iteration to check for negative weight cycles. If a distance can still be updated in this iteration, a negative cycle exists in the graph.

3. Algorithm Pseudocode

function BellmanFord(graph, V, source):
    // Step 1: Initialize distances from source to all other vertices
    distance = array of size V, initialized to ∞
    distance[source] = 0

    // Step 2: Relax all edges V-1 times
    for i from 1 to V-1:
        for each edge (u, v) with weight w in graph:
            if distance[u] + w < distance[v]:
                distance[v] = distance[u] + w

    // Step 3: Check for negative weight cycles
    for each edge (u, v) with weight w in graph:
        if distance[u] + w < distance[v]:
            print("Graph contains negative weight cycle")
            return

    return distance

4. Example

Consider a graph with 5 vertices and the following weighted edges:

  • (0) --1--> (1)
  • (0) --4--> (2)
  • (1) --2--> (2)
  • (1) --7--> (3)
  • (2) --3--> (3)
  • (3) --1--> (4)
  • (4) ---5--> (1)

Let's apply the Bellman-Ford algorithm starting from vertex 0:

  1. Initially, distances are: [0, ∞, ∞, ∞, ∞]
  2. After the first iteration, distances are updated to: [0, 1, 4, ∞, ∞]
  3. After the second iteration, distances become: [0, 1, 3, 7, ∞]
  4. Continued iterations will adjust distances until the shortest paths are identified.

5. Complexity Analysis

The time complexity of the Bellman-Ford algorithm is O(V * E), where V is the number of vertices and E is the number of edges. This makes it less efficient for dense graphs compared to Dijkstra's algorithm. However, its ability to handle negative weights and detect negative cycles is a crucial advantage in various applications.

6. Conclusion

The Bellman-Ford algorithm serves as a fundamental algorithm in graph theory, enabling the calculation of shortest paths even in more complex situations where negative weights may be present. Understanding and mastering this algorithm is essential for computer scientists and engineers working with graphs and networks.
