Graph Theory: Undirected Graphs

Introduction to Graphs

Graph theory is a fundamental area of mathematics and computer science that studies the properties and applications of graphs. Graphs are mathematical structures used to model pairwise relations between objects. They consist of vertices (also called nodes) connected by edges (also called links or lines).

Definition: Graph

A graph G is an ordered pair G = (V, E) where:

  • V is a set of vertices (nodes)
  • E is a set of edges, where each edge is a 2-element subset of V
For an undirected graph, edges have no direction and are unordered pairs.

Example: Social Network

Consider a social network where people are represented as vertices and friendships as edges. If Alice and Bob are friends, there is an undirected edge between them. The friendship is mutual, so the edge has no direction.

Undirected Graph Terminology

Adjacent Vertices

Two vertices u and v are called adjacent or neighbors if they are connected by an edge.

Degree of a Vertex

The degree of a vertex is the number of edges incident to it. In an undirected graph, this is equal to the number of adjacent vertices.

Path

A path is a sequence of vertices where each adjacent pair is connected by an edge. The length of a path is the number of edges in it.

Connected Graph

An undirected graph is connected if there is a path between every pair of vertices.

Cycle

A cycle is a path that starts and ends at the same vertex with no repeated edges or vertices (except the start and end vertex).

Graph Representations

Adjacency Matrix

An adjacency matrix is a 2D array of size V×V where V is the number of vertices. The entry A[i][j] is 1 if there is an edge between vertex i and j, and 0 otherwise.

Example Adjacency Matrix

   A B C D
A 0 1 1 0
B 1 0 1 1
C 1 1 0 0
D 0 1 0 0

This represents a graph with 4 vertices (A, B, C, D) where:

  • A is connected to B and C
  • B is connected to A, C, and D
  • C is connected to A and B
  • D is connected only to B

Adjacency List

An adjacency list represents a graph as an array of linked lists. The index represents a vertex, and each element in the linked list represents the other vertices that form an edge with that vertex.

Example Adjacency List

A: B, C
B: A, C, D
C: A, B
D: B

This represents the same graph as the adjacency matrix example above.

Graph Traversal

Breadth-First Search (BFS)

BFS explores all the vertices of a graph in breadth-first order, meaning it explores all vertices at the present "depth" before moving on to vertices at the next depth level.

Algorithm Steps:

  1. Start at the root vertex (or any arbitrary vertex for an undirected graph)
  2. Explore all neighbor vertices at the present depth before moving on to vertices at the next depth level
  3. Use a queue to keep track of vertices to visit
  4. Mark vertices as visited to avoid cycles and redundant processing

Applications:

  • Shortest path in unweighted graph
  • Finding connected components
  • Web crawling
  • Social network analysis

Depth-First Search (DFS)

DFS explores as far as possible along each branch before backtracking. It uses a stack (either explicitly or via recursion) to remember where it should go when it reaches a "dead end".

Algorithm Steps:

  1. Start at the root vertex (or any arbitrary vertex for an undirected graph)
  2. Explore as far as possible along each branch before backtracking
  3. Use a stack (or recursion) to keep track of vertices to visit
  4. Mark vertices as visited to avoid cycles and redundant processing

Applications:

  • Topological sorting
  • Finding connected components
  • Solving puzzles with only one solution (mazes)
  • Detecting cycles in a graph

Graph Properties and Theorems

Handshaking Lemma

The Handshaking Lemma states that the sum of all vertex degrees in an undirected graph is equal to twice the number of edges:
∑ deg(v) = 2|E|
This is because each edge contributes to the degree of two vertices.

Eulerian Path and Circuit

An Eulerian path is a path that visits every edge exactly once.
An Eulerian circuit is an Eulerian path that starts and ends at the same vertex.

Conditions:

  • For an undirected graph to have an Eulerian circuit: All vertices must have even degree
  • For an Eulerian path but not circuit: Exactly two vertices must have odd degree

Hamiltonian Path and Cycle

A Hamiltonian path is a path that visits each vertex exactly once.
A Hamiltonian cycle is a Hamiltonian path that forms a cycle by connecting the last vertex to the first.

Unlike Eulerian paths, there is no known simple necessary and sufficient condition for a Hamiltonian path/cycle.

Special Types of Undirected Graphs

Complete Graph (Kn)

A complete graph is one where every pair of distinct vertices is connected by a unique edge.
A complete graph with n vertices has n(n-1)/2 edges.

Tree

A tree is an undirected graph that is connected and acyclic (contains no cycles).
Properties:

  • Any two vertices are connected by exactly one path
  • Number of edges = Number of vertices - 1 (|E| = |V| - 1)
  • Adding any edge creates exactly one cycle
  • Removing any edge disconnects the graph

Bipartite Graph

A bipartite graph is one whose vertices can be divided into two disjoint sets U and V such that every edge connects a vertex in U to one in V (no edges between vertices in the same set).
A graph is bipartite if and only if it is 2-colorable (can be colored with two colors such that no two adjacent vertices share the same color).

Applications of Undirected Graphs

  • Social Networks: Modeling friendships and connections between people
  • Computer Networks: Representing connections between computers or routers
  • Transportation Networks: Modeling roads between cities or stops in a transit system
  • Molecular Chemistry: Representing molecular structures where atoms are vertices and bonds are edges
  • Recommendation Systems: Finding similar items or users based on connection patterns
  • Circuit Design: Representing electrical connections between components

Conclusion

Undirected graphs are fundamental structures in computer science and mathematics, providing a powerful way to model symmetric relationships. Understanding their properties, representations, and algorithms is essential for solving many real-world problems. From social networks to computer networks, the applications of undirected graphs are vast and varied.

The concepts covered in this document - including graph representations, traversal algorithms, special graph types, and their properties - form the foundation for more advanced topics in graph theory and network analysis.
