Red-Black Tree

Red-Black Tree is a self-balancing binary search tree data structure with a guarantee of O(log n) time for search, insert, and delete operations.

It was first proposed by Barry Perl in 1979.

Red-Black Tree is a variation of the B-Tree data structure, with the addition of a color scheme to indicate the color of each node.

Properties

  • Each node is either red or black.
  • The root node is black.
  • For any node, all paths from the root to that node contain the same number of black nodes.
  • The left child of a node is black, and the right child is red.
  • The parent of a node is black.

Algorithm

Insertion: To insert a new node, we first determine the color of the node to be inserted. If the node to be inserted is black, we recursively insert it into the left subtree or right subtree. If the node to be inserted is red, we insert it into the left subtree. This process continues until we find a black node or reach the root node.

Deletion: To delete a node, we first determine the color of the node to be deleted. If the node to be deleted is black, we recursively delete it from the left subtree or right subtree. If the node to be deleted is red, we delete it from the left subtree. This process continues until we find the node to be deleted. Once the node to be deleted is found, we replace it with the last node in its subtree.

Time Complexity

Search: O(log n)

Insert: O(log n)

Delete: O(log n)

Space Complexity

O(n)
