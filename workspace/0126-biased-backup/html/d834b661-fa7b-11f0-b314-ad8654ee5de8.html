Heap Sort — A comprehensive, pedagogical explanation

Definition, theory, algorithms, step-by-step walkthrough, proofs of complexity, examples, and simple demonstration.

What is Heap Sort?

Heap Sort is a comparison-based sorting algorithm that uses a binary heap data structure to systematically extract the largest (or smallest) element and place it into a sorted region. It is an in-place algorithm (requires only a constant amount of additional memory) and guarantees O(n log n) worst-case time complexity. Heap Sort is not a stable sort.

Core ideas and intuition

The algorithm relies on the binary heap abstraction: a nearly complete binary tree where each parent node satisfies the heap property relative to its children. For a max-heap, each parent is greater than or equal to its children; for a min-heap, each parent is less than or equal to its children. Heap Sort typically builds a max-heap on the input array so that the maximum element is at the root (array index 0). Repeatedly swap the root with the last element of the unsorted portion, reduce the heap size by one, and restore the heap property (heapify) on the new root. Continue until the heap size is 1.

High-level steps

  1. Interpret the array as a binary tree stored in an array (implicit indexing).
  2. Build a max-heap from the array (Floyd's method, time O(n)).
  3. For i from n−1 down to 1:
    • Swap array[0] (max) with array[i].
    • Reduce effective heap size by 1.
    • Heapify (sift down) array[0] to restore max-heap property.
  4. Array is sorted in ascending order.

Array representation of a binary heap

For zero-based arrays:

  • Parent(i) = floor((i − 1) / 2)
  • Left child(i) = 2i + 1
  • Right child(i) = 2i + 2

Key procedures

Heapify (sift-down)

Maintain the heap property for a node i assuming its children are roots of valid heaps. Repeatedly compare the node with its children, swap with the greater child (for max-heap) if the child is larger, and continue downwards.

Build-heap (Floyd's algorithm)

Starting from the last non-leaf node (floor(n/2) − 1) down to 0, call heapify on each node. This constructs a max-heap in linear time.

Pseudocode (concise)

// Heapsort (ascending order)
BUILD-MAX-HEAP(A):
  heap_size = A.length
  for i = floor(heap_size/2) - 1 downto 0:
    MAX-HEAPIFY(A, i, heap_size)

MAX-HEAPIFY(A, i, heap_size):
  while true:
    left = 2*i + 1
    right = 2*i + 2
    largest = i
    if left < heap_size and A[left] > A[largest]: largest = left
    if right < heap_size and A[right] > A[largest]: largest = right
    if largest == i: break
    swap A[i], A[largest]
    i = largest

HEAP-SORT(A):
  BUILD-MAX-HEAP(A)
  for end = A.length - 1 downto 1:
    swap A[0], A[end]
    reduce heap_size by 1
    MAX-HEAPIFY(A, 0, heap_size)

Why build-heap is O(n), not O(n log n)

Each MAX-HEAPIFY operation cost is proportional to the height of the node. There are many nodes near the leaves (small height) and few near the root (large height). If you sum the work across all nodes, you get a telescoping sum that evaluates to O(n). Intuitively:

  • Level 0 (root): 1 node, height ≈ log n
  • Level 1: 2 nodes, height ≈ log n − 1
  • ...
  • Leaves: ~n/2 nodes, height 0
Summing number_of_nodes_at_height * height across all heights yields a total proportional to n.

Time and space complexity

  • Worst-case time: O(n log n)
  • Average-case time: O(n log n)
  • Best-case time: O(n log n) for the general algorithm (build-heap still O(n), but extracting each max takes log n when restructuring is needed)
  • Space complexity: O(1) additional (in-place)
  • Stability: not stable (equal elements may change relative order)

Correctness sketch

We can reason by a loop invariant during the main extraction loop. At the start of each iteration with heap_size = k (meaning indices 0..k−1 are a valid max-heap), the suffix indices k..n−1 contain the k largest elements, sorted in ascending order relative to one another if we proceed correctly. When you swap the root (largest in heap) with A[k−1] and reduce heap_size by 1, you place the next-largest element into the sorted suffix. After heapifying the root of the reduced heap, the heap property is restored. Repeating this n−1 times yields a fully sorted array.

Step-by-step example (detailed)

Sort A = [4, 10, 3, 5, 1] into ascending order using a max-heap.

Step A — Build max-heap (starting from index floor(n/2)-1 = 1):

Array indexing: 0:4, 1:10, 2:3, 3:5, 4:1

  1. Call heapify on i=1 (subtree rooted at index 1). Children: left=3 (5), right=4 (1). largest = 1 (10). No change.
  2. Call heapify on i=0. Children: left=1 (10), right=2 (3). largest = 1 (10). Swap A[0] and A[1]. Array becomes: [10, 4, 3, 5, 1].
  3. Now heapify at index 1 (where 4 moved). Children: left=3(5), right=4(1). largest = 3 (5). Swap A[1] and A[3]. Array: [10, 5, 3, 4, 1]. Heapify at index 3: no children. Done.

After build-heap: [10, 5, 3, 4, 1] (max-heap)

Step B — Extract max repeatedly

  1. Swap A[0]=10 with A[4]=1 → [1, 5, 3, 4, 10]. Reduce heap size to 4. Heapify at 0: left=1(5), right=2(3) → largest=1(5). Swap → [5, 1, 3, 4, 10]. Heapify index 1: left=3(4) → largest=3(4) → swap → [5, 4, 3, 1, 10].
  2. Swap A[0]=5 with A[3]=1 → [1, 4, 3, 5, 10]. Reduce heap size to 3. Heapify at 0: left=1(4), right=2(3) → largest=1(4) → swap → [4, 1, 3, 5, 10]. Heapify index 1: no children.
  3. Swap A[0]=4 with A[2]=3 → [3, 1, 4, 5, 10]. Reduce heap size to 2. Heapify at 0: left=1(1) → largest=0(3) → no swap.
  4. Swap A[0]=3 with A[1]=1 → [1, 3, 4, 5, 10]. Reduce heap size to 1. Done.

Resulting sorted array: [1, 3, 4, 5, 10]

Implementations and notes

Heapsort can be implemented in a few lines. The language-specific details matter for performance, but the algorithm is straightforward. Below are sample implementations and considerations.

In-place JavaScript implementation (concise)

function heapSort(A){
  let n = A.length;

  function heapify(i, heapSize){
    while(true){
      let left = 2*i + 1;
      let right = 2*i + 2;
      let largest = i;
      if(left < heapSize && A[left] > A[largest]) largest = left;
      if(right < heapSize && A[right] > A[largest]) largest = right;
      if(largest === i) break;
      [A[i], A[largest]] = [A[largest], A[i]];
      i = largest;
    }
  }

  // Build max heap
  for(let i = Math.floor(n/2) - 1; i >= 0; i--) heapify(i, n);

  // Extract elements from heap
  for(let end = n - 1; end > 0; end--){
    [A[0], A[end]] = [A[end], A[0]];
    heapify(0, end);
  }
  return A;
}

Python implementation (concise)

def heap_sort(a):
    n = len(a)
    def heapify(i, heap_size):
        while True:
            left = 2*i + 1
            right = 2*i + 2
            largest = i
            if left < heap_size and a[left] > a[largest]:
                largest = left
            if right < heap_size and a[right] > a[largest]:
                largest = right
            if largest == i:
                break
            a[i], a[largest] = a[largest], a[i]
            i = largest

    for i in range(n//2 - 1, -1, -1):
        heapify(i, n)
    for end in range(n-1, 0, -1):
        a[0], a[end] = a[end], a[0]
        heapify(0, end)
    return a

Common questions and variants

Is Heap Sort stable?

No — the procedure of swapping arbitrary elements to restore the heap can change the relative order of equal keys. If you need stability, use Merge Sort or a stable variant with additional bookkeeping (which loses constant extra-space property).

Why prefer Heap Sort over Quick Sort or Merge Sort?

  • Heap Sort offers guaranteed O(n log n) worst-case time and in-place behavior (O(1) extra memory).
  • Quick Sort often has better practical performance (cache behavior, lower constants) but has O(n^2) worst-case without careful pivoting.
  • Merge Sort is stable and predictable O(n log n) but requires O(n) extra memory (unless using complex in-place merges).

Relationship to Priority Queues

Heaps are the canonical array-based implementation of priority queues. Heapsort can be viewed as repeatedly extracting the max from a priority queue and placing it into output; build-heap corresponds to bulk-inserting all elements.

Proof sketch: build-heap O(n)

Consider a complete binary tree of n nodes. Nodes at depth d (root has d=0) have at most ceil(n / 2^(d+1)) nodes. The cost for heapify for a node at depth d is proportional to the height (≈ log n − d). Summing cost over all depths yields:

Sum_{d=0}^{log n} (n / 2^{d+1}) * (log n − d) = O(n) (this is a bounded geometric series). Thus BUILD-MAX-HEAP runs in linear time.

When to teach/introduce Heap Sort?

Heap Sort is an excellent topic to teach data structures + algorithms together: array-based tree representation, binary heap operations, amortized and aggregate analysis (the O(n) build), loop invariants, and trade-offs between time, space, and stability. It connects sorting theory with practical priority queue usage.
