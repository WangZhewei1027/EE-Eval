Static Typing Interactive Exploration

This interactive demonstration lets you experiment with a simplified static typing system. Define custom types, declare variables, assign values, and check whether assignments are valid according to static typing rules. Explore different type definitions, parametric generics, type compatibility, and complex user-driven workflows. 















Documentation & Usage Notes:


    
  
  • Primitive: define basic types (int, string, boolean). Assign primitive JS values.
    • 
  
  • Alias: Define a type as an alias to an existing type name (e.g. "MyInt" alias of "int").
    • 
  
  • Object: Define a type with property names and their types. E.g. {name: string, age: int}
    • 
  
  • Generic: Define a generic base type with one type parameter T (e.g. List<T>). Use T as type variable inside properties. Instantiate concrete generics by aliasing with parameters (e.g. ListString is an alias of List<string>).
    • 
  
  • Union: Define a type as a union of existing types (e.g. int | string).
    • 
  
  • You cannot redefine built-in primitives (int, string, boolean).
    • 
  
  • Assigning a value tries to parse the input: primitives parse automatically, objects expect JSON, generics expect JSON with correct inner types.
    • 
  
  • If static checking is enabled, assignment rejection will prevent data update.
