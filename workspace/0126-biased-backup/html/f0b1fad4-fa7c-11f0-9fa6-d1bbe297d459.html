Heap Sort: A Comprehensive Guide

Introduction to Heap Sort

Heap Sort is a comparison-based sorting algorithm that uses a binary heap data structure to sort elements. It was first proposed by J. W. J. Williams in 1964 and later refined by Robert W. Floyd. Heap Sort is known for its optimal worst-case time complexity of O(n log n), making it efficient for large datasets.

Definition

Heap Sort is an in-place, unstable sorting algorithm that works by first transforming the input array into a binary heap (either max-heap or min-heap) and then repeatedly extracting the root element (which is the maximum or minimum depending on heap type) and rebuilding the heap until all elements are sorted.

Binary Heap Fundamentals

To understand Heap Sort, we must first understand the binary heap data structure:

Binary Heap

A binary heap is a complete binary tree that satisfies the heap property:

  • Max-Heap Property: For every node i (except the root), the value of the parent is greater than or equal to the value of its children (A[parent(i)] ≥ A[i])
  • Min-Heap Property: For every node i (except the root), the value of the parent is less than or equal to the value of its children (A[parent(i)] ≤ A[i])

Heap Sort typically uses a max-heap for sorting in ascending order.

Complete Binary Tree

A complete binary tree is a binary tree where all levels are completely filled except possibly the last level, which is filled from left to right. This property allows heaps to be efficiently represented as arrays.

Array Representation of Heap

A binary heap can be represented as an array where:

  • The root element is at index 0
  • For any node at index i:
    • Left child is at index 2i + 1
    • Right child is at index 2i + 2
    • Parent is at index floor((i-1)/2)

Example: Array Representation

Consider the following max-heap:

                9
               / \
              7   5
             / \ /
            4 6 2

This heap can be represented as the array: [9, 7, 5, 4, 6, 2]

Heap Sort Algorithm

Heap Sort consists of two main phases:

  1. Build Heap: Transform the input array into a max-heap
  2. Sorting: Repeatedly extract the maximum element from the heap and rebuild the heap

1. Building the Heap (Heapify)

The process of converting an array into a heap is called "heapify". The algorithm starts from the last non-leaf node and works its way up to the root, ensuring the heap property is maintained at each step.

// Pseudocode for heapify heapify(array, n, i): largest = i // Initialize largest as root left = 2*i + 1 // left child right = 2*i + 2 // right child // If left child is larger than root if left < n and array[left] > array[largest]: largest = left // If right child is larger than largest so far if right < n and array[right] > array[largest]: largest = right // If largest is not root if largest != i: swap(array[i], array[largest]) // Recursively heapify the affected sub-tree heapify(array, n, largest)

The build heap process starts from the last non-leaf node (which is at index floor(n/2)-1) and works backwards to the root.

// Pseudocode for build heap buildHeap(array, n): // Start from last non-leaf node and heapify all nodes for i from floor(n/2)-1 down to 0: heapify(array, n, i)

2. Sorting Phase

Once the heap is built, the sorting process begins:

  1. Swap the root (maximum element) with the last element of the heap
  2. Reduce the heap size by 1 (effectively removing the last element from the heap)
  3. Heapify the root to maintain the heap property
  4. Repeat until the heap size is 1
// Pseudocode for heap sort heapSort(array, n): // Build max heap buildHeap(array, n) // Extract elements one by one for i from n-1 down to 1: // Move current root to end swap(array[0], array[i]) // Call heapify on the reduced heap heapify(array, i, 0)

Step-by-Step Example

Let's walk through a complete example with the array [12, 11, 13, 5, 6, 7]

1. Build the Max-Heap

Initial array: [12, 11, 13, 5, 6, 7]

We start heapifying from the last non-leaf node (index 2 = floor(6/2)-1):

  1. Heapify index 2 (value 13) - already a max-heap (no children larger)
  2. Heapify index 1 (value 11) - swap with right child (7) → [12, 11, 13, 5, 6, 7] (no change)
  3. Heapify index 0 (value 12) - swap with right child (13) → [13, 11, 12, 5, 6, 7]

Final max-heap: [13, 11, 12, 5, 6, 7]

2. Sorting Phase

Now we repeatedly extract the maximum element:

  1. Swap root (13) with last element (7) → [7, 11, 12, 5, 6, 13], heap size = 5
  2. Heapify root (7) - swap with 12 → [12, 11, 7, 5, 6, 13]
  3. Swap root (12) with last element (6) → [6, 11, 7, 5, 12, 13], heap size = 4
  4. Heapify root (6) - swap with 11 → [11, 6, 7, 5, 12, 13]
  5. Swap root (11) with last element (5) → [5, 6, 7, 11, 12, 13], heap size = 3
  6. Heapify root (5) - swap with 7 → [7, 6, 5, 11, 12, 13]
  7. Swap root (7) with last element (5) → [5, 6, 7, 11, 12, 13], heap size = 2
  8. Heapify root (5) - swap with 6 → [6, 5, 7, 11, 12, 13]
  9. Swap root (6) with last element (5) → [5, 6, 7, 11, 12, 13], heap size = 1

Final sorted array: [5, 6, 7, 11, 12, 13]

Time Complexity Analysis

Time Complexity

  • Heapify: O(log n) - This is the time to heapify a single node
  • Build Heap: O(n) - Although it might seem like O(n log n), careful analysis shows it's actually O(n)
  • Sorting Phase: O(n log n) - We perform heapify n-1 times

Total Time Complexity: O(n log n) in all cases (best, average, worst)

Space Complexity

Heap Sort is an in-place algorithm, meaning it requires only a constant amount O(1) of additional space. The sorting is done within the original array.

Characteristics of Heap Sort

  • In-place: Doesn't require additional memory proportional to input size
  • Unstable: Does not maintain the relative order of equal elements
  • Not adaptive: Its running time doesn't change much between best and worst cases
  • Comparison-based: Relies on comparisons between elements

Advantages and Disadvantages

Advantages:

  • Efficient time complexity of O(n log n) in all cases
  • Memory efficient - sorts in place with O(1) additional space
  • Good for systems with limited memory
  • No worst-case quadratic time like QuickSort

Disadvantages:

  • Unstable sort (relative order of equal elements may change)
  • Slower in practice than well-implemented QuickSort on most machines
  • Not as cache-friendly as other algorithms due to poor locality of reference
  • More complex to implement than simpler algorithms like insertion sort

Applications of Heap Sort

  • Sorting large datasets where O(n log n) performance is crucial
  • Systems with limited memory where in-place sorting is beneficial
  • As a building block in other algorithms (e.g., priority queues)
  • Embedded systems where memory is constrained
  • When worst-case performance is more important than average-case

Comparison with Other Sorting Algorithms

vs. QuickSort:

  • QuickSort is generally faster in practice due to better cache performance
  • HeapSort has guaranteed O(n log n) time while QuickSort has O(n²) worst case
  • QuickSort is typically preferred when stability isn't required

vs. MergeSort:

  • MergeSort is stable while HeapSort is not
  • MergeSort requires O(n) additional space while HeapSort is in-place
  • MergeSort has better parallelization potential

Implementation in JavaScript

// JavaScript implementation of Heap Sort function heapSort(array) { let n = array.length; // Build max heap for (let i = Math.floor(n / 2) - 1; i >= 0; i--) { heapify(array, n, i); } // Extract elements one by one for (let i = n - 1; i > 0; i--) { // Move current root to end [array[0], array[i]] = [array[i], array[0]]; // Heapify the reduced heap heapify(array, i, 0); } return array; } function heapify(array, n, i) { let largest = i; // Initialize largest as root let left = 2 * i + 1; let right = 2 * i + 2; // If left child is larger than root if (left < n && array[left] > array[largest]) { largest = left; } // If right child is larger than largest so far if (right < n && array[right] > array[largest]) { largest = right; } // If largest is not root if (largest !== i) { [array[i], array[largest]] = [array[largest], array[i]]; // Recursively heapify the affected sub-tree heapify(array, n, largest); } }

Conclusion

Heap Sort is a powerful sorting algorithm that combines the efficiency of O(n log n) time complexity with the space efficiency of in-place sorting. While it may not be as fast in practice as QuickSort for average cases, its guaranteed worst-case performance makes it valuable in certain applications. Understanding Heap Sort also provides valuable insight into heap data structures, which have applications beyond sorting in priority queues and graph algorithms.

The key to mastering Heap Sort lies in understanding the heapify operation and how it maintains the heap property. With this foundation, you can appreciate both the elegance and practical utility of this algorithm.
