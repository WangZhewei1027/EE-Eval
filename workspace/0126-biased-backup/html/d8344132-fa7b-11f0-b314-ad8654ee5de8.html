1. Definition & basic vocabulary

An undirected graph G is an ordered pair (V, E) where:

  • V is a finite set of vertices (also called nodes).
  • E is a set of edges where each edge is an unordered pair of vertices {u, v} (u and v ∈ V).

Because edges are unordered pairs, {u, v} represents the same connection as {v, u}. There is no implicit direction on edges in an undirected graph.

Variants and clarifications:

  • Simple graph: no loops (edges like {v, v}) and at most one edge between any pair of vertices.
  • Multigraph: multiple parallel edges between the same pair of vertices are allowed.
  • Weighted graph: each edge carries a numeric weight. In an undirected weighted graph, the edge weight is the same in both directions.

2. Representations and memory trade-offs

Common ways to store an undirected graph:

  • Adjacency List: For each vertex, store a list (or vector) of its neighbors. Efficient for sparse graphs. Time for iterating all neighbors is proportional to degree. Memory is O(V + E) (or O(V + 2E) if each undirected edge is stored in two lists).
  • Adjacency Matrix: A V×V boolean (or numeric) matrix A where A[u][v] = 1 if {u,v} ∈ E (and 0 otherwise). Memory O(V^2). Fast adjacency check (O(1)).
  • Edge List: Just a list of edge pairs. Very compact for some algorithms; O(E) memory. Iterating neighbors requires scanning the whole list unless indexed.
  • Incidence Matrix: A V×E matrix indicating which vertices are incident to which edges. Rare in practice; used in some theoretical contexts.

When to choose which:

  • Adjacency list: default choice for most algorithms (BFS/DFS, Dijkstra, Kruskal, etc.) when E is much smaller than V^2.
  • Adjacency matrix: convenient for dense graphs, constant-time adjacency tests, or certain matrix-based algorithms.

3. Key properties and terminology

  • Degree of a vertex v: number of edges incident to v. For a simple undirected graph, degree(v) = |neighbors(v)|.
  • Path: a sequence of vertices v0, v1, ..., vk such that each consecutive pair {vi, vi+1} is an edge. If all vertices are distinct, it's a simple path.
  • Cycle: a path with k ≥ 3 where v0 = vk and other vertices are distinct.
  • Connected: an undirected graph is connected if there exists a path between every pair of vertices. Otherwise it has multiple connected components.
  • Tree: an undirected connected acyclic graph. A tree with n vertices has exactly n − 1 edges.
  • Forest: a disjoint union of trees (acyclic undirected graph).
  • Bridge (cut-edge): an edge whose removal increases the number of connected components.
  • Articulation point (cut-vertex): a vertex whose removal increases the number of connected components.

Useful identities:

  • Sum of degrees = 2|E| (each undirected edge contributes two to the sum).

4. Traversal: BFS and DFS (concepts and step-by-step)

4.1 Breadth-First Search (BFS)

Purpose: systematic exploration of vertices by increasing distance (in number of edges) from a chosen source vertex s. BFS computes the shortest path distances (in edges) from s to every reachable vertex in an unweighted graph.

High-level idea: use a queue. Start with source s, mark it visited and enqueue it. Repeatedly dequeue a vertex u, then for every unvisited neighbor v of u, mark v visited, set parent[v] = u, and enqueue v. Continue until queue empty.

Pseudocode (textual):

BFS(G, s):
  for each vertex v in G:
    visited[v] = false
    parent[v] = NIL
    distance[v] = INFINITY
  visited[s] = true
  distance[s] = 0
  queue = empty queue
  enqueue(queue, s)

  while queue is not empty:
    u = dequeue(queue)
    for each neighbor v of u:
      if not visited[v]:
        visited[v] = true
        parent[v] = u
        distance[v] = distance[u] + 1
        enqueue(queue, v)
      end if
    end for
  end while

Complexity: Using adjacency lists, each vertex is enqueued at most once and each edge is examined at most twice (once from each endpoint), so time complexity is O(V + E). Space complexity O(V).

Properties and guarantees:

  • distance[v] produced by BFS is the minimum number of edges in any path from s to v.
  • BFS finds a shortest-path tree rooted at s (parent pointers form the tree).

4.2 Depth-First Search (DFS)

Purpose: explore as far as possible along each branch before backtracking. Useful for discovering connected components, detecting cycles, computing topological-like structures (in directed graphs), and computing bridges/articulation points.

High-level idea: use recursion or an explicit stack. Start at a vertex, mark it discovered, then recursively visit each undiscovered neighbor until no more unvisited neighbors exist; then backtrack.

Pseudocode (recursive):

DFS(G):
  for each vertex v:
    color[v] = WHITE
    parent[v] = NIL
  time = 0
  for each vertex v:
    if color[v] == WHITE:
      DFS-Visit(v)

DFS-Visit(u):
  color[u] = GRAY
  time = time + 1
  discovery[u] = time
  for each neighbor v of u:
    if color[v] == WHITE:
      parent[v] = u
      DFS-Visit(v)
    end if
  end for
  color[u] = BLACK
  time = time + 1
  finish[u] = time

Complexity: O(V + E) with adjacency lists. DFS classifies edges (tree, back, forward, cross) which is helpful for cycle detection and other analyses.

Cycle detection in undirected graphs using DFS: if during DFS from u you encounter a neighbor v that is already visited (GRAY) and v is not the parent of u, that indicates a back edge and hence a cycle.

5. Shortest paths & unweighted graphs

In an unweighted undirected graph, the length of a path is the number of edges. For computing shortest paths (minimizing number of edges), BFS is the correct algorithm: it finds shortest distances from a source vertex s to all reachable vertices.

If edges have nonnegative weights, use Dijkstra's algorithm (which works on undirected graphs by treating each undirected edge {u, v} as two directed edges u→v and v→u with the same weight). For possibly negative weights, consider Bellman-Ford (but negative cycles cause shortest paths to be undefined).

Notes:

  • BFS also yields parent pointers that can be used to reconstruct the actual shortest path from the source to any reachable target by following parent links backwards.
  • For all-pairs shortest paths: if the graph is unweighted and sparse, run BFS from each vertex (O(V*(V+E))). For dense graphs, algorithms like Floyd–Warshall (O(V^3)) may be used depending on needs.

6. Spanning trees and Minimum Spanning Tree (MST)

A spanning tree of an undirected connected graph G = (V, E) is a tree that includes all vertices V and a subset of edges T ⊆ E such that (V, T) is connected and acyclic. A graph may have many spanning trees.

If edges have weights, a Minimum Spanning Tree (MST) is a spanning tree whose total edge weight is minimum among all spanning trees.

Kruskal's algorithm (conceptual)

Sort edges by increasing weight. Start with an empty forest. Iterate edges in order; for each edge (u, v), if u and v are currently in different components (disjoint sets), add the edge to the forest (union their components). Continue until you have |V|−1 edges.

Key data structure: Disjoint Set Union (Union-Find) with union by rank and path compression.

Kruskal(G):
  sort edges E by increasing weight
  make a forest F = empty
  for each vertex v: Make-Set(v)
  for each edge (u, v) in sorted order:
    if Find-Set(u) != Find-Set(v):
      add (u, v) to F
      Union(u, v)
  return F

Complexity: O(E log E) for sorting plus nearly O(E α(V)) for union-find operations, so dominated by sorting.

Prim's algorithm (conceptual)

Start from any vertex, grow a tree by repeatedly selecting the smallest-weight edge that connects a vertex in the tree to a vertex outside. Implemented with a priority queue keyed by minimum connecting edge weight.

Complexity: O(E log V) with binary heap; O(E + V log V) with Fibonacci heap (theoretically).

Both Kruskal and Prim produce an MST. Proofs rely on the cut and cycle properties (safe edges and correctness arguments using exchange arguments).

7. Other important algorithms and problems

7.1 Connected components

Run DFS or BFS starting from unvisited vertices to discover each connected component. Each traversal explores exactly one component.

7.2 Bipartite testing

An undirected graph is bipartite if you can color vertices with two colors (say 0 and 1) so that every edge connects opposite colors. Use BFS or DFS to two-color the graph. If you ever find an edge whose endpoints have the same color, the graph is not bipartite.

7.3 Bridges and articulation points (Tarjan's algorithms)

Use DFS with discovery times and low-link values. Low[u] is the earliest discovery time reachable from u by following zero or more tree edges and then at most one back edge. An edge (u, v) (tree edge where u is parent of v) is a bridge if low[v] > discovery[u]. A vertex u is an articulation point if it satisfies certain low/discovery conditions (root with multiple children or a non-root having a child v with low[v] ≥ discovery[u]).

7.4 Union-Find (Disjoint Set Union)

Useful for Kruskal's algorithm and queries about connectivity when edges are added dynamically. Operations: Make-Set, Find-Set, and Union. With path compression and union by rank, amortized time per operation is α(n) (inverse Ackermann function), effectively constant for practical sizes.

8. Examples and worked walkthroughs (step-by-step)

Consider a small undirected graph with vertices {A, B, C, D, E, F} and edges:

  • {A, B}, {A, C}, {B, C}, {B, D}, {C, E}, {D, E}, {D, F}

Adjacency list representation:

A: B, C
B: A, C, D
C: A, B, E
D: B, E, F
E: C, D
F: D

Adjacency matrix (rows/columns ordered A,B,C,D,E,F):

  A B C D E F
A 0 1 1 0 0 0
B 1 0 1 1 0 0
C 1 1 0 0 1 0
D 0 1 0 0 1 1
E 0 0 1 1 0 0
F 0 0 0 1 0 0

BFS from source A — textual step-by-step

Goal: visit vertices in increasing distance from A and compute parent and distance arrays.

Initial state:

visited = {A}
queue = [A]
distance[A] = 0
parent[A] = NIL

Step 1: dequeue A, examine neighbors B and C.

dequeue -> u = A
neighbors: B, C
visit B: visited add B, parent[B] = A, distance[B] = 1, enqueue B
visit C: visited add C, parent[C] = A, distance[C] = 1, enqueue C

queue now: [B, C]
visited: {A, B, C}

Step 2: dequeue B, examine neighbors A, C, D.

dequeue -> u = B
neighbors: A (already visited), C (already visited), D (not visited)
visit D: visited add D, parent[D] = B, distance[D] = 2, enqueue D

queue now: [C, D]
visited: {A, B, C, D}

Step 3: dequeue C, examine neighbors A, B, E.

dequeue -> u = C
neighbors: A (visited), B (visited), E (not visited)
visit E: visited add E, parent[E] = C, distance[E] = 2, enqueue E

queue now: [D, E]
visited: {A, B, C, D, E}

Step 4: dequeue D, examine neighbors B, E, F.

dequeue -> u = D
neighbors: B (visited), E (visited), F (not visited)
visit F: visited add F, parent[F] = D, distance[F] = 3, enqueue F

queue now: [E, F]
visited: {A, B, C, D, E, F}

Step 5: dequeue E, neighbors C and D are visited → nothing new. Then dequeue F, neighbors D visited → nothing new. BFS ends.

Final distances: A:0, B:1, C:1, D:2, E:2, F:3. Parent pointers define a shortest-path tree.

DFS (sketch) on same graph

If we run a DFS starting at A and we choose neighbor order (B then C), one possible discovery order with parent pointers could be:

Visit A (parent NIL)
  Visit B (parent A)
    Visit C (parent B) — note C is neighbor of A and B; when exploring C neighbors, A and B are already discovered, so may backtrack
      Visit E (parent C)
        Visit D (parent E)
          Visit F (parent D)
Finish exploring, backtrack...

DFS explores paths deeply; discovery and finish times can be used to reason about ancestors, low-links, and find bridges or articulation points.

9. Implementation notes & complexity

9.1 Time complexity reference

  • BFS, DFS: O(V + E) with adjacency lists.
  • Adjacency matrix traversal (scanning neighbors): O(V^2) in general.
  • Kruskal: O(E log E) (dominant sorting cost) + union-find costs.
  • Prim (binary heap): O(E log V).
  • Dijkstra (binary heap): O(E log V) for weighted nonnegative edges.

9.2 Practical tips

  • Prefer adjacency lists for most problems unless V is small or graph is dense.
  • Avoid recursion for very deep DFS in languages with limited call stack; use an explicit stack instead.
  • For undirected graphs, when performing DFS cycle detection, always ignore the parent edge (the edge by which you came) to avoid false positives.
  • When reading input edges for an undirected graph, add each edge to both adjacency lists: adj[u].push(v) and adj[v].push(u).

9.3 Memory

Adjacency list uses O(V + E) memory. If each undirected edge is stored twice (one in each endpoint list), there are 2E stored directed adjacency pairs.

10. Exercises (with hints / brief answers)

  1. Exercise: Given a graph with V vertices and E edges, how many connected components can it have? Answer: between 1 and V. Exactly V if E = 0 (each vertex isolated); exactly 1 if connected.
  2. Exercise: Prove that any tree with n vertices has exactly n − 1 edges. Hint: use induction or show a connected acyclic graph must have at least n−1 edges and that any connected graph with n−1 edges is a tree. Answer (short): add edges until connected—minimum edges required is n−1; removing any edge in a tree disconnects it; combine to show equality.
  3. Exercise: How to detect if an undirected graph is bipartite? Answer: BFS/DFS two-coloring; if a same-color edge appears, not bipartite. Also equivalently: graph has no odd-length cycles.
  4. Exercise: Why does BFS produce shortest paths in unweighted graphs? Answer: by layering — BFS visits vertices in increasing distance (number of edges), so the first time we discover a vertex v is via a shortest path from source.
