Understanding AVL Trees: A Comprehensive Guide

1. Introduction to AVL Trees

An AVL Tree is a type of self-balancing binary search tree (BST). It was invented by Georgy Adelson-Velsky and Evgenii Landis in 1962, and the name AVL comes from their initials. The AVL tree is the first data structure of its kind to provide guaranteed logarithmic time complexity for basic dynamic set operations like insertion, deletion, and lookup by automatically maintaining a balanced structure.

Unlike a regular binary search tree, which can become skewed and degrade to a linked list in the worst case, an AVL tree rigorously enforces strict balancing after every insertion or deletion. This guarantees that the tree's height stays logarithmic relative to the number of nodes, ensuring efficient operations.

Key property: For every node in an AVL tree, the heights of its left and right subtrees differ by at most 1.

2. Basic Definitions and Concepts

Binary Search Tree (BST)

A binary search tree is a binary tree where every node satisfies the BST property:

  • All keys in the left subtree are less than the node's key.
  • All keys in the right subtree are greater than the node's key.

This property facilitates efficient search operations.

Height of a Node

The height of a node in a tree is the length of the longest path from that node down to a leaf node. A leaf node (one with no children) has height 0.

Balance Factor

The balance factor of a node is:

balance_factor(node) = height(left_subtree) - height(right_subtree)

For an AVL tree, the balance factor of every node can only be -1, 0, or +1.

A negative balance factor means the right subtree is taller, a positive means the left subtree is taller, and zero means both subtrees have equal height.

3. Why Balance Matters

In a normal binary search tree, operations such as searching, inserting, or deleting nodes can degrade to O(n) time complexity when the tree becomes unbalanced (e.g., behaves like a linked list). This happens when many nodes are inserted or deleted in an ascending or descending order.

An AVL tree avoids this by automatically keeping the tree height minimal — within a strict balance criterion — which guarantees worst-case time complexity of O(log n) for insertion, deletion, and lookup.

Example: Unbalanced vs Balanced BST

  • Unbalanced BST (insertion of increasing values):
10
  \
   20
     \
      30
        \
         40
  • Balanced AVL Tree after rebalancing:
      20
     /  \
   10    30
            \
             40

In the above balanced AVL tree, operations are much more efficient due to smaller height.

4. AVL Tree Operations and Algorithms

Insertion

Insertion in an AVL tree begins the same way as in a regular BST, by locating the correct position for the new node. After insertion, the tree might become unbalanced. The AVL property is then restored through rotations.

Detailed Steps for Insertion

  1. BST Insertion: Insert the new node as a leaf.
  2. Update Heights: Starting from the inserted node's parent, move upward to the root, updating the height of each ancestor node.
  3. Check Balance: Determine the balance factor of each ancestor node.
  4. Rebalance if needed: If a node's balance factor is outside the range [-1, 1], perform one of four rotation operations to restore AVL balance.

Rotation Types

There are four types of rotations that restore balance when the AVL property is violated.

1. Left Rotation (Single Rotation)

Used when a right-heavy subtree causes imbalance (Right-Right case).

   z                              y
  / \                           /   \
 T1  y      Left Rotate(z)     z     x
     / \    - - - - - - ->    / \   / \
    T2  x                   T1  T2 T3 T4
       / \
     T3  T4

2. Right Rotation (Single Rotation)

Used when a left-heavy subtree causes imbalance (Left-Left case).

      z                             y
     / \                          /   \
    y   T4   Right Rotate(z)     x     z
   / \       - - - - - - ->     / \   / \
  x   T3                     T1  T2 T3 T4
 / \
T1  T2

3. Left-Right Rotation (Double Rotation)

Used for Left-Right case (left subtree's right-heavy imbalance).

     z                          z                           x
    / \                        / \                         /  \
   y   T4   Left Rotate(y)    x   T4   Right Rotate(z)   y    z
  / \       - - - - - - ->   / \       - - - - - - ->  / \  / \
T1   x                      y   T3                  T1  T2 T3 T4
     / \                  / \
   T2  T3               T1  T2

4. Right-Left Rotation (Double Rotation)

Used for Right-Left case (right subtree's left-heavy imbalance).

   z                            z                            x
  / \                          / \                          /  \
 T1   y     Right Rotate(y)   T1   x      Left Rotate(z)  z    y
     / \    - - - - - - ->       / \    - - - - - - -> / \  / \
    x   T4                     T2   y                 T1 T2 T3 T4
   / \                             / \
 T2   T3                          T3  T4

Deletion

Deletion in AVL trees also begins like in a normal BST (removing the node by substitution when necessary). After the deletion, nodes might become unbalanced on the path from the deleted node's parent up to the root. The AVL tree fixes any imbalance by recalculating heights and performing rotations, similar to insertion but applied bottom-up.

Search

Search in an AVL tree functions exactly like in a BST and operate in O(log n) time due to balancing.

5. Complexity Analysis

  • Search: O(log n)
  • Insertion: O(log n) average and worst-case (due to rotations)
  • Deletion: O(log n) average and worst-case

The height h of an AVL tree with n nodes satisfies:

h < 1.44 * log₂(n + 2) - 0.328

This tight bound on height ensures logarithmic complexity.
