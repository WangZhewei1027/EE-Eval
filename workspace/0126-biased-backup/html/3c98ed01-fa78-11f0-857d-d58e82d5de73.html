P vs NP

The great unsolved problem in computational complexity

Class P

Problems solvable quickly – formally in polynomial time – by a deterministic computer. Examples: sorting, shortest paths, and simple arithmetic.

Class NP

Problems verifiable quickly (in polynomial time) if a solution is given, but might not be solvable quickly. Examples: Boolean satisfiability, traveling salesman, and graph coloring.

The P set contains problems solvable fast, the NP set those verifiable fast. Their overlap indicates problems quickly solvable and verifiable. Whether these are exactly the same is the millennium question.

At the heart of computational complexity lies the question:

Can every problem with quickly verifiable solutions also be solved quickly?

Class P contains problems for which a deterministic algorithm can find solutions in reasonable (polynomial) time. Class NP contains problems where a given solution can be verified in polynomial time.

All problems in P are in NP (since verification can’t be harder than solving), but whether P = NP or P ≠ NP remains unproven and famously elusive.

If P=NP, it would revolutionize fields like cryptography and optimization — many currently "hard" problems would suddenly become tractable. If P≠NP, such problems remain inherently complex, preserving existing assumptions about computational difficulty.
