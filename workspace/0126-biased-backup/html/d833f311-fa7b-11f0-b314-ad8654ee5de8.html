What is a Max Heap?

A Max Heap is a specialized complete binary tree data structure that satisfies the heap property: every node's value is greater than or equal to the values of its children. Because it is a complete binary tree, it can be compactly stored in an array without explicit child pointers.

Key points:

  • Complete binary tree: all levels are full except possibly the last, which is filled from left to right.
  • Heap property (max): parent >= children. The maximum element is always at the root.
  • Common uses: priority queues, heap sort, scheduling, selection algorithms (e.g., k-largest), maintaining dynamic maxima.

Array representation

Because the tree is complete, we store nodes in an array. There are two common indexing conventions:

  • 1-based indexing: root at index 1. For node at i: left child at 2i, right child at 2i+1, parent at floor(i/2).
  • 0-based indexing: root at index 0. For node at i: left child at 2i+1, right child at 2i+2, parent at floor((i-1)/2).

This guide uses 0-based array examples in demonstrations and pseudocode unless noted otherwise.

Primary Operations

Below are the main operations on a Max Heap with step-by-step descriptions and pseudocode.

1) Insert (often called push)

Algorithm summary: Insert the new value at the end of the array (maintains completeness), then restore the heap property by "sifting up" (also called "bubble up" or "heapify up").

  1. Append value to end: position = size.
  2. While position > 0 and parent(position) < value:
    • Swap value with parent.
    • Update position = parent(position).
  3. Stop when parent >= value or position is root.
// Pseudocode (0-based)
function push(heap, value):
  heap.append(value)
  i = heap.size - 1
  while i > 0:
    p = (i - 1) // 2
    if heap[p] >= heap[i]: break
    swap(heap[p], heap[i])
    i = p

2) Extract-Max (pop)

Algorithm summary: Remove the root (maximum). Replace it with the last element to maintain completeness, then "sift down" (heapify down) to restore the heap property.

  1. If heap empty: error/return sentinel.
  2. Store max = heap[0].
  3. Move last element to heap[0], reduce size by 1.
  4. From index 0, while node has a child:
    • Find larger child (between left and right).
    • If node >= larger child: stop.
    • Else swap node with larger child and continue from that child index.
  5. Return max.
// Pseudocode (0-based)
function pop(heap):
  if heap.size == 0: error
  maxVal = heap[0]
  heap[0] = heap.pop() // move last to root
  i = 0
  while true:
    left = 2*i + 1
    right = 2*i + 2
    largest = i
    if left < heap.size && heap[left] > heap[largest]: largest = left
    if right < heap.size && heap[right] > heap[largest]: largest = right
    if largest == i: break
    swap(heap[i], heap[largest])
    i = largest
  return maxVal

3) Build-Heap (Floyd's algorithm / heapify in linear time)

Given an unsorted array, build a heap in-place in O(n) time using bottom-up heapify:

  1. Start from the last parent index: floor(n/2) - 1 (0-based).
  2. For i from that index down to 0, call sift-down at index i.

Even though an individual sift-down may cost O(log n), most nodes are near the leaves and need little work. The total cost sums to O(n). See complexity section for explanation.

// Pseudocode (build in-place)
function buildHeap(array):
  n = array.size
  for i = floor(n/2) - 1 down to 0:
    siftDown(array, i, n)

function siftDown(array, i, n):
  while true:
    left = 2*i + 1
    right = left + 1
    largest = i
    if left < n && array[left] > array[largest]: largest = left
    if right < n && array[right] > array[largest]: largest = right
    if largest == i: break
    swap(array[i], array[largest])
    i = largest

Step-by-step Example: Insertions and Extracts (textual)

Example sequence of insertions into an initially empty max heap: insert 5, 3, 8, 1, 7.

  1. Insert 5: heap = [5]. (Root is 5)
  2. Insert 3: append -> [5,3]. 3 < parent 5, done.
  3. Insert 8: append -> [5,3,8]. parent of 8 is 5 < 8 → swap → [8,3,5]. (8 becomes root)
  4. Insert 1: append -> [8,3,5,1]. 1 < parent 3, done.
  5. Insert 7: append -> [8,3,5,1,7]. parent of 7 is 3 < 7 → swap → [8,7,5,1,3]. parent of 7 is 8 > 7 → stop.

After these operations, heap array: [8,7,5,1,3].

Now one extract-max:

  1. Remove root 8. Move last element 3 to root: [3,7,5,1].
  2. Sift down root (3): children 7 and 5, larger child 7 > 3 → swap → [7,3,5,1].
  3. Sift further at index 1: children index 3 is 1 < 3 → done.

Returned max: 8. New heap: [7,3,5,1].

Heap Sort (using Max Heap)

Heap sort uses the heap to sort an array in-place.

  1. Build a max heap from the array.
  2. For i from n-1 down to 1:
    • Swap array[0] (current max) with array[i].
    • Reduce heap size by 1 (ignore last slot which now holds a sorted value).
    • Sift-down at index 0.

Result: array sorted in ascending order. Complexity: O(n log n) time and O(1) extra space (in-place).

Note: Heap sort is not a stable sort. Equal elements may change relative order.

Why Build-Heap is O(n)

Intuition / sketch: Most nodes are near the bottom and require only a small number of swaps when sifting down. If we sum the maximum work per node by height, the total work is proportional to n:

Formal sketch: For a complete tree of n nodes, there are at most ceil(n/2^(h+1)) nodes at height h. Each such node's sift-down costs at most O(h). Summing over h from 0..log n yields a bounded geometric sum: sum_h (n/2^h * h) = O(n).

Properties, Variants, and Notes

  • Min-heap: identical idea but parent <= children; smallest at root.
  • Stability: Standard binary heaps are not stable; they don't preserve order among equal keys.
  • d-ary heaps: Generalization: each node has d children. Good for tuning decrease-key vs decrease of tree height.
  • Memory: Using array representation, extra memory is O(1) (in-place). For a pointer-based tree representation, space includes pointers.
  • Priority queue: Heaps implement priority queues efficiently; many languages' standard libraries use heap structures.
  • Decrease/Increase-Key and Delete: To change a key or remove an arbitrary element, modify it in-place, then either sift-up or sift-down depending on whether the key increased or decreased; deleting can be done by replacing with extreme value and popping.
  • Applications: scheduling, event simulation, Dijkstra's algorithm (with decrease-key), sorting, streaming top-k queries, median maintenance (with two heaps), selection algorithms.

Common Mistakes & Tips

  • Mixing up 0-based vs 1-based formulas: verify which convention you use, and consistently apply parent/child index formulas.
  • Off-by-one errors when calculating last parent: for 0-based arrays, last parent index = floor(n/2) - 1.
  • Remember to shrink the heap size (or treat sorted suffix as excluded) during heap sort.
  • Heap is good for repeated extracts of max/min and for dynamic priority management, but not ideal when you need arbitrary search or ordered traversal.

Pseudocode and Language Examples

Concise Python-style implementation of a Max Heap (array-based):

class MaxHeap:
  def __init__(self):
    self.a = []

  def push(self, x):
    self.a.append(x)
    i = len(self.a) - 1
    while i > 0:
      p = (i - 1)//2
      if self.a[p] >= self.a[i]: break
      self.a[p], self.a[i] = self.a[i], self.a[p]
      i = p

  def pop(self):
    if not self.a:
      raise IndexError("pop from empty heap")
    maxv = self.a[0]
    last = self.a.pop()
    if self.a:
      self.a[0] = last
      self._sift_down(0)
    return maxv

  def _sift_down(self, i):
    n = len(self.a)
    while True:
      left = 2*i + 1
      right = left + 1
      largest = i
      if left < n and self.a[left] > self.a[largest]:
        largest = left
      if right < n and self.a[right] > self.a[largest]:
        largest = right
      if largest == i: break
      self.a[i], self.a[largest] = self.a[largest], self.a[i]
      i = largest

Frequently Asked Questions (FAQ)

Q: Is heap sort stable?

A: No. Heap sort is not stable because pairwise swaps between non-adjacent positions can change the relative order of equal keys.

Q: When should I use a heap vs a balanced BST?

A: Use a heap when you need fast access to the max/min element and you frequently insert or extract extremes (priority queue). Use a balanced BST when you need ordered traversal, search by key, predecessor/successor queries, or dynamic order statistics.

Q: Can I find the k largest elements quickly?

A: Yes. You can build a max heap in O(n) and then extract k times (k log n). Alternatively, use a min-heap of size k to scan elements in O(n log k), which is better when k << n.
