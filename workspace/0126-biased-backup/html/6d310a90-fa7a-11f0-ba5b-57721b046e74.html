ACID Property Explanations

Atomicity

Transactions are all-or-nothing. Try disabling atomicity and see partial transactions.

Consistency

Database remains in a consistent state before and after transactions. Try disabling consistency to allow negative balances.

Isolation

Concurrent transactions don't interfere. Try running concurrent transactions with isolation on/off.

Durability

Committed transactions persist even after failures. Simulate a crash to test.
