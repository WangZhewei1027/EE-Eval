Multiset (also called a Bag) — Complete Explanation and Educational Guide

This page is focused on clear, step-by-step textual explanations: what multisets are, how they differ from sets, formal notations, operations and algorithms, complexity, and practical implementations. Interactivity is minimal — a single simple demonstration is provided near the end.

Overview — What is a Multiset?

A multiset (or bag) is like a set but allows repeated elements. Instead of only knowing whether an element exists, a multiset tracks how many copies (the multiplicity) of each element are present.

Formal definition

A multiset M over a universe U is most simply defined as a function:

m: U → {0, 1, 2, 3, …}

For each element x in U, m(x) is the multiplicity (a non-negative integer) of x in the multiset. The multiset is often written as a formal sum:

M = {x1^m(x1), x2^m(x2), …} or by listing "copies": {a, a, b, c, c, c}.

Key differences from ordinary sets

  • Sets: an element is either present (1) or absent (0).
  • Multisets: elements can be present multiple times (0, 1, 2, ...).
  • Set operations (union, intersection) use boolean logic; multiset operations work on multiplicities (min, max, addition, subtraction).

Notation and basic concepts

Multiplicity

The multiplicity of x in multiset M is denoted m_M(x). For example, for M = {a, a, b, c, c, c} we have m_M(a) = 2, m_M(b) = 1, m_M(c) = 3.

Cardinality

The cardinality (size counting duplicates) of a multiset is the sum of multiplicities:

|M| = Σ_{x∈U} m_M(x)

For M = {a, a, b, c, c, c}, |M| = 2 + 1 + 3 = 6.

Equality

Two multisets M and N are equal iff for every x, m_M(x) = m_N(x). Order of listing does not matter.

Common synonyms

Multiset is often called a bag in computer science.

Basic multiset operations — definitions and examples

Let M and N be multisets over the same universe. For each element x:

  • Union (multiset union): m_{M ∪ N}(x) = max(m_M(x), m_N(x)). This keeps the larger multiplicity.
  • Intersection: m_{M ∩ N}(x) = min(m_M(x), m_N(x)).
  • Sum (multiset sum, sometimes called multiset addition): m_{M ⊎ N}(x) = m_M(x) + m_N(x).
  • Difference (M \ N): m_{M\N}(x) = max(0, m_M(x) − m_N(x)).
  • Subset (M ⊆ N as multisets): ∀x, m_M(x) ≤ m_N(x).

Concrete example

Let M = {a, a, b, c, c, c} and N = {a, b, b, c, d}.

Compute multiplicities:

  • m_M: a→2, b→1, c→3, d→0
  • m_N: a→1, b→2, c→1, d→1

Then:

  • Union M ∪ N: a→max(2,1)=2, b→max(1,2)=2, c→3, d→1 → {a,a,b,b,c,c,c,d}
  • Intersection M ∩ N: a→1, b→1, c→1, d→0 → {a,b,c}
  • Sum (⊎): a→3, b→3, c→4, d→1 → {a,a,a,b,b,b,c,c,c,c,d}
  • Difference M \ N: a→1, b→0, c→2, d→0 → {a,c,c}

Use in combinatorics — counting with repetitions

Permutations of multiset

If you form arrangements (linear permutations) of n objects where an element type i appears n_i times (Σ n_i = n), the number of distinct permutations is:

n! / (n_1! n_2! ... n_k!)

Example: for {a,a,b,c,c,c} with counts 2,1,3, the number of distinct orderings is 6! / (2! 1! 3!) = 720 / (2·1·6) = 60.

Combinations with repetitions

To count multisets (combinations) of size r chosen from k types (order doesn't matter, repetitions allowed), use the "stars and bars" formula:

Number = C(r + k − 1, k − 1)

This counts nonnegative integer solutions to x1 + x2 + ... + xk = r (x_i are multiplicities).

Representation & Algorithms — how to implement multisets

A multiset must store multiplicities. Common representations and their tradeoffs:

1) Frequency map (hash table)

Represent as a map/dictionary from element → integer multiplicity. This is the most common general-purpose representation.

Operations:

  • Insert x: increment m[x] (O(1) expected with hash).
  • Remove one copy: decrement m[x], delete map entry if it reaches 0.
  • Cardinality: maintain a running total or compute sum of values.
  • Union/intersection: iterate keys and compute max/min values.

Complexity: typical operations O(1) average, set-like operations O(k) where k = number of distinct keys involved.

2) Sorted list of (element, count) pairs

Useful if you need ordered iteration and the universe is ordered. Union/intersection performed by merging sorted lists (like merge step in merge sort) in O(a + b) where a,b are numbers of distinct keys.

3) Balanced search trees (multiset as BST)

Each node stores element and multiplicity; operations are O(log n) for n distinct keys. Useful when ordering and log-time updates are needed.

4) Arrays or direct index mapping

When the universe is a small integer range (0..U-1), an array of integers can store multiplicities compactly and with constant-time access.

5) Compressed or frequency-sorted forms

If many multiplicities are identical or elements are contiguous, run-length encoding or other compression can help.

Algorithms (pseudocode)

Union (max of multiplicities) using frequency maps:

function multisetUnion(mapA, mapB):
    result = empty map
    for each key in mapA:
        result[key] = mapA[key]
    for each key in mapB:
        if key in result:
            result[key] = max(result[key], mapB[key])
        else:
            result[key] = mapB[key]
    return result

Intersection (min):

function multisetIntersection(mapA, mapB):
    result = empty map
    for each key in mapA:
        if key in mapB:
            result[key] = min(mapA[key], mapB[key])
    return result

Difference (A \ B):

function multisetDifference(mapA, mapB):
    result = empty map
    for each key in mapA:
        count = mapA[key] - (mapB.get(key) or 0)
        if count > 0:
            result[key] = count
    return result

Complexity notes: If m and n are the numbers of distinct keys in A and B, union and difference are O(m + n) when implemented by iterating keys and using hash-table lookups. Intersection is O(min(m, n)) if you iterate the smaller map and lookup in the larger.

Multisets in programming languages and libraries

Python

Use collections.Counter which is a dict subclass mapping elements to counts. Example operations: addition, subtraction, & (intersection, min), | (union, max).

C++

std::multiset is a sorted container that stores multiple copies of values with one entry per copy (not counts). It behaves like a container of n items rather than a map from element to count. For counting multiplicities explicitly, use std::unordered_map<T,int> or std::map<T,int>.

Java

Guava's Multiset interface provides methods to add/remove counts and query multiplicity. Apache Commons also has Bag implementations.

SQL and databases

Database tables naturally represent multisets: each row can be considered one occurrence; GROUP BY with COUNT aggregates multiplicities.

Applications

  • Counting and combinatorics: permutations with repeated elements, combinations with repetition.
  • Text processing: word frequency is a multiset of words.
  • Histograms and frequency analysis.
  • Databases where duplicates matter (multiset semantics vs set semantics).
  • Search engines, inverted indexes often count occurrences per document (a multiset-like structure).
  • Probability distributions over categories (finite multisets with multiplicities as frequencies/probabilities after normalization).

Advanced topics — multiset coefficients and generating functions

Counting multisets of size r from k types (combinations with repetition) is given by the multiset coefficient (often written H(k,r) or C(r+k-1, r)).

Generating functions: The ordinary generating function for multisets where you have unlimited copies of each of k distinct items is:

(1 + x + x^2 + x^3 + …)^k = (1 / (1 − x))^k, and the coefficient of x^r is C(r + k − 1, r).

Step-by-step pedagogical examples

Example A — Building a multiset from a list

Given the list L = [apple, banana, apple, cherry, banana, apple]

Step 1: Start with an empty frequency map {}.

Step 2: Process each item in order:

  1. apple: map = {apple:1}
  2. banana: map = {apple:1, banana:1}
  3. apple: map = {apple:2, banana:1}
  4. cherry: map = {apple:2, banana:1, cherry:1}
  5. banana: map = {apple:2, banana:2, cherry:1}
  6. apple: map = {apple:3, banana:2, cherry:1}

Resulting multiset: {apple×3, banana×2, cherry×1} with cardinality 6.

Example B — Counting distinct permutations

Take M = {A, A, B, C, C} with multiplicities (2,1,2). Total items n = 5. Distinct permutations:

5! / (2! 1! 2!) = 120 / (2·1·2) = 30

Step-by-step reasoning: Choose positions for the two A's (C(5,2) ways), then choose positions for the two C's among remaining 3 positions (C(3,2) ways), then B occupies the final position. Multiply C(5,2)*C(3,2) = 10*3 = 30.

Practical advice and pitfalls

  • Decide whether duplicates should matter early: many APIs default to set semantics and will silently deduplicate data.
  • For large numbers of identical elements, storing multiplicities instead of storing each copy saves memory.
  • When using containers like C++ std::multiset (one node per copy), operations like erase of one copy can be O(log n) but can be more memory-intensive.
  • To check multiset subset (A ⊆ B), iterate distinct keys of A and ensure m_A(x) ≤ m_B(x) for each x (O(|distinct(A)|) lookups).

Summary — what to remember

  • Multisets generalize sets by allowing multiple occurrences; each element has a multiplicity ≥ 0.
  • Common operations act on multiplicities (min, max, addition, subtraction with clamp-to-zero).
  • Representation choices (hash maps, arrays, trees, repeated-element containers) affect performance and memory.
  • Multisets appear naturally in combinatorics, statistics (histograms), and computer science tasks where duplicates matter.
