Comprehensive Guide to Congestion Control

Introduction to Congestion Control

Congestion control is a fundamental concept in computer networking that regulates the amount of data entering the network to prevent network overload and ensure fair resource allocation among users.

Definition

Congestion Control refers to the mechanisms and algorithms used to prevent or alleviate congestion in a network by regulating the rate at which senders transmit data. It aims to maintain network performance and prevent collapse when demand approaches or exceeds capacity.

Congestion occurs when the aggregate demand for network resources (bandwidth, buffer space, processing power) exceeds the available capacity, leading to:

  • Increased packet delay
  • Packet loss
  • Reduced throughput
  • Potential network collapse

Why Congestion Control is Necessary

Without proper control mechanisms, networks can experience congestion collapse where increasing traffic leads to decreasing useful throughput. This occurs because:

Example Scenario

Imagine a highway during rush hour. As more cars enter the highway:

  1. Initially, flows smoothly with high throughput (many cars passing per minute)
  2. As density increases, cars must slow down to maintain safe distances
  3. At critical density, throughput starts decreasing as speed drops
  4. Eventually, traffic may come to a complete standstill (congestion collapse)

Congestion control in networks works similarly, preventing this collapse by regulating the "traffic" (data packets) entering the network.

Key Concepts in Congestion Control

1. Congestion Window (cwnd)

The congestion window is a TCP state variable that limits the amount of data a sender can transmit before receiving an acknowledgment. It's the primary mechanism TCP uses to control congestion.

2. Slow Start

An algorithm that gradually increases the transmission rate to discover the available network capacity without causing congestion.

3. Congestion Avoidance

After slow start reaches a threshold, the algorithm switches to a more conservative approach to avoid congestion.

4. Fast Retransmit

A mechanism that triggers retransmission of a packet after receiving three duplicate ACKs, indicating packet loss.

5. Fast Recovery

After fast retransmit, the algorithm enters this phase to recover from packet loss without a full slow start.

TCP Congestion Control Algorithms

TCP Tahoe

The original TCP congestion control algorithm includes:

  1. Slow Start phase: cwnd grows exponentially
  2. Congestion Avoidance phase: cwnd grows linearly
  3. On packet loss: cwnd reset to 1 and slow start begins again

TCP Reno

An improvement over Tahoe that adds:

  1. Fast Retransmit: Retransmit after 3 duplicate ACKs
  2. Fast Recovery: After retransmission, enter recovery instead of full slow start
  3. cwnd is halved rather than reset to 1 on packet loss

TCP Cubic

A more modern algorithm that uses a cubic function to determine cwnd growth:

  1. After packet loss, cwnd is reduced by a factor
  2. Then grows as a cubic function of time since last congestion event
  3. Designed for high-bandwidth, high-latency networks
Algorithm Key Features Best Use Case
Tahoe Basic slow start, congestion avoidance, full reset on loss Basic networks
Reno Adds fast retransmit and recovery Most general-purpose networks
Cubic Cubic growth function, less aggressive reduction High-speed, long-distance networks

Detailed Operation of TCP Congestion Control

Slow Start Phase

  1. Sender starts with cwnd = 1 segment (typically 1 MSS - Maximum Segment Size)
  2. For each ACK received, cwnd increases by 1 segment
  3. This results in exponential growth: 1, 2, 4, 8, 16, etc.
  4. Continues until cwnd reaches the slow start threshold (ssthresh)

Congestion Avoidance Phase

  1. When cwnd reaches ssthresh, switch to linear growth
  2. cwnd increases by approximately 1 segment per round-trip time (RTT)
  3. This is achieved by increasing cwnd by 1/cwnd for each ACK received
  4. Continues until packet loss is detected

Response to Packet Loss

Different responses based on loss detection method:

  • Timeout: Indicates severe congestion
    • ssthresh set to half of current cwnd
    • cwnd reset to 1
    • Slow start begins again
  • 3 Duplicate ACKs: Indicates mild congestion
    • Fast retransmit of lost packet
    • ssthresh set to half of current cwnd
    • cwnd set to ssthresh + 3
    • Enter fast recovery

Mathematical Foundations

Additive Increase Multiplicative Decrease (AIMD)

The core principle behind TCP's congestion avoidance:

  • Additive Increase: cwnd += 1/cwnd per ACK (approximately +1 per RTT)
  • Multiplicative Decrease: cwnd *= β (typically β = 0.5) on congestion

Throughput Calculation

The average throughput (B) of a TCP connection can be approximated by:

B ≈ (MSS × C) / (RTT × √p)

Where:

  • MSS: Maximum Segment Size
  • C: Constant (typically ~1.22 for TCP Reno)
  • RTT: Round-Trip Time
  • p: Packet loss probability

Advanced Congestion Control Techniques

Explicit Congestion Notification (ECN)

A mechanism that allows network devices to explicitly signal congestion before buffers overflow:

  1. Routers mark packets when congestion is imminent (rather than dropping them)
  2. Receivers echo the congestion indication back to senders
  3. Senders reduce transmission rate in response

Bufferbloat and Active Queue Management

Bufferbloat occurs when excessively large buffers cause high latency:

  • Traditional FIFO queues can fill up, causing long delays
  • Solutions like CoDel (Controlled Delay) and PIE (Proportional Integral controller Enhanced) actively manage queue lengths
  • These algorithms drop or mark packets before queues become too full

Congestion Control in Modern Networks

Datacenter TCP (DCTCP)

Optimized for data center environments with:

  • Very low latency (microsecond RTTs)
  • High bandwidth (10Gbps+)
  • Shallow buffers
  • Uses ECN for precise congestion feedback

BBR (Bottleneck Bandwidth and Round-trip propagation time)

A modern congestion control algorithm developed by Google that:

  • Models the network path to estimate available bandwidth and minimum RTT
  • Seeks to operate at the optimal operating point
  • Doesn't rely solely on packet loss as a congestion signal

Conclusion

Congestion control is a critical component of network protocols that ensures fair and efficient utilization of network capacity. From the basic AIMD principle to modern algorithms like BBR, these mechanisms have evolved to handle increasingly complex network environments while maintaining stability and performance.

Key takeaways:

  • Congestion control prevents network collapse and ensures fair resource allocation
  • TCP uses a combination of slow start, congestion avoidance, and loss recovery mechanisms
  • Different algorithms (Tahoe, Reno, Cubic, BBR) are optimized for different network conditions
  • Modern techniques like ECN and AQM help manage congestion more effectively
