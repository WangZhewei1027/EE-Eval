Comprehensive Guide to Stacks

Introduction to Stacks

A stack is a fundamental linear data structure in computer science that follows a particular order in which operations are performed. The order is commonly known as Last In, First Out (LIFO), meaning the last element added to the stack will be the first one to be removed.

This behavior is analogous to a stack of plates in a cafeteria: you can only take the top plate from the stack, and when adding a new plate, it goes on top of the existing ones.

Basic Stack Operations

Stacks typically support the following fundamental operations:

1. Push

Adds an element to the top of the stack. If the stack is full, this operation results in a stack overflow.

2. Pop

Removes and returns the top element of the stack. If the stack is empty, this operation results in a stack underflow.

3. Peek (or Top)

Returns the top element of the stack without removing it.

4. isEmpty

Returns true if the stack is empty, false otherwise.

5. isFull

Returns true if the stack is full (in case of fixed-size implementations), false otherwise.

Operation Example:

Consider an empty stack. The following operations would result in:

  1. Push(10): Stack → [10]
  2. Push(20): Stack → [20, 10]
  3. Push(30): Stack → [30, 20, 10]
  4. Pop(): Returns 30, Stack → [20, 10]
  5. Peek(): Returns 20, Stack remains [20, 10]
  6. Pop(): Returns 20, Stack → [10]
  7. Pop(): Returns 10, Stack → []

Stack Implementation

Stacks can be implemented using various underlying data structures. The two most common approaches are:

1. Array-based Implementation

Uses a fixed-size or dynamic array to store elements. Requires tracking the top index.

class Stack {
    constructor(capacity) {
        this.items = new Array(capacity);
        this.top = -1;
        this.capacity = capacity;
    }
    
    push(element) {
        if (this.isFull()) {
            throw new Error("Stack overflow");
        }
        this.items[++this.top] = element;
    }
    
    pop() {
        if (this.isEmpty()) {
            throw new Error("Stack underflow");
        }
        return this.items[this.top--];
    }
    
    peek() {
        if (this.isEmpty()) {
            throw new Error("Stack is empty");
        }
        return this.items[this.top];
    }
    
    isEmpty() {
        return this.top === -1;
    }
    
    isFull() {
        return this.top === this.capacity - 1;
    }
}

2. Linked List-based Implementation

Uses nodes connected via pointers. More flexible as it can grow dynamically.

class Node {
    constructor(data) {
        this.data = data;
        this.next = null;
    }
}

class Stack {
    constructor() {
        this.top = null;
    }
    
    push(data) {
        const newNode = new Node(data);
        newNode.next = this.top;
        this.top = newNode;
    }
    
    pop() {
        if (this.isEmpty()) {
            throw new Error("Stack underflow");
        }
        const data = this.top.data;
        this.top = this.top.next;
        return data;
    }
    
    peek() {
        if (this.isEmpty()) {
            throw new Error("Stack is empty");
        }
        return this.top.data;
    }
    
    isEmpty() {
        return this.top === null;
    }
}

Time Complexity Analysis

The time complexity of stack operations is crucial for understanding its efficiency:

Operation Array Implementation Linked List Implementation
Push O(1) (amortized for dynamic arrays) O(1)
Pop O(1) O(1)
Peek O(1) O(1)
isEmpty O(1) O(1)

Note: The O(1) complexity for array push operations assumes a dynamic array implementation where resizing operations (which are O(n)) occur infrequently enough that their cost can be amortized over many operations.

Applications of Stacks

Stacks are widely used in computer science due to their simplicity and efficiency. Some key applications include:

1. Function Call Management

Programming languages use a call stack to manage function calls, local variables, and return addresses. Each function call pushes a new frame onto the stack, and returning pops it off.

2. Expression Evaluation

Stacks are used in evaluating arithmetic expressions (infix, postfix, prefix notation) and for operator precedence handling.

3. Backtracking Algorithms

Algorithms like depth-first search (DFS) use stacks to remember paths and backtrack when needed.

4. Undo Mechanisms

Many applications implement undo functionality by keeping previous states in a stack.

5. Syntax Parsing

Compilers use stacks to check for balanced parentheses, brackets, and other syntax elements.

6. Browser History

Web browsers often use a stack-like structure to manage the back button functionality.

Expression Evaluation Example:

Evaluating the postfix expression "3 4 + 5 *" using a stack:

  1. Push 3 → Stack: [3]
  2. Push 4 → Stack: [4, 3]
  3. Pop 4 and 3, add (3+4=7), push 7 → Stack: [7]
  4. Push 5 → Stack: [5, 7]
  5. Pop 5 and 7, multiply (7*5=35), push 35 → Stack: [35]
  6. Result is 35

Stack Variations

Several variations of the basic stack exist to address specific needs:

1. Double Stack

Two stacks implemented using a single array, growing from opposite ends. Efficient memory usage when two stacks are needed.

2. Min/Max Stack

Maintains additional information to provide O(1) access to the minimum or maximum element in the stack.

3. Monotonic Stack

A stack where elements are in strictly increasing or decreasing order. Useful for certain algorithmic problems.

4. Concurrent Stack

Thread-safe stack implementations for multi-threaded environments.

Common Stack Problems

Understanding these classic problems helps master stack concepts:

1. Balanced Parentheses

Check if a string of parentheses is properly balanced (every opening has a matching closing).

2. Next Greater Element

For each element in an array, find the next element that is greater than it.

3. Stock Span Problem

Calculate the span of stock prices where the current price is greater than previous prices.

4. Largest Rectangle in Histogram

Find the largest rectangular area under a histogram.

5. Tower of Hanoi

The classic recursion problem that can be solved using stack concepts.

Conclusion

Stacks are a fundamental data structure with wide-ranging applications in computer science. Their LIFO property makes them ideal for scenarios where the most recent item needs to be accessed first. Understanding stacks is essential for:

  • Writing efficient algorithms
  • Understanding program execution
  • Solving many common programming problems
  • Designing systems with undo/redo functionality
  • Implementing parsers and compilers

Mastering stack operations and their implementations provides a strong foundation for more advanced data structures and algorithms.
