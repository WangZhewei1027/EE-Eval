NoSQL

A visual exploration of flexible, scalable data architectures that break free from traditional relational models.

📄

Document Stores

Store data in flexible, JSON-like documents with dynamic schemas.

{
  "_id": 1,
  "name": "John",
  "age": 30,
  "hobbies": ["hiking", "photography"]
}
🗄️

Key-Value

Simple data model that maps unique keys to values.

user:12345 → {name: "Alice", email: "alice@example.com"}
product:789 → {title: "Laptop", price: 999.99}
🧮

Column-Family

Organize data into columns rather than rows for efficient querying.

UserID | Name | Email | Age
----------------------------
1 | John | john@... | 30
2 | Sarah | sarah@... | 28
🕸️

Graph

Store entities and relationships as nodes and edges in a graph.

(User)-[FRIENDS_WITH]->(User)
(Product)-[PURCHASED_BY]->(User)

NoSQL databases provide flexible schemas, horizontal scaling, and specialized data models for modern applications.
