Understanding Multisets: Theory, Definitions, and Examples

In this educational article, we explore the concept of multisets, a fundamental structure in mathematics and computer science. We will cover the formal definition, basic properties, practical examples, and relevant algorithms. This page aims to be a thorough textual guide – ideal for students, educators, and anyone seeking a clear understanding of multisets.

1. What is a Multiset?

A multiset (sometimes called a bag) generalizes the concept of a set by allowing elements to appear multiple times. Unlike a traditional set, where the membership of an element only records presence or absence, a multiset keeps track of the multiplicity (the count) of each element.

"In other words, a multiset is a collection in which elements can repeat — but unlike sequences (lists), the order does not matter."

We can view a multiset as a function from elements to the non-negative integers (including zero), representing how many copies of each element are present.

1.1 Formal Definition

Let S be a universe of elements. A multiset over S is a mapping: 

M: S → ℕ₀

where M(x) is the multiplicity (non-negative integer) of the element x ∈ S.

Elements with M(x) = 0 are considered not to be in the multiset, and those with M(x) > 0 appear as many times as the value indicates.

1.2 Simple Example

Consider the multiset 

M = {a, a, b, c, c, c}

Here the multiplicities are:

  • M(a) = 2
  • M(b) = 1
  • M(c) = 3

Unlike a set {a,b,c}, where each element appears once, a multiset stores how many copies of each element it contains.

2. Difference Between Sets, Multisets, and Sequences

It's important to distinguish multisets from other collection types:

  • Set: Elements are unique. Order does not matter. Membership is binary: either an element is in or out.
  • Multiset: Order does not matter. Elements can repeat, and the multiplicity counts how many times.
  • Sequence/List/Array: Elements can repeat, and order matters.

For example, the multiset {a, a, b} is considered identical to {a, b, a} because order does not matter, but different from {a, b} since the second a is missing.

3. Basic Operations on Multisets

Multisets admit a set of natural operations analogous to those on sets, but adapted to multiplicities.

3.1 Membership

The multiplicity M(x) tells how many times an element x appears.

If M(x) > 0, then x is in the multiset. If zero, it is not.

3.2 Union

The union of two multisets M₁ and M₂ is obtained by taking the maximum multiplicity for each element:

(M₁ ∪ M₂)(x) = max(M₁(x), M₂(x))

3.3 Intersection

The intersection is given by the minimum multiplicity:

(M₁ ∩ M₂)(x) = min(M₁(x), M₂(x))

3.4 Sum (Multiset Addition)

The sum of multisets adds the multiplicities:

(M₁ + M₂)(x) = M₁(x) + M₂(x)

3.5 Difference

The difference subtracts the multiplicities, but clipping at zero to avoid negative counts:

(M₁ - M₂)(x) = max(M₁(x) - M₂(x), 0)

4. Representing Multisets Computationally

In practice, multisets can be represented in various ways in software:

  • Map / Dictionary: A common and straightforward approach is to map elements to their counts. For example, in Python: {"a": 2, "b":1, "c":3}.
  • List / Array with duplicates: Simply storing repeated elements, but then extracting multiplicity requires counting.
  • Sorted list with multiplicities: To optimize certain algorithms, you may keep elements grouped with known counts.

Most programming languages provide data structures for multisets or bags. For example, Python's collections.Counter acts as a multiset.

5. Common Algorithms Involving Multisets

Many classical algorithms and problems rely on the concept of multisets. Some examples:

5.1 Counting Multiplicities

Given a list of elements, counting how many times each element occurs is a basic multiset construction.

Algorithm: Counting Multiplicities
Input: A list of elements L = [e₁, e₂, ..., e_n]
Output: Multiset M representing multiplicities
Steps:
1. Initialize empty dictionary M.
2. For each element e in L:
   a. If e not in M, set M[e] = 1
   b. Else, increment M[e] by 1
3. Return M

This algorithm runs in O(n) time for n elements.

5.2 Multiset Inclusion Check

Given two multisets M₁ and M₂, to check if M₁ is a subset of M₂ (meaning for every element x: M₁(x) ≤ M₂(x)), one can:

Algorithm: Multiset Subset Test
Input: Two multisets M₁ and M₂
Output: True if M₁ ⊆ M₂, False otherwise
Steps:
1. For each element x in M₁:
   a. If M₁(x) > M₂(x) (or x not in M₂), return False
2. Return True

5.3 Generating Multisets of Fixed Size

Another interesting problem involves generating all multisets containing elements from a set S, of fixed total size (total multiplicity sum). This is related to combinations with repetition, which has combinatorial formulas and backtracking algorithms.

6. Applications and Examples

Multisets appear naturally in many domains:

  • Counting Problems: E.g., counting how many times an event occurs.
  • Polynomial Algebra: Multisets correspond to exponents in monomials where repeated variables are allowed.
  • Database Queries: Bags (multisets) are common when duplicates are allowed.
  • Combinatorics: Counting combinations with repeated elements.
  • NLP & Text Processing: Word frequency counts form multisets of words.

6.1 Counting Letters Example

Consider the word "MISSISSIPPI". The multiset of letters is:

  • M: 1
  • I: 4
  • S: 4
  • P: 2

Unlike a set with only unique letters, the multiset records these frequencies, which are crucial for problems like counting permutations of letters.

8. Summary

To recap:

  • A multiset generalizes sets by allowing multiple instances of the same element.
  • Multiplicity indicates how many times each element appears.
  • Basic multiset operations are shaped by the behavior of multiplicities (max for union, min for intersection, sum for addition, etc.).
  • Multisets are pervasive across mathematics, computer science, and practical applications such as counting, algebra, and text processing.
  • Computational representations typically use maps/dictionaries to keep track of element counts efficiently.

By understanding multisets, you gain deeper insight into counting problems, combinatorics, and data structures that go beyond classical sets.
