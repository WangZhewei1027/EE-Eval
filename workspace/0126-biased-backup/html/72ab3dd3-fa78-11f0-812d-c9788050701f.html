Space Complexity

Understanding how algorithms utilize memory resources

O(1) - Constant Space

The algorithm uses a fixed amount of memory regardless of input size. Memory usage remains flat and predictable.

function sum(a, b) {
  const result = a + b;
  return result;
}
// Only uses memory for variables a, b, and result

O(n) - Linear Space

Memory usage grows proportionally with input size. Common when storing each input element in a data structure.

function duplicateArray(arr) {
  const newArray = [];
  for (let i = 0; i < arr.length; i++) {
    newArray.push(arr[i]);
  }
  return newArray;
}
// Memory usage grows with input array size

O(n²) - Quadratic Space

Memory usage grows with the square of input size. Often seen with nested data structures or recursive algorithms.

function generateMatrix(n) {
  const matrix = [];
  for (let i = 0; i < n; i++) {
    matrix[i] = [];
    for (let j = 0; j < n; j++) {
      matrix[i][j] = i * j;
    }
  }
  return matrix;
}
// Memory grows exponentially with input size
