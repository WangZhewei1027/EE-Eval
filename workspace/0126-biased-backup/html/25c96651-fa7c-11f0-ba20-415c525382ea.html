Understanding the Min Heap Data Structure

A heap is a specialized tree-based data structure that satisfies the heap property. In a Min Heap, this property ensures that the key (value) of the parent node is always less than or equal to the keys of its children. Thus, the minimum element is always found at the root of the tree.

What is a Min Heap?

A Min Heap is a complete binary tree where every node's value is less than or equal to the values of its children. It can be visualized as a binary tree where:

  • The tree is complete — all levels are fully filled except possibly for the lowest level, which is filled from the left up to a point.
  • The heap property holds: each parent node's value is smaller than or equal to each of its children’s values.

This property guarantees that the smallest element in the heap is always at the root node, enabling efficient access to the minimum element.

Terminology

  • Root: The top-most node, containing the minimum value in a Min Heap.
  • Parent: Any node except the root has a parent node.
  • Child: The nodes directly reachable from a node (in binary trees, typically up to two children: left and right).
  • Leaf nodes: Nodes without any children, typically located at the lowest level.

Key Properties of a Min Heap

  1. Heap order property: Every parent node is less than or equal to its children.
  2. Complete binary tree: The tree is perfectly balanced except possibly for the last level, which is filled from left to right.

These properties allow efficient operations to access, insert, and delete the minimum value.

How is a Min Heap Represented?

Although we often conceptualize heaps as tree structures, in computer science, heaps are commonly implemented using arrays. This approach eliminates the need for pointers and links, making memory access cache-friendly and efficient.

Array Representation of a Min Heap

For a node stored at index i (assuming zero-based indexing), its relatives can be found by the formulas:

  • Parent index: Math.floor((i-1)/2)
  • Left child index: 2*i + 1
  • Right child index: 2*i + 2

This means a binary tree is "flattened" into a linear array while preserving the parent-child relationships implicitly by index calculations.

Example: Array Representation

Consider the following min heap tree:

         4
       /   \
      10    15
     /  \  
    20  30

The array representation (zero-based) is:

[4, 10, 15, 20, 30]

- The root 4 is at index 0.
- Its left child 10 is at index 1: 2*0 + 1 = 1
- Its right child 15 is at index 2: 2*0 + 2 = 2
- Node 10's children 20 and 30 are at indices 3 and 4 respectively.

Min Heap Operations

The following operations are fundamental to a Min Heap:

1. Peek (Find Minimum)

Retrieve the minimum element without removing it. This is efficient because the minimum is always at the root — located at array index 0.

Time complexity: O(1) ---

2. Extract Min (Remove Minimum)

This operation removes the root (minimum element) from the heap and returns it. Since removing the root breaks the heap’s structure, the heap must be restored afterward, preserving the min-heap property.

Steps:

  1. Replace the root element with the last element in the heap (the one at the highest index in the array).
  2. Reduce the size of the heap by one, effectively removing the last element.
  3. Heapify down (sift down): Starting from the root, compare with its children and swap with the smaller child if the root is greater — repeat until the heap property is restored.
Time complexity: O(log n) where n is the number of nodes in the heap. ---

3. Insert (Add an element)

Inserting a new value involves adding it to the heap while maintaining the min heap property.

Steps:

  1. Add the new element at the end of the array (the next available leaf position).
  2. Heapify up (sift up): Compare the added element with its parent; if the child is smaller, swap them. Repeat until the min-heap property is reestablished or the root is reached.
Time complexity: O(log n) ---

4. Build Heap

Given an arbitrary array of elements, you can convert it into a min heap in-place using a process called heapification:

  1. Start from the last non-leaf node (which is at index Math.floor(n/2) - 1).
  2. Apply heapify down (sift down) to this node.
  3. Move backwards, applying heapify down to each node until the root.
Time complexity: O(n) — more efficient than repeatedly inserting elements.

The Heapify Algorithm (Sift Down)

The heapify or sift down process restores the min-heap property starting from a node and going downwards.

Given a node index i in the heap array:

  1. Find the indices of left and right children: left = 2*i + 1, right = 2*i + 2.
  2. Find the smallest value among the current node and its children.
  3. If either child is smaller than the current node, swap the node with the smallest child.
  4. Recursively perform heapify on the affected subtree.
function heapify(array, n, i) {
  let smallest = i;
  let left = 2 * i + 1;
  let right = 2 * i + 2;

  if (left < n && array[left] < array[smallest]) smallest = left;
  if (right < n && array[right] < array[smallest]) smallest = right;

  if (smallest !== i) {
    swap(array, i, smallest);
    heapify(array, n, smallest);
  }
}
Note: This algorithm is the core mechanism used during both extract-min and build-heap operations.

Heapify Up (Sift Up) Algorithm

Heapify up is used after inserting a new node at the end of the heap to restore the heap property by moving the new element upwards.

Algorithm steps:

  1. Compare the inserted node with its parent.
  2. If the inserted node’s value is smaller, swap with the parent.
  3. Repeat the comparison and swapping up the tree until the root is reached or heap property holds.
function heapifyUp(array, i) {
  while (i > 0) {
    let parent = Math.floor((i - 1) / 2);
    if (array[parent] <= array[i]) break;
    swap(array, parent, i);
    i = parent;
  }
}

Applications of Min Heap

  • Priority Queues: Min heaps implement priority queues efficiently where the element with the smallest priority is always served first.
  • Dijkstra's shortest path algorithm: Often uses a min heap to retrieve the vertex with the smallest tentative distance.
  • Heap Sort: Sorting algorithm based on building a heap and repeatedly removing the minimum element.
  • Order Statistics: Efficiently find the kth smallest element in a dataset.
  • Event simulation: Process events in chronological order by their time stamp.
