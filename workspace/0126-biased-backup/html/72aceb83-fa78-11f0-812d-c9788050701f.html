The Art of Refactoring

Transforming code while preserving functionality

Extract Method

Turn a code fragment into a method with a name that explains the purpose. This improves readability and makes the code more maintainable by breaking down complex operations into named, reusable components.

The new method name serves as documentation for what the code does, making it easier to understand at a glance.

function processOrder(order) {
    // Validate order
    if (!order.items || order.items.length === 0) {
        throw new Error("Order must have items");
    }
    // Calculate total
    let total = 0;
    for (const item of order.items) {
        total += item.price * item.quantity;
    }
    // Apply discount
    if (order.customer.isPremium) {
        total *= 0.9;
    }
    // ... more processing ...
}
function processOrder(order) {
    validateOrder(order);
    const total = calculateTotal(order);
    // ... more processing ...
}

function validateOrder(order) {
    if (!order.items || order.items.length === 0) {
        throw new Error("Order must have items");
    }
}

function calculateTotal(order) {
    let total = 0;
    for (const item of order.items) {
        total += item.price * item.quantity;
    }
    if (order.customer.isPremium) {
        total *= 0.9;
    }
    return total;
}

Replace Temp with Query

Replace temporary variables with method calls. This eliminates variables that are only used once and can make the code more expressive by giving meaningful names to calculations.

By moving the calculation into a method, it becomes reusable and the intention becomes clearer through the method name.

function calculatePrice(order) {
    const basePrice = order.quantity * order.itemPrice;
    if (basePrice > 1000) {
        return basePrice * 0.95;
    }
    else {
        return basePrice * 0.98;
    }
}
function calculatePrice(order) {
    if (basePrice(order) > 1000) {
        return basePrice(order) * 0.95;
    }
    else {
        return basePrice(order) * 0.98;
    }
}

function basePrice(order) {
    return order.quantity * order.itemPrice;
}

Introduce Parameter Object

When a group of parameters naturally go together, replace them with an object. This reduces parameter lists and makes the relationship between parameters explicit.

Parameter objects often become meaningful concepts in your codebase, making the code more expressive and easier to understand.

function bookConcertTickets(
    customerName, 
    customerEmail, 
    concertId, 
    seatNumbers, 
    isVIP, 
    paymentMethod
) {
    // booking logic...
}
function bookConcertTickets(
    customer, 
    bookingDetails
) {
    // booking logic...
}
