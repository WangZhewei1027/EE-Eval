Floyd-Warshall Algorithm

Floyd-Warshall Algorithm is a classic algorithm in computer science and graph theory used to find the shortest paths between all pairs of vertices in a weighted graph (directed or undirected). Unlike algorithms such as Dijkstra’s, which find the shortest path from a single source to all other vertices, the Floyd-Warshall algorithm considers every possible pair of nodes simultaneously, producing a matrix of minimum path distances between all vertices.

1. Background & Problem Definition

Given a weighted graph G = (V, E) where V is the set of vertices and E is the set of weighted edges, the goal is to compute the shortest path distance d(u, v) for every pair of vertices (u, v). This means finding the minimal cost to travel from u to v considering all possible paths.

The weights on edges can be positive, zero, or even negative as long as there are no negative weight cycles (cycles whose total weight sums to a negative number), since that would make shortest path computations undefined (paths could loop indefinitely decreasing cost).

Key Points:

  • Input: Weighted directed or undirected graph (negative weights allowed, but no negative cycles).
  • Output: Matrix D of size n × n where D[i][j] gives length of shortest path from vertex i to vertex j.
  • Handles all pairs simultaneously, unlike single source shortest path algorithms.

2. Algorithm Intuition & Principle

The Floyd-Warshall algorithm uses a dynamic programming approach based on gradually improving estimates of shortest path distances. It works by considering an incremental set of intermediate vertices through which paths are allowed to travel. Initially, it considers no intermediate vertices, then it adds vertex 1, then vertex 2, and so forth up to vertex n.

Mathematically, define D^k[i][j] as the shortest path distance from vertex i to vertex j using only intermediate vertices from the set {1, 2, ..., k}.

- For k = 0 (using no intermediate vertices), D^0[i][j] is simply the direct edge weight between i and j (or infinity if no edge exists). - For each subsequent k, we check if using vertex k as an intermediate vertex can produce a shorter path:

D^k[i][j] = min( D^{k-1}[i][j], D^{k-1}[i][k] + D^{k-1}[k][j] )

Intuitively, this means: the shortest path from i to j using vertices {1...k} either:

  • Does not use vertex k as an intermediate vertex (distance remains D^{k-1}[i][j])
  • Or does use vertex k as an intermediate vertex: the path goes from i to k, then from k to j (D^{k-1}[i][k] + D^{k-1}[k][j]).

By iteratively applying the formula for all pairs (i, j) and all intermediate vertices k, the algorithm ultimately finds the shortest path between every pair.

3. Detailed Step-by-Step Explanation

  1. Initialization:
    Create a 2D matrix D where D[i][j] is initialized to the weight of the edge from i to j, or infinity if no such edge exists. If i == j, then D[i][j] = 0 since distance from a vertex to itself is zero.
  2. Iteration over intermediate vertices:
    For each vertex k (from 1 to n), update the distance matrix D considering the possibility of taking paths that go through k.
  3. Distance update:
    For every pair (i, j), update D[i][j] as:
    D[i][j] = min(D[i][j], D[i][k] + D[k][j])
  4. Repeat:
    Repeat step 3 for each k until all intermediate vertices have been considered.
  5. Result:
    After all iterations, D[i][j] contains the shortest-path distance between i and j.

4. Pseudocode

function FloydWarshall(weights)
    n = number of vertices
    let D be an n×n matrix

    for i from 1 to n:
        for j from 1 to n:
            if i == j then
                D[i][j] := 0
            else if edge (i, j) exists then
                D[i][j] := weight(i, j)
            else
                D[i][j] := ∞

    for k from 1 to n:
        for i from 1 to n:
            for j from 1 to n:
                if D[i][k] + D[k][j] < D[i][j]:
                    D[i][j] := D[i][k] + D[k][j]

    return D

6. Algorithm Characteristics and Complexity

  • Time complexity: O(n³) because of three nested loops over vertices.
  • Space complexity: O(n²) due to storage of distance matrix.
  • Negative edges: Can handle negative edge weights as long as there are no negative cycles.
  • Negative cycles detection: If D[i][i] < 0 for any vertex i after running the algorithm, it indicates a negative weight cycle reachable from that vertex.
  • Use cases: Network routing, transitive closure, pathfinding in graphs, optimizing travel routes, etc.

7. Additional Notes

The Floyd-Warshall algorithm is conceptually simple and elegant, derived from Dynamic Programming principles. It can be adapted to solve not only shortest path distance problems but also other graph-related queries such as:

  • Transitive closure: determine which vertices are reachable from others by treating distances as booleans.
  • Path reconstruction: maintain a predecessor matrix to reconstruct the actual shortest paths, not just distances.

Despite its cubic time complexity, Floyd-Warshall shines for small to medium graphs where a full all-pairs solution is required, or where edge weights can be negative.
