Selection Sort — Comprehensive Explanations and Teaching Material

A thorough textual guide to Selection Sort: definition, algorithm, step‑by‑step walkthroughs, complexity analysis, pedagogical notes, code examples, common pitfalls and exercises. Minimal interactivity: one simple demonstration button.

High-level definition

Selection Sort is a simple comparison-based sorting algorithm. The core idea: repeatedly select the smallest (or largest) element from the unsorted portion and place it at the next position in the sorted portion. The array is conceptually divided into a left (sorted) part and a right (unsorted) part. At each step the algorithm expands the sorted part by one element by selecting the correct element from the unsorted part and swapping it into place.

Key properties (summary)

  • Time complexity: O(n^2) comparisons (best, average, worst) — quadratic.
  • Number of swaps: at most n−1 (one swap per position), often fewer if you avoid swapping when the minimum is already in place.
  • Space complexity: O(1) extra — in-place (only constant extra memory).
  • Stability: not stable by default for array implementations (swapping distant elements can change relative order of equal keys).
  • Adaptive: no — it does not take advantage of existing order; running time is the same regardless of input order.

Algorithm description and invariant

Let A be an array of n elements. The algorithm uses an index i (0-based) which splits the array: A[0..i-1] is sorted and contains the i smallest elements (in final positions). A[i..n-1] is unsorted. Initially i = 0 and the sorted portion is empty. For each i from 0 to n-2:

  1. Find index m of the minimum element in A[i..n-1].
  2. Swap A[i] and A[m] (if i != m). Now A[0..i] is sorted.
  3. Increment i and repeat.

The loop invariant: before iteration i, A[0..i-1] are the i smallest elements in sorted order, and they occupy their final positions.

Pseudocode (clear, classic)

selectionSort(A, n):
  for i from 0 to n-2:
    // find index of minimum in A[i..n-1]
    minIndex = i
    for j from i+1 to n-1:
      if A[j] < A[minIndex]:
        minIndex = j
    // swap A[i] and A[minIndex] if needed
    if minIndex != i:
      swap(A[i], A[minIndex])

Step-by-step textual walkthrough (example)

We will sort the array A = [64, 25, 12, 22, 11] into non-decreasing order using the standard selection-sort (select minimum).

Throughout the walkthrough we explicitly state what the algorithm is doing at each major step.

Initial array (i = 0): A = [64, 25, 12, 22, 11]

Find the minimum element in A[0..4]. Compare 64 with 25, 12, 22, 11 → minimum is 11 at index 4. Swap A[0] and A[4]. After swap: A = [11, 25, 12, 22, 64].

After first pass (i = 1): A = [11, 25, 12, 22, 64]

Now find minimum in A[1..4]. Compare 25 with 12, 22, 64 → minimum is 12 at index 2. Swap A[1] and A[2]. After swap: A = [11, 12, 25, 22, 64].

After second pass (i = 2): A = [11, 12, 25, 22, 64]

Find minimum in A[2..4]. Compare 25 with 22 and 64 → minimum is 22 at index 3. Swap A[2] and A[3]. After swap: A = [11, 12, 22, 25, 64].

After third pass (i = 3): A = [11, 12, 22, 25, 64]

Find minimum in A[3..4]. Compare 25 with 64 → minimum is 25 at index 3 (same as i). No swap (or swap with itself). Array unchanged.

After fourth pass (i = 4): Sorting complete. A = [11, 12, 22, 25, 64]

Counts for this example (if swaps only when needed): comparisons = 10 (n(n−1)/2 with n=5), swaps = 3 (swap at i=0,i=1,i=2).

Exact count of comparisons

Selection sort always performs the same number of comparisons regardless of input: for each i from 0 to n−2 it compares the candidate minimum with each remaining element. Total comparisons = (n−1) + (n−2) + ... + 1 = n(n−1)/2 = Θ(n^2).

Number of swaps

Selection sort performs at most (n−1) swaps because, for each position i, it performs at most one swap: putting the chosen minimum into A[i]. If you avoid swapping when the minimum is already at i, then the actual swap count can be less than n−1. This property makes selection sort useful where swaps (writes) are very expensive: it minimizes the number of writes compared to algorithms that move elements more often.

Stability (why not stable by default)

A sorting algorithm is stable if, for equal keys, their relative order in the input is preserved in the output. Selection sort is not stable by default because it swaps the current i element with the found minimum, which might be equal in key to some earlier element but located later in the array, thereby moving it before other equal elements that were earlier. Example: 

// Example showing instability:
records = [(5, 'a'), (3, 'x'), (5, 'b')]
If we sort by the first field:
After selection sort (may produce): [(3,'x'), (5,'b'), (5,'a')]
The two 5-records changed order: instability.

A stable variant requires careful handling — e.g., after finding the minimum you can remove it and shift elements right until inserting it at position i (this preserves relative order), but that increases the number of writes (worse performance for writes) and makes the algorithm not purely "one swap per i".

Proof sketch of O(n^2) run time

For each index i (0..n−2) the algorithm scans the remaining unsorted part A[i..n−1] to find the minimum. The sizes of these scans sum to (n−1)+(n−2)+...+1 = n(n−1)/2 = Θ(n^2). Swapping is O(1) and happens at most n−1 times. Therefore total time is Θ(n^2).

When (if ever) is Selection Sort useful?

  • Small arrays: for very small n (e.g., n < 10) the simplicity may be acceptable.
  • When write/assignment cost dominates: selection sort minimizes the number of writes (swaps) to O(n), which can matter on devices where writes are slow or scarce (flash memory, EEPROM).
  • Teaching: good for explaining sorting fundamentals and loop invariants due to its conceptual simplicity.
  • Rarely used in production for large arrays because other algorithms (quicksort, mergesort, heapsort) are dramatically faster.

Variants and related ideas

  • Selection of maximum instead of minimum (works symmetrically, can fill array from end backwards).
  • Stable selection sort: instead of swapping, remove minimum and insert at position i by shifting; stable but costs O(n^2) moves.
  • Selection on linked lists: you can select the min by traversing the remaining list and move nodes; with careful pointer manipulation you can perform selection sort in O(n^2) time and O(1) space while being stable (if you re-link rather than swap values).
  • Selection for partial sorting (Selection for top-k): to find k smallest elements you can run selection steps for i in 0..k-1, which costs O(nk) in naive form, or use selection algorithms/heap for better complexity.

Code examples

Python (concise):

def selection_sort(a):
    n = len(a)
    for i in range(n - 1):
        min_index = i
        for j in range(i + 1, n):
            if a[j] < a[min_index]:
                min_index = j
        if min_index != i:
            a[i], a[min_index] = a[min_index], a[i]

JavaScript (classic):

function selectionSort(arr){
  const n = arr.length;
  for(let i = 0; i < n - 1; i++){
    let minIdx = i;
    for(let j = i + 1; j < n; j++){
      if(arr[j] < arr[minIdx]) minIdx = j;
    }
    if(minIdx !== i) {
      const tmp = arr[i];
      arr[i] = arr[minIdx];
      arr[minIdx] = tmp;
    }
  }
  return arr;
}

C (in-place):

void selection_sort(int a[], int n){
  for(int i=0;i<n-1;i++){
    int minIdx=i;
    for(int j=i+1;j<n;j++){
      if(a[j]<a[minIdx]) minIdx=j;
    }
    if(minIdx!=i){
      int tmp=a[i]; a[i]=a[minIdx]; a[minIdx]=tmp;
    }
  }
}

Teaching tips and common pitfalls

  • Emphasize loop invariant: A[0..i-1] are correct and final. Asking students to prove the invariant helps understanding correctness.
  • Distinguish between "swap every iteration" and "swap only when minIndex != i" — this affects swap count but not asymptotic comparisons.
  • Show comparison count formula derivation step-by-step: sum of arithmetic series.
  • Use a carefully chosen example with duplicate keys to surface stability issues.
  • Avoid trying to "optimize" by early exiting — selection sort cannot save comparisons even if array is sorted (not adaptive).

Exercises (with answers)

  1. How many comparisons will selection sort perform on an array of length 10?
  2. Given array [3,1,2,1], show the array after each swap for selection sort and determine whether it is stable.
  3. How would you modify selection sort to be stable? What is the cost?

Answers

1) n(n−1)/2 with n=10 → 10×9/2 = 45 comparisons.

2) Array: [3,1,2,1]

i=0: min in [0..3] is 1 at index 1 (first 1). Swap A[0] and A[1] → [1,3,2,1]. i=1: min in [1..3] is 1 at index 3. Swap A[1] and A[3] → [1,1,2,3]. At the end equal elements with value 1: original order was (1 at index1) then (1 at index3). After sorting they appear as (1 from idx1) then (1 from idx3) — this example preserved relative order by chance. But selection sort can break stability in other arrangements where swapping moves later equal element ahead of earlier ones.

3) To be stable: instead of swapping, store the min value (or node) and shift all elements between i and minIndex right by one, then insert the min value at A[i]. Cost: still O(n^2) time but increases number of element moves (writes) to O(n^2) in worst case.

Invariants, correctness, and short proof of correctness

We show correctness by establishing and maintaining the loop invariant: Prior to iteration i, A[0..i-1] are the i smallest elements of the original array and appear in sorted order in their final positions. Initialization: before i=0 the empty prefix is trivially correct. Maintenance: at iteration i, we find smallest element in A[i..n-1] and place it at A[i] by swap, so A[0..i] are the i+1 smallest elements, sorted. Termination: after i reaches n−1 the invariant implies the entire array is sorted.

Further study and exercises for students

  • Implement stable selection sort and measure number of assignments vs. standard selection sort for random arrays of increasing sizes.
  • Analyze selection sort on a linked list: implement selection by relinking nodes and observe stability and write counts.
  • Compare selection sort with heap sort and explain why heap sort reduces comparisons but requires extra structure.
  • Prove formally that selection sort performs exactly n(n−1)/2 comparisons independent of input arrangement.
