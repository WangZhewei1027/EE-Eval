Comprehensive Guide to Routing in Computer Networks

Routing is the process of selecting a path for traffic in a network or between or across multiple networks. It is performed for many kinds of networks, including circuit-switched networks, such as the public switched telephone network (PSTN), and computer networks, such as the Internet.

Fundamental Concepts of Routing

Packet Switching

Routing is fundamental to packet-switched networks, where data is divided into packets that are individually routed from source to destination. Each packet may take a different path through the network.

Routing vs. Forwarding

While often used interchangeably, routing and forwarding are distinct concepts:

  • Routing refers to the process of determining the optimal path for data to travel.
  • Forwarding is the actual process of sending packets to their next hop based on routing decisions.

Routing Metrics

Routing algorithms use various metrics to determine the best path:

  • Hop count: Number of routers a packet must traverse
  • Bandwidth: Capacity of the link
  • Delay: Time taken for a packet to travel
  • Load: Current traffic on the link
  • Reliability: Error rate of the link
  • Cost: Monetary or administrative cost

Routing Algorithms

Static Routing

Static routing involves manually configured routes in routing tables. These routes don't change unless manually updated by a network administrator.

Example of Static Routing

In a small office network, the administrator might configure:

route add 192.168.2.0 mask 255.255.255.0 192.168.1.2

This command tells the router that to reach the 192.168.2.0 network, send packets to the router at 192.168.1.2.

Dynamic Routing

Dynamic routing protocols allow routers to automatically learn about network paths and adjust to changes in network conditions.

Distance Vector Routing

Distance vector protocols (e.g., RIP) determine the best path based on distance (usually hop count) and direction (next hop router).

Distance Vector Algorithm Steps:

  1. Each router maintains a table with distances to all destinations
  2. Periodically, routers exchange their entire routing tables with neighbors
  3. When a router receives an update, it compares with its current table
  4. If a better path is found, the table is updated
  5. The process continues until all routers have consistent information

Link State Routing

Link state protocols (e.g., OSPF) build a complete map of the network topology and calculate the shortest path to all destinations.

Link State Algorithm Steps:

  1. Each router discovers its neighbors and measures link costs
  2. This information is flooded to all other routers
  3. Each router builds a complete topology map of the network
  4. Using Dijkstra's algorithm, each router computes shortest paths to all destinations
  5. Routing tables are built from these computations

Routing Protocols in Practice

Interior Gateway Protocols (IGPs)

Used within an autonomous system (a network under single administrative control):

  • RIP (Routing Information Protocol): Simple distance-vector protocol with hop count limit of 15
  • OSPF (Open Shortest Path First): Link-state protocol that divides networks into areas
  • EIGRP (Enhanced Interior Gateway Routing Protocol): Cisco's advanced distance-vector protocol

Exterior Gateway Protocols (EGPs)

Used between autonomous systems:

  • BGP (Border Gateway Protocol): The de facto standard for Internet routing, using path-vector algorithm

Advanced Routing Concepts

Hierarchical Routing

To scale routing to large networks, hierarchical organization is used:

  • Networks are divided into areas or domains
  • Routers only need detailed information about their local area
  • Summarized information is shared between areas

Multicast Routing

Specialized routing for one-to-many or many-to-many communication:

  • Source-based trees: Separate shortest path tree for each source
  • Shared trees: Single distribution tree for all sources
  • Protocols include PIM (Protocol Independent Multicast) and DVMRP

Quality of Service (QoS) Routing

Routing that considers service quality parameters:

  • Bandwidth requirements
  • Delay constraints
  • Jitter limitations
  • Packet loss thresholds

Routing in Modern Networks

Software-Defined Networking (SDN)

SDN separates the control plane (routing decisions) from the data plane (packet forwarding):

  • Centralized controller makes routing decisions
  • Routers/switches become simple forwarding devices
  • Enables more flexible and programmable networks

IPv6 Routing

Routing considerations for IPv6 networks:

  • Larger address space changes addressing schemes
  • Modified versions of OSPF (OSPFv3) and BGP (MP-BGP) support IPv6
  • Simplified header format improves routing efficiency
