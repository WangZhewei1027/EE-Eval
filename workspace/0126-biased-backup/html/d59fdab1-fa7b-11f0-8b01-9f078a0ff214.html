Understanding Linked Lists

A linked list is a linear data structure where elements are not stored in contiguous memory locations. Instead, they are represented as a sequence of nodes, where each node contains a data field and a reference (or link) to the next node in the sequence.

Basic Terminology

Before delving into the linked list, let’s clarify some essential terms:

  • Node: The basic unit of a linked list that contains the data and a reference to the next node.
  • Head: The first node in a linked list. If the list is empty, the head will be null.
  • Tail: The last node in a linked list, which points to null (or no next node).
  • Link: A reference from one node to the next node.

Types of Linked Lists

Linked lists can be categorized into several types:

  • Singly Linked List: Each node contains data and a link to the next node. Traversal is only in one direction.
  • Doubly Linked List: Each node contains data and two links: one to the next node and another to the previous node, allowing bi-directional traversal.
  • Circular Linked List: Similar to a singly or doubly linked list, but the last node points back to the first node, forming a circle.

How Linked Lists Work

Linked lists work by connecting nodes through their links. Let's see how a singly linked list operates:

Structure of a Singly Linked List Node

    class Node {
        int data;
        Node next;
    }

Insertion in a Linked List

To insert a new node in a singly linked list, we have to adjust the link of the previous node to point to the new node. The new node's link then points to the subsequent node. There are three main scenarios:

  1. Inserting at the Head: Assign the new node as the head and point it to the old head.
  2. Inserting at the Tail: Traverse to the end of the list and set the current tail's next link to the new node, then set the new node's next link to null.
  3. Inserting in the Middle: Find the appropriate position, change the link of the previous node to the new node, and set the new node's link to the next node.

Deletion in a Linked List

Similar to insertion, deletion involves adjusting links. You need to sever the link of the node before the one to be deleted and link it to the one after the deletion point. Here are common scenarios:

  1. Deleting the Head: Just move the head pointer to the next node.
  2. Deleting the Tail: Traverse to the second last node and set its next link to null.
  3. Deleting a Middle Node: Find the node before the target, update its link to skip the target node, effectively removing it from the list.

Advantages and Disadvantages

Advantages:

  • Dynamic Size: Linked lists can grow or shrink in size easily without reallocating or freeing memory.
  • Efficient Insertions/Deletions: Adding or removing elements can be done without shifting other elements as in arrays.

Disadvantages:

  • Memory Usage: Additional memory space is required for storing the link/reference.
  • Sequential Access: Nodes need to be accessed sequentially, not allowing random access as in arrays.

Example Implementation

Below is a simple demonstration of how a linked list can be implemented and how to perform basic operations:

    class LinkedList {
        Node head; // head of list
        
        // Insertion at the beginning
        void insertAtHead(int data) {
            Node newNode = new Node();
            newNode.data = data;
            newNode.next = head;
            head = newNode;
        }

        // Traversing the list
        void printList() {
            Node current = head;
            while (current != null) {
                System.out.print(current.data + " -> ");
                current = current.next;
            }
            System.out.println("null");
        }
    }

This is a simple demonstration of the linked list operations. For understanding purposes, particular methods can be implemented and tested.
