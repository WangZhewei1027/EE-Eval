Floyd–Warshall Algorithm — A Thorough Explanation

Purpose: compute shortest paths between all pairs of vertices in a weighted graph (directed or undirected). Handles positive and negative edge weights; detects negative cycles.

What problem does Floyd–Warshall solve?

Floyd–Warshall is an all-pairs shortest-path (APSP) algorithm. Given a graph with V vertices and weighted edges, it computes, for every pair of vertices (i, j), the minimum cost to travel from i to j. The graph may contain negative edge weights, but if there exists a negative-weight cycle reachable from some vertex, then some shortest distances are undefined (they can be arbitrarily negative). Floyd–Warshall can detect such cycles.

Key idea (dynamic programming)

The algorithm incrementally considers an increasing set of allowed intermediate vertices. Let D(k)[i][j] be the shortest-path distance from vertex i to j such that all intermediate vertices (if any) are chosen from the set {0, 1, ..., k−1}. Initially (k=0) no intermediate vertices are allowed, so D(0) is just the adjacency matrix (direct edge weights, zero on the diagonal, and INF where no direct edge exists). To allow vertex k as an additional intermediate vertex we update: 

D(k+1)[i][j] = min( D(k)[i][j], D(k)[i][k] + D(k)[k][j] )

Intuition: either the best path from i to j does not go through k, or it does go through k at least once; if it does, then that path can be decomposed into a best path from i to k and a best path from k to j, each of which uses only intermediate vertices numbered less than k.

Pseudocode (classic form)

procedure FloydWarshall(weightMatrix):
    // n = number of vertices
    // weightMatrix[i][j] = direct edge weight or +INF if no edge
    let dist = copy(weightMatrix)
    for v in 0..n-1:
        dist[v][v] = min(dist[v][v], 0)   // ensure diagonal initialized

    for k in 0..n-1:
        for i in 0..n-1:
            for j in 0..n-1:
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]

    // After this, dist[i][j] is shortest path weight or +INF.
    // If any dist[v][v] < 0 then a negative cycle exists that is reachable from v.
    return dist

Correctness (sketch)

The proof is by induction on k. Base case k=0: D(0) allows no intermediate vertices, so it is the adjacency matrix — correct. Inductive step: Assume D(k) is correct for all pairs when intermediate vertices are restricted to {0,...,k-1}. Consider D(k+1). Any shortest path from i to j whose intermediate vertices lie in {0,...,k} either does not use k (so it is present in D(k)), or it uses k at least once. If it uses k, take the first and last occurrence of k in the path to split into i→k and k→j subpaths that use only vertices from {0,...,k−1}. So the recurrence considers both possibilities, and thus D(k+1) is correct. After n steps (k=0..n-1) we allow all vertices as intermediates.

Time and space complexity

  • Time: O(n^3) — three nested loops over vertices.
  • Space: O(n^2) to store the distance matrix. If path reconstruction is required, an additional O(n^2) "next" (or "predecessor") matrix is used.
  • If the graph is sparse and n is large, Johnson's algorithm (reweight + n × Dijkstra) is often faster in practice.

Handling negative cycles

After the main loops, if any dist[v][v] < 0 then there is a reachable negative-weight cycle involving v. In that case shortest path distances that can reach that cycle are undefined (drive to −∞). Floyd–Warshall can be extended to mark any pair (i, j) whose shortest path can be made arbitrarily small by going through a negative cycle: if there exists a vertex v with dist[v][v] < 0 and dist[i][v] < INF and dist[v][j] < INF, then the shortest i→j distance is −∞.

Path reconstruction

To reconstruct paths (not just distances), maintain a matrix next[i][j] that stores the next vertex to move to from i when heading to j along a shortest path. Initialization: if there is an edge i→j, next[i][j] = j; if i==j set next[i][i] = i; and if there is no edge, next[i][j] = null. When you perform an update: 

if dist[i][k] + dist[k][j] < dist[i][j]:
    dist[i][j] = dist[i][k] + dist[k][j]
    next[i][j] = next[i][k]   // path goes i -> ... -> k -> ... -> j

To build the actual path from u to v afterward, follow next[u][v], next[next[u][v]][v], ... until you reach v. If next[u][v] is null, no path exists.

When to use Floyd–Warshall

  • Use when you need all-pairs shortest paths on dense graphs or when n ≤ a few hundred (n^3 may be acceptable).
  • It is simple, compact, and handles negative edge weights and detection of negative cycles.
  • For large sparse graphs prefer repeated Dijkstra (with reweighting = Johnson) or other specialized methods.

Variations and related algorithms

  • Transitive closure of a directed graph (reachability): simple boolean variant of Floyd–Warshall.
  • All-pairs shortest paths (APSP) over semirings: replace (+, min) with other operators for other problems (e.g., path counting with (+, *), where combinatorial variations appear).
  • Johnson's algorithm for sparse graphs: reweight edges using Bellman–Ford then run Dijkstra from each vertex.

Step-by-step textual explanation of the worked example

1) Initialization: each cell dist[i][j] stores the direct edge weight (if present) or INF. Diagonal is zero. This reflects the shortest path length from i to j using no intermediate vertices.

2) k = 0 (allow A as an intermediate): we consider whether going from i to j via A improves dist[i][j]. Only row D changed: D -> B becomes 5 (D->A + A->B = 2 + 3 = 5). D -> D becomes -2 (D->A + A->D = 2 + (-4) = -2). Those are strictly better than the direct values.

3) k = 1 (allow B): using B as intermediate improves several entries. For instance A->C becomes 4 (A->B + B->C = 3 + 1 = 4), and C->D becomes 11 (C->B + B->D = 4 + 7 = 11). The matrix now reflects shortest paths that may use A or B as intermediate vertices.

4) k = 2 (allow C): in this particular graph allowing C does not change any values further (no beneficial new combined routes are created).

5) k = 3 (allow D): allowing D updates many distances by using D as a hub. For example A->A improves via A->D->A to -2, demonstrating a negative cycle A→D→A. After this final step we have the final shortest-path distances considering all intermediate vertices.

Implementation notes and pitfalls

  • Represent INF with a large value (e.g., +1e18) that cannot be exceeded by any sum of real weights. Beware integer overflow in languages with fixed-size integers.
  • If weights are floating-point, be mindful of precision and comparisons (use EPS appropriately).
  • To reconstruct paths, maintain next[i][j] as described above. If a negative cycle exists, path reconstruction may be meaningless for affected pairs.
  • Floyd–Warshall's three loops can be ordered in either (k,i,j) or (i,j,k) style — the (k,i,j) ordering is the standard which uses the D(k) → D(k+1) progression.

Short reference implementations (conceptual)

Distance-only (concise, Python-like)

INF = 10**15
dist = [[INF]*n for _ in range(n)]
for i: dist[i][i] = 0
for (u,v,w) in edges:
    dist[u][v] = w

for k in range(n):
    for i in range(n):
        for j in range(n):
            if dist[i][k] + dist[k][j] < dist[i][j]:
                dist[i][j] = dist[i][k] + dist[k][j]

With path reconstruction (next matrix)

next = [[None]*n for _ in range(n)]
for i in range(n):
    for j in range(n):
        if there is an edge i->j:
            next[i][j] = j
        if i==j:
            next[i][i] = i

for k in range(n):
    for i in range(n):
        for j in range(n):
            if dist[i][k] + dist[k][j] < dist[i][j]:
                dist[i][j] = dist[i][k] + dist[k][j]
                next[i][j] = next[i][k]    # go from i to k first

Summary: Floyd–Warshall is a simple and powerful dynamic programming algorithm for APSP that is easy to implement, tolerant of negative edges, and able to detect negative cycles.
