Logistic Regression — Comprehensive Explanation

A detailed, pedagogical guide to intuition, math, algorithms, interpretation, extensions, and practice

Overview — What is Logistic Regression?

Logistic regression is a fundamental supervised learning method used for classification tasks. Despite the name, it performs classification (most commonly binary classification), not regression in the classical sense. The model estimates the probability that a given input belongs to a target class. It does this by applying a logistic (sigmoid) function to a linear combination of the input features.

Use cases: medical diagnosis (disease / no disease), spam detection (spam / not spam), credit approval (default / no default), binary outcomes in experiments, and as a base model for more complex systems.

Key ideas in one sentence

Build a linear score z = w·x + b, convert it to a probability p = σ(z) using the sigmoid, then choose a threshold (often 0.5) to produce a class label.

Mathematical Formulation

Model: hypothesis and sigmoid

For binary classification with feature vector x ∈ R^d, we compute a linear score: 

z = wᵀ x + b

Convert z to a probability using the sigmoid (also called logistic) function: 

σ(z) = 1 / (1 + exp(−z))

The model's predicted probability that y = 1 (positive class) given x is: 

P(y=1 | x) = σ(wᵀ x + b)

Decision rule

To convert probabilities into labels, use a threshold τ (commonly 0.5): 

ŷ = 1 if σ(wᵀx + b) ≥ τ, else 0

The threshold can be changed when the costs of false positives and false negatives differ.

Loss: negative log-likelihood (binary cross-entropy)

Given N independent observations (x_i, y_i) with y_i ∈ {0,1}, the negative log-likelihood (the common training loss) is: 

L(w,b) = − sum_{i=1..N} [ y_i log p_i + (1−y_i) log(1−p_i) ]

where p_i = σ(wᵀ x_i + b). Minimizing this loss is equivalent to maximizing the likelihood of the parameters w,b under the Bernoulli model.

Gradient (for optimization)

The gradient of the loss with respect to parameters is straightforward: 

∂L/∂w = sum_i (p_i − y_i) x_i
∂L/∂b = sum_i (p_i − y_i)

Here p_i − y_i is the residual error between predicted probability and true label.

Interpretation of coefficients

Each coefficient w_j represents the log-odds change per unit change in x_j, all else equal. 

logit(p) = log( p / (1−p) ) = wᵀ x + b

If you increase x_j by one unit, the odds multiply by exp(w_j). For small w_j, w_j ≈ change in log-odds.

Concise Reference

Symbols

  • x : input vector (features)
  • w : weight vector (parameters)
  • b : bias (intercept)
  • σ(z) : sigmoid function
  • p : predicted probability P(y=1|x)

Common variants

  • Regularized logistic regression (L2 or L1)
  • Multiclass extension: softmax (a.k.a. multinomial logistic regression)
  • One-vs-rest (OvR) strategy for multiclass using binary classifiers

Quick tips

- Use feature scaling for gradient-based solvers to improve convergence.
- L2 regularization helps when data are nearly separable to prevent huge weights.
- Consider class weighting or resampling when classes are imbalanced.
- Evaluate with ROC AUC and calibration (reliability diagrams) for probability quality.

Training Algorithms — How We Find the Parameters

1) Gradient-based optimization

The most common approach is to minimize the negative log-likelihood (cross-entropy) with gradient descent or a variant (stochastic, mini-batch, momentum, Adam).

Basic batch gradient descent update: 

w ← w − α * ∂L/∂w
b ← b − α * ∂L/∂b

where α is the learning rate. In stochastic gradient descent (SGD) we update using one example at a time; in mini-batch gradient descent we update using a subset.

2) Newton-Raphson / IRLS (second-order method)

For convex logistic loss, second-order methods such as Newton-Raphson or the Iteratively Reweighted Least Squares (IRLS) algorithm converge quickly for moderate-size problems.

Newton update: 

θ ← θ − H^{-1} g

where g is the gradient and H is the Hessian matrix of the loss. Because H is of size d×d, this is expensive for very high-dimensional datasets.

Which solver to choose?

  • Small to medium d: Newton/IRLS (fast, few iterations, robust).
  • Large d or streaming: SGD or quasi-Newton (LBFGS) or specialized solvers.
  • Sparse data: use solvers that exploit sparsity to store and compute efficiently.

Pseudocode: batch gradient descent version

Initialize w ← 0, b ← 0
repeat until convergence:
  compute p_i = σ(wᵀ x_i + b) for all i
  compute gradient g_w = (1/N) Σ (p_i − y_i) x_i
  compute gradient g_b = (1/N) Σ (p_i − y_i)
  update w ← w − α * g_w
  update b ← b − α * g_b
end

Practical Considerations & Diagnostics

Feature engineering and preprocessing

- Scale features (standardize or normalize) when using gradient-based solvers. This makes the optimization landscape better conditioned.
- Create interaction terms and polynomial features if decision boundary is nonlinear in original features. Logistic regression is linear in parameters, but nonlinear in transformed features.
- For categorical variables, use one-hot encoding; beware of dummy variable trap (drop one category or use regularization).

Regularization

Regularization penalizes large weights to prevent overfitting (especially when features are many or nearly collinear).

  • L2 (ridge): add λ/2 * ||w||^2 to loss. Encourages small distributed weights, keeps all features but shrinks them.
  • L1 (lasso): add λ * ||w||_1 to loss. Encourages sparsity (feature selection) by driving some weights to zero.
  • Elastic net: combination of L1 and L2.

Class imbalance

When one class is much rarer, standard accuracy is misleading. Remedies:

  • Use class weights in the loss (increase weight of minority class).
  • Resample the dataset (oversample minority or undersample majority).
  • Optimize metrics that focus on performance for minority class (precision/recall, F1, AUC).

Evaluation beyond accuracy

- Confusion matrix, precision, recall, F1-score.
- ROC curve and AUC for threshold-agnostic ranking performance.
- Precision-Recall curve especially useful for imbalanced datasets.
- Calibration plots (reliability diagrams) to check if predicted probabilities match observed frequencies.

Numerical stability

- For large positive or negative z, exp(−z) can overflow or underflow. Implementations clamp or use log-sum-exp tricks for stability.
- Use stable formulas for log-likelihood terms: e.g. log(1+exp(−|z|)) forms or specialized library routines.

Extensions & Connections

Multiclass classification

- One-vs-Rest (OvR): train K binary logistic models, one per class vs all others. For prediction, pick class with highest score or probability.
- Multinomial logistic regression (softmax): generalizes logistic regression to K classes by modeling 

P(y=k | x) = exp(w_kᵀ x + b_k) / Σ_{j=1..K} exp(w_jᵀ x + b_j)

Softmax directly optimizes the multinomial negative log-likelihood and is typically preferred when classes are mutually exclusive.

Probabilistic and Bayesian perspectives

- Maximum Likelihood Estimation (MLE) yields the standard logistic regression solution.
- Bayesian logistic regression places priors on w (e.g. Gaussian for L2-like behaviour) and computes posterior distributions over parameters. Exact inference is not conjugate; use approximations (Laplace, variational inference, MCMC).

Relation to other models

  • Perceptron: linear classifier that uses hinge-like updates but does not produce probabilities.
  • Linear discriminant analysis (LDA): probabilistic linear classifier derived from Gaussian assumptions about features per class.
  • Support Vector Machines (SVM): focus on maximizing margin; can be seen as a related linear classifier with different objective.

Step-by-Step Example (Textual Walkthrough)

Problem setup

Suppose we want to predict whether an email is spam (1) or not spam (0) based on a single feature x: proportion of suspicious words in the message.

1) Choose model

Use logistic regression: p = σ(w·x + b).

2) Initialize

Start with w = 0, b = 0 (predicts p = 0.5 for every example).

3) Compute predictions and loss

For each example (x_i, y_i): compute z_i = w x_i + b and p_i = σ(z_i). Compute average cross-entropy loss.

4) Compute gradient

g_w = (1/N) Σ (p_i − y_i) x_i ; g_b = (1/N) Σ (p_i − y_i). The direction shows how to change parameters to reduce loss.

5) Update

Choose a learning rate α (small, e.g. 0.1) and update w ← w − α g_w, b ← b − α g_b.

6) Iterate until convergence

Stop when changes in loss or parameters are below a small threshold, or after a specified number of iterations.

7) Make predictions

For a new email with x*, compute p* = σ(w x* + b). If p* ≥ 0.5, predict spam; otherwise predict not spam. If false positives are costly, increase threshold.

Note: With more features, the steps are identical: compute the linear combination wᵀx + b and apply the sigmoid. The gradient and updates become vectorized operations.

Common Pitfalls and How to Address Them

Pitfall: Perfect separation

If classes are perfectly separable, the MLE for logistic regression has no finite solution: weights diverge to ±∞. Regularization (L2) stabilizes the solution by preventing weights from blowing up.

Pitfall: Multicollinearity

Highly correlated features can inflate variance of coefficient estimates. Remedies: remove or combine correlated features, use L2 regularization, or apply dimensionality reduction (PCA).

Pitfall: Misinterpreting probabilities

Predicted probabilities depend on the model and the training data. Check calibration — a well-calibrated model's predicted probability of 0.8 should correspond to roughly 80% of such cases being positive.

Pitfall: Using accuracy on imbalanced data

Use precision/recall, F1, and domain-specific cost-aware measures instead.
