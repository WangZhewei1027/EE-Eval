STATIC TYPING

The art of catching errors before runtime

<TypeScript Example>
function sum(a: number, b: number): number {
  return a + b;
}

let total: number = sum(10, 20);

// The following line would cause a compile-time error
// let wrong: number = sum('10', 20);

Static Typing is a programming paradigm where variable types are known at compile-time, not at runtime.

This means:

  • Every variable and function parameter has a fixed type.
  • Errors such as type mismatches are caught before running the program.
  • Improves code reliability, readability, and developer confidence.

Languages like TypeScript, Rust, and Swift embrace static typing.
