HTTP is a stateless application-level protocol used to transfer hypermedia documents, such as HTML. A client sends a request; the server returns a response with a status code and content.

Quick Reference

200 OK
404 Not Found
500 Server Error

How it works

1. The client constructs an HTTP request with a method (GET/POST), target path and headers. 2. The request travels over TCP/IP (often TLS). 3. The server processes and responds with a status code and body.
