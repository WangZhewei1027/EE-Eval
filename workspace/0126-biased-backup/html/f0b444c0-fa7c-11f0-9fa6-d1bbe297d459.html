Socket Programming: A Comprehensive Guide

Introduction to Socket Programming

Socket programming is a method of communication between two computers using a network protocol, typically TCP/IP. It allows processes running on different machines to exchange data in a client-server architecture.

Basic Socket Communication Flow:
Client → Create Socket → Connect → Send/Receive → Close
Server → Create Socket → Bind → Listen → Accept → Send/Receive → Close

Sockets provide an interface between application processes and network protocol stacks in the operating system. They abstract the complexities of network communication into simple operations like reading and writing to a file.

Key Concept: A socket is identified by a unique combination of:
  • IP address (identifies the host)
  • Port number (identifies the service/application)
  • Protocol (usually TCP or UDP)

Types of Sockets

Stream Sockets (SOCK_STREAM)

These sockets use TCP (Transmission Control Protocol) and provide:

  • Reliable, connection-oriented communication
  • In-order data delivery
  • Error checking and correction
  • Flow control

Datagram Sockets (SOCK_DGRAM)

These sockets use UDP (User Datagram Protocol) and provide:

  • Connectionless communication
  • No guarantee of delivery or ordering
  • Lower overhead than TCP
  • Faster transmission for time-sensitive applications
When to use which:
Use TCP for applications requiring reliability (web, email, file transfer).
Use UDP for applications where speed is critical (video streaming, online games, DNS).

Socket API Functions

Core Functions

  • socket() - Creates a new communication endpoint
  • bind() - Associates a socket with a specific port and IP address
  • listen() - Marks a socket as passive (server-side)
  • accept() - Accepts an incoming connection (server-side)
  • connect() - Initiates a connection to a remote socket (client-side)
  • send()/recv() - Exchange data (TCP)
  • sendto()/recvfrom() - Exchange data (UDP)
  • close() - Terminates the connection

TCP Socket Lifecycle

  1. Server: socket() → bind() → listen() → accept() → send()/recv() → close()
  2. Client: socket() → connect() → send()/recv() → close()

UDP Socket Lifecycle

  1. Server: socket() → bind() → recvfrom() → sendto() → close()
  2. Client: socket() → sendto() → recvfrom() → close()

Address Structures

Sockets use special structures to represent network addresses:

sockaddr_in (IPv4)

struct sockaddr_in {
    short            sin_family;   // AF_INET
    unsigned short   sin_port;     // Port number
    struct in_addr   sin_addr;     // IP address
    char             sin_zero[8];  // Padding
};

sockaddr_in6 (IPv6)

struct sockaddr_in6 {
    u_int16_t       sin6_family;   // AF_INET6
    u_int16_t       sin6_port;     // Port number
    u_int32_t       sin6_flowinfo; // IPv6 flow information
    struct in6_addr  sin6_addr;     // IPv6 address
    u_int32_t       sin6_scope_id; // Scope ID
};
Note: The sockaddr structure is a generic structure that can represent both IPv4 and IPv6 addresses. Functions like bind() and connect() take a pointer to sockaddr.

TCP Socket Programming Example

TCP Server

#include <sys/socket.h>
#include <netinet/in.h>
#include <stdio.h>
#include <string.h>

int main() {
    int sockfd = socket(AF_INET, SOCK_STREAM, 0);
    
    struct sockaddr_in serverAddr;
    serverAddr.sin_family = AF_INET;
    serverAddr.sin_port = htons(8080); // Port 8080
    serverAddr.sin_addr.s_addr = INADDR_ANY; // Any incoming interface
    
    bind(sockfd, (struct sockaddr*)&serverAddr, sizeof(serverAddr));
    listen(sockfd, 10); // Queue of 10 connections
    
    printf("Server listening on port 8080...\n");
    
    while(1) {
        struct sockaddr_in clientAddr;
        socklen_t clientLen = sizeof(clientAddr);
        int clientSocket = accept(sockfd, (struct sockaddr*)&clientAddr, &clientLen);
        
        char buffer[1024];
        recv(clientSocket, buffer, 1024, 0);
        printf("Received: %s\n", buffer);
        
        char* response = "Hello from server!";
        send(clientSocket, response, strlen(response), 0);
        
        close(clientSocket);
    }
    
    return 0;
}

TCP Client

#include <sys/socket.h>
#include <netinet/in.h>
#include <stdio.h>
#include <string.h>

int main() {
    int sockfd = socket(AF_INET, SOCK_STREAM, 0);
    
    struct sockaddr_in serverAddr;
    serverAddr.sin_family = AF_INET;
    serverAddr.sin_port = htons(8080);
    serverAddr.sin_addr.s_addr = inet_addr("127.0.0.1"); // Connect to localhost
    
    connect(sockfd, (struct sockaddr*)&serverAddr, sizeof(serverAddr));
    
    char* message = "Hello from client!";
    send(sockfd, message, strlen(message), 0);
    
    char buffer[1024];
    recv(sockfd, buffer, 1024, 0);
    printf("Received: %s\n", buffer);
    
    close(sockfd);
    
    return 0;
}

UDP Socket Programming Example

UDP Server

#include <sys/socket.h>
#include <netinet/in.h>
#include <stdio.h>
#include <string.h>

int main() {
    int sockfd = socket(AF_INET, SOCK_DGRAM, 0);
    
    struct sockaddr_in serverAddr;
    serverAddr.sin_family = AF_INET;
    serverAddr.sin_port = htons(8080);
    serverAddr.sin_addr.s_addr = INADDR_ANY;
    
    bind(sockfd, (struct sockaddr*)&serverAddr, sizeof(serverAddr));
    
    printf("UDP Server listening on port 8080...\n");
    
    while(1) {
        struct sockaddr_in clientAddr;
        socklen_t clientLen = sizeof(clientAddr);
        char buffer[1024];
        
        recvfrom(sockfd, buffer, 1024, 0, 
                (struct sockaddr*)&clientAddr, &clientLen);
        printf("Received: %s\n", buffer);
        
        char* response = "UDP Server response";
        sendto(sockfd, response, strlen(response), 0,
              (struct sockaddr*)&clientAddr, clientLen);
    }
    
    return 0;
}

UDP Client

#include <sys/socket.h>

Advanced Topics

Non-blocking Sockets

By default, socket operations block until they complete. Non-blocking sockets return immediately, allowing for more efficient I/O handling.

// Set socket to non-blocking mode
int flags = fcntl(sockfd, F_GETFL, 0);
fcntl(sockfd, F_SETFL, flags | O_NONBLOCK);

Select() for Multiple Sockets

The select() function allows monitoring multiple sockets for activity:

fd_set readfds;
FD_ZERO(&readfds);
FD_SET(sockfd1, &readfds);
FD_SET(sockfd2, &readfds);

int maxfd = (sockfd1 > sockfd2) ? sockfd1 : sockfd2;

select(maxfd + 1, &readfds, NULL, NULL, NULL);

if (FD_ISSET(sockfd1, &readfds)) {
    // sockfd1 has data to read
}
if (FD_ISSET(sockfd2, &readfds)) {
    // sockfd2 has data to read
}

Socket Options

Various socket options can be set using setsockopt():

// Enable socket reuse
int optval = 1;
setsockopt(sockfd, SOL_SOCKET, SO_REUSEADDR, &optval, sizeof(optval));

// Set receive timeout
struct timeval tv;
tv.tv_sec = 5;  // 5 second timeout
tv.tv_usec = 0;
setsockopt(sockfd, SOL_SOCKET, SO_RCVTIMEO, &tv, sizeof(tv));

Common Issues and Solutions

Address Already in Use

Solution: Set SO_REUSEADDR socket option before binding.

Connection Reset

Occurs when the remote peer closes the connection unexpectedly. Solution: Implement proper error handling.

Blocking Calls

Solution: Use non-blocking sockets or implement timeouts.

Portability Issues

Different systems may have variations in socket implementations. Solution: Use cross-platform libraries or conditional compilation.

Conclusion

Socket programming is a fundamental skill for network application development. Understanding sockets allows you to create various networked applications, from simple client-server models to complex distributed systems.

Key takeaways:

  • Sockets provide an interface between applications and the network stack
  • TCP provides reliable, ordered communication while UDP offers faster, connectionless communication
  • Proper error handling and resource management are crucial in socket programming
  • Advanced techniques like non-blocking I/O and multiplexing enable scalable applications
