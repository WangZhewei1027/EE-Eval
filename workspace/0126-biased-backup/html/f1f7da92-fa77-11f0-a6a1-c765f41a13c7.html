Atomicity

All-or-nothing: a transaction either fully completes or is fully rolled back. No partial results.

Atomicity ensures a transaction is indivisible — either everything is applied, or nothing changes.

Consistency

Preserves rules & constraints: transactions move the system from one valid state to another.

Consistency enforces integrity — constraints, invariants, and rules remain true before and after a transaction.

Isolation

Concurrent transactions don’t interfere with each other — each behaves as if it's alone.

Isolation prevents interference: intermediate states of transactions are hidden from others.

Durability

Once committed, the effects are permanent — changes survive crashes and failures.

Durability guarantees persistence — committed transactions are safely recorded.
