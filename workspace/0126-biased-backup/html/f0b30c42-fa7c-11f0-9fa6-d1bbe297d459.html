Sliding Window Technique

Introduction

The sliding window technique is a method used in computer science and algorithm design to efficiently solve problems involving arrays, strings, or other linear data structures. It involves maintaining a subset of data (the "window") that slides through the larger data structure, typically from left to right, while performing calculations or checks on the window's contents.

This technique is particularly useful for problems that require examining all possible contiguous subarrays or substrings of a certain size, or for problems where we need to find a subarray that meets certain criteria (e.g., maximum sum, minimum length, etc.).

When to Use Sliding Window

The sliding window technique is applicable when:

  • The problem involves arrays, strings, or linear data structures
  • You need to find a subrange (contiguous elements) that meets certain criteria
  • The solution requires examining all possible subarrays of a given size
  • Brute-force solutions would result in O(n²) or worse time complexity

Common problem types that benefit from sliding window include:

  • Finding maximum/minimum sum subarrays of size k
  • Finding the longest substring with k distinct characters
  • Finding the smallest subarray with a sum greater than a given value
  • String permutation problems
  • Problems involving consecutive elements

Types of Sliding Windows

1. Fixed-Size Sliding Window

In this variant, the window size remains constant as it slides through the data structure. The window moves one element at a time, maintaining its size throughout the operation.

Example: Find the maximum sum of any contiguous subarray of size 3 in the array [2, 1, 5, 1, 3, 2].

Possible subarrays of size 3:

  • [2, 1, 5] → sum = 8
  • [1, 5, 1] → sum = 7
  • [5, 1, 3] → sum = 9
  • [1, 3, 2] → sum = 6

The maximum sum is 9.

2. Variable-Size Sliding Window

In this variant, the window size can grow or shrink based on certain conditions. The window typically starts small and expands until a condition is violated, then contracts from the left until the condition is satisfied again.

Example: Find the length of the longest substring with no more than two distinct characters in "eceba".

Possible substrings:

  • "ece" → length 3 (e, c, e)
  • "ceb" → length 3 (c, e, b)
  • "eba" → length 3 (e, b, a)

The longest substring is 3 characters long.

Algorithmic Approach

Fixed-Size Window Algorithm

  1. Initialize windowStart = 0 and calculate the sum of the first window (0 to k-1)
  2. Store this sum as the maximum sum found so far
  3. Iterate through the array from index k to n-1:
    • Add the current element to the window sum
    • Subtract the element leaving the window (at windowStart)
    • Increment windowStart
    • Update the maximum sum if the current window sum is greater
  4. Return the maximum sum
function maxSumSubarray(arr, k) { let maxSum = 0; let windowSum = 0; let windowStart = 0; for (let windowEnd = 0; windowEnd < arr.length; windowEnd++) { windowSum += arr[windowEnd]; if (windowEnd >= k - 1) { maxSum = Math.max(maxSum, windowSum); windowSum -= arr[windowStart]; windowStart++; } } return maxSum; }

Variable-Size Window Algorithm

  1. Initialize windowStart = 0 and create a frequency map/hash to track elements
  2. Iterate through the array with windowEnd:
    • Add the current element to the frequency map
    • While the current window violates the condition:
      • Remove the element at windowStart from the frequency map
      • Increment windowStart
    • Update the result if the current window is valid and better
  3. Return the result
function longestSubstringWithKDistinct(str, k) { let windowStart = 0; let maxLength = 0; const charFrequency = {}; for (let windowEnd = 0; windowEnd < str.length; windowEnd++) { const rightChar = str[windowEnd]; charFrequency[rightChar] = (charFrequency[rightChar] || 0) + 1; while (Object.keys(charFrequency).length > k) { const leftChar = str[windowStart]; charFrequency[leftChar]--; if (charFrequency[leftChar] === 0) { delete charFrequency[leftChar]; } windowStart++; } maxLength = Math.max(maxLength, windowEnd - windowStart + 1); } return maxLength; }

Time Complexity Analysis

The sliding window technique typically reduces the time complexity from O(n²) (in brute-force approaches) to O(n) for many problems.

Fixed-Size Window

Time Complexity: O(n) - Each element is processed exactly twice (once when added to the window and once when removed)

Space Complexity: O(1) - Uses constant extra space

Variable-Size Window

Time Complexity: O(n) - Each element is processed at most twice (added and removed from the window)

Space Complexity: O(k) - Where k is the number of distinct characters/elements in the window

Note: The sliding window technique is particularly effective because it avoids recalculating overlapping window sums or properties from scratch, instead efficiently updating the window's properties as it slides.

Common Mistakes and Pitfalls

  1. Incorrect window boundaries: Off-by-one errors in setting window start/end indices
  2. Premature window adjustment: Adjusting the window before all necessary calculations are done
  3. Overlooking edge cases: Not considering empty arrays, windows larger than array size, etc.
  4. Inefficient updates: Recalculating window properties from scratch instead of updating incrementally
  5. Infinite loops: In variable-size windows, not properly handling the window contraction condition

Conclusion

The sliding window technique is a powerful algorithmic pattern that can dramatically improve the efficiency of solutions to many array and string problems. By maintaining a window that slides through the data structure and efficiently updating window properties, we can often reduce time complexity from O(n²) to O(n).

Key takeaways:

  • Identify when a problem can benefit from sliding window (contiguous subarrays/substrings)
  • Determine whether you need a fixed-size or variable-size window
  • Carefully manage window boundaries and update conditions
  • Practice common patterns to recognize sliding window opportunities
