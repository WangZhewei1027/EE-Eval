Directed Graphs (Digraphs) — Comprehensive Guide

Thorough definitions, representations, properties, and classic algorithms for directed graphs with step-by-step explanations and worked examples.

1. What is a Directed Graph?

A directed graph (or digraph) is an ordered pair G = (V, E) where:

  • V is a finite set of vertices (also called nodes).
  • E is a set of ordered pairs of vertices, called directed edges or arcs. An edge (u, v) indicates a connection going from u to v.

Notation: (u → v) denotes a directed edge that starts at u and points to v. Edges in a directed graph are not symmetric by default: (u, v) does not imply (v, u).

2. Basic Concepts and Terminology

  • Out-degree of a vertex: number of edges leaving the vertex (count of (v, x)).
  • In-degree of a vertex: number of edges entering the vertex (count of (x, v)).
  • Path: a sequence of vertices v0, v1, ..., vk such that (vi, vi+1) is an edge for each i. The path is directed and respects edge directions.
  • Simple path: a path where all vertices (and thus edges) are distinct.
  • Cycle: a path with k ≥ 1 where v0 = vk and vertices (except the start/end) are distinct. For directed graphs we require all edges to follow directions.
  • Directed Acyclic Graph (DAG): a directed graph that contains no directed cycles.
  • Reachability: vertex v is reachable from u if there exists a directed path from u to v.
  • Strongly Connected: u and v are strongly connected if each is reachable from the other. A strongly connected component (SCC) is a maximal set of vertices that are pairwise strongly connected.

3. Representations of Directed Graphs

Common ways to represent a directed graph for algorithmic use:

  • Adjacency list: for each vertex, store a list (or vector) of outgoing neighbors. Efficient for sparse graphs; iterating outgoing edges is O(out-degree).
  • Adjacency matrix: a |V|×|V| boolean or weighted matrix where M[u][v] is true or contains the edge weight if (u, v) exists. Good for dense graphs; constant-time edge queries but O(|V|^2) memory.
  • Edge list: simply a list of directed edges (u, v). Useful when edges are the primary input (e.g., Bellman-Ford).

Memory and time trade-offs: adjacency lists have O(|V| + |E|) memory and are generally preferred when |E| ≪ |V|^2. Adjacency matrices are useful when quick existence checks are required or when implementing certain algebraic operations.

4. Fundamental Operations and Properties

4.1 Reachability and Traversal

Two canonical traversals are Breadth-First Search (BFS) and Depth-First Search (DFS). Both can be used on directed graphs by following only outgoing edges.

Breadth-First Search (BFS)

Use BFS to discover shortest paths in an unweighted directed graph (shortest measured in number of edges) from a source s to all reachable vertices.

// BFS (directed)
procedure BFS(graph G, source s)
  for each vertex v in V:
    v.visited = false
    v.dist = ∞
    v.parent = NIL
  s.visited = true
  s.dist = 0
  queue Q
  enqueue(Q, s)
  while Q not empty:
    u = dequeue(Q)
    for each v in Adj[u]:        // only outgoing edges u->v
      if not v.visited:
        v.visited = true
        v.dist = u.dist + 1
        v.parent = u
        enqueue(Q, v)
  return distances and parents

Time complexity: O(|V| + |E|). Space: O(|V|).

Depth-First Search (DFS)

DFS explores as far as possible along each outgoing edge before backtracking. Useful for topological sort (via post-order), cycle detection, and computing strong components.

// DFS (recursive style)
procedure DFS(G)
  for each vertex u in V:
    u.color = WHITE
    u.parent = NIL
  time = 0
  for each vertex u in V:
    if u.color == WHITE:
      DFS-Visit(u)

procedure DFS-Visit(u)
  u.color = GRAY
  time = time + 1
  u.discover = time
  for each v in Adj[u]:
    if v.color == WHITE:
      v.parent = u
      DFS-Visit(v)
  u.color = BLACK
  time = time + 1
  u.finish = time

Time complexity: O(|V| + |E|). Finish times and discovery times are used in several proofs and algorithms.

5. Topological Sorting (DAGs)

A topological ordering of a directed graph is a linear ordering of its vertices such that for every directed edge (u, v), u comes before v in the ordering. Only possible if the graph is acyclic (DAG).

Two common algorithms

Kahn's Algorithm (BFS-like)

Kahn's Algorithm (sketch)
  compute in-degree for every vertex
  put all vertices with in-degree 0 into queue Q
  order = []
  while Q not empty:
    u = pop(Q)
    append u to order
    for each v in Adj[u]:
      in-degree[v] -= 1
      if in-degree[v] == 0:
        push v into Q
  if size(order) == |V|:
    return order  // valid topological order
  else:
    return error (graph has a cycle)

Time: O(|V| + |E|). Intuitive: repeatedly remove sources (vertices with no incoming edges).

Topological Sort by DFS (postorder)

Topological Sort via DFS
  perform DFS on the graph
  when each vertex finishes (after exploring all outgoing edges), push it onto a stack
  after DFS, pop vertices from stack in order — that is a topological order

DFS-based method relies on finish times. Also runs in O(|V| + |E|).

6. Strongly Connected Components (SCCs)

SCCs partition the vertex set into maximal groups where each vertex is reachable from each other vertex in the same group. Two popular algorithms:

Kosaraju's Algorithm (two-pass DFS)

Kosaraju's algorithm
  1. Run DFS on G, keep vertices ordered by finish time (push onto stack on finish).
  2. Compute transpose graph G^T (reverse every edge).
  3. Pop vertices from stack; for each popped vertex, if not visited in G^T, run DFS on G^T starting from it. The set of vertices reached is one SCC.
  Time: O(|V| + |E|) (building G^T costs O(|V| + |E|)).

Tarjan's Algorithm (single-pass DFS)

Tarjan's algorithm (sketch)
  Use a DFS with timestamps and a stack. For each vertex u track:
    index[u] = time of first visit
    lowlink[u] = smallest index reachable from u by following zero or more tree edges then at most one back edge
  When lowlink[u] == index[u], u is root of an SCC: pop stack until u to form the component.
  Time: O(|V| + |E|), space O(|V|).

Tarjan's algorithm is efficient and commonly used because it finds SCCs in a single DFS traversal without explicitly transposing the graph.

7. Shortest Paths in Directed Graphs

Which algorithm to use depends on edge weights and graph properties.

Dijkstra's algorithm

For directed graphs with non-negative edge weights, Dijkstra finds shortest paths from a single source. Implement with a priority queue (min-heap).

Dijkstra (sketch)
  initialize dist[s]=0, dist[v]=∞ for v≠s
  push (0, s) into min-heap
  while heap not empty:
    (d, u) = pop min
    if d > dist[u]: continue
    for each edge u->v with weight w:
      if dist[v] > dist[u] + w:
        dist[v] = dist[u] + w
        push (dist[v], v) into heap
  Time: O((|V| + |E|) log |V|) with binary heap

Bellman-Ford algorithm

Handles negative weights (but no negative cycles reachable from the source). Also detects negative cycles.

Bellman-Ford (sketch)
  initialize dist[s]=0, dist[v]=∞ for v≠s
  repeat |V|-1 times:
    for each edge u->v with weight w:
      if dist[u] + w < dist[v]:
        dist[v] = dist[u] + w
  // Optionally, check for negative cycles:
  for each edge u->v with weight w:
    if dist[u] + w < dist[v]:
      report negative cycle reachable from s

DAG shortest paths

For a DAG we can compute single-source shortest paths in O(|V| + |E|) by processing vertices in topological order and relaxing outgoing edges once.

8. Cycle Detection and Special Paths

Cycle detection

To detect if a directed graph has a directed cycle, run DFS and look for back edges (an edge to a currently gray vertex). Alternatively, run Kahn's algorithm: if Kahn's ends with fewer than |V| nodes in the produced order, a cycle exists.

Directed Eulerian path/circuit

Conditions for Eulerian circuit (every edge used exactly once and start=end) in a directed graph:

  • Every vertex has equal in-degree and out-degree;
  • All vertices with non-zero degree belong to a single strongly connected component (ignoring edge direction, we often require "strong connectivity" in the directed sense except possibly for isolated vertices).

Directed Eulerian path (start ≠ end) requires: exactly one vertex with out-degree = in-degree + 1 (possible start), exactly one vertex with in-degree = out-degree + 1 (possible end), and all other vertices have in-degree = out-degree, plus connectivity constraints.

9. Complexity Summary

  • DFS/BFS: O(|V| + |E|)
  • Kahn's topological sort: O(|V| + |E|)
  • Tarjan's/Kosaraju's SCC: O(|V| + |E|)
  • Dijkstra (binary heap): O((|V| + |E|) log |V|)
  • Bellman-Ford: O(|V| × |E|)
  • Adjacency matrix operations: O(|V|^2) memory; many algorithms degrade to O(|V|^2) if matrix used

10. Worked Example: Step-by-step Kahn's Topological Sort (textual)

We repeat the example here and explain each step without the button (the button above produces the same steps interactively):

Graph — vertices: A, B, C, D, E. Edges: A→B, A→C, B→D, C→D, D→E.

  1. Compute in-degrees:
    • in(A)=0
    • in(B)=1 (from A)
    • in(C)=1 (from A)
    • in(D)=2 (from B and C)
    • in(E)=1 (from D)
  2. Initialize queue with in-degree 0 vertices: Q = [A]. Order=[]
  3. Pop A; append to order => Order=[A]. Remove outgoing edges A→B and A→C:
    • in(B) becomes 0 — add B to Q
    • in(C) becomes 0 — add C to Q
    Q might be [B, C] (order between them depends on implementation; both lead to valid topological sorts).
  4. Pop B; Order=[A, B]. Remove B→D:
    • in(D) becomes 1 (still >0) — not added yet.
  5. Pop C; Order=[A, B, C]. Remove C→D:
    • in(D) becomes 0 — add D to Q.
  6. Pop D; Order=[A, B, C, D]. Remove D→E:
    • in(E) becomes 0 — add E to Q.
  7. Pop E; Order=[A, B, C, D, E]. Now Q empty and we have all 5 vertices in order. This is a valid topological order.

Note: A different valid topological order is [A, C, B, D, E] — Kahn's algorithm can produce any topological order depending on queue ordering for zero in-degree vertices.

11. Pedagogical Notes — How to Reason About Directed Graphs

Here are guidelines and mental models that often help:

  • Think of edges as one-way streets: reachability respects direction.
  • For algorithms, always decide if you are iterating outgoing or incoming edges. Many representations (adjacency lists) are outgoing-focused, so computing in-degrees may require a prepass.
  • DFS discovery/finish times give powerful ordering information: finishing later usually means "depended-on" earlier in topological contexts.
  • Condensation graph: compress each SCC into a single node; the resulting graph is a DAG. This helps for reasoning about high-level structure: how strongly connected groups relate acyclically.
  • Test small examples by hand: draw a graph and walk through in-degrees, finish times, and relaxations to build intuition.

12. Common Pitfalls

  • Assuming an edge (u, v) implies (v, u). That's only if both directed edges exist explicitly.
  • Using Dijkstra with negative edges — incorrect results. Use Bellman-Ford or DAG shortest path (if acyclic).
  • When implementing Kahn's algorithm, forgetting to recompute in-degrees or to copy them if you need the original graph later.
  • When checking Eulerian conditions, also verify connectivity requirements; degree conditions alone are not sufficient in general.

13. Exercises and Practice Problems (with hints)

  1. Given a directed graph, implement a function that returns true if there exists a cycle. Hint: use DFS and track recursion stack (gray nodes) or use Kahn's algorithm and see if all nodes are removed.
  2. Given an adjacency list, compute the condensation DAG (one node per SCC). Hint: find SCCs via Kosaraju or Tarjan, then collapse edges between components.
  3. Given a DAG, find the length of the longest path. Hint: topologically order vertices and perform relaxation with negative weights or dynamic programming (store longest distance to each vertex).
  4. Prove that a topological order exists iff the graph is acyclic. Hint: if DAG then Kahn's algorithm produces an order; conversely, if order exists, no edge can go backwards, so no cycle.

14. Applications of Directed Graphs

  • Dependency resolution (package managers, build systems): model dependencies as edges A→B meaning A depends on B or vice versa depending on convention; topological sort determines build/install order.
  • Web links and authority/hub ranking (PageRank uses directed link structure).
  • Compiler analysis: control-flow graphs, data-flow, SSA form involve directed edges.
  • Network flows (directed capacities) and routing.
  • Modeling one-way relationships like follower graphs in social networks.
