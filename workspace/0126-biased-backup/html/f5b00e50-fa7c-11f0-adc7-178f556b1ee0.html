Binary Search Tree

Binary Search Tree (BST) is a data structure in which each node has at most two children (i.e., left child and right child) and each node represents a value. The left subtree of a node contains only values less than the node's value, while the right subtree of a node contains only values greater than the node's value.

Here's a step-by-step explanation of how a Binary Search Tree works:

  1. Insert a new node with a value into the tree.
  2. Recursively search for the value in the left subtree and the right subtree.
  3. If the value is found in the left subtree, the left subtree is searched again.
  4. If the value is found in the right subtree, the right subtree is searched again.
  5. Continue this process until the value is found or the search is exhausted.

Here's a simple example of a Binary Search Tree in JavaScript:

            let bst = new BinarySearchTree();
            bst.insert(5);
            bst.insert(2);
            bst.insert(8);
            bst.insert(3);
            bst.insert(9);
            
            console.log("Searching for 8:");
            console.log(bst.search(8));
            
            console.log("Searching for 1:");
            console.log(bst.search(1));

Types of Binary Search Trees

  • Binomial Tree
  • Binary Search Tree (BST)
  • Binary Search Tree (AVL) Tree

Advantages and Disadvantages

  • Advantages:
    • Search, Insert, and Delete operations are O(log n)
    • Binary Search Tree is self-balancing, ensuring efficient search and insertion
    • Binary Search Tree is a good choice for large datasets
  • Disadvantages:
    • Insertion and deletion operations can be O(n)
    • Binary Search Tree requires extra memory to store the tree

Common Operations

  • Insert a new node with a value into the tree.
  • Search for a value in the tree.
  • Delete a node from the tree.
