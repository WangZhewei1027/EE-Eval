Two Pointers — In-Depth Guide

A detailed, pedagogical explanation of the Two Pointers technique, its variants, invariants, algorithms, examples, and common interview problems. Static, reading-first resource.

What is Two Pointers?

Two Pointers is an algorithmic technique in which two indices (pointers) traverse a data structure—typically an array or a linked list—either from opposite ends or together in a coordinated way to solve problems efficiently without nested loops.

Key idea

Rather than using nested loops (O(n^2)), Two Pointers often allows you to solve problems in linear time O(n) or linearithmic time O(n log n) with a simple linear pass. The two pointers maintain an invariant that gradually moves toward a goal (e.g., find a target sum, partition elements, or detect cycles).

When to consider Two Pointers

  • The array is sorted (or can be sorted) and you need to find pairs or triplets meeting a condition.
  • You're dealing with subsequences, subarrays, or windows where two indices can define a range.
  • You need to do in-place rearrangements (e.g., remove duplicates, partition by property).
  • There is a linked list problem where you need two different traversal speeds (slow/fast pointers).

Variants and Patterns

1. Opposite-direction (left/right) on arrays

Classic usage on a sorted array: put one pointer at the start and one at the end. Move them inward based on comparisons to target values. Common for 2-sum, counting pairs, or removing elements.

2. Same-direction with two indices (sliding window / slow & fast)

Pointers both move to the right but at different rates. Often used to keep a subarray/window with certain properties or to overwrite elements in place (slow = write position, fast = read position).

3. Fast and slow (linked lists)

One pointer moves one step at a time (slow), the other moves two steps (fast). This is used for cycle detection, finding middle element, or removing nth node from end.

4. Partitioning two pointers

Use pointers to partition an array by a predicate—elements that should be on the left vs right—often used by quicksort's partition or the Dutch National Flag problem.

5. Two pointers with hashing or extra memory

Sometimes augmented with extra memory for counts or positions, but the core pointer movement idea remains.

Common concrete problems

  • Sorted two-sum: find two numbers that add up to target (left/right).
  • Remove duplicates from sorted array (slow/fast in-place overwrite).
  • Reverse array in-place (swap with two pointers from ends).
  • Check palindrome in a string (left/right compare).
  • Longest subarray with conditions (sliding window).
  • Cycle detection in linked list (fast/slow).

Quick intuition

Two pointers exploit ordered structure or monotonic properties. If an operation at one end moves you closer to the goal, move that pointer. Always maintain a clear invariant.

Tip: Before using Two Pointers ask:
  • Is the input ordered or can be ordered cheaply?
  • What invariant will I maintain?
  • Which pointer(s) move in response to which condition?

Detailed Examples and Step-by-Step Walkthroughs

Example 1 — Two-sum on a sorted array (left/right)

Problem: Given a sorted array nums[] and a target value T, find indices i & j such that nums[i] + nums[j] = T. Return a pair or indicate none exist.

Algorithm (high level)

  1. Initialize left = 0, right = n-1.
  2. While left < right:
    • sum = nums[left] + nums[right]
    • If sum == T, return (left, right)
    • If sum < T, left++ (we need a larger sum)
    • If sum > T, right-- (we need a smaller sum)
  3. If loop ends, no pair found.

Pseudocode

function twoSumSorted(nums, T):
  left = 0
  right = len(nums) - 1
  while left < right:
    s = nums[left] + nums[right]
    if s == T:
      return (left, right)
    else if s < T:
      left = left + 1
    else:
      right = right - 1
  return null

Step-by-step example

nums = [1, 2, 3, 4, 6, 8], T = 10

  1. left=0 (1), right=5 (8) → sum=9 < 10 → left++
  2. left=1 (2), right=5 (8) → sum=10 == 10 → found (1,5)

Why it works (invariant)

At every step, any valid pair must involve a number between left and right inclusive. When sum < T, moving right inward cannot create a solution with current left because right only decreases (values are non-increasing). Therefore we must increase left to try larger values.

Complexity

Time: O(n). Space: O(1). If the array is not sorted and you need to sort first, time becomes O(n log n).

Example 2 — Remove duplicates in-place (slow/fast)

Problem: Given a sorted array, remove duplicates in-place so that each element appears once and return the new length.

Algorithm idea

Use slow = index of last unique element, fast = scanning index. When nums[fast] != nums[slow], increment slow and write nums[slow] = nums[fast].

Pseudocode

function removeDuplicates(nums):
  if nums is empty: return 0
  slow = 0
  for fast from 1 to len(nums)-1:
    if nums[fast] != nums[slow]:
      slow = slow + 1
      nums[slow] = nums[fast]
  return slow + 1  // new length

Explanation

slow always marks the last index of the unique prefix; fast explores the rest and copies new unique values forward. This is O(n) time and O(1) space.

Example 3 — Reverse array in-place

Problem: Reverse an array using two pointers.

Algorithm

left = 0
right = n - 1
while left < right:
  swap(nums[left], nums[right])
  left++
  right--

Notes

Simple invariant: positions left..right are yet to be reversed. Stop when pointers meet or cross.

Proofs, Invariants, and Correctness

Maintain a clear invariant

Every Two Pointers solution must define an invariant: a relationship that holds between pointers at the start of each loop iteration and that gets strictly closer to termination when a pointer moves. Examples:

  • Sorted two-sum: any solution must use elements in [left, right].
  • Remove duplicates: slow is last unique index of processed prefix.
  • Reverse array: positions outside [left, right] are already finalized.

Typical correctness proof structure

  1. Initialization: invariant holds before loop starts.
  2. Maintenance: show that after one iteration the invariant still holds and pointers make progress.
  3. Termination: when pointers meet/ cross, show invariant implies desired result.

Example correctness sketch (two-sum)

Invariant: If there is any pair summing to T, at the start of every iteration both elements of such pair are within indices [left, right]. Initially true. If sum < T, we increment left — any pair with left as first element and any j ≤ right had sum < T (since array sorted), so cannot be a solution; thus any remaining valid pair remains in new interval. Similar for sum > T. Therefore when we find sum == T, it's correct.

Common Pitfalls and Tips

  • If the array is unsorted, the left/right method is not valid unless you sort (which may change indices or cost time).
  • Be careful with equality conditions (e.g., left < right vs left ≤ right), off-by-one errors are common.
  • When modifying the array in-place (slow/fast), ensure read/write pointers don't overwrite needed data.
  • For problems with more than two items (3-sum), a frequent pattern is to fix one index and run two-pointers on the remaining subarray.
  • Edge cases: empty inputs, single element arrays, all duplicates, negative numbers (sorting handles sign), very large numbers (watch overflow).

Practice Problems (with brief strategies)

  • Two Sum (sorted): left/right — O(n)
  • 3Sum: sort, then for each i run two-pointer on remaining array — O(n^2)
  • Container With Most Water (max area): two-pointer moving the smaller height pointer inward — O(n)
  • Longest Substring Without Repeating Characters: sliding window (dictionary + two pointers) — O(n)
  • Linked List Cycle: fast/slow pointer to detect cycle; to find start, reset one pointer and move both at speed 1 — O(n)
  • Dutch National Flag: three-way partition with low/mid/high pointers — O(n)

For each problem: identify if data is ordered or can be made ordered, define invariant, decide pointer movement rules, and reason about termination.

Implementation Notes & Patterns

1. Pair sum on sorted array

// left/right pattern
left = 0
right = n-1
while left < right:
  // evaluate and adjust

// Always decide clearly: when to increment left and when to decrement right

2. Slow/Fast for in-place overwrite

// Example remove duplicates
slow = 0
for fast in 1..n-1:
  if nums[fast] != nums[slow]:
    slow++
    nums[slow] = nums[fast]

3. Sliding window

left = 0
for right in 0..n-1:
  add nums[right] to window
  while window violates constraint:
    remove nums[left]
    left++

4. Fast/slow in linked lists

slow = head
fast = head
while fast != null and fast.next != null:
  slow = slow.next
  fast = fast.next.next
// If fast hits null -> no cycle; if fast == slow -> cycle detected

Summary Checklist

  • Identify if Two Pointers is applicable: look for order or a sliding-range structure.
  • Decide pointer roles (left/right, slow/fast, low/mid/high).
  • Define and document the invariant explicitly in your head or code comments.
  • Ensure progress: each iteration must move pointers closer to termination.
  • Handle edge cases: empty input, one element, repeated values.
  • Consider whether sorting is allowed/affordable if input is unsorted.
