Understanding Directed Graphs (Digraphs)

A graph is one of the fundamental abstract data structures used in computer science and mathematics to represent pairwise relationships between objects. In this document, we focus exclusively on directed graphs, also known as digraphs.

1. What is a Directed Graph?

A directed graph is a graph in which edges have a direction associated with them. Formally, a directed graph G can be defined as an ordered pair:

G = (V, E), where:

  • V is a set of vertices (also called nodes).
  • E is a set of ordered pairs of vertices called edges or arcs.

An edge (u, v) in E signifies a connection from vertex u to vertex v. The direction is important — an edge from u to v is not the same as an edge from v to u.

In a directed graph:

  • Edges are arrows pointing from one vertex to another.
  • Paths follow the direction of edges.
  • It’s possible that an edge from u to v exists without an edge from v to u.

Example: Consider vertices V = {A, B, C} and edges E = {(A, B), (B, C), (C, A)}. This forms a directed cycle between the three nodes.

1.1 Visual Representation

Directed graphs are often visualized as a set of labeled points (vertices) with arrows between them indicating edges:

  • A → B means an edge from A pointing to B.
  • Loops (edges from a vertex to itself) are allowed.

Note: In diagrams, arrows illustrate edge directions and thus help to understand properties like reachability.

2. Important Terminology in Directed Graphs

  • Vertex (Node): An individual element in the graph.
    Example: A, B.
  • Edge (Arc): A directed connection from one vertex to another.
    Example: (A, B).
  • In-degree: Number of edges incoming to a vertex.
    Example: If edges are (X, A) and (Y, A), then in-degree(A) = 2.
  • Out-degree: Number of edges outgoing from a vertex.
    Example: If edges are (A, X) and (A, Y), then out-degree(A) = 2.
  • Path: A sequence of vertices connected by a sequence of edges following direction.
    Example: A → B → C is a path if edges (A, B) and (B, C) exist.
  • Cycle: A path starting and ending on the same vertex with at least one edge.
  • Strongly Connected: A directed graph is strongly connected if a path exists from every vertex to every other vertex.
  • DAG (Directed Acyclic Graph): A directed graph with no cycles.

3. Representing Directed Graphs in Computer Programs

There are two predominant ways to represent graphs in computer programs:

3.1 Adjacency List

An adjacency list represents a graph as an array or map of lists. Each vertex stores a list of vertices reachable directly from it (via outgoing edges).

// Example Adjacency List Representation in JavaScript:
const graph = {
  A: ['B', 'C'],
  B: ['C'],
  C: ['A']
};

This representation is memory-efficient for sparse graphs. To find all vertices reachable from A, look at graph['A'].

3.2 Adjacency Matrix

An adjacency matrix is a 2D array where rows represent source vertices and columns represent destination vertices. If there's an edge from vertex i to vertex j, the element matrix[i][j] is true (or 1), else false (or 0).

// Example Adjacency Matrix Representation:
   A B C
A [0 1 1]
B [0 0 1]
C [1 0 0]

This method uses more memory when the graph is large but provides constant-time edge queries.

4. Core Algorithms on Directed Graphs

Directed graphs are widely used in numerous algorithms. We will explain some fundamental algorithms with step-by-step textual descriptions.

4.1 Depth-First Search (DFS)

Goal: Explore as far along each branch before backtracking.

Step-by-step:

  1. Start at a chosen node S.
  2. Mark S as visited.
  3. For each outgoing edge (S, V) to a vertex V not yet visited, recursively perform DFS starting at V.
  4. Continue until all reachable vertices are visited.

DFS can be used to detect cycles or to generate a topological ordering if the graph is acyclic.

4.2 Breadth-First Search (BFS)

Goal: Explore vertices in order of shortest path distance from the start vertex, layer by layer.

Step-by-step:

  1. Begin at vertex S. Mark it visited and enqueue it in a queue.
  2. While queue is not empty:
    • Dequeue a vertex U.
    • For every outgoing edge (U, V), if V is unvisited,
      mark V visited and enqueue it.

BFS finds the shortest path in terms of edge count in an unweighted directed graph.

4.3 Detecting Cycles in Directed Graphs

Cycles in directed graphs affect algorithmic decisions (for example, topological sorting requires an acyclic graph). One common method uses DFS with a recursion stack.

Step-by-step approach:

  1. Maintain two arrays (or sets):
    • visited: vertices already fully explored.
    • recStack: vertices currently in the recursion stack of DFS.
  2. For each vertex v, if not visited, invoke DFS.
  3. In DFS visit of v:
    • Mark v as visited and add it to recStack.
    • For every adjacent vertex u,
      • If u has not been visited, recursively visit u.
      • If u is in recStack, a cycle is detected.
    • Remove v from recStack after DFS calls complete.

4.4 Topological Sorting

Topological sorting orders vertices linearly such that for every edge (u, v), u appears before v in the order. Only possible if the directed graph is a Directed Acyclic Graph (DAG).

Two popular methods:

a) DFS Based Method

  1. Perform DFS on every vertex not yet visited.
  2. When DFS finishes exploring a vertex (i.e., all its descendants are explored), push it onto a stack.
  3. At the end, reverse the stack contents; this is the topological order.

b) Kahn's Algorithm (BFS Based)

  1. Initialize a queue with all vertices having in-degree 0.
  2. While queue not empty:
    • Remove a vertex u from the queue, append it to topological order.
    • For each vertex v adjacent from u:
      • Decrement in-degree(v) by 1.
      • If in-degree(v) becomes 0, add it to the queue.
  3. If topological order contains all vertices, graph is DAG; else a cycle exists.

5. Applications of Directed Graphs

Directed graphs model many real-world systems because direction often matters:

  • Modeling Relations: e.g. Twitter "Follower" relationships where A → B means "A follows B".
  • Task Scheduling: Tasks with dependencies form DAGs; topological sort helps find execution order.
  • Finite State Machines (FSMs): States as vertices, transitions (with direction) as edges.
  • Webpage link structure: Pages link to other pages directedly (edges show hyperlinks).
  • Shortest path finding: Navigation systems often use directed graphs to model one-way streets.

7. Summary

  • Directed graphs contain edges with direction, allowing modeling of asymmetrical relationships.
  • Key concepts include in-degree, out-degree, paths, cycles, strongly connected components, and DAGs.
  • They are commonly represented via adjacency lists or matrices in memory.
  • Core algorithms such as DFS, BFS, cycle detection, and topological sort utilize directed graph properties.
  • Applications span networks, scheduling, web structure, FSMs, shortest-path computations, and more.
