Floyd-Warshall Algorithm

Definition: The Floyd-Warshall algorithm is a dynamic programming algorithm for finding shortest paths in a weighted graph with positive or negative edge weights (but with no negative cycles). It can also be used to detect the presence of negative cycles in a graph.

Introduction

The Floyd-Warshall algorithm, developed by Robert Floyd and Stephen Warshall in 1962, is an algorithm for finding shortest paths in a weighted graph. Unlike Dijkstra's algorithm, which finds shortest paths from a single source vertex to all other vertices, Floyd-Warshall finds shortest paths between all pairs of vertices.

Key characteristics of the algorithm:

  • Works with directed and undirected graphs
  • Handles both positive and negative edge weights
  • Detects negative cycles in the graph
  • Time complexity: O(V³) where V is the number of vertices
  • Space complexity: O(V²)

Algorithm Overview

The Floyd-Warshall algorithm is based on the principle of dynamic programming. It progressively improves an estimate on the shortest path between two vertices by considering all possible intermediate vertices.

The algorithm maintains a distance matrix dist where dist[i][j] represents the shortest distance from vertex i to vertex j. Initially, this matrix is filled with:

  • 0 for the diagonal (distance from a vertex to itself)
  • The weight of the edge between vertices if an edge exists
  • Infinity if no edge exists between the vertices

Core Idea

The algorithm considers each vertex as an intermediate point in the shortest path between every pair of vertices. For each intermediate vertex k, it checks if going through k gives a shorter path from i to j than the current known path.

The update rule is:

if (dist[i][k] + dist[k][j] < dist[i][j])
    dist[i][j] = dist[i][k] + dist[k][j]

Step-by-Step Algorithm

  1. Initialize the distance matrix with the edge weights between vertices.
  2. For each vertex k (from 1 to V):
    1. For each pair of vertices i and j:
      1. If vertex k is on the shortest path from i to j, then update the value of dist[i][j]
  3. After all iterations, the distance matrix contains the shortest paths between all pairs of vertices.

Pseudocode

let dist be a |V| × |V| array of minimum distances initialized to ∞ (infinity)
for each edge (u, v) do
    dist[u][v] ← w(u, v) // The weight of the edge (u, v)
for each vertex v do
    dist[v][v] ← 0

for k from 1 to |V|
    for i from 1 to |V|
        for j from 1 to |V|
            if dist[i][j] > dist[i][k] + dist[k][j]
                dist[i][j] ← dist[i][k] + dist[k][j]
            end if

Detecting Negative Cycles

The Floyd-Warshall algorithm can also detect negative cycles in a graph. A negative cycle is a cycle whose edges sum to a negative value.

To detect a negative cycle:

  1. Run the Floyd-Warshall algorithm as usual
  2. After completion, check the diagonal of the distance matrix
  3. If any diagonal entry dist[i][i] is negative, then the graph contains a negative cycle that includes vertex i

Applications

  • Finding shortest paths in road networks
  • Routing protocols in computer networks
  • Solving the transitive closure of a graph
  • Computing similarity between graphs
  • Inversion of real matrices

Comparison with Other Algorithms

Algorithm Best Case Worst Case Space Graph Type
Floyd-Warshall O(V³) O(V³) O(V²) All pairs, positive/negative weights
Dijkstra's O(E + V log V) O(E + V log V) O(V) Single source, non-negative weights
Bellman-Ford O(VE) O(VE) O(V) Single source, negative weights

Limitations

  • High time complexity (O(V³)) makes it impractical for very large graphs
  • Not suitable for graphs that change frequently (dynamic graphs)
  • For single-source shortest path problems, Dijkstra's or Bellman-Ford are more efficient

Conclusion

The Floyd-Warshall algorithm is a powerful all-pairs shortest path algorithm that works with graphs containing negative edge weights (but no negative cycles). While its cubic time complexity makes it less efficient than other algorithms for single-source problems or sparse graphs, it remains the best choice for dense graphs where we need shortest paths between all pairs of vertices.
