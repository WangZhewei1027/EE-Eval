Congestion Control

Maintaining network efficiency through intelligent traffic management

Congestion Detected

Slow Start

TCP begins with a conservative window size and exponentially increases it until congestion is detected or a threshold is reached.

Avoidance Phase

After reaching the threshold, TCP switches to additive increase, growing the window size linearly to maximize throughput without causing congestion.

Congestion Response

When packet loss is detected, TCP dramatically reduces its window size and begins building back up, creating the characteristic sawtooth pattern.
