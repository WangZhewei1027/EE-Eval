Queue Data Structure

Introduction to Queues

A queue is a fundamental linear data structure that follows the First-In-First-Out (FIFO) principle. This means the first element added to the queue will be the first one to be removed, similar to a real-world queue of people waiting in line.

Queues are used extensively in computer science for:

  • Operating system process scheduling
  • Handling requests on a shared resource (like a printer)
  • Breadth-first search algorithms
  • Buffering data streams
  • Simulating real-world queues

Queue Operations

The primary operations that can be performed on a queue are:

1. Enqueue (Add/Insert)

Adds an element to the rear (also called tail) of the queue.

2. Dequeue (Remove/Delete)

Removes an element from the front (also called head) of the queue.

3. Peek/Front

Returns the element at the front of the queue without removing it.

4. IsEmpty

Checks if the queue is empty.

5. IsFull

Checks if the queue is full (for fixed-size implementations).

Queue Operation Example

Consider a queue of integers:

  1. Enqueue(10): Queue becomes [10]
  2. Enqueue(20): Queue becomes [10, 20]
  3. Enqueue(30): Queue becomes [10, 20, 30]
  4. Dequeue(): Returns 10, queue becomes [20, 30]
  5. Peek(): Returns 20, queue remains [20, 30]

Queue Implementation

Queues can be implemented using various underlying data structures:

1. Array Implementation

A simple but fixed-size implementation using an array:

class Queue { constructor(capacity) { this.items = new Array(capacity); this.front = 0; this.rear = -1; this.size = 0; } enqueue(item) { if (this.isFull()) throw new Error: Queue is full"); this.rear = (this.rear + 1) % this.items.length; this.items[this.rear] = item; this.size++; } dequeue() { if (this.isEmpty()) throw new Error("Queue is empty"); const item = this.items[this.front]; this.front = (this.front + 1) % this.items.length; this.size--; return item; } peek() { if (this.isEmpty()) throw new Error("Queue is empty"); return this.items[this.front]; } isEmpty() { return this.size === 0; } isFull() { return this.size === this.items.length; } }

2. Linked List Implementation

A dynamic implementation using nodes:

class Node { constructor(data) { this.data = data; this.next = null; } } class LinkedListQueue { constructor() { this.front = null; this.rear = null; this.size = 0; } enqueue(item) { const newNode = new Node(item); if (this.rear === null) { this.front = this.rear = newNode; } else { this.rear.next = newNode; this.rear = newNode; } this.size++; } dequeue() { if (this.isEmpty()) throw new Error("Queue is empty"); const item = this.front.data; this.front = this.front.next; if (this.front === null) this.rear = null; this.size--; return item; } peek() { if (this.isEmpty()) throw new Error("Queue is empty"); return this.front.data; } isEmpty() { return this.front === null; } }

Circular Queue

A circular queue (or ring buffer) is an efficient array implementation where the rear and front pointers wrap around to the beginning of the array when they reach the end.

Advantages:

  • Better memory usage as it reuses empty spaces created after dequeues
  • Constant time O(1) for both enqueue and dequeue operations

Circular Queue Example

Consider a circular queue of size 4:

  1. Enqueue A: [A, _, _, _] (front=0, rear=0)
  2. Enqueue B: [A, B, _, _] (front=0, rear=1)
  3. Enqueue C: [A, B, C, _] (front=0, rear=2)
  4. Dequeue: Returns A, [_, B, C, _], front=1, rear=2)
  5. Enqueue D: [_, B, C, D] (front=1, rear=3)
  6. Enqueue E: [E, B, C, D] (front=1, rear=0)

Priority Queue

A priority queue is a special type where elements are dequeued based on priority rather than just arrival order. Higher priority elements are dequeued before lower priority ones.

Implementation options:

  • Heap: Most efficient implementation (O(log n) for enqueue/dequeue)
  • Ordered array/list: O(n) for insertion, O(1) for removal
  • Unordered array/list: O(1) for insertion, O(n) for removal
// Simple priority queue using an array class PriorityQueue { constructor() { this.items = []; } enqueue(item, priority) { const queueElement = { item, priority }; let added = false; for (let i = 0; i < this.items.length; i++) { if (queueElement.priority < this.items[i].priority) { this.items.splice(i, 0, queueElement); added = true; break; } } if (!added) this.items.push(queueElement); } dequeue() { if (this.isEmpty()) throw new Error("Queue is empty"); return this.items.shift().item; } isEmpty() { return this.items.length === 0; } }

Applications of Queues

1. CPU Scheduling

Operating systems use queues to manage processes waiting for CPU time. Common scheduling algorithms like Round Robin use queues.

2. Breadth-First Search (BFS)

Queues are essential for BFS graph traversal, where nodes are explored level by level.

3. Printer Spooling

Print jobs are managed in a queue to handle multiple print requests fairly.

4. Call Center Systems

Incoming calls are placed in a queue until a representative becomes available.

5. Buffering

Queues buffer data in I/O operations, network communications, and streaming applications.
