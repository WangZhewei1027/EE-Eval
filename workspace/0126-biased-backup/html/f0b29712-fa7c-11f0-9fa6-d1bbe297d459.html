Kruskal's Algorithm

Definition

Kruskal's Algorithm is a greedy algorithm in graph theory that finds a minimum spanning tree (MST) for a connected, undirected graph with weighted edges. The algorithm works by sorting all the edges from the lowest weight to highest weight, then picking the smallest edge and adding it to the spanning tree if it doesn't form a cycle.

Key Concepts

Minimum Spanning Tree (MST)

A minimum spanning tree is a subset of the edges of a connected, edge-weighted undirected graph that connects all the vertices together, without any cycles, and with the minimum possible total edge weight.

Greedy Algorithm

Kruskal's algorithm is a classic example of a greedy algorithm. Greedy algorithms make the locally optimal choice at each stage with the hope of finding a global optimum. In this case, at each step we select the cheapest available edge that doesn't form a cycle.

Union-Find Data Structure

The algorithm uses the union-find (disjoint set) data structure to efficiently keep track of which vertices are in which components and to determine whether adding an edge would create a cycle.

Algorithm Steps

1. Sort all the edges in non-decreasing order of their weight 2. Initialize the MST as an empty set 3. For each edge in the sorted list: a. Check if adding the edge to the MST would form a cycle b. If no cycle is formed, add the edge to the MST c. If a cycle would be formed, discard the edge 4. Repeat step 3 until there are (V-1) edges in the MST (where V is the number of vertices in the graph)

Detailed Explanation

Initial Setup

Before beginning the algorithm, we need to represent our graph. Typically, we represent the graph as a list of edges, where each edge has a source vertex, destination vertex, and weight. We also need to initialize a disjoint set (union-find) data structure where each vertex is initially in its own set.

Edge Sorting

The first step is to sort all edges in increasing order of their weight. This allows us to always consider the smallest available edge next, which is crucial for the greedy approach to work correctly.

Cycle Detection

For each edge in the sorted list, we need to determine if adding it to our growing MST would create a cycle. This is done using the union-find data structure:

  • Find the root parent of both vertices of the edge
  • If the roots are the same, the vertices are already connected (adding the edge would create a cycle)
  • If the roots are different, we can safely add the edge and union the two sets

Termination Condition

The algorithm terminates when we've added (V-1) edges to the MST, where V is the number of vertices in the graph. This is because any spanning tree of a graph with V vertices must have exactly (V-1) edges.

Example Walkthrough

Consider the following weighted, undirected graph with 4 vertices (A, B, C, D) and 5 edges:

  • A-B: weight 10
  • A-C: weight 6
  • A-D: weight 5
  • B-D: weight 15
  • C-D: weight 4

Step 1: Sort edges by weight

Sorted order: C-D(4), A-D(5), A-C(6), A-B(10), B-D(15)

Step 2: Initialize MST and Union-Find

MST: empty

Sets: {A}, {B}, {C}, {D}

Step 3: Process edges

Edge C-D (4): Roots are C and D (different) → Add to MST, union sets

MST: C-D

Sets: {A}, {B}, {C, D}

Edge A-D (5): Roots are A and D (different) → Add to MST, union sets

MST: C-D, A-D

Sets: {A, C, D}, {B}

Edge A-C (6): Roots are A and C (same) → Skip (would form cycle A-D-C-A)

Edge A-B (10): Roots are A and B (different) → Add to MST, union sets

MST: C-D, A-D, A-B

Sets: {A, B, C, D}

We now have V-1 = 3 edges in the MST, so we're done.

Final MST

Edges: C-D (4), A-D (5), A-B (10)

Total weight: 19

Time Complexity Analysis

Operation Time Complexity Explanation
Sorting edges O(E log E) Using an efficient comparison-based sorting algorithm
Union-Find operations O(E α(V)) Where α is the inverse Ackermann function (very slow growing)
Total O(E log E) Dominant term is the sorting step

Since E is at most V², we can also express the complexity as O(E log V).

Space Complexity

The space complexity is O(V + E) to store the graph and the union-find data structure.

Applications

  • Network design (telephone, electrical, hydraulic, TV cable, computer, road)
  • Approximation algorithms for NP-hard problems like the traveling salesman problem
  • Cluster analysis
  • Image segmentation
  • Circuit design

Comparison with Prim's Algorithm

While both Kruskal's and Prim's algorithms find a minimum spanning tree, they differ in their approach:

  • Kruskal's: Works with edges, grows a forest of trees that eventually merge
  • Prim's: Works with vertices, grows a single tree by always adding the cheapest edge connected to the current tree
  • Kruskal's is generally better for sparse graphs (few edges), while Prim's is better for dense graphs (many edges)
  • Kruskal's requires sorting all edges initially, while Prim's uses a priority queue

Proof of Correctness

Kruskal's algorithm produces a minimum spanning tree because:

  1. It produces a spanning tree (connects all vertices without cycles)
  2. The tree is minimal because:
    • At each step, it adds the smallest available edge that doesn't form a cycle
    • This is equivalent to not rejecting any edge that must be part of some MST (cut property)
    • By induction, the algorithm maintains the invariant that the partial solution can be extended to an MST

Limitations

  • Only works for connected graphs (for disconnected graphs, it finds a minimum spanning forest)
  • Requires the graph to be undirected
  • Edge weights must be comparable (can be negative though)
