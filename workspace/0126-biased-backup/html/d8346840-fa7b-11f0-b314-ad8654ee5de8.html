Bubble Sort — Comprehensive Explanation and Guide

A detailed, pedagogical description of the Bubble Sort algorithm: what it is, how it works, why it behaves the way it does, and when and why you would (or would not) use it.

What is Bubble Sort?

Bubble Sort is one of the simplest comparison-based sorting algorithms. It repeatedly steps through the list, compares adjacent elements and swaps them if they are in the wrong order. This process is repeated until the list is sorted.

When do we meet Bubble Sort?

  • Often introduced as the first example of a sorting algorithm in computer science education because it is intuitive and easy to implement.
  • Useful to learn fundamentals: comparisons, swaps, loops, invariants, and optimization via early exit.
  • Not recommended for large-scale production sorting because of inferior time complexity compared to more advanced algorithms (e.g., quicksort, mergesort, timsort).
Key idea: With each full pass through the array, the largest unsorted element "bubbles up" to its correct position at the end of the array.

Formal definition (informal)

Given an array A of n elements, Bubble Sort repeatedly compares pairs (A[i], A[i+1]) for i from 0 to n-2 and swaps them if A[i] > A[i+1]. Each repetition (pass) moves the next largest element into its final position at the end.

Algorithm and Pseudocode

Below is the classic Bubble Sort pseudocode for sorting in non-decreasing order (smallest first).

procedure bubbleSort(A[0..n-1])
  for pass from 0 to n-2                  // need at most n-1 passes
    for i from 0 to n-2-pass              // last pass elements already in place
      if A[i] > A[i+1] then
        swap A[i] and A[i+1]
      end if
    end for
  end for
end procedure

Optimized version (early exit)

If during a pass no swaps occur, the array is already sorted and we can stop early. This makes Bubble Sort adaptive for nearly-sorted inputs.

procedure bubbleSortOptimized(A[0..n-1])
  for pass from 0 to n-2
    swapped = false
    for i from 0 to n-2-pass
      if A[i] > A[i+1] then
        swap A[i] and A[i+1]
        swapped = true
      end if
    end for
    if not swapped then
      break   // array is sorted
    end if
  end for
end procedure

Correctness argument (short)

Each comparison and possible swap enforces that after the first pass, the largest element is at the last index. After k passes, the k largest elements are in final positions at the ends of the array. Eventually, after at most n-1 passes, all elements are sorted.

Time and Space Complexity

Time complexity

  • Worst-case: O(n^2). Example: reverse-sorted array; every pass compares and swaps maximally.
  • Average-case: O(n^2). Random data leads to quadratic work on average.
  • Best-case: O(n) for the optimized version with an early-exit flag when the array is already sorted (one pass with zero swaps). The unoptimized version remains O(n^2) even if already sorted.

Why O(n^2)? (intuitive justification)

In the basic algorithm we perform about n passes and each pass makes up to n comparisons (actually n-pass). The total number of comparisons is roughly the sum (n-1)+(n-2)+...+1 = O(n^2). Swaps may be at most that many too.

Space complexity

Bubble Sort is in-place: O(1) auxiliary space (ignoring input storage). It only needs a few extra variables (counters and a temporary variable for swapping).

Properties, Stability, and Variants

Stability

Bubble Sort is stable: equal elements maintain their original relative order because swaps only occur when A[i] > A[i+1], not when equal.

Adaptive behavior

With the early-exit optimization, Bubble Sort becomes adaptive: it runs faster on nearly-sorted input, potentially O(n) in the ideal case.

Variants

  • Cocktail Shaker Sort (bidirectional bubble): Instead of always traversing left-to-right, alternate directions on each pass to move both large elements to the end and small elements to the front more quickly.
  • Odd-Even Sort: Compare and swap pairs (1,2), (3,4), ... then (2,3), (4,5), ...; repeat until sorted. This is useful in parallel settings.

When to use Bubble Sort?

  • As a teaching tool to introduce sorting fundamentals.
  • When n is tiny (e.g., n ≤ 10) and simplicity is important.
  • When you specifically need a stable, in-place, very simple algorithm and data is almost sorted.
  • Not suitable for performance-critical or large datasets.

Step-by-step Walkthrough (textual)

We will illustrate how classic (unoptimized) Bubble Sort works on a short example. The textual explanation lists steps and demonstrates the invariant behavior.

Example array

A = [5, 1, 4, 2, 8]

Pass 1 (compare adjacent pairs left to right)

  1. Compare A[0]=5 and A[1]=1 → 5 > 1 → swap → array becomes [1, 5, 4, 2, 8]
  2. Compare A[1]=5 and A[2]=4 → 5 > 4 → swap → array becomes [1, 4, 5, 2, 8]
  3. Compare A[2]=5 and A[3]=2 → 5 > 2 → swap → array becomes [1, 4, 2, 5, 8]
  4. Compare A[3]=5 and A[4]=8 → 5 < 8 → no swap → array stays [1, 4, 2, 5, 8]
Result after pass 1: the largest element 8 is in its final place at index 4.

Pass 2

  1. Compare A[0]=1 and A[1]=4 → 1 < 4 → no swap → [1, 4, 2, 5, 8]
  2. Compare A[1]=4 and A[2]=2 → 4 > 2 → swap → [1, 2, 4, 5, 8]
  3. Compare A[2]=4 and A[3]=5 → 4 < 5 → no swap → [1, 2, 4, 5, 8]
Result after pass 2: second-largest element 5 is at index 3.

Pass 3

  1. Compare A[0]=1 and A[1]=2 → no swap
  2. Compare A[1]=2 and A[2]=4 → no swap
The array is now [1, 2, 4, 5, 8], already sorted. An optimized algorithm would detect no swaps and stop early; the unoptimized one will still continue passes but make no swaps.

Code Examples (illustrative)

Simple implementation (JavaScript)

// Classic bubble sort (non-optimized)
function bubbleSort(arr) {
  const n = arr.length;
  for (let pass = 0; pass < n - 1; pass++) {
    for (let i = 0; i < n - 1 - pass; i++) {
      if (arr[i] > arr[i + 1]) {
        // swap
        const tmp = arr[i];
        arr[i] = arr[i + 1];
        arr[i + 1] = tmp;
      }
    }
  }
  return arr;
}

Optimized JavaScript (early exit)

function bubbleSortOptimized(arr) {
  const n = arr.length;
  for (let pass = 0; pass < n - 1; pass++) {
    let swapped = false;
    for (let i = 0; i < n - 1 - pass; i++) {
      if (arr[i] > arr[i + 1]) {
        const tmp = arr[i];
        arr[i] = arr[i + 1];
        arr[i + 1] = tmp;
        swapped = true;
      }
    }
    if (!swapped) break; // array is sorted
  }
  return arr;
}
These implementations show the idea in an imperative language. Many languages will have the same structure with nested loops and a swap when adjacent items are out of order.

Further Discussion, Exercises, and Teaching Tips

Comparisons with other sorts

  • Quicksort, mergesort, and heapsort give better average/worst cases for larger inputs (O(n log n)).
  • Insertion sort is often better than Bubble Sort for small/mostly-sorted arrays and is also stable and in-place.
  • Timsort (used in many standard libraries) is adaptive and fast on real-world data.

Teaching tips

  • Use physical cards or colored blocks to show the "bubbling" action — seeing the largest block move to the end helps learners form intuition.
  • Ask students to simulate by hand on a small array and record swaps — this improves understanding of invariants.
  • Introduce the optimized early-exit version and ask when it helps: demonstrate best-case O(n) on already sorted inputs.

Exercises

  1. Show the sequence of arrays produced by Bubble Sort (classic and optimized) on the input [3, 2, 1, 4].
  2. Prove that Bubble Sort is stable: explain why equal elements never change relative order.
  3. Implement the Cocktail Shaker variant and compare number of swaps on example arrays.
  4. For a reversed array of size n, compute the exact number of swaps Bubble Sort performs.
Quick exercise answer hint: For a reverse-sorted array, the number of swaps equals the number of inversions. For a reverse-sorted list of n distinct elements the number of inversions is n*(n-1)/2.
