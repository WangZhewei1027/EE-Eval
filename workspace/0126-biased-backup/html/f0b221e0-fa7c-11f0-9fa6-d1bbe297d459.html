Counting Sort: A Comprehensive Guide

Introduction to Counting Sort

Counting Sort is a non-comparison based sorting algorithm that operates in linear time, making it exceptionally efficient for certain use cases. Unlike comparison-based sorts (e.g., QuickSort, MergeSort) that have a lower bound of O(n log n), Counting Sort can achieve O(n) time complexity by making assumptions about the input data.

Key Characteristic: Counting Sort is particularly effective when sorting integers within a known, limited range.

How Counting Sort Works

The algorithm works by counting the number of occurrences of each distinct element in the input array, then using arithmetic to determine the positions of each element in the sorted output array.

Core Steps:

  1. Determine the Range: Find the maximum value in the input array to determine the range of values.
  2. Count Occurrences: Create a count array to store the count of each unique value.
  3. Calculate Cumulative Counts: Modify the count array to store cumulative counts.
  4. Build the Output Array: Use the count array to place elements in their correct sorted positions.

Detailed Algorithm Explanation

1. Input Constraints

Counting Sort requires that:

  • The input consists of integers (or data that can be mapped to integers)
  • The range of possible values is known and not excessively large

2. Pseudocode

function countingSort(array, maxValue) { // Initialize count array with zeros count = array of (maxValue + 1) zeros // Count occurrences of each value for i from 0 to length(array) - 1 do count[array[i]] += 1 // Calculate cumulative counts for i from 1 to maxValue do count[i] += count[i - 1] // Build the output array output = array of same length as input array for i from length(array) - 1 downto 0 do output[count[array[i]] - 1] = array[i] count[array[i]] -= 1 return output }

3. Time Complexity Analysis

The time complexity of Counting Sort is:

  • Best Case: O(n + k)
  • Average Case: O(n + k)
  • Worst Case: O(n + k)

Where:

  • n = number of elements in the input array
  • k = range of input (max value - min value + 1)
Note: When k is O(n), Counting Sort runs in linear time, making it asymptotically faster than comparison-based sorts.

4. Space Complexity

Counting Sort requires additional space for:

  • The count array (size k)
  • The output array (size n)

Thus, the space complexity is O(n + k).

Step-by-Step Example

Let's sort the following array: [4, 2, 2, 8, 3, 3, 1]

Step 1: Find the Maximum Value

maxValue = 8

Step 2: Initialize Count Array

Create count array of size 9 (0 to 8): [0, 0, 0, 0, 0, 0, 0, 0, 0]

Step 3: Count Occurrences

After counting each element:

Index (Value) 0 1 2 3 4 5 6 7 8
Count 0 1 2 2 1 0 0 0 1

Step 4: Calculate Cumulative Counts

Modify count array to store cumulative counts:

Index (Value) 0 1 2 3 4 5 6 7 8
Count 0 1 3 5 6 6 6 6 7

Step 5: Build Output Array

Place elements in their correct positions:

  1. Start from the end of the input array (value 1)
  2. count[1] = 1 → output[0] = 1, decrement count[1] to 0
  3. Next value 3 → count[3] = 5 → output[4] = 3, decrement count[3] to 4
  4. Next value 3 → count[3] = 4 → output[3] = 3, decrement count[3] to 3
  5. Continue this process completes...

Final sorted array: [1, 2, 2, 3, 3, 4, 8]

Properties of Counting Sort

Advantages

  • Linear Time Complexity: O(n + k) when k is O(n)
  • Stable Sort: Maintains relative order of equal elements
  • No Comparisons: Doesn't rely on element comparisons

Limitations

  • Integer Restriction: Only works with integer keys (or data mappable to integers)
  • Range Dependency: Performance degrades when k is large compared to n
  • Space Requirements: Needs additional space proportional to the range of data

Variations and Related Concepts

1. Radix Sort

Radix Sort uses Counting Sort as a subroutine to sort numbers digit by digit, allowing it to handle larger ranges efficiently.

2. Bucket Sort

Bucket Sort divides the input into buckets and then sorts each bucket, potentially using Counting Sort for individual buckets.

3. Negative Number Handling

Counting Sort can be modified to handle negative numbers by:

  1. Finding the minimum value
  2. Shifting all values by the minimum to make them non-negative
  3. Performing standard Counting Sort
  4. Shifting values back in the final output

Practical Applications

  • Histogram Generation: Counting occurrences of values in data analysis
  • Small Integer Ranges: Sorting arrays with small integer values (e.g., ages, test scores)
  • As a Subroutine: Used in more complex algorithms like Radix Sort
  • Frequency Counting: Finding frequencies of elements in a dataset

Conclusion

Counting Sort is a powerful algorithm for specific use cases where the input range is known and limited. While it has constraints that prevent it from being a general-purpose sorting solution, its linear time complexity makes it invaluable in the right circumstances. Understanding Counting Sort also provides foundational knowledge for more complex algorithms like Radix Sort.

When to Use: Consider Counting Sort when sorting integers with a limited range, especially when n is large and k is O(n).
