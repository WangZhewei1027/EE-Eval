Conceptual Notes
Dynamic Programming reduces repeated computation by caching subproblem solutions. The visualization shows calls (left) and memo cache population (right).
• Recursion tree expands exponentially without memoization.
• When a subproblem is computed once, subsequent calls reuse the cached result.
• Bottom-up DP fills table iteratively; memoization is its top-down twin.
