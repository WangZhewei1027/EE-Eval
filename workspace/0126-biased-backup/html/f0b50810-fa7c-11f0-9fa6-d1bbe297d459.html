Logistic Regression: A Comprehensive Guide

Introduction to Logistic Regression

Logistic regression is a fundamental statistical and machine learning technique used for binary classification problems. Despite its name containing "regression," it is primarily used for classification tasks where the outcome is categorical.

Definition: Logistic regression is a statistical model that uses a logistic function to model a binary dependent variable. It estimates the probability that a given input belongs to a particular category.

Unlike linear regression which predicts continuous values, logistic regression predicts the probability that an instance belongs to a particular class. The output is transformed using the logistic function (also called the sigmoid function) to ensure the predicted values lie between 0 and 1.

Example Application: Predicting whether an email is spam (1) or not spam (0) based on features like word frequency, sender information, etc.

The Logistic Function (Sigmoid Function)

At the heart of logistic regression is the logistic function, which maps any real-valued number into a value between 0 and 1. This function is also known as the sigmoid function.

σ(z) = 1 / (1 + e-z)

Where:

  • σ(z) is the output (probability between 0 and 1)
  • e is the base of the natural logarithm (Euler's number, ~2.718)
  • z is the input to the function (often called the log-odds or logit)

The sigmoid function has an S-shaped curve (sigmoid curve) with the following properties:

  • As z approaches +∞, σ(z) approaches 1
  • As z approaches -∞, σ(z) approaches 0
  • At z = 0, σ(z) = 0.5

Logistic Regression Model

The logistic regression model combines the linear regression model with the logistic function to produce probabilities.

P(Y=1) = σ(β0 + β1X1 + β2X2 + ... + βnXn)

Where:

  • P(Y=1) is the probability of the positive class
  • β0 is the intercept term (bias)
  • β1 to βn are the coefficients for features X1 to Xn
  • X1 to Xn are the input features

Decision Boundary

The model makes classifications by comparing the predicted probability to a threshold (typically 0.5):

  • If P(Y=1) ≥ 0.5, predict class 1
  • If P(Y=1) < 0.5, predict class 0
Note: The threshold can be adjusted based on the specific problem requirements, especially when the costs of false positives and false negatives are not equal.

Model Training: Maximum Likelihood Estimation

Logistic regression models are typically trained using maximum likelihood estimation (MLE), which finds the parameter values that make the observed data most probable.

Likelihood Function

For a dataset with m examples, the likelihood is:

L(β) = ∏i=1m P(yi|xi; β) = ∏i=1m (σ(βTxi))yi (1 - σ(βTxi))1-yi

Log-Likelihood

It's often easier to work with the log-likelihood:

ℓ(β) = ∑i=1m [yi log(σ(βTxi)) + (1 - yi) log(1 - σ(βTxi))]

The goal is to find the parameters β that maximize this log-likelihood function.

Optimization: Gradient Descent

In practice, we use optimization algorithms like gradient descent to find the parameters that maximize the log-likelihood.

Gradient of the Log-Likelihood

The gradient (derivative) of the log-likelihood with respect to the j-th parameter βj is:

∂ℓ/∂βj = ∑i=1m (yi - σ(βTxi))xi,j

Gradient Descent Update Rule

The parameters are updated iteratively using:

βj := βj + α ∑i=1m (yi - σ(βTxi))xi,j

Where α is the learning rate.

Note: In practice, variants like stochastic gradient descent (SGD) or mini-batch gradient descent are often used for large datasets.

Multiclass Logistic Regression

While basic logistic regression is for binary classification, it can be extended to multiclass problems using:

One-vs-Rest (OvR) Approach

Train one classifier per class, with the samples of that class as positive samples and all other samples as negatives.

Softmax Regression (Multinomial Logistic Regression)

A more direct approach that generalizes logistic regression to multiple classes. The probability that an input x belongs to class k is:

P(y=k|x) = eβkTx / (∑j=1K eβjTx)

Model Evaluation

Common metrics for evaluating logistic regression models include:

  • Accuracy: (TP + TN) / (TP + TN + FP + FN)
  • Precision: TP / (TP + FP)
  • Recall (Sensitivity): TP / (TP + FN)
  • F1 Score: 2 * (Precision * Recall) / (Precision + Recall)
  • ROC Curve and AUC: Measures the trade-off between true positive rate and false positive rate
  • Log-Loss: Measures the performance of a classifier where the predicted output is a probability

Assumptions of Logistic Regression

Logistic regression makes several key assumptions:

  1. Binary outcome: The dependent variable should be binary (for standard logistic regression)
  2. Linearity of independent variables and log odds: The relationship between the logit of the outcome and each predictor is linear
  3. No multicollinearity: Independent variables should not be too highly correlated with each other
  4. Large sample size: Requires a sufficiently large number of cases per independent variable
  5. No extreme outliers: The model can be sensitive to extreme values in predictors

Advantages and Disadvantages

Advantages:

  • Simple and efficient to implement
  • Provides probabilities for outcomes
  • Works well with linearly separable data
  • Less prone to overfitting with L1/L2 regularization
  • Easy to interpret (coefficients represent log odds ratios)

Disadvantages:

  • Assumes linear decision boundary
  • May underperform with non-linear problems
  • Sensitive to outliers
  • Requires feature scaling for some optimization methods
  • Not suitable for very large feature spaces without regularization

Practical Considerations

Feature Scaling

While logistic regression doesn't require feature scaling, it's often beneficial for:

  • Faster convergence in gradient descent
  • Better performance with regularization

Regularization

Common regularization techniques to prevent overfitting:

  • L1 (Lasso): Encourages sparsity (some coefficients become exactly zero)
  • L2 (Ridge): Shrinks coefficients toward zero but doesn't eliminate them
  • Elastic Net: Combination of L1 and L2 regularization

Handling Imbalanced Data

For imbalanced datasets:

  • Adjust class weights
  • Use different classification thresholds
  • Resample the data (oversampling minority class or undersampling majority class)

The sigmoid function transforms any real number into a value between 0 and 1:

For z = 0: σ(0) = 1 / (1 + e0) = 0.5
For z = 2: σ(2) = 1 / (1 + e-2) ≈ 0.88
For z = -1: σ(-1) = 1 / (1 + e1) ≈ 0.27

This transformation allows us to interpret the output as a probability.

Conclusion

Logistic regression remains one of the most widely used algorithms in statistics and machine learning due to its simplicity, interpretability, and effectiveness for binary classification problems. While it has limitations, particularly with non-linear decision boundaries, it serves as an excellent baseline model and is often the first algorithm to try for classification tasks.

Understanding logistic regression provides a strong foundation for learning more complex machine learning models, as many concepts (like maximum likelihood estimation, gradient descent, and regularization) extend to other algorithms.
