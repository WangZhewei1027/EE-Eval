Binary Tree — Comprehensive Educational Guide

Purpose: a thorough, textual explanation of the binary tree concept, properties, algorithms, and common techniques. This page is intentionally mostly static and explanatory. A single simple demo is provided at the end for illustration.

1. What is a Binary Tree?

A binary tree is a hierarchical data structure in which each node has at most two children. These two children are conventionally called the left child and the right child. A binary tree consists of nodes connected by edges; one node is designated as the root. Nodes with no children are called leaf nodes.

Important components and vocabulary:

  • Root: the topmost node of the tree.
  • Parent: a node that has children.
  • Child: a node directly connected to another node when moving away from the root.
  • Leaf: a node with no children.
  • Subtree: any node with all its descendants forms a subtree rooted at that node.
  • Depth (or level): number of edges from the root to the node.
  • Height: maximum depth among all nodes; sometimes defined as number of edges on the longest path to a leaf, or number of nodes on that path (be explicit which you use).

2. Types and special forms of binary trees

There are many specialized binary tree classifications. Important ones:

  • Full (proper or strict) binary tree: every node has either 0 or 2 children (no nodes with exactly one child).
  • Complete binary tree: all levels except possibly the last are completely filled, and all nodes in the last level are as far left as possible.
  • Perfect binary tree: all internal nodes have two children and all leaves are at the same level. Equivalent to a complete tree with all levels full.
  • Balanced binary tree: the heights of left and right subtrees of every node differ by at most a small constant (varies by definition; AVL trees use 1).
  • Binary search tree (BST): a binary tree where for each node, values in the left subtree are less than the node's value and values in the right subtree are greater (or depending on equality policy).

3. Representations

Binary trees are commonly represented in two main ways:

Pointer-based (linked) representation

Each node stores a value and two pointers/references: left and right. Example structure in pseudocode:

Node {
  value
  left  -> Node or null
  right -> Node or null
}

This is flexible, suits dynamic trees (insertion/deletion), and is typical in high-level languages.

Array (implicit) representation

For complete binary trees (especially heaps), use an array where for index i (1-based):

  • left child index = 2i
  • right child index = 2i + 1
  • parent index = floor(i / 2)

This representation is memory-efficient for complete trees and simple to navigate by index arithmetic.

4. Common properties and formulas

  • Maximum nodes at height h (where height counts edges from root starting 0): for a perfect binary tree: nodes = 2^(h+1) - 1.
  • Minimum height for n nodes: floor(log2(n)).
  • In a perfect binary tree with height h, number of leaves = 2^h.
  • In any binary tree, n = i + l where n = total nodes, i = internal nodes, l = leaves. For a non-empty full binary tree: l = i + 1.

5. Traversals — visiting every node

Traversal algorithms define the order in which nodes are visited. The main categories:

  • Depth-first: Pre-order, In-order, Post-order.
  • Breadth-first: Level-order (uses a queue).

Pre-order (root, left, right)

Visit the node first, then recursively traverse left subtree, then right subtree.

// Pre-order pseudocode (recursive)
function preorder(node):
  if node == null: return
  visit(node)
  preorder(node.left)
  preorder(node.right)

In-order (left, root, right)

Traverse left subtree, visit node, then traverse right subtree. For BSTs, in-order traversal yields sorted order.

// In-order pseudocode (recursive)
function inorder(node):
  if node == null: return
  inorder(node.left)
  visit(node)
  inorder(node.right)

Post-order (left, right, root)

Traverse left subtree, then right subtree, then visit the node. Useful for deleting tree or computing sizes bottom-up.

// Post-order pseudocode (recursive)
function postorder(node):
  if node == null: return
  postorder(node.left)
  postorder(node.right)
  visit(node)

Level-order (breadth-first)

Visit nodes level by level from root downward. Implemented using a queue.

// Level-order pseudocode
function levelOrder(root):
  if root == null: return
  q = new Queue()
  q.enqueue(root)
  while not q.empty():
    node = q.dequeue()
    visit(node)
    if node.left != null: q.enqueue(node.left)
    if node.right != null: q.enqueue(node.right)

Iterative alternatives

Recursive traversals are simple and elegant, but can be transformed to iterative using an explicit stack. There are also advanced constant-space techniques like Morris traversal for in-order and pre-order, which temporarily modifies pointers to avoid extra memory (careful handling required).

6. Binary Search Tree (BST)

A BST is a binary tree with the ordering property: for every node n, all values in the left subtree are < n.value and all values in the right subtree are > n.value (ties handled by convention).

BST operations — intuition and pseudocode

Search

// BST search (recursive)
function bstSearch(node, key):
  if node == null: return null  // not found
  if key == node.value: return node
  if key < node.value: return bstSearch(node.left, key)
  else: return bstSearch(node.right, key)

Iterative variant uses a loop and moves left/right accordingly.

Insertion

// BST insert (iterative style)
function bstInsert(root, key):
  if root == null: return new Node(key)
  parent = null
  cur = root
  while cur != null:
    parent = cur
    if key < cur.value: cur = cur.left
    else: cur = cur.right
  if key < parent.value: parent.left = new Node(key)
  else: parent.right = new Node(key)
  return root

Recursive insertion follows the same idea by returning new subtrees up the call chain.

Deletion

Deletion is the trickiest basic BST operation and has three main cases for the node to delete:

  1. Node is a leaf: simply remove it.
  2. Node has one child: replace the node with its child.
  3. Node has two children: replace the node's value with its in-order successor (smallest node in the right subtree) or in-order predecessor (largest in left subtree), then delete that successor/predecessor node from its old position (which will be case 1 or 2).
// BST delete (high-level)
function bstDelete(node, key):
  if node == null: return null
  if key < node.value: node.left = bstDelete(node.left, key)
  else if key > node.value: node.right = bstDelete(node.right, key)
  else:
    // found node to delete
    if node.left == null: return node.right
    if node.right == null: return node.left
    // both children exist:
    succ = min(node.right)
    node.value = succ.value
    node.right = bstDelete(node.right, succ.value)
  return node

7. Complexity analysis

Let n be the number of nodes in the tree, and h the height (maximum number of edges from root to leaf):

  • Traversal (visit all nodes): Time = O(n). Space = O(h) for recursion stack (or O(n) if you include output storage).
  • BST search/insert/delete (average case for random tree or balanced tree): Time = O(h) = O(log n). Worst case (degenerate tree like a linked list): O(n).
  • Level-order traversal: Time = O(n). Extra space = O(width) where width is maximum number of nodes at a level, worst-case O(n).

Balancing strategies (AVL, Red-Black) ensure h = O(log n) guaranteeing logarithmic time for search/insert/delete.

8. Balanced trees — why and how

Unbalanced BSTs can degrade to linked lists in worst-case insertion sequences (e.g., inserting sorted keys). Balanced trees maintain structural properties that keep height proportional to log(n).

AVL Trees

AVL trees maintain the invariant that the heights of left and right subtrees of any node differ by at most 1. Insert/delete operations may trigger rotations (single or double) to restore balance. Rotations are local adjustments that preserve BST order while reducing height.

Red-Black Trees

Red-black trees use color bits on nodes and a set of invariants (e.g., root is black, red nodes have black children, all root-to-leaf paths have same number of black nodes). They guarantee O(log n) height with fewer rotations on average than AVL.

Both structures are widely used in libraries (map/dictionary/set implementations) where consistent performance is required.

9. Common algorithmic patterns and uses

Binary trees are used extensively in computer science. Some common usages and patterns:

  • Expression trees: represent arithmetic expressions where internal nodes are operators and leaves are operands. Post-order can evaluate the expression.
  • Heaps: a kind of binary tree (usually array-based) used for priority queues; not a BST.
  • Segment trees and Fenwick trees: binary-tree-like structures for range queries (segment tree is explicit tree, Fenwick is array-based conceptual tree).
  • Search trees: BSTs, AVL, red-black, B-trees (multi-way), etc.
  • Decision trees: used in machine learning for classification/regression.

10. Educational examples and intuitive reasoning

Example tree (textual diagram):

Root A / \ B C / \ \ D E F / G

For this tree, traversals produce:

  • Pre-order: A B D E C F G
  • In-order: D B E A C G F (note: in-order only yields sorted order if this is a BST with keys arranged accordingly)
  • Post-order: D E B G F C A
  • Level-order: A B C D E F G

Step-by-step thought for pre-order: start at root (A), visit A; go left to B, visit B; left to D, visit D (no children) — back up to B, then right to E, visit E; back to A, go right to C, visit C; right subtree of C -> F, visit F; left of F -> G, visit G.

11. Recursion vs Iteration & practical tips

  • Recursion maps naturally to trees: each node’s operation calls the same operation on its subtrees. Prefer recursion for clarity when depth is moderate.
  • For very deep trees, recursion can cause stack overflow; use iterative algorithms with explicit stacks or iterative transforms like Morris traversal for in-order/pre-order when constant extra space is required (but these modify tree pointers and are more advanced).
  • For interview settings: be explicit about edge cases (empty tree, single node, duplicates policy in BST), articulate base cases and invariants, and explain complexity trade-offs.

12. Pseudocode library (reference)

Compute height (recursive)

function height(node):
  if node == null: return -1         // height in edges; return 0 for nodes count option
  leftH = height(node.left)
  rightH = height(node.right)
  return max(leftH, rightH) + 1

Count nodes

function countNodes(node):
  if node == null: return 0
  return 1 + countNodes(node.left) + countNodes(node.right)

Check if two trees are identical

function equals(a, b):
  if a == null and b == null: return true
  if a == null or b == null: return false
  return a.value == b.value and equals(a.left, b.left) and equals(a.right, b.right)
