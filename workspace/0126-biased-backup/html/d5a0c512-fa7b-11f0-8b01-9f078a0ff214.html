Selection Sort Algorithm

Selection Sort is a simple and intuitive sorting algorithm that works by repeatedly selecting the smallest (or largest, depending on the sorting order) element from the unsorted portion of the list and moving it to the sorted portion of the list. It operates in O(n^2) time complexity, making it inefficient on large lists, and it is not a stable sort.

How Selection Sort Works

The Selection Sort algorithm divides the input list into two parts: the sorted part at the left and the unsorted part at the right. Initially, the sorted part is empty, and the unsorted part contains all the elements.

Step-by-Step Explanation

  1. Start with the first element: Assume the first element is the minimum.
  2. Compare with other elements: Compare the assumed minimum with the next elements of the array. If a smaller element is found, update the minimum.
  3. Swap: After going through all the elements, if you found a smaller element, swap it with the first element (the assumed minimum).
  4. Repeat: Move to the next position in the array and repeat the above steps until the entire array is sorted.

Algorithm Description

SelectionSort(arr)
1.  n = length(arr)
2.  for i from 0 to n-1 do:
3.      minIndex = i
4.      for j from i+1 to n do:
5.          if arr[j] < arr[minIndex] then
6.              minIndex = j
7.      end for
8.      swap(arr[i], arr[minIndex])
9.  end for

Example of Selection Sort

Consider the array: [64, 25, 12, 22, 11]. Here’s how Selection Sort will sort it step by step:

  1. Initial array: [64, 25, 12, 22, 11]
  2. Find the minimum from index 0 to 4, which is 11. Swap it with index 0.
  3. Array now: [11, 25, 12, 22, 64]
  4. Find the minimum from index 1 to 4, which is 12. Swap it with index 1.
  5. Array now: [11, 12, 25, 22, 64]
  6. Find the minimum from index 2 to 4, which is 22. Swap it with index 2.
  7. Array now: [11, 12, 22, 25, 64]
  8. Find the minimum from index 3 to 4, which is 25. No swap needed.
  9. Array now: [11, 12, 22, 25, 64] (already sorted)

Characteristics of Selection Sort

  • Time Complexity: O(n²) in the average and worst cases.
  • Space Complexity: O(1), as it only requires a constant amount of additional space.
  • Stable: Not stable; equal elements may not retain their relative order.
  • In-Place: Yes; it requires only a small, constant amount of additional storage space.

Conclusion

Selection Sort is an excellent introductory algorithm for understanding sorting concepts, though in practice, more efficient algorithms such as Quick Sort or Merge Sort are often preferred for large datasets.
