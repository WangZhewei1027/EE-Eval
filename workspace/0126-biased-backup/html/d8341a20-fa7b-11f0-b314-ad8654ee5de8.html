Priority Queue — Comprehensive Explanation

A complete textual, pedagogical treatment of the priority queue abstract data type — definitions, algorithms, step-by-step examples, complexity, implementations, and common uses.

What is a Priority Queue?

A priority queue is an abstract data type (ADT) similar to a regular queue or stack, but where each element has an associated "priority". Instead of being strictly FIFO or LIFO, the element removed next is the one with the highest priority (for a max-priority queue) or the lowest priority (for a min-priority queue).

Key properties

  • Elements are stored together with a priority key.
  • Primary operations are insertion and removal of the highest/lowest priority element.
  • Priority queues can be implemented in many ways; different implementations offer different time/space tradeoffs.

Common terms

  • Min-priority queue (min-heap): Extracts the element with the smallest priority.
  • Max-priority queue (max-heap): Extracts the element with the largest priority.
  • Key / priority: The value used to order elements.
  • Stability: Whether equal-priority items preserve insertion order (many heap implementations are not stable by default).

Abstract operations and semantics

At the ADT level, the basic operations are:

  • make-queue(): Create an empty priority queue.
  • insert(element, priority): Insert an element with a given priority.
  • peek() / top(): Return (but do not remove) the element with highest (or lowest) priority.
  • extract(): Remove and return the element with highest (or lowest) priority.
  • change-priority(element, newPriority): Change the priority of a given element (sometimes called decrease-key/increase-key in algorithms).
  • merge(q1, q2): Merge two priority queues into one.

Behavioral notes

  • Some implementations support keyed or indexed priorities allowing efficient change-priority operations.
  • Merging (meld) is O(1) for Fibonacci and certain other heap variants; for binary heaps merging is O(n).

Why and where priority queues are used

Priority queues are fundamental building blocks used in many algorithms and real systems:

  • Shortest path algorithms: Dijkstra's algorithm uses a priority queue keyed by tentative distance.
  • Minimum spanning tree: Prim's algorithm similarly uses a priority queue for nearest edges.
  • Event simulation: Next event chosen by smallest timestamp.
  • Scheduling: Operating systems, job schedulers, and task managers select highest-priority jobs.
  • A* search: Open set is often a priority queue keyed by f-score.
  • Data streaming and selection: Keeping the top-k elements in a stream.

Common implementations and their tradeoffs

Binary heap (array-based)

Most commonly used in practice due to simplicity and good practical performance.

  • Structure: a nearly complete binary tree stored in an array.
  • Parent/child indices (0-based array): parent(i) = floor((i-1)/2), left(i)=2i+1, right(i)=2i+2.
  • Operations:
    • insert: O(log n) — bubble-up/sift-up
    • extract-min/max: O(log n) — sift-down
    • peek: O(1)
    • build-heap from an array: O(n) via Floyd's method (sift-down from middle).
    • merge: O(n) (no cheap constant-time meld)

Fibonacci heap

Advanced structure with excellent amortized bounds; used where decrease-key matters (e.g., Dijkstra's theoretical improvement).

  • Amortized complexity: insert O(1), find-min O(1), extract-min O(log n) amortized, decrease-key O(1) amortized.
  • Merge/meld: O(1) amortized.
  • More complex and higher constant factors than binary heaps — rarely used in practical libraries.

Binomial heap, pairing heap, multi-ary (d-ary) heaps

Other variants tradeoff different operation costs and constants. Pairing heaps are simple and perform well in practice; d-ary heaps (e.g., 4-ary) can reduce cache misses.

Language/library support

  • C++ STL: std::priority_queue (max-heap by default) — wrapper around a random-access container (usually vector) using push_heap/pop_heap.
  • Java: java.util.PriorityQueue (min-heap) — binary heap implementation.
  • Python: heapq module — functions to use a binary heap on a list (min-heap semantics).

Detailed step-by-step: Binary min-heap example

We will walk through inserting elements into a binary min-heap stored in an array. Example sequence: [7, 2, 5, 3, 10, 1]

Initial empty heap

[ ]

Insert 7

Place at end, then bubble-up while parent > current. The heap becomes:

[7]

Insert 2

Place at end -> [7, 2]. Compare with parent (7). Since 2 < 7, swap (bubble-up):

[2, 7]

Insert 5

Place at end -> [2, 7, 5]. Parent of 5 is 2, 5 > 2, so no swap. Heap:

[2, 7, 5]

Insert 3

Place at end -> [2, 7, 5, 3]. Parent of 3 is 7 (index 1). Since 3 < 7, swap -> [2, 3, 5, 7]. Parent of 3 is now 2, 3 > 2 no further swaps.

[2, 3, 5, 7]

Insert 10

Append -> [2, 3, 5, 7, 10]. Parent is 3, 10 > 3, no swap.

[2, 3, 5, 7, 10]

Insert 1

Append -> [2, 3, 5, 7, 10, 1]. Parent of 1 is 5 (index 2). 1 < 5, swap -> [2, 3, 1, 7, 10, 5]. Parent of 1 is 2 (index 0). 1 < 2, swap -> [1, 3, 2, 7, 10, 5]. Final heap:

[1, 3, 2, 7, 10, 5]

Extract-min from [1, 3, 2, 7, 10, 5]

We remove the root (1). To restore heap shape, move last element to root (5), shrink array, then sift-down:

Step 1: move last to root -> [5, 3, 2, 7, 10]
Compare 5 with children 3 and 2: smallest child is 2 -> swap with 2
Step 2: [2, 3, 5, 7, 10]
Now 5 is at index 2 with children out of range -> done.
Extracted value: 1
Final heap: [2, 3, 5, 7, 10]

Pseudocode (array-based binary min-heap)

function insert(heap, value):
    heap.append(value)
    i = heap.size - 1
    while i > 0:
        parent = (i - 1) // 2
        if heap[parent] <= heap[i]:
            break
        swap(heap[parent], heap[i])
        i = parent

function extract_min(heap):
    if heap.size == 0:
        error "empty"
    minVal = heap[0]
    heap[0] = heap.pop()  // move last to root
    sift_down(heap, 0)
    return minVal

function sift_down(heap, i):
    n = heap.size
    while true:
        left = 2*i + 1
        right = 2*i + 2
        smallest = i
        if left < n and heap[left] < heap[smallest]:
            smallest = left
        if right < n and heap[right] < heap[smallest]:
            smallest = right
        if smallest == i:
            break
        swap(heap[i], heap[smallest])
        i = smallest

Complexities: insert O(log n), extract-min O(log n), peek O(1), build-heap O(n).

Amortized vs worst-case complexities

Binary heap gives worst-case O(log n) for insert and extract. Fibonacci heap gives amortized bounds: insert O(1) amortized, extract-min O(log n) amortized, decrease-key O(1) amortized. "Amortized" means averaged over a sequence of operations; individual operations may be more expensive.

When does decrease-key matter?

Algorithms such as Dijkstra frequently perform decrease-key operations. If decrease-key is frequent and performance-critical, an indexed priority queue (supporting decrease-key efficiently) or Fibonacci heap might be used. In practice, many implementations sidestep decrease-key by inserting duplicates (with better priority) and ignoring stale entries on extraction — this is simpler and often fast enough when constants are small.

Indexed / supporting change-priority

An indexed priority queue associates each element with an index and maintains a mapping from index to position in the heap array. This enables change-priority or remove operations for arbitrary elements in O(log n) time (bubble-up or sift-down from the element's index) because you can locate the element quickly.

Use-cases

  • Dijkstra with decrease-key: label each vertex with its heap index.
  • Systems where priorities change frequently for existing items (job priority updates).

Detailed comparison: Binary heap vs Fibonacci heap

  • Binary heap: Simple, low constants, good cache behavior. Typical choice in standard libraries.
  • Fibonacci heap: Better asymptotic amortized complexities for decrease-key and merge. Complex to implement and higher constants; often not worth it in practice except for theoretical guarantees.

Summary table (informal)

Operation        Binary heap (worst)    Fibonacci heap (amortized)
insert               O(log n)                    O(1)
find-min             O(1)                        O(1)
extract-min          O(log n)                    O(log n)
decrease-key         O(log n)                    O(1)
merge                O(n)                        O(1)

Practical tips and pitfalls

  • For many applications, use the library-provided priority queue (e.g., heapq in Python, PriorityQueue in Java, std::priority_queue in C++).
  • Binary heaps are not stable. If you need stability, include a tie-breaking counter that increments on insert and compare by (priority, counter).
  • For mutable priorities, prefer indexed heaps or the "insert duplicate with new priority" trick plus lazy deletion (mark stale entries and skip when extracted).
  • Memory: binary heap uses O(n) space. Fibonacci heaps use more complex node structures and use more memory per element.
  • In multithreaded contexts, protect the structure with synchronization; many language libraries provide thread-safe variants or wrappers.

Step-by-step narrative example: Dijkstra with a priority queue

Suppose we want to find shortest paths from source vertex s. We maintain a min-priority queue keyed by tentative distances, dist[v].

  1. Initialize dist[s]=0, dist[others]=∞. Insert all vertices (or insert as discovered) into the priority queue keyed by dist.
  2. While the priority queue is not empty:
    1. Extract vertex u with smallest dist[u]. If this dist is stale (larger than current dist[u]), skip it.
    2. For each neighbor v of u with weight w(u,v), if dist[u] + w < dist[v], set dist[v] = dist[u] + w and perform decrease-key on v (or insert duplicate entry with new smaller key).

Run-time depends on priority queue: with binary heap O((V+E) log V), with Fibonacci heap O(E + V log V).

Step-by-step small graph example

Graph (edges with weights):

s --1--> a
s --4--> b
a --2--> b
a --5--> t
b --1--> t

Initialize: dist[s]=0, push (s,0).

Extract s -> relax edges: push (a,1), push (b,4). Extract a (dist 1) -> relax: update b to 3 (decrease-key or push (b,3)), push (t,6). Extract b (dist 3) -> relax: update t to 4 (push (t,4)). Extract t (dist 4) -> done.

Note: Lazy insertion of duplicates requires skipping stale entries on extraction.

Implementations: language notes & idioms

Python (heapq)

import heapq

heap = []
heapq.heappush(heap, (priority, item))
priority, item = heapq.heappop(heap)  # min-heap behavior

# For max-heap, push (-priority, item) or use custom wrapper.
# For stable behavior with equal priorities, include a counter:
counter = 0
heapq.heappush(heap, (priority, counter, item))

C++ (std::priority_queue)

#include <vector>

std::priority_queue pq; // max-heap of int

// For min-heap:

Java (PriorityQueue)

import java.util.PriorityQueue;

PriorityQueue<Integer> pq = new PriorityQueue<>(); // min-heap
pq.offer(5);
int x = pq.poll();

Advanced topics (brief)

Concurrent priority queues

Concurrent priority queues (multiple producers/consumers) are tricky because operations like extract-min require coordination. Many practical concurrent implementations use approximate priority queues or relaxed semantics for higher throughput.

External (disk-based) priority queues

For huge datasets, external priority queues (external memory heaps) are designed to minimize I/O. Examples include tournament trees and buffer-heap variants.

Approximate/relaxed queues

Some systems accept slightly relaxed ordering (e.g., k-relaxed), which allows higher concurrency with controlled deviation from strict priority order.

FAQs and common misunderstandings

Is a priority queue the same as a heap?

Not exactly. A priority queue is an abstract data type. A heap (binary heap, Fibonacci heap, etc.) is a concrete data structure that implements a priority queue.

Are priority queues stable?

Not by default. If you must preserve insertion order among equal priorities, include a tie-breaker (e.g., an insertion counter) in the ordering key.

What if I need to frequently change priorities?

Use an indexed priority queue or a heap variant that supports decrease-key efficiently, or use the lazy-insert-with-stale-skip pattern.

Summary

Priority queues are a versatile, widely used ADT. For most practical uses, a binary heap delivers strong performance with simple code. For theoretical bounds on decrease-key and merge, Fibonacci heaps and related structures are relevant. Understanding both the abstract operations and the concrete implementations allows you to choose the right tool for your needs.

If you use library heaps, remember the semantics (min vs max) and whether you need stability or efficient decrease-key.
