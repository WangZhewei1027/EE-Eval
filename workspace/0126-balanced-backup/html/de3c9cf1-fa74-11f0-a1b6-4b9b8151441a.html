Understanding NP-Completeness

What is NP-Completeness?

NP-Completeness is a classification of computational problems that are:

  1. In NP (nondeterministic polynomial time): Solutions can be verified quickly
  2. NP-Hard: At least as hard as the hardest problems in NP

If any NP-Complete problem can be solved in polynomial time, then all NP problems can. This is the P vs NP question.

Example Problems

Boolean Satisfiability Problem (SAT)

The first problem proven to be NP-Complete. Given a boolean formula, is there an assignment of variables that makes the whole formula true?

Try this simple example: (A ∨ B) ∧ (¬A ∨ C)

Traveling Salesman Problem (TSP)

Given a list of cities and distances between them, find the shortest possible route that visits each city exactly once and returns to the origin city.

Complexity Classes

Class Description Example Problems
P Problems solvable in polynomial time Sorting, Shortest path
NP Problems verifiable in polynomial time SAT, TSP, Graph coloring
NP-Complete Hardest problems in NP (all NP problems can be reduced to them) SAT, TSP, Knapsack
NP-Hard At least as hard as NP-Complete problems (may not be in NP) Halting problem, Chess

Why Does It Matter?

NP-Complete problems appear everywhere in real-world applications:

  • Scheduling and logistics (TSP)
  • Circuit design (SAT)
  • Protein folding
  • Cryptography
  • Network design

The fact that no efficient solution is known for these problems (and that they're all equivalent in difficulty) is one of the most important unsolved questions in computer science.
