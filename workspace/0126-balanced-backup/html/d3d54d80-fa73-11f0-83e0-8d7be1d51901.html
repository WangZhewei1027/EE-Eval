About this demo

This demo implements a Binary Search Tree (BST). Inserted values are placed so that for any node, left children are smaller and right children are larger. Traversals show different visiting orders:

  • Preorder: Node → Left → Right
  • Inorder: Left → Node → Right (sorted order)
  • Postorder: Left → Right → Node
  • Level-order: Breadth-first by levels
Click a node to highlight it. Use "Check Balanced" to test if the tree is height-balanced (difference of heights ≤ 1 at every node).

Quick tips
  • Insert duplicate values are ignored.
  • Export gives you JSON of nodes (preorder) to restore later.
  • Try Randomize to create varied trees and then visualize traversals.
Example

Try inserting: 50, 30, 70, 20, 40, 60, 80. Then click Inorder — you'll see a sorted sequence.
