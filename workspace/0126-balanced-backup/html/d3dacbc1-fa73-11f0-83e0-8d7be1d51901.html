About this demo

This single-file demo applies a few automated, safe refactors to a sample invoicing function. The transforms are simple, pattern-based, and meant to illustrate ideas — real-world refactors typically use AST tools (TypeScript, Babel, clang) and tests to ensure safety.

What is Refactoring?

Refactoring is the disciplined technique of restructuring existing code — changing its internal structure without altering external behavior. Benefits:
  • Improve readability and intent
  • Reduce duplication
  • Isolate responsibilities (single responsibility)
  • Make testing and future changes easier
  • Enable safer enhancements

Refactor patterns shown

  • Introduce guard clause
  • Extract function
  • Replace magic numbers with named constants
  • Use let/const
  • Basic formatting & clearer variable names

Safety tips

  1. Have tests (unit/integration) covering behavior before refactor
  2. Refactor in small steps and commit often
  3. Prefer renames & extractions over rewriting logic
  4. Run tests after each change
This demo runs tests automatically and compares the outputs before and after refactor to demonstrate behavior preservation for these examples.
