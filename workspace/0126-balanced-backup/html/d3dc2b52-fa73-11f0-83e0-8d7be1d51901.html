Symmetric Cryptography — Interactive Demo

This page demonstrates symmetric encryption using the Web Crypto API (AES‑GCM). Symmetric cryptography uses the same key for encryption and decryption. You can generate a key, derive one from a passphrase, encrypt text, and decrypt it back.

Algorithm: AES‑GCM, 256‑bit key. If deriving from a passphrase we use PBKDF2 + SHA‑256. IV is randomly generated per encryption (12 bytes) and prefixed to ciphertext.
You can copy this base64 key and reuse it. If you derived a key from a passphrase, a salt is shown below — you must keep the salt to derive the same key again.
- Symmetric keys must be kept secret and shared securely between sender and receiver.
- AES‑GCM provides confidentiality and authenticity (an authentication tag). Tampering the ciphertext will cause decryption to fail.
- This demo prefixes the IV (initialization vector) to the ciphertext so the recipient can extract it to decrypt. The IV must be unique for each encryption with the same key.
- Deriving keys from passphrases requires a salt (random value) and many iterations to slow attackers performing guesses. Keep salt and parameters with the ciphertext or send them alongside via a secure channel.
