Big-Theta (Θ) Notation Explained

What is Big-Theta Notation?

Big-Theta notation (Θ) describes the tight bound of an algorithm's growth rate. It means the algorithm's running time grows at the same rate as the function inside the Θ.

Formally, we say that f(n) = Θ(g(n)) if there exist positive constants c₁, c₂, and n₀ such that:
0 ≤ c₁·g(n) ≤ f(n) ≤ c₂·g(n) for all n ≥ n₀

Example: Linear Search

Let's analyze the linear search algorithm:

function linearSearch(arr, target) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) return i;
    }
    return -1;
}

In the best case, the target is the first element (Θ(1)).
In the worst case, the target isn't in the array (Θ(n)).
But Big-Theta describes the exact bound when best and worst cases are the same.

Demonstration: Finding Θ for a Function

Let's analyze the function: f(n) = 3n² + 2n + 1

We can show this is Θ(n²) because:

  • Upper bound (O): 3n² + 2n + 1 ≤ 6n² for n ≥ 1
  • Lower bound (Ω): 3n² + 2n + 1 ≥ 3n² for n ≥ 0

Thus, 3n² ≤ 3n² + 2n + 1 ≤ 6n² for n ≥ 1, so f(n) = Θ(n²).

Common Big-Theta Notations

  • Θ(1): Constant time (array access, arithmetic operations)
  • Θ(log n): Logarithmic time (binary search)
  • Θ(n): Linear time (simple search through array)
  • Θ(n log n): Linearithmic time (efficient sorting algorithms)
  • Θ(n²): Quadratic time (nested loops, bubble sort)
  • Θ(2ⁿ): Exponential time (recursive Fibonacci)
