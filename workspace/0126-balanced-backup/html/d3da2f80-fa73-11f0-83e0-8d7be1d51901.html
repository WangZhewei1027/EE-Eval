Router Implementation (concept)

This demo shows a tiny hash-based router with: route parameters (e.g. /users/:id), query parsing, nested routes, and a 404 fallback.

/**
 * Routes are defined as: { path: '/users/:id', render: (params, query) => '...' }
 * The router converts path patterns into RegExp to extract params.
 * It listens to "hashchange" and renders the matching route.
 */
