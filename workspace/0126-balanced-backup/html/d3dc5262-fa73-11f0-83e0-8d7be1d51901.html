How it works (simplified):
  • Generate ECDSA key pair (P-256). Private key stays in your browser memory unless you export it.
  • To sign: the message is hashed (SHA-256) and the private key creates a signature over that hash.
  • To verify: the verifier uses the signer's public key to check the signature against the message hash. If even one bit of the message or signature changed, verification fails.
Note: This demo uses the Web Crypto API. Keys and operations happen in your browser; no data is sent anywhere.
