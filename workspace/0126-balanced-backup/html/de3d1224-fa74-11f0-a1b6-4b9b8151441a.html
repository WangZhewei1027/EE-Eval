OSI Model Demonstration

The OSI (Open Systems Interconnection) model is a conceptual framework used to understand network interactions in 7 layers.

7. Application Layer

Provides network services directly to user applications. Protocols: HTTP, FTP, SMTP.

Example: Your web browser communicating with a web server.

6. Presentation Layer

Translates data between the application and network formats. Handles encryption, compression.

Example: Converting data to JSON/XML, SSL/TLS encryption.

5. Session Layer

Manages connections between applications. Establishes, manages, and terminates sessions.

Example: RPC (Remote Procedure Call), NetBIOS.

4. Transport Layer

Provides reliable data transfer. Handles flow control, error checking, and data segmentation.

Protocols: TCP (reliable), UDP (unreliable).

3. Network Layer

Handles logical addressing and routing. Determines the best path for data to travel.

Protocols: IP, ICMP. Devices: Routers.

2. Data Link Layer

Provides node-to-node data transfer. Handles physical addressing (MAC) and error detection.

Protocols: Ethernet, PPP. Devices: Switches, Bridges.

1. Physical Layer

Transmits raw bit stream over physical medium. Defines electrical, mechanical specifications.

Example: Cables, hubs, network interface cards.
