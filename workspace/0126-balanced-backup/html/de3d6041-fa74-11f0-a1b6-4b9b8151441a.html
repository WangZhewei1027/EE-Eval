How This Works:

This simulation demonstrates how load balancers distribute incoming requests across multiple servers.

Try different algorithms to see how they affect request distribution:

  • Round Robin: Distributes requests sequentially to each server in order
  • Least Connections: Sends new requests to the server with the fewest active connections
  • Random: Randomly selects a server for each request
  • IP Hash: Uses client IP to consistently route to the same server (simulated with random IPs)
