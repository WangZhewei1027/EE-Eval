Quick explanation
This demo implements the Interpreter pattern:
  • Context holds variable values.
  • Expression objects (Terminal & Non-Terminal) implement interpret(context).
  • Parser builds an AST from text, then the root.interpret(context) evaluates it.

Implementation (key parts)
/* Expression base */
class Expression {
  interpret(context){ throw 'abstract'; }
}

/* Terminal expressions */
class VariableExpression extends Expression {
  constructor(name){super(); this.name=name;}
  interpret(ctx){ return ctx.get(this.name); }
}
class ConstantExpression extends Expression {
  constructor(value){super(); this.value=value;}
  interpret(ctx){ return this.value; }
}

/* Non-terminals */
class AndExpression extends Expression {
  constructor(a,b){super(); this.a=a; this.b=b; }
  interpret(ctx){ return this.a.interpret(ctx) && this.b.interpret(ctx); }
}
class OrExpression extends Expression {
  constructor(a,b){super(); this.a=a; this.b=b; }
  interpret(ctx){ return this.a.interpret(ctx) || this.b.interpret(ctx); }
}
class NotExpression extends Expression {
  constructor(expr){super(); this.expr=expr; }
  interpret(ctx){ return !this.expr.interpret(ctx); }
}

/* Context: variable storage */
class Context {
  constructor(vars){ this.vars = vars; }
  get(name){ if(!(name in this.vars)) throw 'Unknown variable '+name; return !!this.vars[name]; }
}
Implementation note: this demo uses a simple tokenizer + recursive-descent parser to build the AST; each Expression implements interpret(context) — the heart of the Interpreter pattern.
