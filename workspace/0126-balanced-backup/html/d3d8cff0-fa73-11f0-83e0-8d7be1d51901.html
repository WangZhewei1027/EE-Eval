Process definition (short):
A process is an instance of a running program with its own memory space, registers, and OS bookkeeping. Processes go through states (new → ready → running → waiting/blocked → terminated). This demo simulates scheduling, blocking for I/O, and context switching at tick granularity.

Concept notes
- Processes have private memory and kernel bookkeeping. Threads share memory within a process.
- Scheduling decides which process runs next; preemption may interrupt running process.
- Blocking occurs when process performs I/O and waits — it cannot run until I/O completes.
- Fork creates a child process with (simulated) copied state.
