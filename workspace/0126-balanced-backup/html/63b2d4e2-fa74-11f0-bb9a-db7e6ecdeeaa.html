Relational Database Concept Demo

What is a Relational Database?
It stores data in tables (relations) consisting of rows and columns. Each table represents an entity, and tables can be connected using keys to model relationships.
This demonstration shows a simple relational database with 3 tables: Customers, Orders, and Products. We will see the data and query a result by joining tables.

Tables

Customers

CustomerID (PK)NameEmail

Products

ProductID (PK)NamePrice ($)

Orders

OrderID (PK)CustomerID (FK)ProductID (FK)Quantity

Relationships

Primary Key (PK)
Uniquely identifies each row in a table.
Examples: CustomerID, ProductID, OrderID
Foreign Key (FK)
Refers to a primary key in another table to establish a relationship.
Examples: Orders.CustomerID → Customers.CustomerID
                          Orders.ProductID → Products.ProductID

Query: List all Orders with Customer name and Product details
