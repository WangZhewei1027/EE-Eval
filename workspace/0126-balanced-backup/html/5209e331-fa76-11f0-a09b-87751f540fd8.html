Big-Omega Notation

Big-Omega Notation is used to describe the time or space complexity of an algorithm.

It is a measure of the worst-case scenario, i.e., the most time-consuming or space-consuming part of the algorithm.

It is usually denoted by the symbol O

Examples

1. Sorting algorithms: O(n^2)

2. Searching algorithms: O(n)

3. String matching algorithms: O(n*m)

4. Binary search: O(log n)

Code

        
            function binarySearch(arr, target) {
                let left = 0;
                let right = arr.length - 1;
                while (left <= right) {
                    let mid = Math.floor((left + right) / 2);
                    if (arr[mid] === target) {
                        return mid;
                    } else if (arr[mid] < target) {
                        left = mid + 1;
                    } else {
                        right = mid - 1;
                    }
                }
                return -1;
            }

Time Complexity

Time complexity is the number of operations performed by the algorithm as the input size increases.

It is usually denoted by the symbol O

Examples

1. Time complexity of binary search: O(log n)

2. Time complexity of bubble sort: O(n^2)

3. Time complexity of merge sort: O(n log n)

Space Complexity

Space complexity is the amount of additional memory required by the algorithm.

It is usually denoted by the symbol O

Examples

1. Space complexity of binary search: O(1)

2. Space complexity of bubble sort: O(n)

3. Space complexity of merge sort: O(n)
