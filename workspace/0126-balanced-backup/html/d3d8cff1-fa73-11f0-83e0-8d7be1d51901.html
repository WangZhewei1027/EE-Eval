Thread concept demo (Web Workers)

This single file demonstrates the "thread" concept in browsers using Web Workers. You can spawn workers (background threads), run CPU-bound tasks (prime counting), cancel them, create a small thread pool and—when available—use a SharedArrayBuffer + Atomics to show atomic increments shared between threads. If SharedArrayBuffer is not available in your environment, a safe fallback is used.

What this shows
  • Web Workers run off the main UI thread and communicate via postMessage.
  • Workers can be created from blob URLs (all code is in this file).
  • SharedArrayBuffer + Atomics allow multiple threads to operate on shared memory atomically (requires cross-origin isolation).
  • Fallback uses a manager worker to simulate atomic increments when SharedArrayBuffer isn't available.
Try this
  1. Choose number of workers and ranges, click "Spawn Pool & Start Work".
  2. Watch progress bars and logs. Try "Stop / Terminate All".
  3. Use "Atomic Increment" repeatedly to observe atomic/shared increments.
Note: Browsers expose "threads" via Web Workers. This demo avoids requiring server configuration by creating worker code dynamically.
