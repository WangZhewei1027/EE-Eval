Encryption Concepts — Interactive Demo

Try simple ciphers and real Web Crypto (AES-GCM, RSA-OAEP). Learn symmetric vs asymmetric and see how hybrid encryption works.

Why encryption?

Encryption transforms readable information (plaintext) into an unreadable format (ciphertext) so only authorized parties can reverse it (decrypt).

Symmetric
Same secret key encrypts and decrypts. Fast for large data (e.g., AES).
Asymmetric
Public/private key pair. Public key encrypts, private key decrypts (e.g., RSA). Useful for key exchange and signatures.

How this demo is organized

  • Simple ciphers: Caesar and Vigenère (educational).
  • Symmetric: AES-GCM using Web Crypto API — generate key, encrypt/decrypt, show IV and ciphertext.
  • Asymmetric: RSA-OAEP key pair, encrypt with public key, decrypt with private key.
  • Hybrid: encrypt AES key with RSA public key — common real-world pattern.

Notes

This page uses the browser's Web Crypto API (crypto.subtle). Keys marked "extractable" are exportable for demo — in production private keys should be kept non-extractable and protected.
