Understanding Weighted Graphs: A Comprehensive Explanation

Graphs are fundamental structures in computer science and mathematics. Among many varieties, weighted graphs are especially important as they represent networks where connections have different "costs," "distances," or "capacities."

1. What Is a Weighted Graph?

A weighted graph is a graph in which each edge (the connection between two vertices or nodes) has an associated numerical value called a weight. This weight can represent various real-world factors, such as:

  • Distance or length between cities on a map
  • Cost of travel or shipping between nodes
  • Bandwidth or capacity in network links
  • Time required to complete an activity

Formally: A weighted graph can be represented as G = (V, E, w), where:

  • V is the set of vertices (nodes)
  • E is the set of edges (connections), typically pairs of vertices: (u, v)
  • w: E → ℝ is the weight function assigning a real number weight to every edge

Weighted graphs can be either directed or undirected:

  • Undirected: edges have no direction; the weight applies both ways (e.g., a road you can travel both directions, same cost)
  • Directed: edges have a direction, and weights apply in that direction only (e.g., one-way streets or asymmetric costs)

3. Why Are Weights Important?

Weights add rich meaning to graph connections. They allow us to:

  • Measure cost or benefit: Find the “cheapest” or “shortest” path between two nodes
  • Model real conditions: Travel times, network bandwidth, risk, etc., quantified as edge weights
  • Perform optimizations: Minimize total cost or maximize efficiency in routing, scheduling, and resource allocation

Unweighted graphs only represent whether a connection exists or not. Weighted graphs also tell us how good, bad, or costly an edge is.

4. Common Representations of Weighted Graphs

Weighted graphs can be stored in multiple ways, depending on the use case and efficiency needs. The two most common are:

a) Adjacency Matrix with Weights

In an adjacency matrix, rows and columns correspond to graph vertices. Each cell matrix[i][j] holds the weight of the edge from vertex i to vertex j. If no edge exists, a special marker (often ∞ or a null value) is used.

Example adjacency matrix for the example graph:

ABC
A0510
B507
C1070

Here, 0 usually means no cost to stay on the same vertex.

b) Adjacency List with Weights

An adjacency list stores for each vertex a list of pairs: connected neighbor vertex and the edge's weight. This representation is usually more memory-efficient for sparse graphs.

Example adjacency list for the example graph:

A: (B, 5), (C, 10)
B: (A, 5), (C, 7)
C: (A, 10), (B, 7)

5. Fundamental Algorithms on Weighted Graphs

One of the primary reasons we study weighted graphs is to solve search and optimization problems, like finding the shortest or least costly path.

a) Dijkstra’s Algorithm (Shortest Path)

This algorithm finds the shortest path from a source vertex to all other vertices in a graph with non-negative edge weights.

  1. Initialize: Set the tentative distance to 0 for the source node and ∞ for all others. Mark all nodes as unvisited.
  2. Visit the unvisited node with the smallest tentative distance.
  3. Update its neighbors: For each neighbor, calculate the distance through this node. If it is less than the currently recorded distance, update it.
  4. Mark the current node as visited.
  5. Repeat steps 2-4 until all nodes have been visited or the shortest path to the desired target is established.

Dijkstra’s algorithm ensures the shortest path is found efficiently, especially with data structures like priority queues.

b) Bellman-Ford Algorithm

This algorithm can also find shortest paths but can handle graphs with negative edge weights (as long as there are no negative cycles).

It relaxes edges repeatedly and detects negative-weight cycles.

c) Minimum Spanning Tree (MST) Algorithms

Weighted undirected graphs enable algorithms to find the MST — a subset of edges that connect all vertices with minimal total weight:

  • Prim’s algorithm: Grows the MST from a starting vertex, always adding the smallest weight edge that connects new vertices
  • Kruskal’s algorithm: Sorts all edges by weight and adds them incrementally, avoiding cycles, until all vertices are connected

6. Applications of Weighted Graphs

  • Transportation Networks: Road distances, airline routes with costs, shipping logistics
  • Telecommunications: Network design with bandwidth/cost constraints
  • Social Networks: Strength or frequency of relationships
  • Machine Learning: Graph-based clustering and shortest path in feature spaces
  • Resource Scheduling: Task dependencies with weights representing time or cost
