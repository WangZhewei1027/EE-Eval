Union-Find (Disjoint Set) Data Structure

The Union-Find data structure, also known as the Disjoint Set data structure, is a powerful and efficient tool commonly used in computer science for managing and querying partitioned sets. It provides a way to keep track of a collection of non-overlapping sets and supports two primary operations:

  • Find: Determine which subset a particular element is in. This can be used to check if two elements are in the same subset.
  • Union: Join two subsets into a single subset.

This structure is instrumental in many algorithms, including Kruskal's algorithm for finding Minimum Spanning Trees, network connectivity, image processing, and more.

Core Concepts and Motivation

Imagine you have a set of objects, and you want to group them into several disjoint (non-overlapping) sets. Over time, you need to perform two types of operations:

  1. Check whether two objects belong to the same group (set).
  2. Merge two groups into one when you discover a connection that relates them.

The naive approach to manage such sets would be to store each object with a label or pointer to the set it belongs to and update labels after each merge. However, if many unions occur, this approach can be very inefficient, especially if implemented with simple arrays or lists.

The Union-Find data structure is optimized to perform a series of unions and finds very efficiently, making it nearly constant time on average (amortized) per operation.

Formal Definition

The Union-Find structure manages n elements (usually indexed from 0 to n-1) partitioned into disjoint sets. Initially, each element is in its own distinct set.

Operations:

  • Find(x): Returns the identifier (usually called a representative or root) of the set to which x belongs.
  • Union(x, y): Merges the sets containing x and y into a single set.

How is Union-Find Implemented?

The most common implementation uses an integer array parent[] where each element stores either:

  • Its own set representative (if it is the root of its set, typically indicated by parent[i] = i), or
  • The index of its parent element in the tree representation of the set.

The sets are represented as rooted trees, where the root is the representative of the set. Initially, each element is its own parent (a tree of one element).

Find Operation

The Find(x) operation travels up the tree from x to find the root (the representative). To optimize repeated queries, two techniques are used:

  • Path Compression: During Find, every node traversed points directly to the root. This flattens the tree structure, improving subsequent Find operations.

Example: If a path of nodes 5 → 3 → 1 → 1 is traversed, then after compression, nodes 5 and 3 will point directly to 1 (the root).

Union Operation

The Union(x, y) operation merges two trees by connecting the root of one to the root of the other. To prevent tall trees (which increase Find costs), two strategies are commonly applied:

  • Union by Rank (or Height): Attach the smaller tree to the root of the larger tree.
  • Union by Size: Attach the tree with fewer nodes to the root of the tree with more nodes.

This ensures the trees remain shallow, maintaining efficient Find queries.

Illustrated Example

Let’s begin with 7 disjoint sets containing elements 0, 1, 2, 3, 4, 5, 6:

Sets: {0}, {1}, {2}, {3}, {4}, {5}, {6}

Perform the following operations:

  1. Union(0, 1) → Merge sets containing 0 and 1.
  2. Union(1, 2) → Merge sets containing 1 and 2.
  3. Find(0) → Returns representative of the set containing 0.
  4. Union(3, 4) → Merge sets containing 3 and 4.
  5. Find(4) → Returns representative of the set containing 4.
  6. Union(2, 4) → Merge sets containing 2 and 4.
  7. Find(3) → Returns representative of the set containing 3.

After these operations, the sets are:

{0, 1, 2, 3, 4}, {5}, {6}

The Find operation will return the root representative of the set, which might be any member of the merged group depending on union rules, but commonly the root of the first union or balanced by rank/size.

Union-Find Pseudocode

Initialization

for i in 0 to n-1:
    parent[i] = i
    rank[i] = 0  # rank or size to assist union

Find with Path Compression

function Find(x):
    if parent[x] != x:
        parent[x] = Find(parent[x])   # Path compression step
    return parent[x]

Union by Rank

function Union(x, y):
    rootX = Find(x)
    rootY = Find(y)
    if rootX == rootY:
        return  # Already in the same set
    if rank[rootX] < rank[rootY]:
        parent[rootX] = rootY
    else if rank[rootX] > rank[rootY]:
        parent[rootY] = rootX
    else:
        parent[rootY] = rootX
        rank[rootX] += 1

Time Complexity

Thanks to path compression and union by rank, m operations (Find and Union combined) over n elements run in nearly amortized O(m α(n)) time, where α(n) is the inverse Ackermann function, a quantity so small it is effectively constant for all realistic input sizes.

This makes Union-Find extremely efficient and suitable for performance-critical applications involving dynamic connectivity.

Real-World Applications

  • Minimum Spanning Trees (Kruskal's Algorithm): Efficiently detect cycles to avoid adding edges that cause loops.
  • Network Connectivity: Quickly answer whether two computers are in the same network.
  • Image Processing: Group connected components or regions in images.
  • Percolation Theory: Determine connectivity in models of porous materials.
  • Computational Geometry: Manage sets of connected points.
