Understanding Deque: Theory, Concepts, and Algorithms

In computer science, a deque (pronounced "deck") is a specialized data structure that allows insertion and deletion of elements from both ends efficiently. The term deque stands for double-ended queue. Unlike a standard queue which is first-in-first-out (FIFO) and only allows operations at one end for insertion and the other for deletion, a deque enables more flexible operations.

1. Defining the Deque

A deque is an abstract data type that generalizes a queue, supporting insertion and deletion operations at both the front and the rear ends of the list. Imagine it as a queue that lets you enqueue or dequeue from either end.

Formally, a deque supports the following operations:
  • push_front(x): Insert element x at the front.
  • push_back(x): Insert element x at the back.
  • pop_front(): Remove and return the element at the front.
  • pop_back(): Remove and return the element at the back.
  • front(): Retrieve (without removing) the element at the front.
  • back(): Retrieve (without removing) the element at the back.
  • is_empty(): Check if the deque is empty.
  • size(): Return the current number of elements.

2. Why Use a Deque?

The deque's versatility makes it an excellent choice for many algorithmic needs. Some common cases include:

  • Palindrome checking: Quickly compare front and back characters.
  • Sliding window algorithms: Efficiently maintain max/min values over a window.
  • Breadth-first search variants: Some BFS variants use deques to push neighbors at different ends.
  • Undo/Redo buffers: Handling operations from both ends of the history.

3. Deque Implementations

Because of its requirement to efficiently operate at both ends, ordinary data structures like arrays or linked lists require some design choices:

3.1 Using Arrays (Circular Buffers)

By using a fixed-size array as a circular buffer, we can represent a deque efficiently:

  • Maintain two indices: front and rear.
  • Insertion or deletion at front means moving front backward or forward circularly.
  • Insertion or deletion at rear means moving rear forward or backward accordingly.

This approach offers O(1) average time complexity for all operations, but requires a predefined capacity or dynamic resizing strategies.

3.2 Using Doubly Linked Lists

A doubly linked list naturally supports insertion and deletion at both ends:

  • Each element (node) stores a reference to the previous and next node.
  • push_front inserts a new node before the current head node.
  • pop_back removes the tail node.

This approach guarantees O(1) time complexity per operation and is flexible for dynamicly sized deques. The tradeoff is slightly increased memory usage for pointers.

4. Step-by-Step Algorithmic Description of Key Operations

4.1 Operation push_front(x)

  1. Create a new node with value x.
  2. Set this node's next pointer to the current head node.
  3. If the deque is empty, set both head and tail pointers to this new node.
  4. Otherwise, update the previous pointer of the old head node to this new node.
  5. Update the head pointer to the new node.
  6. Increase the size count by 1.

4.2 Operation pop_back()

  1. If deque is empty, raise underflow error or return null.
  2. Retrieve the tail node's value.
  3. Update the tail pointer to the previous node of the current tail.
  4. If tail becomes null, set head to null as deque is now empty.
  5. Otherwise, set the new tail's next pointer to null.
  6. Decrease the size count by 1.
  7. Return the retrieved value.

4.3 Example: Using a Circular Buffer Array

Imagine an array size of 5, initially empty. You represent front and rear indices and wrap around when reaching the end (using modulo arithmetic): 

Index:  0   1   2   3   4
Array: [_,  _,  _,  _,  _]
front=0, rear=0 (empty)

push_back(10):
  arr[rear] = 10;
  rear = (rear + 1) % 5; // Rear = 1

push_front(20):
  front = (front - 1 + 5) % 5; // front = 4
  arr[front] = 20;

Elements are now at arr[4] = 20 and arr[0] = 10.

As elements are added or removed from either end, the indices move circularly through the array, ensuring full usage of space.

5. Real-World Examples for Deque Usage

5.1 Palindrome Checker

The palindrome checking algorithm can employ a deque such that characters are inserted at the back and compared from front and back until the deque is empty or a mismatch is found. 

Example:
Input: "radar"
Steps:
- Insert r, a, d, a, r into deque
- While deque size > 1:
    Compare front and back → both 'r'? yes
    pop_front and pop_back
    Next compare front and back → both 'a'? yes
    pop_front and pop_back
- The remaining element is 'd', confirms palindrome

5.2 Sliding Window Maximum Problem

This is a classic use of a deque:

  • You maintain indices of elements, ensuring the deque is sorted decreasingly by element value.
  • For each new element, you pop from the back all smaller elements.
  • The front always holds the index of the current window's maximum.

This gives an overall linear time algorithm (O(n)), since each element is pushed and popped at most once.
