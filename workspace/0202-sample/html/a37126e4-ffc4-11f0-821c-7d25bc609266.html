Prim’s Algorithm: A Comprehensive Guide

Prim’s Algorithm is a classic algorithm in graph theory used to find a Minimum Spanning Tree (MST) for a weighted, connected, undirected graph. Its primary utility is in network design, such as laying out electrical wiring, computer networks, or road maps, where the aim is to connect all points (nodes) together with the least total cost (weight).

What is a Minimum Spanning Tree?

Consider a connected, undirected graph where each edge has a weight or cost associated with it. A spanning tree is a subset of the graph’s edges that:

  • Connects all vertices together.
  • Contains no cycles (is a tree).

Among all possible spanning trees, a minimum spanning tree is the one whose total edge weight sum is the smallest.

“A Minimum Spanning Tree ensures every point is connected in the least costly way possible without forming loops.”

Why is the Minimum Spanning Tree Problem Important?

MST algorithms optimize resource usage in a number of practical applications:

  • Design of Communication Networks: Minimizing cable length between hubs.
  • Electrical Grid Layouts: Minimizing the cost of wiring.
  • Clustering Analysis: Organizing data points by proximity.
  • Approximation of Other Network Problems: Such as traveling salesman approximations.

Overview of Prim’s Algorithm

Prim’s Algorithm is a greedy algorithm. It builds the minimum spanning tree incrementally by:

  1. Starting from an arbitrary vertex.
  2. At each step, adding the shortest edge that connects a vertex already in the MST to a vertex outside it.
  3. Repeating until all vertices are included.

It differs from another popular MST algorithm, Kruskal’s Algorithm, which considers edges globally in order of weight and connects disjoint components.

Key Idea

Always add the lowest weight edge that expands the current tree without creating a cycle.

Step-by-Step Breakdown of Prim’s Algorithm

Assume we have a weighted graph G(V, E) with vertices V and edges E.

  1. Initialization: Choose an arbitrary vertex v0 to start the MST. Initialize two sets:
    • MST_Set = {v0} — contains vertices already in MST
    • Edge_Set = ∅ — stores edges included in MST
  2. Find the Minimum Weight Edge: Among all edges that have one endpoint in MST_Set and the other outside it, find the edge with the smallest weight.
  3. Add this edge and its vertex: Add the chosen edge to Edge_Set and include the new vertex in MST_Set.
  4. Repeat steps 2 and 3 until MST_Set contains all vertices.

Formal Pseudocode


Input: Connected, weighted graph G(V, E)
Output: Minimum Spanning Tree (MST) set of edges

1. Initialize an empty set MST_Set for vertices included in MST
2. Initialize empty set Edge_Set for edges included in MST
3. Choose arbitrary start vertex u and add it to MST_Set
4. While MST_Set does not contain all vertices:
     a. Among all edges with one vertex in MST_Set and the other not in MST_Set,
        pick the edge (u, v) with minimum weight
     b. Add vertex v to MST_Set
     c. Add edge (u, v) to Edge_Set
5. Return Edge_Set as MST

Example

Consider this graph with vertices and weighted edges:

Edge Weight
A - B4
A - H8
B - C8
B - H11

C - I2
D - E9
D - F14

G - I6

Let's build the MST starting from vertex A:

  1. Start: MST_Set = {A}
  2. Step 1: Edges from A are (A-B:4) and (A-H:8). Choose (A-B:4). MST_Set = {A,B}.
  3. Step 2: Next edges crossing MST boundary:
    • (B-C:8), (B-H:11), (A-H:8)
    Pick (A-H:8) or (B-C:8) — choose (B-C:8). MST_Set = {A,B,C}
  4. Step 3: Now consider edges from {A,B,C} outward:
    • (B-H:11), (C-D:7), (C-F:4), (C-I:2), (A-H:8)
    Pick the smallest: (C-I:2). MST_Set = {A,B,C,I}
  5. Step 4: Edges now:
    • (B-H:11), (C-D:7), (C-F:4), (I-G:6), (A-H:8)
    Pick (C-F:4). MST_Set = {A,B,C,I,F}
  6. Step 5: Edges:
    • (B-H:11), (F-G:2), (C-D:7), (D-E:9), (D-F:14), (A-H:8), (I-G:6)
    Pick (F-G:2). MST_Set = {A,B,C,I,F,G}
  7. Step 6: Edges:
    • (G-H:1), (B-H:11), (C-D:7), (D-E:9), (D-F:14), (A-H:8), (I-G:6)
    Pick (G-H:1). MST_Set = {A,B,C,I,F,G,H}
  8. Step 7: Edges:
    • (C-D:7), (D-E:9), (D-F:14), (B-H:11)
    Pick (C-D:7). MST_Set = {A,B,C,I,F,G,H,D}
  9. Step 8: Edges:
    • (D-E:9), (D-F:14), (B-H:11)
    Pick (D-E:9). MST_Set = {A,B,C,I,F,G,H,D,E}

At this point all vertices are included, and the MST edges sum to a minimal total weight.

Properties of Prim’s Algorithm

  • Greedy Method: Chooses local optimal edge at each step hoping for a global optimum.
  • Works only with connected graphs: A spanning tree must connect all vertices.
  • Edge weights must be non-negative: Typically positive weights are assumed.
  • Deterministic: Given the same graph and starting vertex, it produces the same MST.
  • Time Complexity: Varies by data structure used:
    • Using adjacency matrix and simple arrays: O(V²)
    • Using adjacency list and min-priority queue (binary heap): O(E log V)
    • Using Fibonacci heaps (advanced): O(E + V log V)

Algorithm Implementation Notes

To efficiently execute Prim’s Algorithm, the following data structures are commonly used:

  • Priority Queue (Min-Heap): Keeps track of edges crossing the MST boundary sorted by weight.
  • Array or Set for MST Membership: Quickly check if a vertex is already part of the MST.

The algorithm continuously updates the priority queue with edges from the newly included vertex.
