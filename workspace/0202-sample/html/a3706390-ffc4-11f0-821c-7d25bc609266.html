Understanding Binary Trees

Introduction

A binary tree is a fundamental data structure in computer science, widely used in various algorithms, data organization, and problem-solving scenarios. It is a hierarchical structure in which each node has at most two children, commonly referred to as the left child and the right child.

This simplicity enables binary trees to efficiently represent sorted data, hierarchical relationships, and facilitate fast search, insert, and delete operations in some tree variations like binary search trees.

Definition and Terminology

A binary tree can be formally defined as a finite set of nodes with the following properties:

  • There is a distinguished node called the root.
  • The remaining nodes are partitioned into two disjoint subsets, each of which is a binary tree called the left subtree and the right subtree of the root.
  • Each node contains data and references (or pointers) to its children (if any).
Key Terms:
  • Node: An element containing a value and pointers to child nodes.
  • Root: The top node of the tree without a parent.
  • Child: A node connected directly below another node.
  • Parent: A node directly connected above another node.
  • Leaf Node: A node with no children (end of a branch).
  • Subtree: A tree consisting of a node and its descendants.
  • Height/Depth: The length of the longest path from the node down to a leaf.

Structure Illustration

Below is a conceptual example of a binary tree:

         10
        /  \
       5    15
      / \     \
     3   7     20

Explanation:

  • 10 is the root.
  • 5 and 15 are children of 10.
  • 3 and 7 are children of 5, making 5 their parent.
  • 20 is the right child of 15, and 15 is its parent.
  • 3, 7, and 20 are leaf nodes (no children).

Binary Tree Properties

Some important theoretical properties of binary trees include:

  • The maximum number of nodes at level l is 2^(l - 1), where the root is at level 1.
  • The maximum number of nodes in a binary tree of height h is 2^h - 1.
  • A binary tree with n nodes has n + 1 null child pointers (empty children).
  • The depth of a node is the length of the path from the root to the node.

Types of Binary Trees

The term "binary tree" is a general structure. Various specific types arise by placing more constraints:

  • Full Binary Tree: Every node has either 0 or 2 children.
  • Complete Binary Tree: All levels, except possibly the last, are completely filled, and all nodes in the last level are as far left as possible.
  • Perfect Binary Tree: A full binary tree in which all leaves are at the same level.
  • Balanced Binary Tree: Binary tree where height differences of left and right subtrees of any node are small (specific criteria vary).
  • Binary Search Tree (BST): A binary tree where the left subtree of a node contains only nodes with keys less than the node’s key, and right subtree only nodes with keys greater than the node’s key.

Binary Tree Traversals

Traversal refers to visiting all the nodes in a binary tree in some order. The main types of depth-first traversals are:

  1. Inorder Traversal (Left, Root, Right):

    Visit the left subtree, then the root, then the right subtree.

    For a binary search tree, inorder traversal visits nodes in ascending order.

  2. Preorder Traversal (Root, Left, Right):

    Visit the root first, then the left subtree, followed by the right subtree.

    This traversal is often used to copy or serialize the tree.

  3. Postorder Traversal (Left, Right, Root):

    Visit the left subtree, then the right subtree, and finally the root.

    This is useful when deleting or freeing the tree from leaf-to-root.

Example Tree: 
         1
        / \
       2   3
      / \
     4   5

Traversals produce:

  • Inorder: 4, 2, 5, 1, 3
  • Preorder: 1, 2, 4, 5, 3
  • Postorder: 4, 5, 2, 3, 1

Algorithms for Common Operations

1. Searching (in a Binary Search Tree)

The binary search tree property allows efficient searching for a value:

  1. Start at the root.
  2. If the current node’s value is equal to the target, return success.
  3. If the target is less than the current node’s value, go to the left child subtree.
  4. If greater, go to the right child subtree.
  5. If a null pointer is reached, the target is not present.

2. Insertion (in a Binary Search Tree)

  1. Search for the appropriate null child pointer where the new node belongs, following the search logic.
  2. Create a new node and link it at that position.
  3. No re-balancing happens in a basic BST insertion (may cause tree imbalance).

3. Deletion (in a Binary Search Tree)

Deletion depends on node’s children:

  • No child (Leaf node): Simply remove the node.
  • One child: Replace the node with its child.
  • Two children:
    1. Find the minimum value node in the right subtree (inorder successor) or maximum in left subtree (inorder predecessor).
    2. Replace the node’s value with that minimum/maximum node.
    3. Delete the successor/predecessor node from the subtree.

4. Computing Height of a Binary Tree

Height is the maximum depth from root to any leaf node.

Algorithm (recursive):

function height(node) {
  if (node == null) 
    return 0;
  else {
    leftHeight = height(node.left);
    rightHeight = height(node.right);
    return 1 + max(leftHeight, rightHeight);
  }
}

Use Cases and Applications

Binary trees have a wide variety of applications:

  • Expression Trees: Used to represent mathematical expressions with operators and operands.
  • Hierarchical Data: File systems, organization charts, and XML/HTML DOM parsing.
  • Searching and Sorting: Binary Search Trees, AVL trees, and Red-Black trees improve search/insertion times.
  • Priority Queues: Implemented using Binary Heap, a special case of binary tree.
  • Decision Trees: Used in machine learning for classification and regression.
