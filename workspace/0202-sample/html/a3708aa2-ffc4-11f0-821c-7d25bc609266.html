Understanding Graphs: Directed and Undirected

Graph theory is a fundamental area of discrete mathematics and computer science. It provides a powerful abstraction for modeling pairwise relationships between objects. This document offers a comprehensive textual explanation of graphs—specifically Directed and Undirected graphs—along with descriptions of their characteristics, common representations, and key algorithms.

1. What is a Graph?

Formally, a graph is a collection of points, called vertices (or nodes), connected by lines called edges. The edges represent relationships or connections between vertices.

Definition: A graph G is an ordered pair G = (V, E), where:
  • V is a finite set of vertices.
  • E is a set of edges, which are pairs of vertices.

Depending on how these edges are structured and interpreted, graphs can be broadly categorized into two types:

  • Undirected Graphs – edges have no direction. They simply connect two vertices.
  • Directed Graphs (or digraphs) – edges have a direction, from one vertex to another.

2. Undirected Graphs

In an undirected graph, edges represent bidirectional relationships. That means if vertex u is connected to vertex v, then v is also connected to u.

Formal Definition: For an undirected graph G = (V, E), each edge in E is an unordered pair of vertices {u, v}, where u, v ∈ V.

Example:

Vertices: V = {A, B, C, D}
Edges: E = { {A, B}, {A, C}, {B, D} }

This means:

  • There is an edge connecting A and B.
  • There is an edge connecting A and C.
  • There is an edge connecting B and D.

Since the graph is undirected, {A, B} means the same as {B, A}.

Applications of Undirected Graphs:

  • Modeling social networks where friendship is mutual.
  • Representing road maps where roads allow two-way traffic.
  • Modeling trees, such as file directory structures.

Undirected Graph Properties:

  • Degree of a vertex: The number of edges incident on it.
  • Connectedness: Whether every vertex is reachable from every other.
  • Cycles: Paths that start and end at the same vertex without repeating edges/vertices.

3. Directed Graphs (Digraphs)

In a directed graph, edges have a direction associated with them. Each edge is represented as an ordered pair (u, v), indicating an edge from vertex u (source) to vertex v (target).

Formal Definition: For a directed graph G = (V, E), each edge in E is an ordered pair (u, v), where u, v ∈ V.

Example:

Vertices: V = {1, 2, 3}
Edges: E = { (1, 2), (2, 3), (3, 1) }

Interpretation:

  • An edge from vertex 1 to vertex 2.
  • An edge from vertex 2 to vertex 3.
  • An edge from vertex 3 to vertex 1.

This creates a cycle: 1 → 2 → 3 → 1.

Applications of Directed Graphs:

  • Representing web page links (hyperlinks from one page to another).
  • Modeling workflows and task dependencies.
  • Representing one-way streets in traffic maps.

Directed Graph Properties:

  • In-degree: Number of edges arriving at a vertex.
  • Out-degree: Number of edges leaving a vertex.
  • Strong connectivity: Every vertex reachable from every other via directed paths.
  • Cycles: Directed cycles represent sequences of edges following arrows that start and end at the same vertex.

4. Common Graph Representations

Efficiently representing graphs in a computer program is vital for performing algorithms. Two common methods are:

4.1 Adjacency Matrix

An adjacency matrix is a 2D array (or matrix) of size |V| × |V| where each element indicates presence or absence of edges between vertices.

  • For undirected graphs, the matrix is symmetric: matrix[u][v] = matrix[v][u].
  • For directed graphs, matrix[u][v] is 1 if there is an edge from vertex u to v, else 0.

  • Fast edge lookup: checking if an edge exists is O(1).
  • Simple and easy to implement.

Disadvantages:

  • Consumes O(|V|²) space, inefficient for sparse graphs.

4.2 Adjacency List

An adjacency list stores, for each vertex, a list (or array) of adjacent vertices. This is more space-efficient for sparse graphs.

  • 0: [1]
  • 1: [2]

  • Efficient use of space: O(|V| + |E|).
  • Easy to iterate over neighbors of a vertex.

Disadvantages:

  • Checking edge existence can be slower: O(k) where k is the number of adjacent edges.

5. Important Graph Algorithms (Textual Description)

Many algorithms operate on graphs. Here are some fundamental ones with step-by-step textual descriptions.

5.1 Depth-First Search (DFS)

DFS explores as far as possible along each branch before backtracking.

Step 1: Start at a chosen vertex and mark it as visited.
Step 2: For each neighbor of the current vertex, if it is unvisited, recursively perform DFS on it.
Step 3: Backtrack when no unvisited neighbors remain.
Step 4: DFS finishes when all reachable vertices from the start are visited.

Uses: Finding connected components, detecting cycles, topological sort (in DAGs), pathfinding.

5.2 Breadth-First Search (BFS)

BFS explores all neighbors at the current depth before going deeper.

Step 1: Start at a chosen vertex, mark it visited, and insert it into a queue.
Step 2: While the queue is not empty:
    a) Dequeue a vertex.
    b) For each unvisited neighbor, mark it visited and enqueue it.
Step 3: BFS finishes when the queue is empty.

Uses: Finding shortest paths (in unweighted graphs), connectivity testing.

5.3 Dijkstra's Algorithm (for shortest path in weighted graphs)

Dijkstra's algorithm finds the shortest path from a start vertex to all others in a graph with non-negative edge weights.

Step 1: Assign to every vertex a tentative distance value: 0 for the start vertex, infinity for others.
Step 2: Set all vertices as unvisited and create a set of all unvisited vertices.
Step 3: While unvisited vertices remain:
    a) Select the unvisited vertex with the smallest tentative distance.
    b) For the current vertex, consider all its unvisited neighbors and calculate their tentative distances via the current vertex.
    c) If a new tentative distance is smaller, update it.
    d) Mark the current vertex as visited.
Step 4: Repeat until all vertices are visited.

Note: This algorithm requires weighted edges and no negative weights to function correctly.

6. Additional Concepts

6.1 Weighted vs Unweighted Graphs

Edges in graphs may have an associated weight, usually representing cost, distance, or capacity:

  • Unweighted graph: Edges have no weight or all weights are implicitly equal.
  • Weighted graph: Each edge carries a numeric value.

6.2 Cycles and Acyclic Graphs

A cycle is a path of edges and vertices wherein a vertex is reachable from itself by a non-trivial path.

  • Cyclic graph: Contains at least one cycle.
  • Acyclic graph: Contains no cycles.

Directed Acyclic Graphs (DAGs) are important in computer science for modeling dependencies (e.g., task scheduling).

6.3 Connectivity in Graphs

  • Undirected Graph Connectivity: A graph is connected if there exists a path between every pair of vertices.
  • Strong connectivity in Directed Graphs: A directed graph is strongly connected if there is a directed path from any vertex to any other vertex.
