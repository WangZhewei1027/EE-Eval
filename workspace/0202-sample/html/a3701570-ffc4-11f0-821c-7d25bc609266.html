Understanding Stacks: Concept, Theory, and Algorithms

In computer science, a stack is one of the most fundamental and important abstract data types. It represents a collection of elements with two principal operations: pushing (adding an element to the collection) and popping (removing the most recently added element). The stack follows a strict ordering principle: the Last-In, First-Out (LIFO) method. This guide aims to provide a comprehensive explanation of stacks, covering their theoretical foundations, typical use cases, step-by-step descriptions of core operations, and algorithms.

1. What Is a Stack?

A stack is an abstract data type that models a collection where elements are added and removed according to the LIFO principle. You can think of a stack as a vertical pile of objects where you can only:

  • Add a new object on the top
  • Remove the object from the top

This is just like stacking a pile of plates — you can only take or add the top plate, and not the ones underneath without removing those first.

Key Terminology:
- Push: Add an element to the top of the stack.
- Pop: Remove the element currently at the top.
- Top or Peek: Inspect (without removing) the element at the top.
- Empty Stack: When no elements are stored.

2. Theoretical Foundations of Stacks

A stack is a linear data structure typically implemented using arrays or linked lists. It maintains the order of insertion strictly via the LIFO discipline. The precise behavior is defined by:

  • Axiom 1 (Push): Pushing places the new element on the top of the stack.
  • Axiom 2 (Pop): Popping removes the current top and exposes the previously inserted element.
  • Axiom 3 (Peek): Peeking allows viewing the top element without changing the stack.

Because the stack operates only at one end (the top), it does not allow access to elements beneath until the top elements are popped. This restrictive access pattern makes stacks simpler but very powerful for certain algorithms and system functions.

2.1 Mathematical Definition

Formally, a stack S is an ordered collection that supports two operations:

  • push(x): S := S ∪ {x} where x is added at the end of S.
  • pop(): removes and returns the last element added to S.

If S = [a, b, c], with a being the bottom and c the top, performing pop() yields: c and the stack becomes [a, b].

3. Fundamental Stack Operations Explained Step-by-Step

3.1 Push Operation

The push operation inserts a new element onto the top of the stack.
Step-by-step description:

  1. Check if the stack has capacity (if implemented using an array with fixed size).
  2. If space is available, place the new element at the position above the current top.
  3. Update the pointer/reference marking the current top to the newly added element.

After the push, the new element becomes the top of the stack.

3.2 Pop Operation

The pop operation removes the element at the top from the stack.
Step-by-step description:

  1. Verify the stack is not empty to avoid underflow errors.
  2. Retrieve the element at the top position.
  3. Remove it by updating the top pointer to the next element down.
  4. Return the retrieved element to the caller.

If the stack is empty, popping is not possible and usually raises an error or returns a sentinel value.

3.3 Peek (Top) Operation

The peek or top operation allows inspecting the element at the top without removing it.
Step-by-step description:

  1. Check that the stack is not empty.
  2. Return the current top element without modifying the stack.
Note about Stack Overflow and Underflow:
  • Stack Overflow: Happens if pushing is attempted on a full stack (in fixed size implementations).
  • Stack Underflow: Happens if popping is attempted on an empty stack.

4. Examples to Illustrate Stack Behavior

Imagine a stack of numbers and these operations: 

Initial Stack (empty): []

Operation: push(5)
Stack: [5]

Operation: push(10)
Stack: [5, 10]  (10 is on top)

Operation: peek()
Returns: 10
Stack remains: [5, 10]

Operation: pop()
Returns: 10
Stack now: [5]

Operation: pop()
Returns: 5
Stack now: []

Operation: pop()
Error: Stack is empty (Cannot pop)

4.1 Real-World Analogies

  • Stack of trays: You place trays on top and remove only the top tray.
  • Undo feature in text editors: Each change is pushed; undo pops the last change.
  • Backtracking algorithms: Decisions are stacked and popped when backtracking.

5. Common Implementation Strategies

Stacks can be implemented mainly using:

  • Arrays: Fixed size or dynamically resizable arrays keep elements sequentially, with a variable to track the top element index.
  • Linked lists: Each node stores a value and a reference to the next, with the top always pointing to the head node.

Array Implementation Example (Conceptual): 

Stack: array[size]
topIndex = -1  // initially empty

function push(value):
    if topIndex == size - 1:
        error "Stack Overflow"
    else
        topIndex = topIndex + 1
        array[topIndex] = value

function pop():
    if topIndex == -1:
        error "Stack Underflow"
    else
        val = array[topIndex]
        topIndex = topIndex - 1
        return val

function peek():
    if topIndex == -1:
        error "Stack is empty"
    else
        return array[topIndex]

Linked List Implementation Example (Conceptual): 

Node structure:
    value
    next

top = null  // initially empty

function push(value):
    newNode = Node(value)
    newNode.next = top
    top = newNode

function pop():
    if top == null:
        error "Stack Underflow"
    else
        val = top.value
        top = top.next
        return val

function peek():
    if top == null:
        error "Stack is empty"
    else
        return top.value

6. Applications of Stacks in Computer Science

Stacks are widely used behind the scenes in various algorithms and systems:

  • Expression evaluation and syntax parsing: For example, evaluating arithmetic expressions in postfix notation.
  • Function call management: The call stack keeps track of active subroutines and local variables during program execution.
  • Depth-First Search (DFS): Graph traversal algorithms use stacks to explore nodes.
  • Undo mechanisms: Applications record actions in a stack to enable undoing them.
  • Balancing symbols and syntax checking: Matching parentheses, brackets, and braces involves stacks.

6.1 Example: Checking Balanced Parentheses Using a Stack

A common use for stacks is verifying whether an expression has balanced parentheses. For example, in ( [ ] { } ), every opening symbol must be properly closed.

Algorithm (step-by-step):

  1. Initialize an empty stack.
  2. Scan the expression from left to right.
  3. When an opening symbol ((, [, {) is encountered, push it onto the stack.
  4. When a closing symbol (), ], }) is encountered:
    • If the stack is empty, it means no matching opening symbol → unbalanced.
    • Otherwise, pop the top of the stack and check if it matches the closing symbol type.
    • If they do not correspond, the expression is unbalanced.
  5. After finishing scanning, if the stack is not empty, then some opening symbols have not been closed → unbalanced.
  6. If none of the above errors occurred, the expression is balanced.

8. Summary

The stack data structure is a simple yet powerful tool that relies on the Last-In, First-Out (LIFO) principle. It operates through the fundamental operations of push, pop, and peek. Due to its controlled access and predictable behavior, stacks play a critical role in computer science — from managing function calls in programming languages to evaluating expressions and traversing structures.

Mastery of stacks and their properties is foundational knowledge for anyone delving deeper into data structures and algorithms.
