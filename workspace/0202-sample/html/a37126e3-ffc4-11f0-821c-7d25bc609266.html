Kruskal’s Algorithm

Kruskal’s Algorithm is a fundamental algorithm in computer science and graph theory used to find a Minimum Spanning Tree (MST) of a weighted, undirected graph. An MST is a subset of edges that connects all the vertices together without any cycles and with the minimum possible total edge weight.

What Is a Minimum Spanning Tree?

Consider a graph defined by:

  • Vertices (nodes) representing points or locations.
  • Edges representing connections between the nodes.
  • Weights, a numerical value assigned to each edge, often representing cost, distance, or time.

A spanning tree is a subgraph that includes all vertices with the minimum number of edges needed to keep the graph connected, without forming any cycles. For a graph with V vertices, a spanning tree always has V-1 edges.

Among all spanning trees, a minimum spanning tree has the least total weight when summing all edge weights.

Note: MSTs have broad applications in network design (e.g., computer, telecommunication networks), clustering algorithms, and approximate solutions to optimization problems.

Background & Historical Context

Kruskal’s algorithm was devised by Joseph B. Kruskal Jr. in 1956 and is recognized as one of the earliest and simplest greedy algorithms for MST problems.

It belongs to the family of greedy algorithms — methods that make locally optimal choices at each step, hoping that these choices lead to a globally optimal solution.

Preconditions

Kruskal’s algorithm assumes:

  • The graph is undirected.
  • Edges have well-defined weights (non-negative values are typical, but negative weights can also be handled).
  • The graph is connected or at least we want to find a spanning forest if the graph is disconnected.

Intuition Behind Kruskal’s Algorithm

Imagine you have a network of cities connected by roads, each road having a toll cost. You want to connect all cities with roads so that total toll cost is minimized, and there is a path between every pair of cities.

Kruskal’s approach is:

  • Start with no roads selected.
  • Pick the road with the smallest toll.
  • Keep adding the smallest toll roads that don't form loops until all cities are connected.

This approach ensures cost efficiency and avoids cycles.

Detailed Step-by-Step Algorithm Description

  1. Sort all the edges in non-decreasing order of their weight.
    Arrange edges from the smallest to the largest weight.
  2. Initialize a forest: treat each vertex as a separate tree (initially, no edges are selected).
  3. Iterate over the sorted edges:
    • For each edge, check whether the ends belong to different trees.
    • If they are in different trees, select this edge and merge the trees into a single tree.
    • If they are in the same tree, ignore the edge to avoid cycles.
  4. Repeat until the spanning tree is complete: The MST will have V-1 edges where V is the number of vertices.

Core Principle: Avoiding Cycles

Detecting whether adding an edge creates a cycle is crucial. To efficiently perform these checks, Kruskal’s algorithm commonly uses a Disjoint Set Union (DSU) or Union-Find data structure:

  • Find operation: Determines which subset a particular element is in.
  • Union operation: Merges two subsets into a single subset.

With Union-Find, the algorithm queries if two vertices belong to the same set (tree). If yes, adding the edge would create a cycle; if no, the sets are united using the edge.

Pseudocode of Kruskal’s Algorithm


function Kruskal(graph):
  MST = empty set
  create sets for each vertex (Disjoint Sets)

  sort all edges of graph by ascending weight

  for each edge (u, v) in sorted edges:
    if findSet(u) ≠ findSet(v):    // no cycle formed
      MST.add((u, v))
      unionSet(u, v)

  return MST

Example Walkthrough

Consider a graph with vertices A, B, C, D, E and weighted edges:

  • (A, B) = 1
  • (B, C) = 4
  • (A, C) = 3
  • (C, D) = 2
  • (D, E) = 5
  • (B, E) = 7

Step 1: Sort edges by weight:

  1. AB(1), CD(2), AC(3), BC(4), DE(5), BE(7)

Step 2: Initialize sets: {A}, {B}, {C}, {D}, {E}

Step 3: Iterate through edges:

  • (A,B)=1 connects {A} and {B} → add edge, sets merged: {A,B}
  • (C,D)=2 connects {C} and {D} → add edge, sets merged: {C,D}
  • (A,C)=3 connects {A,B} and {C,D} → add edge, sets merged: {A,B,C,D}
  • (B,C)=4 connects vertices in same set {A,B,C,D} → skip (would create cycle)
  • (D,E)=5 connects {A,B,C,D} and {E} → add edge, all vertices connected
  • (B,E)=7 skip (all vertices already connected)

Resulting MST edges: (A,B), (C,D), (A,C), (D,E)

Why Kruskal’s Algorithm Works (Correctness)

Kruskal’s makes locally optimal choices by including the smallest edge that connects two different components. By doing this repeatedly, it ensures the final tree is minimal. The algorithm adheres to the greedy choice property and exhibits optimal substructure.

Furthermore, the algorithm avoids cycles by design (with Union-Find), ensuring the subset remains a forest until it becomes a spanning tree.

Time Complexity

The runtime primarily depends on two factors:

  • Sorting edges: Sorting all E edges takes O(E log E) time.
  • Union-Find operations: Performing E union/find operations can be done in near-constant amortized time O(α(V)), where α is the inverse Ackermann function, an extremely slow-growing function, practically constant.

Overall complexity: O(E log E), which is efficient especially for sparse graphs.

Comparison with Other MST Algorithms

  • Prim’s Algorithm: Also finds MST but incrementally grows a single tree starting from one vertex by selecting the minimal edge connecting the tree to a new vertex.
  • Kruskal’s Algorithm: Grows a forest by adding edges in ascending order, connecting disjoint trees until the MST is complete.

Generally, Kruskal's is simpler to implement and can be faster for sparse graphs. Prim's is often preferred for dense graphs.

Practical Applications

  • Network design: Minimizing cable lengths or costs in telecommunication, computer networks, or electrical grids.
  • Clustering algorithms: MSTs can form the basis of hierarchical clustering.
  • Approximation algorithms: Useful in NP-hard problems like the Traveling Salesman Problem (as part of heuristics).
