Breadth-First Search (BFS): A Detailed Explanation

Breadth-First Search (BFS) is one of the fundamental graph traversal algorithms widely used in computer science and various applications such as networking, pathfinding, and artificial intelligence. It systematically explores the vertices (or nodes) of a graph level by level, ensuring that all nodes at one “distance” from the start are explored before moving on to nodes further away.

What is Breadth-First Search?

BFS is an algorithm for traversing or searching tree or graph data structures. Starting from a chosen source node, it explores all of the neighbor nodes at the present depth prior to moving on to nodes at the next depth level.

This approach guarantees the shortest path from the starting node to all other reachable nodes in an unweighted graph.

Key Characteristics of BFS

  • Level-Order Traversal: BFS visits nodes in layers or levels, starting with the given start node.
  • Queue-Based Implementation: BFS uses a queue data structure to keep track of nodes to visit next.
  • Shortest Path: In unweighted graphs, BFS produces the shortest path to all reachable nodes.
  • Time Complexity: O(V + E), where V is vertices and E is edges.
  • Space Complexity: O(V), to store the queue and visited nodes.

Terminology and Definitions

  • Graph: A set of nodes (vertices) connected by edges. Graphs can be directed or undirected.
  • Node (Vertex): A fundamental unit of a graph.
  • Edge: A connection from one node to another.
  • Neighbors: Nodes directly connected to a given node by an edge.
  • Queue: A First-In-First-Out (FIFO) data structure used to track nodes to visit next.
  • Visited: Nodes that have been examined to avoid repeated processing and infinite cycles.

How BFS Works: Conceptual Overview

  1. Initialization: Start by marking the starting node as visited and enqueue it.
  2. Exploration: Dequeue the front node from the queue and examine all of its neighbors.
  3. Visiting Neighbors: For each neighbor, if it has not yet been visited, mark it as visited and enqueue it.
  4. Repeat: Continue dequeuing nodes from the queue and visiting their neighbors until the queue is empty.

This structure ensures that nodes are visited in order of their distance from the starting node, i.e., all nodes at distance 1 are visited before nodes at distance 2, and so on.

Step-by-Step Example

Consider the following undirected graph:

Graph Description:
Nodes: A, B, C, D, E, F
Edges: A–B, A–C, B–D, C–E, D–F, E–F

We want to perform BFS starting from node A.

  1. Start by visiting A. Enqueue A.
    Queue: [A]
    Visited: {A}
  2. Dequeue A. Explore neighbors B and C. Enqueue both and mark visited.
    Queue: [B, C]
    Visited: {A, B, C}
  3. Dequeue B. Explore neighbors: A (already visited), D (new). Enqueue D.
    Queue: [C, D]
    Visited: {A, B, C, D}
  4. Dequeue C. Explore neighbors: A (visited), E (new). Enqueue E.
    Queue: [D, E]
    Visited: {A, B, C, D, E}
  5. Dequeue D. Explore neighbors: B (visited), F (new). Enqueue F.
    Queue: [E, F]
    Visited: {A, B, C, D, E, F}
  6. Dequeue E. Explore neighbors: C (visited), F (visited). No new enqueue.
    Queue: [F]
    Visited: {A, B, C, D, E, F}
  7. Dequeue F. Explore neighbors: D, E both visited. No enqueue.
    Queue: []
    Visited: {A, B, C, D, E, F}

BFS completes when the queue is empty, having visited all nodes reachable from A.

Algorithm Pseudocode

function BFS(Graph, startNode):
    create a queue Q
    mark startNode as visited
    enqueue startNode onto Q

    while Q is not empty:
        currentNode = dequeue from Q
        process currentNode (e.g., print it or record it)

        for each neighbor n of currentNode in Graph:
            if n has not been visited:
                mark n as visited
                enqueue n onto Q

Important Notes

  • Visited Marking: It's essential to mark nodes visited as soon as they are enqueued to prevent duplicates in the queue.
  • Graph Representations: BFS can work equally well on adjacency lists or adjacency matrices, but adjacency lists are generally more efficient for sparse graphs.
  • Directed vs Undirected: BFS works similarly on both, but consider edge direction when exploring neighbors in directed graphs.
  • Applications:
    • Finding shortest path in an unweighted graph
    • Checking connectivity
    • Peer-to-peer networking (e.g., finding people in social networks)
    • Web crawling
    • Level order traversal of trees
    • Solving puzzles and games by exploring legal moves
