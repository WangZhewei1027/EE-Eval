Understanding the Bellman-Ford Algorithm

The Bellman-Ford algorithm is a fundamental algorithm in graph theory used primarily to find the shortest paths from a single source vertex to all other vertices in a weighted graph. Unlike algorithms such as Dijkstra’s, the Bellman-Ford algorithm can handle graphs that contain negative edge weights, making it a powerful tool in scenarios where costs or distances may decrease.

Background and Motivation

Shortest path problems are central to computer science and many application domains such as network routing, urban planning, and transportation logistics.

The classical Dijkstra’s algorithm efficiently finds shortest paths but only works correctly when all edge weights are non-negative. However, many real-world scenarios can have negative edge weights, for example:

  • Currency exchange with arbitrage opportunities.
  • Some cost reduction computations in scheduling.
  • Graphs modeling debts or penalties.

In such cases, Bellman-Ford is often the preferred algorithm because of its ability to handle negative edges and detect negative weight cycles.

What Does the Algorithm Do?

Given a graph G = (V, E) with vertices V and edges E, along with a source vertex s, the Bellman-Ford algorithm computes shortest distances from s to every other vertex.

It also determines whether the graph contains any negative weight cycles reachable from the source. A negative weight cycle is a cycle whose sum of edge weights is negative, causing shortest paths to be undefined (because looping around the cycle reduces path length indefinitely).

Algorithm Overview

The Bellman-Ford algorithm proceeds as follows:

  1. Initialization: Set the distance to the source vertex s as 0, and to all other vertices as ∞ (infinity), representing unknown distances initially.
  2. Relaxation: For each edge (u, v) with weight w, update the distance to v if going through u offers a shorter path.
  3. Repeat this relaxation step |V| - 1 times, where |V| is the number of vertices in the graph.
  4. Check for negative-weight cycles: Perform one more relaxation step over all edges. If any distance is updated, it indicates a negative-weight cycle exists.

Formal Pseudocode

function BellmanFord(G, w, s):
    # G = graph, w = edge weights dictionary (or function), s = source vertex
    initialize distance:
        for each vertex v in G:
            dist[v] ← ∞
            predecessor[v] ← null
        dist[s] ← 0
  
    # Relax edges repeatedly
    for i from 1 to |V| - 1:
        for each edge (u, v) in E:
            if dist[u] + w(u, v) < dist[v]:
                dist[v] ← dist[u] + w(u, v)
                predecessor[v] ← u

    # Check for negative-weight cycles
    for each edge (u, v) in E:
        if dist[u] + w(u, v) < dist[v]:
            return "Graph contains a negative-weight cycle"

    return dist, predecessor

Explanation of Steps

  • Distances Initialization: All distances start at infinity except the source vertex, which has distance zero because the shortest path from itself is zero by definition.
  • Relaxation: Relaxing an edge means trying to improve the currently known shortest distance to the destination vertex via that edge. If you find a shorter distance, update it.
  • Repeating |V|-1 times: The reasoning behind |V|-1 iterations is that the longest possible path without cycles between any two vertices involves at most |V|-1 edges. Repeating the relaxation process enough times ensures all such shortest paths can be found.
  • Negative Cycle Check: If after all these relaxations another improvement is possible, it means a cycle exists where you can reduce the distance indefinitely — a negative-weight cycle.

Computational Complexity

Its time complexity is:

  • O(V × E), where V is the number of vertices and E is the number of edges.

This is less efficient than Dijkstra’s algorithm for graphs without negative edges, but it is more general.

Example Walkthrough

Consider the following directed graph (edges represented as (u → v, w)):

  • (A → B, 4)
  • (A → C, 2)
  • (B → C, 3)
  • (B → D, 2)
  • (B → E, 3)
  • (C → B, 1)
  • (C → D, 4)
  • (C → E, 5)
  • (E → D, -5) (negative edge)

We want the shortest paths from vertex A to all other vertices.

Step-by-step reasoning

  1. Initialize distances: dist[A] = 0; dist[B] = ∞; dist[C] = ∞; dist[D] = ∞; dist[E] = ∞.
  2. 1st iteration of relaxation:
    • Relax (A→B,4): dist[B] = min(∞, 0 + 4) = 4
    • Relax (A→C,2): dist[C] = min(∞, 0 + 2) = 2
    • Relax (B→C,3): dist[C] = min(2, 4 + 3) = 2 (unchanged)
    • Relax (B→D,2): dist[D] = min(∞, 4 + 2) = 6
    • Relax (B→E,3): dist[E] = min(∞,4 + 3) = 7
    • Relax (C→B,1): dist[B] = min(4, 2 + 1) = 3
    • Relax (C→D,4): dist[D] = min(6, 2 + 4) = 6 (unchanged)
    • Relax (C→E,5): dist[E] = min(7, 2 + 5) = 7 (unchanged)
    • Relax (E→D,-5): dist[D] = min(6, 7 + (-5)) = 2
  3. 2nd iteration: Repeat relaxation steps with updated distances:
    • Relax (A→B,4): dist[B] = min(3, 0 + 4) = 3 (unchanged)
    • Relax (A→C,2): dist[C] = 2 (unchanged)
    • Relax (B→C,3): dist[C] = min(2, 3 + 3) = 2 (unchanged)
    • Relax (B→D,2): dist[D] = min(2, 3 + 2) = 2 (unchanged)
    • Relax (B→E,3): dist[E] = min(7, 3 + 3) = 6
    • Relax (C→B,1): dist[B] = min(3, 2 + 1) = 3 (unchanged)
    • Relax (C→D,4): dist[D] = 2 (unchanged)
    • Relax (C→E,5): dist[E] = min(6, 2 + 5) = 6 (unchanged)
    • Relax (E→D,-5): dist[D] = min(2, 6 - 5) = 1
  4. 3rd iteration: Continuing:
    • Relaxing edges again reduces dist[D] further to 0 via E→D edge.
    • Subsequent relaxations will not reduce distances anymore.

After |V|-1 = 4 iterations, the shortest distances stabilize:

  • dist[A] = 0
  • dist[B] = 3
  • dist[C] = 2
  • dist[D] = 0
  • dist[E] = 6

Detecting Negative Cycles

If during the final cycle of edge relaxations any distance value decreases, then the graph contains a negative weight cycle reachable from the source. For example, if you had an edge forming part of a cycle that sums to negative weight, the distances would keep decreasing every iteration.

Note: Bellman-Ford can detect negative weight cycles, but cannot produce meaningful shortest paths in their presence since such paths are not well-defined.

Summary

  • Bellman-Ford finds shortest paths from one source vertex to all other vertices in graphs with negative edge weights.
  • It works by iteratively relaxing edges up to |V| - 1 times.
  • It detects negative weight cycles reachable from the source.
  • Its worst-case runtime is O(V × E), less efficient than Dijkstra's in graphs without negative edges.
