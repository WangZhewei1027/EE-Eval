Counting Sort: A Thorough Explanation and Educational Guide

Counting Sort is an integer sorting algorithm that operates by counting the number of occurrences of each distinct element in the input list. Unlike comparison-based sorting algorithms (like QuickSort, MergeSort, or HeapSort), Counting Sort leverages the numeric nature and bounded range of input data to achieve a very efficient runtime in certain scenarios.

1. What is Counting Sort?

Counting Sort is a non-comparative sorting algorithm that sorts elements by counting the frequency of each unique element in the array. Using these frequencies, it calculates the positions of each element in the sorted output.

It is especially effective when:

  • The range of input data (k) is not significantly larger than the number of elements (n).
  • The input elements are integers (or can be mapped to integers) within a known, fixed range.

Because it relies on counting rather than element-to-element comparison, Counting Sort can run in O(n + k) time, which is linear relative to the input size and the range of data.

2. Conceptual Overview

Suppose you have a list of integers to sort, and you know the minimum and maximum values in that list. Counting Sort proceeds as follows:

  1. Determine the range of input values, from a minimum to a maximum.
  2. Create a "count array" that can track how many times each value within the range occurs.
  3. Count the occurrences of each element by traversing the input array.
  4. Modify the count array so that each element at index i contains the sum of counts up to i — this step determines the position of each element in the output array.
  5. Build the output sorted array by iterating over the input in reverse order, placing each element at its correct position, and decreasing the count accordingly.
  6. Copy the sorted elements back to the original array (if needed).

Why iterate backwards when building the output?

This ensures stability of the sorting algorithm. Stability means that if two elements have the same value, the one appearing earlier in the input array will appear earlier in the sorted array as well.

3. Detailed Step-By-Step Description

Let's say you want to sort the array:

arr = [4, 2, 2, 8, 3, 3, 1]

  1. Find minimum and maximum values:
    • min = 1
    • max = 8
  2. Create count array: Since max = 8 and min = 1, the size will be max - min + 1 = 8. Initialize all to 0.
    3. count = [0, 0, 0, 0, 0, 0, 0, 0]  // Index 0 corresponds to value 1
  3. Count each element in the original array:
    • For 4 → count[4-1] += 1 → count[3] = 1
    • For 2 → count[2-1] += 1 → count[1] = 1
    • Next 2 → count[1] = 2
    • For 8 → count[7] = 1
    • For 3 → count[2] = 1
    • Next 3 → count[2] = 2
    • For 1 → count[0] = 1

    The count array after counting all elements:

    count = [1, 2, 2, 1, 0, 0, 0, 1]

    Indices correspond to values from 1 to 8.

  4. Modify the count array by cumulative sum: Transforms count[i] into the position of the last occurrence of value i + min in the output array. 
    i:     0  1  2  3  4  5  6  7
count:  1, 2, 2, 1, 0, 0, 0, 1

Cumulative sum:
count[0] = 1
count[1] = 1 + 2 = 3
count[2] = 3 + 2 = 5
count[3] = 5 + 1 = 6
count[4] = 6 + 0 = 6
count[5] = 6 + 0 = 6
count[6] = 6 + 0 = 6
count[7] = 6 + 1 = 7

Final count:
[1, 3, 5, 6, 6, 6, 6, 7]
  5. Build the output array by placing elements: Iterate the original array backward:
    • Start with last element = 1 (index 6), value = 1 → position = count[0] = 1 Place 1 at index 0 in output (1-1 because zero based indexing), decrement count[0] → 0
    • Next element = 3, count[2] = 5 → place at index 4, decrement to 4
    • Next element = 3, count[2] = 4 → place at index 3, decrement to 3
    • Next element = 8, count[7] = 7 → place at index 6, decrement to 6
    • Next element = 2, count[1] = 3 → place at index 2, decrement to 2
    • Next element = 2, count[1] = 2 → place at index 1, decrement to 1
    • Next element = 4, count[3] = 6 → place at index 5, decrement to 5

    Output array after placement:

    [1, 2, 2, 3, 3, 4, 8]

4. Pseudocode

function countingSort(arr)
    minVal = minimum value in arr
    maxVal = maximum value in arr

    countRange = maxVal - minVal + 1
    count = array of size countRange initialized with 0
    output = array of same size as arr

    // Step 1: Count frequencies
    for element in arr
        count[element - minVal] += 1

    // Step 2: Cumulative sum for counts
    for i from 1 to countRange - 1
        count[i] = count[i] + count[i - 1]

    // Step 3: Build output array (iterate backwards for stability)
    for i from length(arr) - 1 down to 0
        element = arr[i]
        count[element - minVal] -= 1
        position = count[element - minVal]
        output[position] = element

    // Step 4: Copy output back to arr
    for i from 0 to length(arr) - 1
        arr[i] = output[i]

5. Properties and Performance

Property Description
Type Non-comparison integer sorting algorithm
Time Complexity O(n + k), where n = number of elements, k = range of input
Space Complexity O(n + k) for output and count arrays
Stable Yes, it preserves order of equal elements
In-place No, requires additional output array
Suitable for Sorting integers with limited range

6. When to Use Counting Sort?

Counting Sort is best suited for situations where:

  • The input consists of integers (or discrete keys) within a small or manageable range.
  • Stability is required for sorting.
  • A linear time sorting algorithm is desired for certain constrained datasets.

It is not efficient when:

  • The input values range widely (e.g., large integers spanning millions or billions), making the count array excessively large (space-inefficient).
  • The input consists of floating-point numbers or other data structures without a direct, small-range integer key.

7. Comparison with Other Sorting Algorithms
