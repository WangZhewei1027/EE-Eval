Understanding Merge Sort: A Comprehensive Explanation

Merge Sort is a fundamental algorithm in computer science used for sorting arrays or lists of elements. It is an example of the divide and conquer strategy, breaking the problem into smaller parts, solving those recursively, and then combining the results.

What is Merge Sort?

Merge Sort is a comparison-based sorting algorithm that divides the unsorted list into smaller sublists until each sublist contains only one element (which by definition is sorted), and then merges those sublists to produce new sorted sublists until there is only one sorted list remaining.

It was invented by John von Neumann in 1945 and remains one of the most efficient general-purpose sorting methods, especially well-suited for sorting linked lists and external data (e.g., data that doesn't fit into memory).

Key Characteristics of Merge Sort

  • Time complexity: O(n log n) in best, average, and worst cases.
  • Space complexity: O(n) auxiliary space (due to the merge process).
  • Stability: Merge sort is a stable sort — it preserves the relative order of equal elements.
  • Method: Divide and conquer.

High-Level Idea

"Split the data into halves, sort each half, then merge the sorted halves."

The process consists of two main phases:

  1. Divide: Recursively split the list into two halves until each sublist has a single element.
  2. Conquer (Merge): Merge the sorted sublists two at a time to produce new sorted sublists until the entire list is sorted.

Detailed Step-by-Step Explanation

Given an array: [38, 27, 43, 3, 9, 82, 10]

  1. Divide the array into smaller subarrays:
    Split into two halves:
    [38, 27, 43] and [3, 9, 82, 10]
  2. Recursively divide each half:
    - [38, 27, 43] splits into [38] and [27, 43]
    - [27, 43] splits into [27] and [43]
    - [3, 9, 82, 10] splits into [3, 9] and [82, 10]
    - [3, 9] into [3] and [9]
    - [82, 10] into [82] and [10]
  3. Start merging sorted arrays:
    - Merge [27] and [43] → [27, 43]
    - Merge [38] and [27, 43] → Sort and merge to get [27, 38, 43]
    - Merge [3] and [9] → [3, 9]
    - Merge [82] and [10] → [10, 82]
    - Merge [3, 9] and [10, 82] → [3, 9, 10, 82]
  4. Merge the two big sorted halves:
    - Merge [27, 38, 43] and [3, 9, 10, 82] → [3, 9, 10, 27, 38, 43, 82]

The final merged array is sorted!

Why Does Merge Sort Work Efficiently?

Merge Sort leverages the fact that merging two already sorted lists is much faster than sorting an unsorted list from scratch. The steps are logarithmic in nature because the array is reduced by half at each division, making the overall time complexity O(n log n).

How Does the Merge Step Work?

The merge step is the core operation of the merge sort. It takes two sorted arrays and combines them into a single sorted array without many comparisons.

Suppose we have two sorted arrays:

  • Left: [3, 27, 38]
  • Right: [9, 10, 82]

The merge process:

  1. Compare the first elements of each array: 3 (Left) and 9 (Right). Since 3 < 9, place 3 into the result.
  2. Next compare 27 and 9. 9 < 27, so 9 goes next in the result.
  3. Then compare 27 and 10. 10 < 27, so 10 goes next.
  4. Compare 27 and 82. 27 < 82, add 27.
  5. Compare 38 and 82. 38 < 82, add 38.
  6. Right array now only has 82 left, add it.

Final merged array: [3, 9, 10, 27, 38, 82].

Merge Sort Pseudocode

MergeSort(array):
  if length of array ≤ 1:
    return array

  middle = floor(length of array / 2)
  leftHalf = array[0...middle - 1]
  rightHalf = array[middle...end]

  leftSorted = MergeSort(leftHalf)
  rightSorted = MergeSort(rightHalf)

  return Merge(leftSorted, rightSorted)

Merge(left, right):
  create empty result array
  while left and right are not empty:
    if left[0] ≤ right[0]:
      append left[0] to result
      remove left[0] from left
    else:
      append right[0] to result
      remove right[0] from right
  append remaining elements of left (if any) to result
  append remaining elements of right (if any) to result
  return result

Why is Merge Sort Stable?

Because when comparing two equal elements, merge sort copies the element from the left sublist first, it preserves the relative order of records with equal keys. This property often matters when sorting data with multiple keys.

When to Use Merge Sort?

  • When you need guaranteed O(n log n) runtime regardless of input (e.g., unlike Quick Sort’s worst case).
  • When stability is important.
  • When sorting linked lists (merge sort works great without extra space in linked lists).
  • External sorting (sorting data too large to fit into main memory).
