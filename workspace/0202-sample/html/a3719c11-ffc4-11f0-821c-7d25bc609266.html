K-Nearest Neighbors (KNN)

K-Nearest Neighbors (KNN) is a simple, intuitive, and widely used supervised machine learning algorithm primarily used for classification and regression problems.

Overview and Definition

KNN belongs to the family of instance-based or lazy learning algorithms that do not build an explicit model during training. Instead, they store the training data and make predictions based on the similarity between data points.

At its core, KNN assumes that similar data points exist near each other. By examining the “k” closest data points (neighbors) to a new input, KNN predicts the output—whether a class label in classification or a numeric value in regression.

Key Characteristics

  • Type: Supervised, lazy learning algorithm
  • Tasks: Classification and regression
  • Model: Non-parametric (makes no assumptions about data distribution)
  • Basis: Similarity to neighbors, typically based on distance metrics
  • Training Time: Very fast because there is minimal or no training phase
  • Prediction Time: Relatively slower as it requires computation of distances to all training points

How KNN Works - Step by Step

Let’s go through the KNN algorithm in a stepwise manner:

  1. Select the number of neighbors (k): Choose a positive integer value that defines how many nearest neighbors to consider when predicting. Typical values of k are small, e.g., 1, 3, 5, 7.
  2. Compute distances: Calculate the distance between the new input sample and every single point in the training data. Common distance metrics include:
    • Euclidean distance (most common for continuous variables)
    • Manhattan distance
    • Minkowski distance
    • Hamming distance (for categorical/binary data)
  3. Sort neighbors: Arrange the training data points in order of increasing distance to the new input point.
  4. Identify nearest neighbors: Select the top k points closest in distance.
  5. Predict:
    • Classification: Assign the class that has the majority vote among the neighbors.
    • Regression: Compute the average (mean) or weighted average of the neighbors’ target values.

Distance Metrics Explained

The choice of distance metric affects how similarity is measured between data points.

  • Euclidean distance:
    This is the straight-line or "as-the-crow-flies" distance in n-dimensional space.
    d(x, y) = sqrt( Σ (x_i - y_i)² )
  • Manhattan distance:
    Also called city-block distance, it sums absolute differences of coordinates.
    d(x, y) = Σ |x_i - y_i|
  • Minkowski distance:
    A generalization that includes both Euclidean and Manhattan distances as special cases.
    d(x, y) = ( Σ |x_i - y_i|^p )^(1/p) where p ≥ 1
  • Hamming distance:
    Counts the number of positions where the corresponding elements differ. Common for categorical/binary data.

Choosing the Value of k

Selecting the number of neighbors k is critical to the algorithm’s performance:

  • Small k (e.g., 1): Low bias but high variance. The algorithm can be sensitive to noise and outliers.
  • Large k: Higher bias but lower variance. The classification boundaries become smoother but the algorithm may misclassify points near boundaries.

Often, the value of k is chosen via cross-validation or by experimentation on validation data.

Pros and Cons of KNN

Advantages

  • Easy to understand and implement.
  • Non-parametric - no assumptions on data distribution.
  • Versatile – can be used for classification or regression.
  • Incremental learning - no training required, useful for datasets that change over time.
  • Adaptable with different distance metrics and weighting schemes.

Disadvantages

  • Computationally expensive at prediction time for large datasets.
  • Performance degrades with high-dimensional data ("curse of dimensionality").
  • Choice of k and distance metric significantly affect performance.
  • Does not handle well categorical features unless appropriately encoded.
  • Sensitive to irrelevant or redundant features and noisy data.

Application Example: Classification

Suppose you want to classify a new fruit based on features like weight and color intensity. You have a labeled dataset of fruits identified as either "Apple" or "Orange".

Steps we would follow:

  1. Decide k=3.
  2. Measure distances between the new fruit and all example fruits.
  3. Find the three closest fruits in this feature space.
  4. Among these 3 neighbors, if two are labeled "Apple" and one "Orange", predict "Apple".

Algorithm in Pseudocode

Input: training set of points with labels T, test point x, integer k
Output: predicted label/class for x

1. For each point t in T:
    a. Compute distance d(t, x)
2. Sort all points in T by increasing distance d(·, x)
3. Select top k points from sorted list => N
4. If classification task:
      a. Count the frequency of each label in N
      b. Return label with highest frequency (majority vote)
   Else if regression task:
      a. Compute mean of target values in N
      b. Return mean as predicted value

Variations and Improvements

KNN can be enhanced or adapted in several ways, including:

  • Weighted KNN: Neighbors closer to the query point have a larger influence. For example, weights can be inversely proportional to distance.
  • Feature scaling and normalization: Since distance calculations are sensitive to scale, features are often normalized or standardized before applying KNN.
  • Dimensionality reduction: Techniques like PCA can reduce the number of features to mitigate the curse of dimensionality.
  • Optimized data structures: Using KD-trees or Ball-trees can speed up neighbor searches for large datasets.

Summary

The K-Nearest Neighbors algorithm is a fundamental and intuitive approach in machine learning that eschews building models and instead leverages the principle that “near things are more related than distant things.” It bases its predictions on the labels or values of the closest known examples. Despite its simplicity, KNN is powerful, interpretable, and, with proper tuning, effective for various tasks.

Simple Demonstration (Classification)

Given Points (features x1, x2 and their classes):

  • (1, 2) – Class: Red
  • (2, 3) – Class: Red
  • (3, 3) – Class: Blue
  • (6, 5) – Class: Blue
  • (7, 7) – Class: Blue

Predict class of point (3, 4) with k = 3:

  1. Calculate Euclidean distances to all points:
    • Distance to (1, 2): √((3-1)² + (4-2)²) = √8 ≈ 2.83
    • Distance to (2, 3): √((3-2)² + (4-3)²) = √2 ≈ 1.41
    • Distance to (3, 3): √((3-3)² + (4-3)²) = √1 = 1
    • Distance to (6, 5): √((3-6)² + (4-5)²) = √10 ≈ 3.16
    • Distance to (7, 7): √((3-7)² + (4-7)²) = √25 = 5
  2. 3 nearest neighbors: (3,3 - Blue), (2,3 - Red), (1,2 - Red)
  3. Majority class among neighbors = Red
  4. Prediction: The point (3, 4) is classified as Red
