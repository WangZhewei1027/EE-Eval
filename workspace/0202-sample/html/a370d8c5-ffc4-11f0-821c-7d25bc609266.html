Understanding Radix Sort: A Comprehensive Explanation

1. Introduction

Radix Sort is a non-comparative, integer sorting algorithm that sorts data with integer keys by grouping keys by individual digits which share the same position and value. Unlike comparison-based sorting algorithms (like Quick Sort or Merge Sort), Radix Sort processes digits of the numbers to be sorted. It is particularly efficient when the length of the numbers (or number of digits) and the range of digits are relatively small.

Radix Sort was invented in the 1880s and remains relevant particularly in contexts where integer sorting needs to be very fast and stable. It is widely used in computer science both in theory and practice.

2. Concept and Theory Behind Radix Sort

2.1. Core Idea

Radix Sort sorts numbers by processing each digit separately from least significant digit (LSD) to most significant digit (MSD), or vice versa. The most common approach is Least Significant Digit first (LSD Radix Sort).

At each digit processing phase, it groups numbers by the current digit and rearranges them based on these groups while preserving the relative order of the previous digits — ensuring stability.

Because it processes digit by digit, Radix Sort does not rely on direct comparison between values, and the sorting happens in multiple passes — one pass per digit position.

2.2. Key Properties

  • Stability: The algorithm maintains the relative order of records with equal keys during each digit sorting pass.
  • Non-comparative: Sorting is done by digit grouping, not by comparing each pair.
  • Time complexity dependent on digit length: For n numbers and d digits per number, the complexity is O(d × (n + k)), where k is the base/radix (e.g., 10 for decimal).
  • Suitable For: Integers, strings, or any data that can be represented as fixed length keys with digits or characters over an ordered alphabet.

3. Detailed Algorithm Description (LSD Radix Sort)

We will illustrate Radix Sort using base 10 (decimal system) integers. The algorithm proceeds as follows:

  1. Identify the maximum number in the list to know the number of digits, d, we need to sort by.
  2. Starting from the least significant digit (rightmost digit), sort all numbers based on that single digit. This sorting is typically done using a stable sorting algorithm — commonly Counting Sort because the digit range (0-9) is small and fixed.
  3. Move to the next more significant digit (one digit to the left) and again sort the entire list based on this digit, maintaining stability.
  4. Repeat step 3 until sorting is done on the most significant digit (leftmost digit).
  5. After processing all digits, the list is sorted.

Note: The stable sorting at each digit is crucial. If sorting at any digit is unstable, the final result will be incorrect.

4. Implementation Details

The core challenge in Radix Sort is sorting the list by each digit with stability. The typical approach is to use Counting Sort as the stable subroutine because digits range over known fixed intervals (for decimal digits, 0–9).

Counting Sort Used in Radix Sort

Counting Sort works by counting occurrences of each digit and using cumulative counts to assign correct positions to elements. It runs in linear time relative to input size and digit range.

Pseudocode for Radix Sort

function radixSort(array, base = 10) {
  maxNumber = maximum value in array
  exp = 1    // exponent, 1 = units place, 10 = tens place, etc.
  
  while maxNumber / exp > 0:
    array = countingSortByDigit(array, exp, base)
    exp *= base
  
  return array
}

function countingSortByDigit(array, exp, base) {
  output = array of the same length
  count = array of length base filled with 0
  
  // Count occurrences of digit at exp
  for each number in array:
    digit = (number // exp) mod base
    count[digit]++
  
  // Modify count to represent position
  for i from 1 to base-1:
    count[i] = count[i] + count[i - 1]
  
  // Build output array backwards for stability
  for i from length(array)-1 down to 0:
    digit = (array[i] // exp) mod base
    output[count[digit]-1] = array[i]
    count[digit]--
  
  return output
}

5. Advantages and Disadvantages

Advantages

  • Efficiency: Can sort n numbers in O(d × (n + k)) time, which can be faster than comparison sorts for fixed-length integers or strings.
  • Stability: Maintains input order for equal keys.
  • Non-comparative: Avoids expensive comparison operations.

Disadvantages

  • Limited to keys with fixed or bounded length: Performance depends on digit length.
  • Additional memory: Needs extra space for counting arrays and output.
  • Not suitable for floating-point or complex keys: Requires modifications or different approaches.

6. Variants of Radix Sort

  • LSD Radix Sort: Processes digits from least significant to most significant. Most common and simpler to implement.
  • MSD Radix Sort: Processes digits from most significant to least significant. Used when sorting strings or variable length numbers, often implemented with recursive calls.
  • Radix Choice: Though base 10 (decimal) is intuitive, other bases (e.g., base 2 for binary digits, base 256 for bytes) may be used to optimize performance.

8. Summary

  

    Radix Sort is a powerful integer sorting algorithm that achieves efficient, stable sorting by processing each digit of the numbers. Its fundamental strength lies in non-comparative digit handling and stable sorting. While it may not replace general-purpose comparison sorts in every case, it shines where input key sizes are predictable and bounded — such as sorting phone numbers, strings of fixed length, or large datasets where numeric keys dominate.
  

  

    Understanding Radix Sort also prepares programmers to appreciate non-comparison-based algorithms and offers practical insight into the tradeoffs between algorithm speed, memory usage, and input characteristics.
