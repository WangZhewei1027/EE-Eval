Heap Sort: A Comprehensive Explanation

Heap Sort is a comparison-based sorting technique based on a special data structure called a heap. It is an efficient, in-place sorting algorithm with a guaranteed time complexity of O(n log n) in all cases (worst, best, and average). This explanation delves deeply into the concepts, theory, and algorithmic steps behind heap sort to equip you with a solid understanding.

1. What Is a Heap?

A heap is a specialized complete binary tree that satisfies the heap property: for every node i other than the root, the value of i is less than or equal to (in a max heap) or greater than or equal to (in a min heap) the value of its parent.

  • Max Heap: Parent nodes have values greater than or equal to their children.
  • Min Heap: Parent nodes have values less than or equal to their children.

Heap Sort typically uses a max heap because it sorts in ascending order by repeatedly removing the largest element.

A complete binary tree means all levels except possibly the last are completely filled, and the last level has all keys as left as possible.

Heap Representations

Heaps are traditionally implemented using arrays. For an element at index i (0-based), in a binary heap:

  • Left child index = 2i + 1
  • Right child index = 2i + 2
  • Parent index = (i - 1) >> 1 or Math.floor((i - 1)/2)

This array-based representation avoids explicit node objects and pointer manipulation, making it highly space-efficient and cache-friendly.

2. Fundamental Operations on a Heap

Two key operations are used to maintain the heap property:

heapify (also called siftDown or maxHeapify)

This function ensures the subtree rooted at a given index follows the max heap property. If the root violates the max heap property (is smaller than any of its children), heapify swaps it with the largest child and recursively heapifies the affected subtree.

buildHeap

This procedure converts an unordered array into a max heap by iteratively calling heapify from the last non-leaf node up to the root node.

3. How Does Heap Sort Work?

Heap Sort consists of two main phases:

  1. Build a max heap from the input array.
  2. One by one, extract the maximum element (root of the heap) and place it at the end of the array, then heapify the reduced heap again.

This works because the maximum element is always at the root of a max heap. By repeatedly swapping the root with the last element and shrinking the heap size, the array becomes sorted in-place.

Step-by-Step Outline:

  1. Transform the array into a max heap.
    • Start from the last non-leaf node and call heapify up to the root.
  2. Swap the root (max element) with the last element of the array.
  3. Reduce the effective heap size by one (exclude the last element now considered sorted).
  4. Call heapify on the root to restore the max heap property.
  5. Repeat steps 2 to 4 until the heap size is reduced to 1.

4. Algorithm Complexity

Time Complexity:

  • Building the heap: O(n) — More efficient than performing heapify from the root repeatedly.
  • Extracting elements: O(n log n) — n removals, each requiring heapify of O(log n).
  • Total: O(n log n)

Space Complexity: O(1) additional space beyond the input array, as heap sort is an in-place algorithm.

5. Illustrative Example

Consider the array: [4, 10, 3, 5, 1]

Step 1: Build max heap

  • Start heapify at index 1 (value 10): children are at indices 3 and 4 (values 5 and 1).
  • 10 is already >= children, no swap needed.
  • Move to index 0 (value 4): children at indices 1 and 2 (values 10 and 3).
  • 10 is largest; swap 4 and 10.
  • Heapify subtree rooted at index 1 (now 4): its children 5 and 1.
  • 5 is largest child, swap with 4.

Resulting heap array: [10, 5, 3, 4, 1]

Step 2: Extract max repeatedly

  1. Swap root (10) with last element (1): [1, 5, 3, 4, 10]
  2. Heap size reduced to 4. Heapify root (index 0, value 1). Largest child is 5 (index 1), swap.
  3. Heapify subtree index 1 (value 1), swap with 4 (index 3).
  4. Heap is now [5, 4, 3, 1, 10]
  5. Swap root (5) with index 3 (1): [1, 4, 3, 5, 10]
  6. Heap size 3. Heapify root (1). Largest child 4, swap.
  7. Heapify index 1 (1) no further swaps needed.
  8. Heap now [4, 1, 3, 5, 10]
  9. Swap root (4) with index 2 (3): [3, 1, 4, 5, 10]
  10. Heap size 2. Heapify root (3). Largest child 1, no swap since 3 > 1.
  11. Swap root (3) with index 1 (1): [1, 3, 4, 5, 10]
  12. Heap size 1. Sorting finished.

Final sorted array:

[1, 3, 4, 5, 10]

6. Pseudocode

function heapSort(array):
    n = length(array)

    // Build max heap
    for i = floor(n / 2) - 1 down to 0:
        heapify(array, n, i)

    // Extract elements from heap
    for i = n - 1 down to 1:
        swap array[0] and array[i]  // Move max to the end
        heapify(array, i, 0)         // Heapify reduced heap

function heapify(array, heapSize, rootIndex):
    largest = rootIndex
    left = 2 * rootIndex + 1
    right = 2 * rootIndex + 2

    if left < heapSize and array[left] > array[largest]:
        largest = left

    if right < heapSize and array[right] > array[largest]:
        largest = right

    if largest != rootIndex:
        swap array[rootIndex] and array[largest]
        heapify(array, heapSize, largest)

7. Key Points and Educational Insights

  • Why Build Heap from Bottom-Up? Calling heapify on non-leaf nodes from bottom up ensures that subtrees are correct heaps before their parents are heapified, guaranteeing overall correctness and efficiency.
  • Complete binary tree nature: Using an array efficiently represents a heap without pointer overhead.
  • In-Place: Heap sort does not require additional memory beyond input, unlike merge sort, making it attractive for memory-sensitive applications.
  • Stable? Heap sort is generally not stable because it can change original relative order of equal elements during heapify swapping.
  • Use cases: Useful when guaranteed O(n log n) time is required and space is at a premium.
