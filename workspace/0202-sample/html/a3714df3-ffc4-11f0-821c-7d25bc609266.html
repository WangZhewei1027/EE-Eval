Understanding the Longest Common Subsequence (LCS)

A comprehensive textual exploration of the Longest Common Subsequence problem, its theory, algorithms, and examples.

1. Introduction to the Longest Common Subsequence

The Longest Common Subsequence (LCS) problem is a classic computer science and algorithmic challenge that appears frequently in diverse areas such as text comparison, bioinformatics, and version control systems.

But what exactly is a subsequence? A subsequence of a sequence is a new sequence generated from the original by deleting some elements without changing the order of the remaining elements.

For example, consider the sequence:

ABCBDAB

Some subsequences of this sequence are:

  • AB
  • ACB
  • BCDA
  • BDAB
  • ABCBDAB (the sequence itself)

Notice that the order of characters remains the same as in the original sequence, but elements can be skipped.

Given two sequences, the goal of the LCS problem is to find the longest sequence that is a subsequence of both sequences.

2. Formal Definition

Let X and Y be two sequences of lengths m and n respectively: 

X = x1, x2, ..., xm
Y = y1, y2, ..., yn

A sequence Z = z1, z2, ..., zk is a subsequence of both X and Y if there exist indices i1 < i2 < ... < ik in X and indices j1 < j2 < ... < jk in Y such that:

zt = xit = yjt for all t = 1, 2, ..., k

The LCS problem asks for such a sequence Z with the maximum possible value of k (length).

The analogy is often illustrated with strings, but the sequences can be generalized to any ordered list of elements.

3. Intuition and Examples

Let’s understand LCS by an example:

Consider two strings:

X = "ABCBDAB"

Y = "BDCABA"

What is the longest common subsequence between these two sequences?

Some common subsequences and their lengths:

  • BCA (length 3)
  • BDAB (length 4)
  • BCAB (length 4)
  • BCBA (length 4)

The length of the LCS is 4. One possible LCS is BCAB, but others also exist.

Note: The LCS is not necessarily unique. Multiple different subsequences with the same maximum length can exist.

4. Algorithmic Approach: Dynamic Programming

The most common efficient approach to solve the LCS problem is Dynamic Programming (DP).

The key idea is to solve smaller subproblems and build up to the solution for the full sequences via tabulation.

4.1 Defining Subproblems

Let dp[i][j] represent the length of the LCS of the prefixes X[1..i] and Y[1..j].

That is, dp[i][j] stores the length of the longest common subsequence of the first i characters of X and the first j characters of Y.

Our goal is to fill a 2D array dp of dimensions (m+1) x (n+1), where dp[m][n] gives the final answer.

4.2 Recurrence Relation

The value of dp[i][j] can be computed as follows:

  • If X[i] == Y[j], then the current character contributes to the LCS:
    dp[i][j] = dp[i-1][j-1] + 1
  • Otherwise, if the characters do not match:
    dp[i][j] = max(dp[i-1][j], dp[i][j-1])

This recurrence captures that either the current character is part of the LCS (if they match), or we drop one from either X or Y and take the best solution so far.

4.3 Initialization

Initialize the first row and first column of dp with zeros because an empty sequence compared to any sequence has LCS length zero:

  • dp[0][j] = 0 for all j
  • dp[i][0] = 0 for all i

5. Reconstructing the Actual Subsequence

Often, the length of the LCS is insufficient; we want to find the actual longest common subsequence string.

After filling the dp table, we can reconstruct the subsequence by tracing back from dp[m][n]:

  1. Start from i = m, j = n.
  2. If X[i] == Y[j], the character is part of the LCS. Prepend this character to the result and move diagonally to dp[i-1][j-1].
  3. If not equal, move to the cell with the greater value among dp[i-1][j] and dp[i][j-1]. This indicates which subsequence is longer and hence a better path.
  4. Repeat until either i or j reaches zero.

This process yields one valid longest common subsequence.

6. Detailed Step-By-Step Example of Reconstruction

Using the strings X = "ABCBDAB" and Y = "BDCABA", assume you have computed the dp table as explained.

Starting from dp[7][6] = 4:

  1. At i=7 (B), j=6 (A): Characters differ (B != A).
    Compare dp[6][6] = 4 and dp[7][5] = 4. Both equal, arbitrary choose: move up to dp[6][6].
  2. At i=6 (A), j=6 (A): Characters match.
    Add A to LCS.
    Move diagonally to dp[5][5].
  3. At i=5 (D), j=5 (B): Characters differ.
    Compare dp[4][5]=3 and dp[5][4]=2. Move to dp[4][5] (left.)
  4. At i=4 (B), j=5 (B): Characters match.
    Add B to LCS.
    Move diagonally to dp[3][4].
  5. At i=3 (C), j=4 (A): Characters differ.
    Compare dp[2][4]=1 and dp[3][3]=2. Move to dp[3][3].
  6. At i=3 (C), j=3 (C): Characters match.
    Add C to LCS.
    Move diagonally to dp[2][2].
  7. At i=2 (B), j=2 (D): Characters differ.
    Compare dp[1][2]=0 and dp[2][1]=1. Move to dp[2][1].
  8. At i=2 (B), j=1 (B): Characters match.
    Add B to LCS.
    Move diagonally to dp[1][0] which ends the backtracking.

The characters added (in reverse order) are: A, B, C, B → reversed: BCBA.

Hence, BCBA is one valid longest common subsequence of length 4.

7. Pseudocode for LCS Algorithm

// Let X be of length m, Y be of length n
function LCSLength(X, Y)
  let dp = array of size (m+1) x (n+1) initialized with 0
  
  for i from 1 to m
    for j from 1 to n
      if X[i-1] == Y[j-1]
        dp[i][j] = dp[i-1][j-1] + 1
      else
        dp[i][j] = max(dp[i-1][j], dp[i][j-1])
  return dp

function ReconstructLCS(dp, X, Y)
  i = length of X
  j = length of Y
  lcs = empty list

  while i > 0 and j > 0
    if X[i-1] == Y[j-1]
      prepend X[i-1] to lcs
      i = i - 1
      j = j - 1
    else if dp[i-1][j] > dp[i][j-1]
      i = i - 1
    else
      j = j - 1
  return concatenate elements of lcs into string

8. Time and Space Complexity Analysis

The LCS dynamic programming algorithm runs with:

  • Time complexity: O(m * n), where m and n are the lengths of the input sequences.
  • Space complexity: O(m * n) for storing the DP table.

There exist space-optimized algorithms that reduce space complexity to O(min(m, n)), but with more complex reconstruction processes.

9. Practical Applications

The concept and algorithms for LCS are widely used in:

  • Version Control Systems: Tools like Git use LCS-like algorithms to identify changes between file versions.
  • Bioinformatics: To compare DNA, RNA, or protein sequences to identify similarities, evolutionary connections, or conserved motifs.
  • Text Comparison: Spell checking, diff tools, and plagiarism detection.
  • Data Comparison: To find patterns or alignments in ordered datasets.
