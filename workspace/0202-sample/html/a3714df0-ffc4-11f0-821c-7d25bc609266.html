Comprehensive Explanation of Topological Sort

Topological Sort (or topological ordering) is an algorithmic technique used to linearly order the vertices of a Directed Acyclic Graph (DAG) such that for every directed edge uv from vertex u to vertex v, u comes before v in the ordering.

1. What Is a Topological Sort?

Given a graph where edges indicate dependencies — for example, a task A must be done before task B — a topological sort provides a linear sequence of those tasks respecting all the dependencies.

More formally, if the graph is denoted as G = (V, E) where V is the set of vertices and E the set of directed edges, a topological ordering is a permutation v1, v2, ..., vn of V such that for every edge (vi, vj) ∈ E, i < j. In other words, every directed edge goes from an earlier vertex in the ordering to a later vertex.

Key Characteristics

  • The graph must be acyclic: no cycles allowed. Presence of a cycle means no valid topological order exists.
  • A DAG can have more than one topological ordering.
  • Not all directed graphs can be topologically sorted — only DAGs.

2. Why Is Topological Sort Important?

Many practical problems require ordering tasks or elements based on dependency constraints:

  • Build systems: Compile source files respecting dependency order
  • Course scheduling: Prerequisite courses must be taken before subsequent ones
  • Project task scheduling: Some tasks must precede others
  • Resolving symbol dependencies: In linkers or loaders

Thus, topological sort solves a wide class of problems where linearization of dependencies is required.

3. Definition of a Directed Acyclic Graph (DAG)

A DAG is a graph that is directed and contains no cycles. Formally:

A cycle exists if there is a path from a vertex back to itself following the directed edges.

The absence of cycles means there is at least one linear ordering of vertices that respects all edge directions — this linear ordering is what topological sort finds.

4. Algorithms for Topological Sort

There are several classical algorithms to compute a topological ordering. Two of the most common are:

4.1 Kahn's Algorithm (Source Removal Algorithm)

This algorithm repeatedly removes vertices that have no incoming edges (called "sources") and adds them to the ordering. The steps are:

  1. Initialize an empty list L that will contain the sorted elements.
  2. Identify all vertices with in-degree 0 (no incoming edges) and add them to a queue or set S.
  3. While S is not empty:
    1. Remove a vertex u from S and append it to L.
    2. For each vertex v adjacent to u (i.e., edge u → v):
      • Remove the edge u → v from the graph.
      • If v now has no incoming edges, add v to S.
  4. If the graph still has edges at the end, it means there was a cycle (no topological sort exists).

Intuition: Always pick vertices that do not depend on any others (no inbound edges), remove them, and continue with remaining graph.

4.2 Depth-First Search (DFS) Based Algorithm

The alternative approach to topological sort uses DFS to explore nodes, and on recursive backtracking, add vertices to the output in reverse finishing order.

  1. Mark all vertices as unvisited.
  2. For each unvisited vertex, run DFS:
    • When a vertex finishes exploring all its descendants, add it to the front of a list (or push to a stack).
  3. After DFS completes for all vertices, the list or stack contains a topological order.

Important: If a cycle is encountered during DFS (a back edge to an already visiting node), topological ordering is not possible.

5. Step-by-Step Example

Consider the following DAG describing build dependencies between modules:

  • Module A must be compiled before C and D
  • Module B must be compiled before C
  • Module C must be compiled before E
  • Module D must be compiled before F
  • Module E must be compiled before F

The edges of the graph are (A → C), (A → D), (B → C), (C → E), (D → F), (E → F).

Applying Kahn's Algorithm:

  • Calculate in-degree for each vertex:
    • A: 0 (no incoming edges)
    • B: 0
    • C: 2 (from A and B)
    • D: 1 (from A)
    • E: 1 (from C)
    • F: 2 (from D and E)
  • Initialize S = {A, B} (vertices with in-degree 0)
  • Remove A from S and append to L: L = [A]
    • Remove edges A → C and A → D
    • Update in-degree: C:1, D:0
    • Add D to S
  • Remove B from S: L = [A, B]
    • Remove edge B → C
    • Update in-degree: C:0
    • Add C to S
  • Remove D from S: L = [A, B, D]
    • Remove edge D → F
    • Update in-degree: F:1
    • F not added yet
  • Remove C from S: L = [A, B, D, C]
    • Remove edge C → E
    • Update in-degree: E:0
    • Add E to S
  • Remove E from S: L = [A, B, D, C, E]
    • Remove edge E → F
    • Update in-degree: F:0
    • Add F to S
  • Remove F from S: L = [A, B, D, C, E, F]

The resulting topological order is: A, B, D, C, E, F.

6. Limitations and Important Notes

  • If the graph contains a cycle, it is impossible to perform a topological sort because no linear ordering can satisfy all dependency constraints.
  • Topological sorting is only possible for DAGs.
  • Different runs or algorithms might produce different valid topological orders.

7. Code Snippets for Reference

Kahn’s Algorithm in Pseudocode


KahnTopologicalSort(Graph):
    L ← Empty list to hold the sorted elements
    S ← Set of all nodes with no incoming edges

    while S is not empty do
        remove a node n from S
        add n to tail of L
        for each node m with an edge e from n to m do
            remove edge e from the graph
            if m has no other incoming edges then
                insert m into S

    if graph has edges then
        return error (graph has at least one cycle)
    else
        return L (a topologically sorted order)

DFS Based Approach in Pseudocode


visited ← set of nodes
stack ← empty stack

function DFS(node):
    if node is temporarily marked then
        report cycle (not a DAG)
    if node is not visited then
        mark node temporarily
        for each neighbor in adjacency list of node do
            DFS(neighbor)
        mark node permanently
        push node onto stack

for each node in graph do
    if node is not visited then
        DFS(node)

topological order is contents of stack in reverse order
