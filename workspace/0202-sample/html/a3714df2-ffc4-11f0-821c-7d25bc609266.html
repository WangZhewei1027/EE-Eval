The Knapsack Problem

The Knapsack Problem is a classic algorithmic problem in the fields of computer science, mathematics, and optimization theory. It is a fundamental example of combinatorial optimization. The problem derives its name from the scenario of a thief who is going to rob a store and wants to maximize the value of the loot while carrying it in a knapsack (backpack) with limited capacity.

Problem Definition

You are given a set of n items. Each item has two attributes:

  • Weight (or size)
  • Value (or profit)

The goal is to select a subset of these items to include in a knapsack so that:

  • The total weight of the selected items does not exceed the knapsack's capacity W.
  • The total value of the selected items is as large as possible.

Formally, if the items are represented as arrays weights[] and values[], find a subset S ⊆ {1, ... n} such that:

  • ∑i∈S weights[i] ≤ W
  • Maximize ∑i∈S values[i]

Types of Knapsack Problems

1. 0/1 Knapsack Problem

The most classical version. Each item can be either taken whole or not taken at all (0 or 1). You cannot take fractional parts of an item.

2. Fractional Knapsack Problem

The items can be broken into smaller pieces. You can take any fraction of an item to fill the knapsack optimally. This makes the problem solvable in greedy fashion in polynomial time.

3. Unbounded Knapsack Problem

You can take any number of copies of each item; the restriction is only the total capacity.

This guide primarily focuses on the 0/1 Knapsack Problem, which is widely studied and not solvable in polynomial time by greedy methods.

Fundamental Computational Challenge

The 0/1 Knapsack Problem is a classic example of an NP-complete problem. Although a brute force approach would be to evaluate all subsets (which is an exponential time approach, O(2^n)), efficient approximate and exact algorithms exist to solve the problem for reasonably sized inputs.

Approaches to the 0/1 Knapsack Problem

1. Brute Force Enumeration

One can enumerate every subset of the n items and choose the best valid subset under the weight constraint. However, this approach has a time complexity of O(2ⁿ) and is impractical for moderate or larger inputs.

2. Dynamic Programming Solution

A classic and practical approach to the problem uses dynamic programming (DP). The insight is to build a table that keeps track of the maximum value achievable with a subset of items and a bounded weight.

DP Subproblem Definition:

  • dp[i][w]: maximum value achievable using items 1..i with a weight limit w.

DP Recurrence Formula:

For each item i (indexed 1 to n) and weight w (from 0 to W): 

dp[i][w] = {
  dp[i-1][w]                           if weights[i] > w
  max(dp[i-1][w], dp[i-1][w - weights[i]] + values[i])  otherwise
}

- If the weight of the current item exceeds the current weight limit w, we cannot include it, so the best value is the same as without this item.

- Otherwise, consider the maximum of two options:

  • Exclude the item: dp[i-1][w]
  • Include the item: dp[i-1][w - weights[i]] + values[i]

DP Algorithm Complexity:

  • Time Complexity: O(nW)
  • Space Complexity: O(nW) (can be optimized to O(W))

Example Walkthrough (Step-by-step):

Consider 3 items with:

  • Weights = [1, 3, 4]
  • Values = [1500, 2000, 3000]
  • Knapsack capacity = 4

We build a DP table where rows correspond to items, columns to capacity from 0 to 4.

Step 1: For item 1 (weight 1, value 1500)

  • Capacity 0: 0 (can't carry anything)
  • Capacity 1: 1500 (can carry item 1)
  • Capacity 2 to 4: 1500 (only item 1 is available)

Step 2: For item 2 (weight 3, value 2000)

  • Capacity 0,1,2: same as before (can't fit item 2)
  • Capacity 3: max of not taking item 2 (1500) or taking it (2000) = 2000
  • Capacity 4: max of not taking item 2 (1500) or taking it + remaining capacity (1500+2000=3500, but capacity after weight 3 is 1) actually we check dp[1][1]+2000=1500+2000=3500

Step 3: For item 3 (weight 4, value 3000)

  • Capacity 0 to 3: best from previous step
  • Capacity 4: max of previous (3500) and taking item 3 alone (3000) = 3500

Final maximum value = 3500 by taking items 1 and 2.

Fractional Knapsack Problem and Greedy Algorithm

Unlike the 0/1 knapsack problem, the fractional knapsack allows taking fractions of items.

Greedy Approach:

  1. Calculate value per unit weight for each item: value[i]/weight[i]
  2. Sort items by this ratio in descending order.
  3. Start picking items from the sorted list by taking as much as you can until the knapsack is full.

This greedy algorithm provides the optimal solution efficiently in O(n log n) due to sorting.

Example: Suppose two items:

  • Item A: weight=10, value=60 (value/weight=6)
  • Item B: weight=20, value=100 (value/weight=5)

Knapsack capacity = 15

Greedy picks as much as possible from Item A first because of the higher value-to-weight ratio:

  • Take entire Item A (weight 10, value 60)
  • Remaining capacity is 5, take 5 units fraction of Item B (value 5/20 * 100 = 25)

Total value = 60 + 25 = 85 (optimal).

Proofs and Complexity Comments

Why Greedy Fails on 0/1 Knapsack

The greedy algorithm based on value-to-weight ratio might select items that do not provide the optimal combination because it cannot take fractions. For example:

  • Item 1: weight 10, value 60 (value/weight = 6)
  • Item 2: weight 20, value 100 (value/weight = 5)
  • Knapsack capacity: 20

Greedy picks Item 1 first (weight=10, value=60). Remaining capacity 10 cannot fit Item 2, so total value=60.

However, the optimal solution is just taking Item 2 (weight=20, value=100).

This simple example shows greedy fails on 0/1 knapsack.

Computational Complexity:

  • 0/1 Knapsack is NP-complete as it is reducible from the Subset Sum problem.
  • Dynamic programming solutions run in pseudo-polynomial time (polynomial in both the number of items and the capacity value).

Summary and Key Takeaways

  • The Knapsack Problem illustrates a fundamental challenge in optimization and algorithm design.
  • 0/1 Knapsack requires careful combinatorial examination or dynamic programming; greedy solutions do not work.
  • Fractional Knapsack can be solved optimally using a greedy approach.
  • Dynamic programming provides exact solutions efficiently when capacity and number of items are not too large.
  • Understanding the problem helps grasp core algorithmic paradigms: greedy algorithms, dynamic programming, complexity theory.
