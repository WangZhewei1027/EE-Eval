Understanding Heaps (Min/Max)

A heap is a specialized tree-based data structure that satisfies the heap property. It is widely used in computer science for implementing priority queues, efficient sorting algorithms (like Heap Sort), and many graph algorithms.

1. What is a Heap?

A heap is a complete binary tree that can be visualized as a tree where every level, except possibly the last, is completely filled, and all nodes are as far left as possible. Unlike a binary search tree, a heap is not ordered in the same way; instead, it enforces the heap property.

The Heap Property

  • Min-Heap: For every node i other than the root, the value of i is greater than or equal to the value of its parent. This means the smallest element is always at the root.
  • Max-Heap: For every node i other than the root, the value of i is less than or equal to the value of its parent. This means the largest element is always at the root.
Note: The heap property ensures that the root is always the min (in min-heap) or max (in max-heap), but it doesn’t guarantee any ordering between sibling nodes.

2. Why Use a Heap?

Heaps allow efficient implementation of priority queues where quick access to the minimum or maximum element is required. They provide:

  • Fast access to the extreme element: The min or max element is always at the root, accessible in O(1) time.
  • Efficient insertion and deletion: New elements can be inserted and the extreme element can be deleted in O(log n) time.
  • Space efficiency: Heaps are usually implemented with arrays, allowing compact storage without pointers.

3. Representation of Heaps

Array Representation

A heap is often implemented as an array rather than pointers and nodes. The relationship between parent and children is determined by indices:

  • For a node at index i (0-based indexing):
    • Left child is at index 2i + 1
    • Right child is at index 2i + 2
    • Parent is at index floor((i - 1) / 2)

This array form corresponds to a complete binary tree, stored level by level.

4. Heap Operations

a) Insertion

When inserting a new value into a heap:

  1. Insert the new element at the end of the heap (the array’s last position).
  2. Heapify-Up (also called percolate up or bubble up): Compare the inserted element with its parent; if the heap property is violated, swap them.
  3. Repeat the comparison and swap up the tree until the heap property is restored or the root is reached.

This ensures the heap remains complete and satisfies the heap property after insertion.

b) Deletion of Root (Extract Min or Max)

Removing the root element (minimum in min-heap or maximum in max-heap) is the primary way to retrieve priority elements:

  1. Remove the root value (element at index 0 in the array).
  2. Move the last element of the heap to the root position to fill the gap.
  3. Heapify-Down (also called percolate down or bubble down): Compare the root with its children; if the heap property is violated, swap it with the appropriate child (min or max, depending on heap type).
  4. Repeat the process down the tree until the heap property is restored.

5. Heapify Algorithm

The heapify operation is fundamental to maintaining the heap property. It either moves a node up or down the tree to its correct position.

Heapify-Down (used after root removal)

Given a node that may violate the heap property, to restore heap:

  1. Compare the node with its children.
  2. Swap it with the smallest (min-heap) or largest (max-heap) child if heap property is violated.
  3. Continue this process recursively on the child node’s new position.

Heapify-Up (used after insertion)

Given a newly inserted node at the bottom of the heap, to restore heap:

  1. Compare the node with its parent.
  2. If heap property is violated, swap with parent.
  3. Continue recursively up the tree until heap property is restored or root is reached.

6. Building a Heap from an Unordered Array

To build a heap efficiently from an unordered array of n elements:

  • Start from the last non-leaf node (at index floor(n/2)-1).
  • Apply heapify-down to each non-leaf node going backwards to the root.

This approach runs in O(n) time—not O(n log n)—and is the basis of the Heap Sort algorithm.

7. Example: Min-Heap Construction

Suppose we want to build a min-heap from the array:

[9, 5, 6, 2, 3]
  1. Start by inserting all elements as if forming a complete binary tree:
    • Index 0: 9 (root)
    • Index 1: 5 (left child of root)
    • Index 2: 6 (right child of root)
    • Index 3: 2 (left child of node at index 1)
    • Index 4: 3 (right child of node at index 1)
  2. Heapify from last non-leaf node (index 1):
    • Node at index 1 is 5, children are 2 (index 3) and 3 (index 4).
    • Smallest child is 2; since 2 < 5, swap them.
  3. Heap now looks like: [9, 2, 6, 5, 3]
  4. Heapify down at the new position of 5 (index 3), has no children so stop.
  5. Heapify at root (index 0):
    • Children: 2 (index 1) and 6 (index 2).
    • Smallest child is 2; since 2 < 9, swap.
  6. Heap is now: [2, 9, 6, 5, 3]
  7. Heapify down at index 1 (now containing 9): children are 5 and 3.
    • Smallest child is 3; since 3 < 9, swap.
  8. Heap final form: [2, 3, 6, 5, 9]

8. Practical Applications

  • Priority Queues: Heaps allow efficient insertion and extraction of the min/max element, perfect for task scheduling or bandwidth management.
  • Heap Sort: A comparison-based sorting technique based on a binary heap. Runs in O(n log n) time.
  • Graph Algorithms: Used in Dijkstra’s shortest path and Prim’s minimum spanning tree algorithms to quickly access the next vertex with minimum distance.
  • Median Maintenance: Using two heaps (min and max), find the median of a stream of numbers efficiently.

9. Summary of Time Complexities

Operation Time Complexity
Insert O(log n)
Extract Min/Max (delete root) O(log n)
Peek (get root) O(1)
Build Heap (from unordered array) O(n)
