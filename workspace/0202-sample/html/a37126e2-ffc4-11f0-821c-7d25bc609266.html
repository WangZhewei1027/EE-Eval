Floyd-Warshall Algorithm

Introduction

The Floyd-Warshall algorithm is a classic algorithm in computer science used to find the shortest paths between all pairs of vertices in a weighted graph. It works for graphs with positive or negative edge weights but requires that no negative weight cycles exist.

Unlike algorithms such as Dijkstra's, which find the shortest path from a single source to all other vertices, Floyd-Warshall efficiently computes the shortest paths between every pair of nodes in a graph. It is a prominent example of a dynamic programming algorithm.

Problem Definition

Given a graph G = (V, E) with edge weights w(u, v), the goal is to determine the shortest paths d(i, j) between every pair of vertices i, j ∈ V.

Specifically, for every pair (i, j), we want the minimum sum of edge weights among all paths from i to j.

The graph may be directed or undirected. Edges can have negative weights as long as no negative cycles are present (a cycle whose total weight sum is negative, which would otherwise cause shortest paths to be undefined).

Key Ideas and Concepts

1. Dynamic Programming Approach

The Floyd-Warshall algorithm uses dynamic programming by incrementally improving the solution considering intermediate vertices. In other words, it tries to find the shortest path between two vertices by progressively allowing paths that include a subset of intermediate nodes.

2. Intermediate Vertices

Imagine the vertices labeled 1, 2, ..., n. Define d_k(i, j) as the shortest distance from vertex i to vertex j that only allows intermediate vertices from the subset {1, 2, ..., k}.

Then the algorithm tries all possibilities for k from 0 to n, where:

  • d_0(i, j) is the direct edge cost from i to j if it exists, else infinity.
  • d_n(i, j) is the shortest distance considering all vertices as possible intermediate stops.

This leads to the key recurrence:

d_k(i, j) = min(d_{k-1}(i, j), d_{k-1}(i, k) + d_{k-1}(k, j))

Meaning: while considering vertices up to k, the shortest path from i to j is either the same as without k as an intermediate vertex, or a path that goes through k.

3. Detecting Negative Cycles

If after the process, any diagonal element d(i, i) is negative, then the graph contains a negative weight cycle reachable from vertex i. The algorithm can report such cases as shortest paths are ill-defined.

Algorithm Pseudocode


Let n be the number of vertices.
Initialize matrix d of size n×n:

for i from 1 to n:
  for j from 1 to n:
    if i == j:
      d[i][j] = 0
    else if edge (i, j) exists:
      d[i][j] = weight(i, j)
    else:
      d[i][j] = ∞

for k from 1 to n:
  for i from 1 to n:
    for j from 1 to n:
      d[i][j] = min(d[i][j], d[i][k] + d[k][j])

Check for negative cycles:
for i from 1 to n:
  if d[i][i] < 0:
    report "Graph contains a negative weight cycle"

Running time complexity: O(n³), due to three nested loops over the vertices.

Step-by-Step Explanation of the Algorithm

  1. Initialization: Create a 2D matrix d that will store shortest path distances. Set the distance from each vertex to itself as 0. For other pairs, if there is a direct edge, initialize with its weight; otherwise, initialize with infinity (meaning no direct path known initially).
  2. Iterative Improvement: For each vertex k from 1 to n, iterate over every vertex pair (i, j) and update d[i][j] by checking if going through vertex k improves the path: i.e., if d[i][k] + d[k][j] < d[i][j], update d[i][j].
  3. Detect Negative Cycles: After all iterations, check the diagonal of matrix d. If any d[i][i] < 0, it indicates a negative weight cycle reachable from i, hence shortest paths can't be accurately computed.

Understanding the Updates

The algorithm considers each vertex in turn as an intermediate vertex k, checking if the path from i to j can be improved by going through k.

For example, when k = 4, the algorithm checks for all i, j pairs if d[i][4] + d[4][j] provides a shorter path than the currently known d[i][j]. Because vertex 4 has edges involved with negative weights, it often allows dramatically reducing some path lengths.

Additional Notes

  • Space complexity: The algorithm requires O(n²) space due to the distance matrix.
  • If you want to retrieve not just the distance but also the actual shortest path (i.e., the sequence of vertices), a separate parent (or predecessor) matrix is maintained and updated alongside.
  • Floyd-Warshall is especially practical for dense graphs, or when all-pairs shortest paths are required.
  • Variations of the algorithm exist for specialized scenarios, including on GPUs or with early stopping to optimize performance in some cases.
