Binary Search Tree (BST)

A Binary Search Tree (BST) is a data structure that maintains a sorted order of elements (nodes), allowing for efficient insertion, deletion, and lookup operations. Each node in a BST has a maximum of two children, referred to as the left and right child. The left child must have a value less than its parent node, and the right child must have a value greater than or equal to its parent node. This property enables binary search and ensures that operations can be performed in logarithmic time on average.

Basic Properties of a BST

  • Node: Each element in the tree is called a node, and it contains a key, along with pointers to its left and right children.
  • Height: The height of a tree is defined as the length of the longest path from the root node to a leaf node.
  • Balance: A well-balanced BST ensures that the height of the left and right subtrees of any node differ by at most one, leading to more efficient operations.

Key Operations

The efficient operation of a Binary Search Tree is attributed to three key operations: insertion, deletion, and searching. Here’s how each operation works:

1. Insertion

To insert a new value into a BST, follow these steps:

  1. Start at the root node.
  2. If the value to be inserted is less than the current node's value, move to the left child; if greater, move to the right child.
  3. Repeat the process until you reach a null position (where a child node does not exist).
  4. Create a new node at that position with the value to be inserted.

Example: Inserting the value 7 into the BST:

        10
       /  \
      5    15
     / \
    2   7
Inserting 7: - Start at 10 (7 < 10) - Move left to 5 (7 > 5) - Insert 7 as the right child of 5. 
        10
       /  \
      5    15
     / \
    2   7

2. Search

Searching for a value in a BST follows a similar pattern:

  1. Start from the root node.
  2. If the current node's value is equal to the target value, you've found it.
  3. If the target value is less, move to the left child; if greater, move to the right child.
  4. Repeat until the value is found or a null position is reached, indicating that the value doesn't exist in the tree.

3. Deletion

Deleting a node from a BST involves three potential cases:

  1. Node has no children: Simply remove the node.
  2. Node has one child: Remove the node and link its parent to its child.
  3. Node has two children: Find the in-order successor (the smallest value in the right subtree) or in-order predecessor (the largest value in the left subtree), copy its value to the node to be deleted, and then delete the successor/predecessor.

Complexity Analysis

The average time complexity for lookups, insertions, and deletions in a BST is O(log n), where n is the number of nodes in the tree. However, in the worst case (if the tree becomes unbalanced), the complexity becomes O(n), resembling a linked list. To mitigate this, self-balancing BSTs like AVL trees and Red-Black trees are often employed.

Conclusion

A Binary Search Tree is a fundamental data structure with numerous applications in computer science, particularly in searching and sorting operations. Mastering the BST and understanding its operations lays the groundwork for delving deeper into more complex data structures.
