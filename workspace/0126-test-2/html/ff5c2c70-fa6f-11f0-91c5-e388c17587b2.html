A Binary Search Tree (BST) is a type of binary tree that maintains a specific order among its nodes, allowing for efficient searching, insertion, and deletion of elements. Here are the key characteristics and properties of a BST: ### Properties of a BST 1. **Node Structure**: Each node in a BST contains three components: - A key (or value). - A reference to the left child (less than the key). - A reference to the right child (greater than the key). 2. **Ordering**: - For any given node: - All nodes in the left subtree contain keys that are less than the node’s key. - All nodes in the right subtree contain keys that are greater than the node’s key. 3. **Null Children**: A BST can have null references for child nodes that do not exist. ### Basic Operations 1. **Insertion**: - Start at the root. - Compare the value to be inserted with the current node's key. - If the value is smaller, move to the left child; if larger, move to the right child. - Repeat until a null reference is found, then insert the new node at that position. 2. **Search**: - Start at the root and compare the target value with the current node's key. - If the target is equal to the current node’s key, the search is successful. - If the target is smaller, move to the left child; if larger, move to the right child. - Continue until either the value is found or a null reference is reached (indicating the value is not present). 3. **Deletion**: - Locate the node to be deleted. - There are three cases to consider: 1. If the node is a leaf (no children), simply remove it. 2. If the node has one child, remove it and replace it with its child. 3. If the node has two children, find the node’s in-order predecessor (largest in left subtree) or in-order successor (smallest in right subtree), replace the node with that value, and then delete the predecessor or successor node. ### Time Complexity - **Search Operation**: O(h), where h is the height of the tree. - **Insertion Operation**: O(h) - **Deletion Operation**: O(h) For a balanced BST, the height is approximately log(n) (where n is the number of nodes), leading to average-case time complexities of O(log n) for all operations. However, in unbalanced trees (often resembling a linked list), the height can be as large as n, resulting in O(n) time complexity. ### Balancing BSTs To maintain O(log n) performance guarantees, various self-balancing binary search trees exist, such as: - **AVL Trees** - **Red-Black Trees** - **Splay Trees** - **B-Trees** (used mainly in databases and filesystems) Each of these trees uses different strategies for balancing the tree after insertion and deletion operations. ### Example Here's a simple illustration of a BST: ``` 15 / \ 10 20 / \ / \ 8 12 17 25 ``` In this tree: - The root is 15. - The left subtree contains values less than 15 {8, 10, 12}. - The right subtree contains values greater than 15 {17, 20, 25}. This structure allows for efficient searching, insertion, and deletion operations while maintaining the order property of the BST.
