A Binary Search Tree (BST) is a type of data structure that maintains a collection of items, typically numbers or comparable objects, allowing for efficient searching, insertion, and deletion operations. The key properties that define a BST are: 1. **Node Structure**: Each node in the tree contains a value (or key) and references (or pointers) to two child nodes: left and right. 2. **Left and Right Child Rules**: - The left child's value must be less than its parent's value. - The right child's value must be greater than its parent's value. 3. **Unique Values**: In a standard BST, each value must be unique. ### Basic Operations 1. **Insertion**: - To insert a new value, compare it with the root node and decide whether to go left (if the value is smaller) or right (if the value is larger). - Repeat this process recursively until you find an empty spot where the new node can be placed. 2. **Searching**: - Similar to insertion, start at the root and traverse the tree by moving left or right based on comparisons until you find the value or reach a leaf node. 3. **Deletion**: - Deleting a node can be more complex since there are three cases to consider: - If the node is a leaf (no children), simply remove it. - If the node has one child, remove the node and link its parent directly to its child. - If the node has two children, find its in-order successor (the smallest value in its right subtree) or in-order predecessor (the largest value in its left subtree), replace the node's value with this value, and then delete the successor or predecessor. 4. **Traversal**: - Common tree traversal methods include: - In-order (Left, Node, Right): Yields sorted order of values in a BST. - Pre-order (Node, Left, Right): Used to create a copy of the tree. - Post-order (Left, Right, Node): Used for deleting or freeing nodes in a tree. ### Complexity - **Time Complexity**: - Average case for insertion, searching, and deletion is \(O(\log n)\), where \(n\) is the number of nodes in the tree. - Worst case (when the tree becomes unbalanced and resembles a linked list) is \(O(n)\). - **Space Complexity**: - \(O(n)\) for storing the nodes, but recursive operations can use \(O(h)\) space on the call stack, where \(h\) is the height of the tree. ### Example Here’s a simple example of a BST: ``` 8 / \ 3 10 / \ \ 1 6 14 / \ / 4 7 13 ``` In this tree: - In-order traversal gives us the sorted order: `1, 3, 4, 6, 7, 8, 10, 13, 14`. - If we want to insert `5`, we would compare it to `8`, `3`, and `6` and place it between `4` and `6`. ### Self-Balancing BSTs Several variations of BSTs maintain balance to ensure optimal performance regardless of insertion order. These include: - **AVL Trees** - **Red-Black Trees** - **Splay Trees** By maintaining balance, these trees ensure that the height remains \(O(\log n)\), which keeps operations efficient.
